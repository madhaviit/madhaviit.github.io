Puzzle

First of all grid contains \(169\) cells, assuming somehow we can fill all the cells, we get \(n*(n+1)<=169*2\) where we fill numbers unto \(n.\) This gives us value that \(n\) has to be \(17\) since \(n\) must be integer. Thus we know that all the equations in the grid should resolve to a number between \(1\) to \(17.\) Now we need to solve them for the same.

Few equations that I primaraily take cue from are square roots, log and powers. Since \(b^2\) is a cell, \(b\) has to an integer among \(\pm{1},\pm{2},\pm{3}, \pm{4}.\) but \((b-1)^2\) removes \(-4\) from consideration.

Because \(a-c\) is denominator \(a!=c.\) \(c-1\) is denominator so \(c!=1.\) Now there is a cell \(\sqrt{c-a}\) which means \(c>a\) and in another cell where value is \(b/(a-c),\) \(a-c\) is denominator which is negative, so to make cell value positive, we need to eliminate positive possibilities of \(b.\) So \(b\) is now \(-1/-2/-3.\)

Now there is a element \(c^b\) and \(b\) is negative integer which suggests \(c\) has to be of form \(1/d\) where \(d\) is integer \(>1.\) and since we have a cell with \(\log_{c}{a}\) gives away that \(a=c^k\) where \(k\) is an integer among \(2\) to \(17\) beacuse \(c>a.\) There is a cell \(\frac{\sqrt{a+2}}{a}\) which means \(a\) is positive implying \(c\) is also positive.

There is a cell with \(a^b - 12/a\) where if \(b=-1,\) we have \(-11/a\) which won’t work. So remove \(b=-1.\) Now there is also \(b/a-1.\) Which means if we put \(a\) as \(c^k = (1/d)^k\) we get \(-b*d^k/(d^k-1).\) Now, let’s take \(b=-2\) so \(2*d^k/(d^k-1)\) has to be an integer –> \(d^k\) has to be \(2\) or \(3\) and with current constraints on \(d\) and \(k\) we cannot have it. So \(b=-2\) has to be ruled out. Only \(b=-3\) is left. So we have here \(3*d^k/(d^k-1)\) so \(d^k\) should be either \(2\) or \(4.\) \(d^k=2\) won’t work so \(d^k=4\) and deterministicly we can conclude that \(d=2\) and \(k=2.\) Thus \(c = 1/2\) and \(a = 1/4.\)
\(a= 1/4, b=-3, c=1/2.\) We can now fill labeled cells.

First apporach that comes to my mind was to just code this. We can precompute all \(K\)-ominos and can link them with \(K+1\) as well as \(K-1\)-ominos as it is just adding or removing one of the cell. Then we know at least one cell allocated to each number, we put all variations of \(K\)-ominos in that cell which can be base of \(K+1\)-omino we selected, and only those which satisfy conditions like not overlap with any other existing number we pick.

Another way I can see this unfolds is deducing a bunch of things. Now we have values at \(37\) cells. \(R4C11\) has to be \(12\) otherwise we will cut \(11s\) or \(6s.\) Similarly,\(R11C6\) has to \(14.\) Now just to connect exisiting \(12\) cells we must exhaust all remaining \(12s.\) There are only \(2\) ways to do this. (In 3rd way we cut off \(6s\)). But we can surely confirm \(R5,\)\(R6\) in \(C10\) to be \(12.\) So now table becomes
