Natural Language Processing — From Zero to Expert

This document is built for the Workera / SAGE assessment in Natural Language Processing. It is exhaustive by design: a single-file path from corpus collection and tokenization, through statistical NLP, embeddings, RNNs and Transformers, to fine-tuning, RLHF/DPO, evaluation, interpretability, production monitoring, and bias mitigation.

The narrative interleaves intuition, mathematics, and runnable code (Python with transformers, tokenizers, datasets, spaCy, NLTK, scikit-learn, and PyTorch). Recommended companion reading is sprinkled inline: Jay Alammar’s “Illustrated” series, Lilian Weng’s Lil’Log, Christopher Olah’s “Understanding LSTM Networks”, Sebastian Ruder’s blog, Stanford CS224n, and Jurafsky & Martin SLP3 (3rd ed. draft).

1.1.2 Scraping Responsibly

When you scrape, you are operating in a legal and ethical minefield. The defensible posture:

  1. Respect robots.txt — RobotFileParser in Python’s urllib.robotparser.
  2. Honor rate limits — exponential backoff; identify yourself in the User-Agent.
  3. Cache and deduplicate — to avoid hammering a site twice.
  4. Comply with site Terms of Service — TOS overrides scraping convenience.
  5. Strip PII — never store names, emails, phone numbers, addresses unless legally cleared.
  6. Record provenance — URL, retrieval timestamp, license tag — for every document.
from urllib.robotparser import RobotFileParser
rp = RobotFileParser()
rp.set_url("https://example.com/robots.txt")
rp.read()
allowed = rp.can_fetch("MyBot/1.0", "https://example.com/some/page")

1.1.4 Compliance, Licensing, and Privacy

The four legal pillars to understand:

  • Copyright & fair use — In the US, four-factor test (purpose, nature, amount, market effect). Training is contested; Authors Guild v. Google (search snippets) and ongoing NYT v. OpenAI shape doctrine.
  • Licenses — CC-BY (attribution), CC-BY-SA (share-alike), CC-BY-NC (non-commercial), CC0 (public domain), MIT/Apache-2.0 (permissive code), GPL (copyleft), ODC-BY / ODbL (databases). Mixing a CC-BY-SA corpus with a proprietary model can be incompatible.
  • Privacy regulation — GDPR (EU) Articles 6, 9, 15, 17, 22 (legitimate interest, special categories, access, erasure, automated decisions), CCPA/CPRA (California), HIPAA (US health), PIPL (China), LGPD (Brazil). Right to be forgotten conflicts with model memorization.
  • PII categories — direct identifiers (name, SSN, email), quasi-identifiers (DOB + ZIP + sex re-identifies 87% of US population — Sweeney 2000), sensitive categories (health, race, religion, sexuality, biometrics).

Documentation artifacts to ship with every dataset:

  • Datasheets for Datasets (Gebru et al., 2018): motivation, composition, collection process, preprocessing, uses, distribution, maintenance.
  • Data Statements for NLP (Bender & Friedman, 2018): curation rationale, language variety, speaker demographics, annotator demographics, situation, text characteristics.
  • Dataset Cards (HuggingFace README.md): summary, supported tasks, languages, structure, splits, source, considerations.

MODULE 1.2: Labeled Dataset Creation

Three regimes — manual, semi-automated, programmatic — each with distinct tradeoffs.

1.2.1 Manual Annotation

  • Crowdsourcing: Amazon Mechanical Turk (MTurk), Prolific, Appen, Toloka, Scale AI. Pay per HIT; cheap but noisy.
  • Expert annotation: medical, legal, scientific text — slow, expensive, high quality.
  • Guidelines — write them. Iterate them. They are the actual product.
  • Inter-annotator agreement (IAA) is the only honest quality signal:
    • Cohen’s κ (two annotators): \(\kappa = \frac{p_o - p_e}{1 - p_e}\) where \(p_o\) is observed agreement and \(p_e\) is expected agreement by chance.
    • Fleiss’ κ generalizes to >2 annotators.
    • Krippendorff’s α handles missing data and multiple measurement levels.
    • Interpretation: κ < 0.2 poor, 0.2–0.4 fair, 0.4–0.6 moderate, 0.6–0.8 substantial, > 0.8 almost perfect.

1.2.2 Semi-Automated: Active Learning

Train iteratively; ask the model what to label next.

  • Uncertainty sampling — label points where the model is least confident (low max-prob, high entropy, smallest margin).
  • Query-by-committee — train an ensemble, label points where the committee disagrees most (vote entropy, KL divergence between committee distributions).
  • Expected model change — label points whose gradient would most change the model.
  • BALD (Bayesian Active Learning by Disagreement) — for Bayesian / MC-dropout models, label points with high mutual information between predictions and parameters: \(I(y; \theta | x, D) = H[\mathbb{E}_\theta p(y|x,\theta)] - \mathbb{E}_\theta H[p(y|x,\theta)]\).

1.2.3 Programmatic: Weak Supervision

Snorkel (Ratner et al., Stanford) — write labeling functions (LFs) that programmatically vote on labels; a label model combines them while estimating accuracies and correlations without ground truth.

from snorkel.labeling import labeling_function
from snorkel.labeling.model import LabelModel

@labeling_function()
def lf_money_pattern(x):
    import re
    return 1 if re.search(r"\$\d+", x.text) else -1  # 1 = "finance", -1 = abstain

label_model = LabelModel(cardinality=2)
label_model.fit(L_train, n_epochs=500, seed=42)
y_prob = label_model.predict_proba(L_train)

Other patterns: distant supervision (Mintz et al. — match knowledge base entities to text), heuristics + crowdsourcing hybrids, bootstrapping (label a seed, train, label more, repeat — beware of semantic drift).

PART 2: PREPROCESSING & TOKENIZATION

MODULE 2.1: Text Normalization

2.1.1 Unicode Normalization

Four canonical forms (Unicode TR #15):

Form Description Example
NFC Canonical decomposition + composition “é” stays a single code point
NFD Canonical decomposition only “é” → “e” + combining acute (U+0301)
NFKC Compatibility decomposition + composition “ﬁ” → “fi”; “②” → “2”
NFKD Compatibility decomposition strongest flattening

Rule of thumb: NFC for display, NFKC for ML preprocessing (strips presentation variants).

import unicodedata
unicodedata.normalize("NFKC", "café ﬁsh ②")  # → 'café fish 2'

2.1.2 Casing, Punctuation, Whitespace

  • Lowercasing vs casefolding — "ß".lower() == "ß" but "ß".casefold() == "ss". Casefold for cross-locale matching.
  • Aggressive lowercasing destroys NER signal (“Apple” the company vs “apple” the fruit). For cased models (BERT-cased) keep case.
  • Punctuation — strip for BoW; keep for sequence models (punctuation is information).
  • Whitespace — collapse runs of spaces, normalize line endings.

2.1.3 Sentence Segmentation

Naïve regex on .!? fails on “Dr. Smith”, “U.S.A.”, and “…”.

  • Punkt (NLTK) — unsupervised abbreviation detection.
  • spaCy sentencizer or full statistical model.
  • PySBD — rule-based, robust for legal/medical.

2.1.4 Stopword Removal

Remove high-frequency, low-content tokens (“the”, “is”, “of”). When to use:

  • BoW / TF-IDF — yes, often helpful.
  • Neural models — usually no; “not” matters.
  • Sentiment — be careful: “not bad” means “good”.

2.1.5 Stemming vs Lemmatization

  • Stemming — rule-based suffix stripping; output may not be a real word.
    • Porter (1980) — most popular, English only.
    • Snowball (Porter 2) — multilingual, more aggressive.
    • Lancaster — most aggressive, sometimes too lossy.
  • Lemmatization — dictionary-based; returns the base form (“running” → “run”, “better” → “good”). Needs POS to be accurate. WordNet lemmatizer in NLTK; spaCy’s lemmatizer is faster.
from nltk.stem import PorterStemmer, SnowballStemmer, WordNetLemmatizer
PorterStemmer().stem("running")          # 'run'
SnowballStemmer("english").stem("ponies") # 'poni'
WordNetLemmatizer().lemmatize("better", pos="a")  # 'good'

2.1.6 Language Identification

  • CLD2 / CLD3 (Google) — Compact Language Detector, fast.
  • fastText lid.176.bin — 176 languages, accurate, recommended for production.
  • langid.py — pure Python, slower.

MODULE 2.2: Tokenization — From Whitespace to Subwords

Tokenization is the single most consequential preprocessing decision: it defines the model’s vocabulary, OOV behavior, and what “a token” means.

2.2.1 Whitespace and Rule-Based

  • Whitespace — text.split(). Fails on punctuation ("hello,").
  • Moses tokenizer — classical SMT tokenizer; rules for languages.
  • spaCy — non-destructive (you can recover the original string), language-aware.
  • NLTK — word_tokenize (Punkt + Penn Treebank conventions).
  • Treebank tokenization — "don't" → ["do", "n't"]; "$10.50" → ["$", "10.50"].

2.2.2 Why Subwords Won

Word-level tokenization explodes vocabulary (English Wikipedia: ~10M unique words) and is brittle to OOV. Character-level handles OOV but sequences are very long. Subwords balance both: frequent words stay whole, rare words decompose into known pieces.

2.2.3 Byte-Pair Encoding (BPE) — Sennrich et al. 2016

Algorithm (training):

  1. Start with a vocabulary of all characters in the corpus, append a special end-of-word marker </w> to each word.
  2. Count all adjacent symbol pairs across the corpus.
  3. Merge the most frequent pair into a new symbol.
  4. Repeat until you reach the target vocab size (e.g., 32k merges).

Worked example. Tiny corpus with frequencies:

low      : 5
lower    : 2
newest   : 6
widest   : 3

Initial symbols (each character + </w>):

l o w </w>          5
l o w e r </w>      2
n e w e s t </w>    6
w i d e s t </w>    3

Pair counts: (e, s) = 9, (s, t) = 9, (t, </w>) = 9, (l, o) = 7, (o, w) = 7, …

Merge most frequent (break ties by order): (e, s) → es. Recount; next merge (es, t) → est; then (est, </w>) → est</w>; then (l, o) → lo; then (lo, w) → low; …

At test time you apply learned merges in order to a new word.

from tokenizers import Tokenizer, models, trainers, pre_tokenizers
tokenizer = Tokenizer(models.BPE(unk_token="[UNK]"))
tokenizer.pre_tokenizer = pre_tokenizers.Whitespace()
trainer = trainers.BpeTrainer(vocab_size=30000, special_tokens=["[UNK]", "[PAD]", "[CLS]", "[SEP]", "[MASK]"])
tokenizer.train(files=["corpus.txt"], trainer=trainer)

2.2.4 Byte-Level BPE — Radford et al. (GPT-2)

Operate on raw bytes (256 base symbols) rather than Unicode characters. Pros: no UNK token possible — every byte sequence is representable; handles emojis, code, scripts uniformly. Cons: longer sequences for non-Latin scripts.

2.2.5 WordPiece — Schuster & Nakajima, used by BERT

Same iterative-merge spirit as BPE, but the merge criterion is likelihood rather than raw frequency. Choose the pair \((x, y)\) maximizing

\[ \text{score}(x, y) = \frac{\text{freq}(x y)}{\text{freq}(x) \cdot \text{freq}(y)}. \]

Subwords inside a word are marked with ## (BERT convention): playing → play ##ing.

2.2.6 Unigram Language Model — Kudo 2018, used by SentencePiece / T5 / XLNet

Top-down rather than bottom-up:

  1. Start with a large seed vocabulary (e.g., all substrings, or BPE output).
  2. Assume each token has probability \(p(x_i)\); sentence probability = \(\prod p(x_i)\) over the segmentation.
  3. EM: estimate \(p(x_i)\), compute the loss of removing each token (how much the total log-likelihood drops), prune the bottom 10–20%.
  4. Repeat until target vocab size.

At inference, Viterbi picks the most likely segmentation. Subword regularization (Kudo 2018) samples alternative segmentations during training for robustness.

2.2.7 SentencePiece — Kudo & Richardson 2018

A system, not a single algorithm. Treats input as a raw stream (no whitespace pretokenization), encodes whitespace as the visible character ▁ (U+2581). Trains BPE or Unigram. Used by T5, XLNet, ALBERT, LLaMA, mT5.

2.2.8 Comparison Table

Property BPE Byte-level BPE WordPiece Unigram (SP)

OOV possible impossible possible rare
Pretokenization required optional required optional
Multiple segmentations no no no yes (regularization)

2.2.9 Vocabulary Size — A Tradeoff

Vocab Pros Cons
Small (~8k) Short tokenizer, smaller embedding matrix Longer sequences, more tokens per word
Medium (~32k) BERT/RoBERTa default; balanced “—”
Large (~128k+) LLaMA-3 (~128k), Gemma; shorter sequences Bigger embedding & output matrices

Embedding parameter count: vocab_size × hidden_dim. For vocab_size = 32_000, hidden = 4096: 131M parameters in embeddings alone.

2.2.10 Special Tokens

  • [CLS] / <s> — classification head input (BERT).
  • [SEP] / </s> — sentence boundary (BERT, RoBERTa).
  • [PAD] / <pad> — batch padding.
  • [MASK] — masked language modeling target (BERT).
  • [UNK] — out-of-vocabulary (avoid in production).
  • <bos> / <eos> — beginning / end of sequence (generative).
  • <s>...</s> — RoBERTa fairseq convention.

PART 3: DATA AUGMENTATION FOR TEXT

Augmentation widens training distribution and improves generalization, especially for low-resource tasks.

MODULE 3.1: Token-Level (EDA — Wei & Zou 2019)

Four simple operations applied with probability \(\alpha\):

  1. Synonym Replacement (SR) — replace \(n\) random non-stopwords with WordNet synonyms.
  2. Random Insertion (RI) — pick a non-stopword, insert a random synonym of it at a random position.
  3. Random Swap (RS) — swap positions of two random words; repeat \(n\) times.
  4. Random Deletion (RD) — drop each word with probability \(p\).

EDA gives +0.8 to +3.0 absolute F1 on small datasets (≤ 5k examples); diminishing returns on large datasets.

import random, nltk
from nltk.corpus import wordnet
def synonym_replacement(words, n=1):
    new_words = words.copy()
    candidates = [w for w in words if w.isalpha()]
    random.shuffle(candidates)
    replaced = 0
    for w in candidates:
        syns = wordnet.synsets(w)
        lemmas = [l.name() for s in syns for l in s.lemmas() if l.name() != w]
        if lemmas:
            choice = random.choice(lemmas).replace("_", " ")
            new_words = [choice if x == w else x for x in new_words]
            replaced += 1
        if replaced >= n: break
    return new_words

MODULE 3.2: Embedding-Based Replacement

Replace token \(w\) with a nearest neighbor in word2vec / GloVe / fastText space. Word2vec synonyms can be near-antonyms (“good” and “bad” are both close) — be careful for sentiment tasks.

MODULE 3.3: Contextual Augmentation (Kobayashi 2018; Wu et al. 2019)

Mask a token, let BERT fill the blank. Far higher-quality replacements than WordNet because they respect context.

from transformers import pipeline
fill = pipeline("fill-mask", model="bert-base-uncased")

Translate en → fr → en (or via several pivot languages). Preserves meaning, changes surface form. Strong for paraphrasing, NLI, classification.

MODULE 3.5: Paraphrase Models

Seq2seq paraphrasers — T5-paraphrase, PEGASUS-paraphrase, Parrot-paraphraser — generate \(k\) paraphrases per input. Useful for data augmentation, query reformulation, and adversarial robustness.

MODULE 3.6: Mixup-Style for Text

  • wordMixup / sentMixup (Guo et al. 2019) — interpolate embeddings of two examples and their one-hot labels: \(\tilde{x} = \lambda x_i + (1-\lambda) x_j\), \(\tilde{y} = \lambda y_i + (1-\lambda) y_j\), \(\lambda \sim \text{Beta}(\alpha, \alpha)\).
  • AEDA — only insert punctuation (preserves words).
  • TextAttack library bundles many augmentations and attacks.

PART 4: LINGUISTIC FEATURES

MODULE 4.1: Part-of-Speech (POS) Tagging

4.1.1 Tagsets

  • Penn Treebank (PTB) — 36 tags. Examples: NN (noun, singular), NNS (noun plural), NNP (proper noun), VB (verb base form), VBD (verb past), VBG (verb gerund), VBN (past participle), VBP (non-3sg present), VBZ (3sg present), JJ (adjective), RB (adverb), IN (preposition), DT (determiner), PRP (pronoun), CC (conjunction).
  • Universal POS (UPOS) — 17 coarse tags: NOUN, VERB, ADJ, ADV, PRON, DET, ADP, NUM, CONJ, PRT, PUNCT, X, .... Multilingual.

4.1.2 Approaches

Approach Notes
Rule-based (Brill tagger) Transformation-based learning.
HMM See Module 5.2 below.
MaxEnt / CRF Conditional models; handle overlapping features.
BiLSTM + CRF Strong neural baseline.
Transformer (BERT) token classification SOTA, ~97.7% on PTB.

MODULE 4.2: Syntactic Parsing

4.2.1 Constituency Parsing

  • Output is a phrase-structure tree: (S (NP The cat) (VP sat (PP on (NP the mat)))).
  • Context-Free Grammar (CFG): rules like S → NP VP. PCFG assigns probabilities to rules.
  • CKY (Cocke–Kasami–Younger) algorithm — \(O(n^3 |G|)\) dynamic programming for CNF grammars. Fills a triangular chart c[i][j] of constituents spanning words \(i..j\).
  • Modern: self-attentive parser (Kitaev & Klein 2018), Berkeley Neural Parser.

4.2.2 Dependency Parsing

  • Output is a labeled tree of head-dependent arcs: nsubj(sat, cat), det(cat, The), prep(sat, on), …
  • Universal Dependencies (UD) — cross-lingual standard, ~40 relations.
  • Transition-based (Nivre 2003; Chen & Manning 2014 neural) — stack + buffer + actions (SHIFT, LEFT-ARC, RIGHT-ARC); greedy or beam. Linear time.
  • Graph-based (McDonald 2005; Dozat & Manning 2017 biaffine) — score every potential arc, take maximum spanning tree (Chu-Liu-Edmonds). \(O(n^2)\) scoring + \(O(n^2)\) MST for non-projective.
  • Projective vs non-projective — projective means arcs do not cross when drawn above the sentence; English mostly projective, German and Czech often not.

4.2.3 Chunking / Shallow Parsing

Partial parsing that identifies non-overlapping flat phrases (NP, VP, PP). Trained with BIO tags. Faster than full parsing.

MODULE 4.3: Semantic Layers

  • Semantic Role Labeling (SRL) — predicate-argument structure (“Who did what to whom?”). Uses PropBank (verb senses + numbered Arg0–Arg5) or FrameNet (frames + frame elements). E.g., for “Mary gave John a book”: give.01 → Arg0=Mary (giver), Arg1=a book (thing given), Arg2=John (recipient).
  • Coreference resolution — link mentions to the same entity (“Mary said she was tired” → she = Mary). Mention-pair, mention-ranking, end-to-end neural (Lee et al. 2017), and SpanBERT-based systems.
  • Word senses & WordNet — synsets, hypernyms (more general), hyponyms (more specific), meronyms (part of), holonyms (whole of). Wu-Palmer / Path / Lin similarity.

PART 5: STATISTICAL NLP

MODULE 5.1: Bag-of-Words, n-grams, TF-IDF, BM25

5.1.1 Bag-of-Words

Represent a document as a vector \(\mathbf{d} \in \mathbb{R}^{|V|}\) where \(d_i\) is the count of vocabulary term \(i\). Discards order, keeps frequency.

5.1.2 n-grams and Skip-grams

  • Unigram: single token. Bigram: pairs. Trigram: triples. Higher orders are sparse.
  • Skip-grams: tokens within window allowing skips (“he ___ home” with skip-2 captures distant collocations).
  • Add-one (Laplace) smoothing for n-gram LMs: \(P(w_n | w_{n-1}) = \frac{C(w_{n-1} w_n) + 1}{C(w_{n-1}) + V}\).
  • Kneser-Ney smoothing — uses continuation probability (how many unique contexts a word appears in), not raw frequency. State of the art for n-gram LMs.

5.1.3 TF-IDF — Full Math

Term frequency variants:

  • Raw: \(\text{tf}(t, d) = \text{count}(t, d)\).
  • Log-normalized: \(\text{tf}(t, d) = 1 + \log(\text{count}(t, d))\) if count > 0.
  • Double normalization: \(\text{tf}(t, d) = 0.5 + 0.5 \frac{\text{count}(t, d)}{\max_{t'} \text{count}(t', d)}\).

Inverse document frequency (smoothed, sklearn default):

\[ \text{idf}(t) = \log \frac{1 + N}{1 + \text{df}(t)} + 1 \]

TF-IDF score:

\[ \text{tf-idf}(t, d) = \text{tf}(t, d) \cdot \text{idf}(t). \]

L2-normalize each document vector for cosine similarity.

from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity
corpus = ["the cat sat on the mat", "a dog sat on the log", "cats and dogs"]
vec = TfidfVectorizer(ngram_range=(1,2), min_df=1, sublinear_tf=True)
X = vec.fit_transform(corpus)
cosine_similarity(X[0], X[1])  # 2D array

5.1.4 BM25 — Okapi Best-Match 25

Used in search (Elasticsearch, Lucene). Saturating term frequency + document length normalization:

\[ \text{BM25}(q, d) = \sum_{t \in q} \text{idf}(t) \cdot \frac{\text{tf}(t, d) (k_1 + 1)}{\text{tf}(t, d) + k_1 \left(1 - b + b \frac{|d|}{\text{avgdl}}\right)} \]

Typical \(k_1 \in [1.2, 2.0]\) (saturation rate), \(b \in [0.5, 0.8]\) (length normalization strength). BM25 still beats embedding retrieval on out-of-domain corpora and remains the canonical sparse retriever.

5.1.5 PMI / PPMI

Pointwise mutual information measures association between two words:

\[ \text{PMI}(x, y) = \log \frac{P(x, y)}{P(x) P(y)}. \]

PPMI clamps negatives to zero: \(\text{PPMI}(x, y) = \max(\text{PMI}(x, y), 0)\). PPMI matrices were the precursor to learned word embeddings; SVD on PPMI is essentially a count-based embedding.

MODULE 5.2: Hidden Markov Models for POS Tagging

An HMM models a sequence of hidden states \(S = s_1, ..., s_T\) (POS tags) emitting observations \(O = o_1, ..., o_T\) (words).

5.2.1 Parameters

  • Initial distribution: \(\pi_i = P(s_1 = i)\).
  • Transition probabilities: \(a_{ij} = P(s_t = j | s_{t-1} = i)\).
  • Emission probabilities: \(b_j(o) = P(o_t = o | s_t = j)\).

Three classic problems (Rabiner 1989):

  1. Evaluation — compute \(P(O | \lambda)\) via the Forward algorithm.
  2. Decoding — find best state sequence via Viterbi.
  3. Learning — estimate \(\lambda\) via Baum-Welch (EM).

5.2.2 Viterbi Algorithm

Define \(v_t(j)\) = probability of the best path ending in state \(j\) at time \(t\):

\[ v_1(j) = \pi_j \cdot b_j(o_1), \] \[ v_t(j) = \max_{i} \left[ v_{t-1}(i) \cdot a_{ij} \right] \cdot b_j(o_t), \] \[ P^* = \max_j v_T(j), \quad s^*_T = \arg\max_j v_T(j). \]

Backpointers \(\psi_t(j) = \arg\max_i v_{t-1}(i) a_{ij}\) allow path recovery. Complexity \(O(T \cdot N^2)\) for \(N\) states. Work in log-space to avoid underflow.

5.2.3 Forward / Backward

Forward: \(\alpha_t(j) = P(o_1, ..., o_t, s_t = j | \lambda) = \left[ \sum_i \alpha_{t-1}(i) a_{ij} \right] b_j(o_t)\).

Backward: \(\beta_t(i) = P(o_{t+1}, ..., o_T | s_t = i, \lambda) = \sum_j a_{ij} b_j(o_{t+1}) \beta_{t+1}(j)\).

State posterior: \(\gamma_t(i) = \frac{\alpha_t(i) \beta_t(i)}{\sum_j \alpha_t(j) \beta_t(j)}\).

5.2.4 Baum-Welch (EM)

E-step computes \(\gamma_t(i)\) and \(\xi_t(i, j) = P(s_t = i, s_{t+1} = j | O, \lambda)\). M-step re-estimates:

\[ \hat{a}_{ij} = \frac{\sum_t \xi_t(i, j)}{\sum_t \gamma_t(i)}, \quad \hat{b}_j(o) = \frac{\sum_{t: o_t = o} \gamma_t(j)}{\sum_t \gamma_t(j)}. \]

import numpy as np
def viterbi(obs, states, start_p, trans_p, emit_p):
    T, N = len(obs), len(states)
    V = np.full((T, N), -np.inf)
    bp = np.zeros((T, N), dtype=int)
    for j in range(N):
        V[0, j] = np.log(start_p[j]) + np.log(emit_p[j][obs[0]])
    for t in range(1, T):
        for j in range(N):
            scores = V[t-1] + np.log(trans_p[:, j])
            bp[t, j] = np.argmax(scores)
            V[t, j] = scores[bp[t, j]] + np.log(emit_p[j][obs[t]])
    best = [int(np.argmax(V[T-1]))]
    for t in range(T-1, 0, -1):
        best.append(int(bp[t, best[-1]]))
    return best[::-1]

HMMs achieve ~96% on PTB POS tagging; neural sequence models cross 97.5%.

5.2.5 Discriminative Sequence Models

  • Maximum Entropy Markov Model (MEMM) — \(P(s_t | s_{t-1}, o_t)\). Suffers label bias.
  • Conditional Random Field (CRF) (Lafferty et al. 2001) — globally normalized:

\[ P(s | o) = \frac{1}{Z(o)} \exp\left( \sum_t \sum_k \lambda_k f_k(s_{t-1}, s_t, o, t) \right). \]

Forward-backward and Viterbi adapt to CRF inference. BiLSTM-CRF (Huang et al. 2015) and BERT + CRF remain standard for NER.

PART 6: WORD AND SENTENCE EMBEDDINGS

Best companion: Jay Alammar — “The Illustrated Word2vec”.

MODULE 6.1: word2vec — Mikolov et al. 2013

Two architectures, one shallow neural network each.

6.1.1 Skip-Gram

Predict surrounding context words given a center word. For center \(w_t\) and window size \(c\):

\[ \mathcal{L} = -\frac{1}{T} \sum_{t=1}^{T} \sum_{-c \le j \le c, j \neq 0} \log P(w_{t+j} | w_t), \]

with

\[ P(w_O | w_I) = \frac{\exp(\mathbf{v}'_{w_O}{}^\top \mathbf{v}_{w_I})}{\sum_{w \in V} \exp(\mathbf{v}'_w{}^\top \mathbf{v}_{w_I})}. \]

6.1.2 CBOW

Predict the center word from the (averaged) context.

6.1.3 Negative Sampling

The softmax over \(|V|\) is prohibitive. Replace it: for each positive pair \((w, c)\) sample \(k\) “negative” words from a noise distribution and discriminate:

\[ \log \sigma(\mathbf{v}'_c{}^\top \mathbf{v}_w) + \sum_{i=1}^{k} \mathbb{E}_{w_i \sim P_n}[\log \sigma(-\mathbf{v}'_{w_i}{}^\top \mathbf{v}_w)]. \]

The noise distribution is \(P_n(w) \propto U(w)^{3/4}\) — unigram raised to 3/4 power down-weights very frequent words. Typical \(k = 5\) for large data, 15–25 for small.

6.1.4 Hierarchical Softmax

Alternative to negative sampling: arrange vocabulary in a Huffman tree, compute \(P(w | c)\) as product of binary classifications along the path. \(O(\log |V|)\) per update.

6.1.5 Subsampling Frequent Words

Drop word \(w\) with probability \(P(w) = 1 - \sqrt{t / f(w)}\) for \(t \approx 10^{-5}\). Speeds up training and improves rare-word quality.

6.1.6 The Famous Analogy

\[ \mathbf{v}(\text{king}) - \mathbf{v}(\text{man}) + \mathbf{v}(\text{woman}) \approx \mathbf{v}(\text{queen}). \]

Geometry encodes gender, plurality, capital-of-country, verb-tense, comparative/superlative, and more.

from gensim.models import Word2Vec
sents = [["the", "cat", "sat"], ["the", "dog", "ran"]]
model = Word2Vec(sents, vector_size=100, window=5, min_count=1,
                 sg=1, negative=10, epochs=5)
model.wv.most_similar("cat")

MODULE 6.2: GloVe — Pennington et al. 2014

GloVe is count-based but factorizes a log-co-occurrence matrix rather than predicting context.

Define \(X_{ij}\) = number of times word \(j\) appears in the context of word \(i\). The model fits

\[ \mathbf{w}_i^\top \tilde{\mathbf{w}}_j + b_i + \tilde{b}_j = \log X_{ij} \]

with weighted least squares

\[ J = \sum_{i, j: X_{ij} > 0} f(X_{ij}) \left( \mathbf{w}_i^\top \tilde{\mathbf{w}}_j + b_i + \tilde{b}_j - \log X_{ij} \right)^2, \]

\[ f(x) = \begin{cases} (x / x_{\max})^\alpha & x < x_{\max} \\ 1 & \text{otherwise} \end{cases}, \]

typically \(x_{\max} = 100\), \(\alpha = 3/4\). Final embedding \(\mathbf{w}_i + \tilde{\mathbf{w}}_i\) averages both vectors.

MODULE 6.3: fastText — Bojanowski et al. 2017

word2vec ignores morphology — “running” and “run” are unrelated. fastText represents each word as the sum of its character n-gram embeddings (n = 3..6), then trains a skip-gram-style model on these. Out-of-vocabulary handling becomes essentially free: compute n-grams of the new word.

from gensim.models import FastText
ft = FastText(sents, vector_size=100, min_n=3, max_n=6, sg=1, epochs=5)
ft.wv["runnning"]  # works even though "runnning" never appeared

MODULE 6.4: ELMo — Peters et al. 2018

A two-layer biLSTM language model trained on a large corpus; each token gets a contextual embedding by a learned weighted sum of:

  • the character-CNN embedding (layer 0),
  • the first-layer LSTM hidden state,
  • the second-layer LSTM hidden state.

Same word, different sentences → different vectors. This was the breakthrough that ended the static-embeddings era.

Recommended: Jay Alammar’s “The Illustrated BERT, ELMo, and co.”.

MODULE 6.5: BERT — Devlin et al. 2018

A bidirectional transformer encoder pretrained with two objectives.

6.5.1 Masked Language Modeling (MLM)

Randomly mask 15% of WordPiece tokens. Of those: 80% replaced with [MASK], 10% with a random token, 10% kept unchanged. Predict the original.

6.5.2 Next Sentence Prediction (NSP)

Given pair (A, B), predict whether B follows A. Later papers (RoBERTa) showed NSP adds little value — removing it improves downstream performance.

6.5.3 Inputs

  • Token embeddings (WordPiece, 30k vocab).
  • Segment embeddings (A or B).
  • Learned positional embeddings (up to 512).
  • Special tokens: [CLS] at start (its final hidden state is the sentence rep for classification), [SEP] between segments.

6.5.4 Sizes

  • BERT-base: 12 layers, 768 hidden, 12 heads, 110M params.
  • BERT-large: 24 layers, 1024 hidden, 16 heads, 340M params.

MODULE 6.6: Variants of BERT

  • RoBERTa (Liu et al. 2019) — drops NSP, dynamic masking (re-mask every epoch), bigger batches, more data, byte-level BPE; consistently beats BERT.
  • DistilBERT (Sanh et al. 2019) — knowledge-distilled from BERT; 40% smaller, 60% faster, retains 97% of GLUE.
  • ALBERT (Lan et al. 2019) — factorized embedding \(V \times E \times H\) (E ≪ H), cross-layer parameter sharing, replaces NSP with Sentence-Order Prediction.
  • DeBERTa (He et al. 2020) — disentangled attention: separate content and position vectors with three attention scores (content-content, content-position, position-content). State of the art among encoders.
  • XLNet (Yang et al. 2019) — permutation language modeling. Trains \(P(x_{\sigma(t)} | x_{\sigma(<t)})\) over random permutations.
  • ELECTRA (Clark et al. 2020) — replaced-token detection (binary “is this real or generated by a small MLM?”); far more sample-efficient than MLM.
  • SpanBERT (Joshi et al. 2020) — mask contiguous spans; predict via boundary tokens.
  • MPNet (Song et al. 2020) — combines MLM + PLM.

MODULE 6.7: Sentence-BERT — Reimers & Gurevych 2019

BERT [CLS] embeddings are bad for semantic similarity. Sentence-BERT fine-tunes BERT in a siamese architecture with a contrastive / triplet loss to produce sentence embeddings that work with cosine similarity.

Variants and follow-ups:

  • MPNet / all-MiniLM-L6-v2 — small and fast, very popular.
  • E5 (Wang et al. 2022) — query: / passage: prefixes; weakly supervised then SFT.
  • BGE (BAAI General Embedding) — Chinese-strong, strong English.
  • Instructor (Su et al. 2022) — instruction-conditioned embeddings.
  • GTE, Cohere Embed v3, OpenAI text-embedding-3-large — production-grade closed embeddings.
  • ColBERT (Khattab & Zaharia 2020) — late-interaction; per-token MaxSim retrieval.

model = SentenceTransformer("sentence-transformers/all-MiniLM-L6-v2")
emb = model.encode(["I love NLP.", "Natural language processing is great."])
util.cos_sim(emb[0], emb[1])

PART 7: CORE NLP TASKS

MODULE 7.1: Named Entity Recognition

7.1.1 BIO and BIOES Tagging

Each token gets a label:

  • B-X — beginning of an entity of type X.
  • I-X — inside an entity of type X.
  • O — outside any entity.

Extended BIOES (a.k.a. BILOU) adds:

  • E-X — end of an entity (last token).
  • S-X — single-token entity.

Example: “Barack Obama visited Paris .”

Token BIO BIOES
Barack B-PER B-PER
Obama I-PER E-PER
visited O O
Paris B-LOC S-LOC
. O O

7.1.2 Architectures

  • HMM/CRF with hand-crafted features (gazetteers, capitalization).
  • BiLSTM-CRF (Huang et al. 2015; Lample et al. 2016) — neural feature extractor + CRF for valid tag sequences.
  • BERT + linear / CRF head — current standard.
  • Span-based NER (Lee et al. 2017 style) — enumerate spans (length ≤ L), classify each.
  • Seq2seq NER — generate annotations as text (T5, GPT). Useful for nested entities.

7.1.3 Evaluation — Token vs Entity F1

Token-level F1 over-counts: getting one token of a 3-token entity right is partial credit. Entity-level F1 (exact span match) is the standard, computed by seqeval.

from seqeval.metrics import classification_report, f1_score
y_true = [["B-PER","I-PER","O","B-LOC","O"]]
y_pred = [["B-PER","I-PER","O","B-LOC","O"]]
f1_score(y_true, y_pred)  # 1.0

7.1.4 Common Pitfalls

  • Boundary errors dominate; partial matches penalized fully.
  • Nested entities (e.g., “Bank of America” contains “America”). Standard BIO can’t represent nesting; use multi-label or span-based.
  • Rare types (“PRODUCT”, “EVENT”) need oversampling or focal loss.
  • Capitalization signal — case-normalized models lose ground.

MODULE 7.2: Sentiment Analysis

7.2.1 Approaches Ladder

  1. Lexicon-based — VADER (rule-based, social-media tuned), AFINN, SentiWordNet, NRC. No training needed; weak on context.
  2. Bag-of-words + classical ML — TF-IDF + LogReg, NaiveBayes, SVM. Surprisingly strong baselines.
  3. Word embeddings + LSTM/CNN — captures local order.
  4. Fine-tuned BERT / RoBERTa / DeBERTa — SOTA on SST-2 (~97%), IMDB (~96%).
  5. LLM zero-shot / few-shot — for novel domains where labeled data is scarce.

7.2.2 Evaluation

  • Confusion matrix — pred × true grid; reveals which class is confused with which.
  • Precision \(= TP / (TP + FP)\) — of predicted positives, how many were right.
  • Recall \(= TP / (TP + FN)\) — of true positives, how many we caught.
  • F1 \(= 2 PR / (P + R)\) — harmonic mean.
  • Macro F1 — unweighted mean of per-class F1; fair when classes are imbalanced.
  • Micro F1 — global TP/FP/FN pooled (equals accuracy in single-label multi-class).
  • Weighted F1 — per-class F1 weighted by support.
  • ROC-AUC — threshold-free; insensitive to class imbalance.
  • PR-AUC — better than ROC under severe imbalance.
  • MCC (Matthews Correlation Coefficient) — balanced, single number from confusion matrix.
from sklearn.metrics import classification_report, confusion_matrix
print(classification_report(y_true, y_pred, digits=3))
print(confusion_matrix(y_true, y_pred))

7.2.3 Class-Imbalance Tricks

  • Class weights in the loss: \(w_c = N / (C \cdot N_c)\).
  • Oversampling minority class (with augmentation to avoid overfitting).
  • Focal Loss (Lin et al. 2017): \(\text{FL}(p_t) = -\alpha_t (1 - p_t)^\gamma \log p_t\) — down-weights easy examples.

PART 8: NEURAL SEQUENCE MODELS — RNN, LSTM, GRU, CNN

Companion reading: Christopher Olah — “Understanding LSTM Networks”.

MODULE 8.1: Vanilla RNN (Elman)

For input \(\mathbf{x}_t\) and previous hidden \(\mathbf{h}_{t-1}\):

\[ \mathbf{h}_t = \tanh(W_{xh} \mathbf{x}_t + W_{hh} \mathbf{h}_{t-1} + \mathbf{b}_h), \quad \mathbf{y}_t = W_{hy} \mathbf{h}_t + \mathbf{b}_y. \]

Trained with Backpropagation Through Time (BPTT). Truncated BPTT limits the backward window to keep memory bounded.

8.1.1 Vanishing / Exploding Gradients

\(\partial \mathcal{L} / \partial \mathbf{h}_k\) involves a product of Jacobians \(\prod_{t=k+1}^{T} \mathbf{J}_t\). If the dominant singular value of \(\mathbf{J}_t\) is \(< 1\), gradients vanish exponentially; if \(> 1\), they explode. Mitigations:

  • Gradient clipping (\(\| \mathbf{g} \| > \tau \implies \mathbf{g} \leftarrow \tau \mathbf{g} / \|\mathbf{g}\|\)).
  • Orthogonal / identity initialization of \(W_{hh}\).
  • Gating — LSTM and GRU.

MODULE 8.2: LSTM — Hochreiter & Schmidhuber 1997

A cell state \(\mathbf{c}_t\) that flows through time with only linear interactions, gated by sigmoids.

\[ \begin{aligned} \mathbf{f}_t &= \sigma(W_f [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_f) \quad &\text{(forget gate)} \\ \mathbf{i}_t &= \sigma(W_i [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_i) \quad &\text{(input gate)} \\ \tilde{\mathbf{c}}_t &= \tanh(W_c [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_c) \quad &\text{(candidate)} \\ \mathbf{c}_t &= \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{c}}_t \quad &\text{(cell state)} \\ \mathbf{o}_t &= \sigma(W_o [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_o) \quad &\text{(output gate)} \\ \mathbf{h}_t &= \mathbf{o}_t \odot \tanh(\mathbf{c}_t) \quad &\text{(hidden state)} \end{aligned} \]

Why it works: the additive \(\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \cdots\) recurrence has a path for gradients to flow unchanged when \(\mathbf{f}_t \approx 1\).

Forget-gate bias initialization to 1 is a classic trick: encourages “remember by default” early in training (Jozefowicz et al. 2015).

MODULE 8.3: GRU — Cho et al. 2014

A simpler gate (no separate cell state):

\[ \begin{aligned} \mathbf{z}_t &= \sigma(W_z [\mathbf{h}_{t-1}, \mathbf{x}_t]) \quad &\text{(update)} \\ \mathbf{r}_t &= \sigma(W_r [\mathbf{h}_{t-1}, \mathbf{x}_t]) \quad &\text{(reset)} \\ \tilde{\mathbf{h}}_t &= \tanh(W [\mathbf{r}_t \odot \mathbf{h}_{t-1}, \mathbf{x}_t]) \\ \mathbf{h}_t &= (1 - \mathbf{z}_t) \odot \mathbf{h}_{t-1} + \mathbf{z}_t \odot \tilde{\mathbf{h}}_t. \end{aligned} \]

Fewer parameters than LSTM, often similar performance.

MODULE 8.4: BiLSTM, Stacked, Encoder-Decoder

  • Bidirectional: concat forward and backward hidden states, \(\mathbf{h}_t = [\overrightarrow{\mathbf{h}}_t; \overleftarrow{\mathbf{h}}_t]\). Standard for tagging tasks where future context is available.
  • Stacked (deep) RNN: outputs of one layer become inputs of the next. Often 2–4 layers.
  • Seq2seq (Sutskever et al. 2014): an encoder RNN compresses the source into a fixed vector, a decoder RNN generates the target conditioned on it. Reverse the source sentence for English-to-X — improves performance.

MODULE 8.5: Bahdanau Attention — 2014/2015

The fixed-vector bottleneck of seq2seq fails on long sequences. Attention lets the decoder peek at all encoder states at every step.

For decoder step \(t\), with encoder states \(\mathbf{h}_1, ..., \mathbf{h}_S\):

\[ e_{t,s} = \mathbf{v}^\top \tanh(W_d \mathbf{s}_{t-1} + W_e \mathbf{h}_s), \quad \alpha_{t,s} = \frac{\exp e_{t,s}}{\sum_{s'} \exp e_{t,s'}}, \quad \mathbf{c}_t = \sum_s \alpha_{t,s} \mathbf{h}_s. \]

The context \(\mathbf{c}_t\) joins the decoder. Luong attention (2015) variants:

  • Dot: \(e_{t, s} = \mathbf{s}_t^\top \mathbf{h}_s\).
  • General: \(e_{t, s} = \mathbf{s}_t^\top W \mathbf{h}_s\).
  • Concat (Bahdanau-like): \(e_{t, s} = \mathbf{v}^\top \tanh(W [\mathbf{s}_t; \mathbf{h}_s])\).

MODULE 8.6: CNNs for Text — TextCNN (Kim 2014)

Even though sequences are 1D, convolutions still work:

  1. Embed tokens to \(\mathbb{R}^{n \times d}\).
  2. Apply \(K\) filters of widths \(w \in \{3, 4, 5\}\), each producing a feature map of length \(n - w + 1\).
  3. Max-over-time pooling picks the strongest activation per filter.
  4. Concatenate, dense layer, softmax.

Fast, strong on short text (tweets, headlines). Doesn’t capture long-range order on its own — combine with attention or stack dilated convolutions (TCN, ConvS2S).

PART 9: ATTENTION & TRANSFORMERS — DEEP DIVE

Companion reading: Jay Alammar — “The Illustrated Transformer” and Lilian Weng — “Attention? Attention!” and “The Transformer Family v2”.

MODULE 9.1: Scaled Dot-Product Attention

Given queries \(Q \in \mathbb{R}^{n_q \times d_k}\), keys \(K \in \mathbb{R}^{n_k \times d_k}\), values \(V \in \mathbb{R}^{n_k \times d_v}\):

\[ \text{Attn}(Q, K, V) = \text{softmax}\!\left( \frac{QK^\top}{\sqrt{d_k}} \right) V. \]

Why divide by \(\sqrt{d_k}\)? For independent zero-mean unit-variance entries, \(\text{Var}(q \cdot k) = d_k\), so dot products grow with \(d_k\). Without scaling, softmax saturates and gradients vanish.

Masked attention: set illegal positions (future tokens for causal LMs; padding tokens) to \(-\infty\) before softmax.

import torch, math
def attn(Q, K, V, mask=None):
    d_k = Q.size(-1)
    scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)
    if mask is not None: scores = scores.masked_fill(mask == 0, float("-inf"))
    return torch.softmax(scores, dim=-1) @ V

MODULE 9.2: Multi-Head Attention

Split the model dim \(d\) into \(h\) heads of dim \(d_h = d / h\). Each head computes its own scaled dot-product attention. Outputs are concatenated and projected:

\[ \text{MHA}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h) W^O, \] \[ \text{head}_i = \text{Attn}(Q W_i^Q, K W_i^K, V W_i^V). \]

Multiple heads learn different attention patterns (syntactic, semantic, positional).

MODULE 9.3: Transformer Block (Vaswani et al. 2017)

A stack of \(N\) identical blocks; each block:

  1. Multi-head self-attention sublayer.
  2. Position-wise feed-forward network: \(\text{FFN}(x) = \max(0, x W_1 + b_1) W_2 + b_2\) (or GELU/SwiGLU).
  3. Residual connections around each sublayer.
  4. LayerNorm.

Two normalization placements:

  • Post-LN (original): \(\text{LN}(x + \text{Sublayer}(x))\). Harder to train very deep.
  • Pre-LN (GPT-2, LLaMA, modern default): \(x + \text{Sublayer}(\text{LN}(x))\). Stable for 100+ layers.

FFN inner dim is usually \(4d\). SwiGLU (Shazeer 2020; used by LLaMA) splits into a gated variant: \(\text{SwiGLU}(x) = (\text{Swish}(x W_1)) \odot (x W_2)\) then \(W_3\).

MODULE 9.4: Positional Encodings

Self-attention is permutation-equivariant — positions must be injected.

9.4.1 Sinusoidal (Vaswani 2017)

\[ \text{PE}_{(\text{pos}, 2i)} = \sin\!\left(\frac{\text{pos}}{10000^{2i/d}}\right), \quad \text{PE}_{(\text{pos}, 2i+1)} = \cos\!\left(\frac{\text{pos}}{10000^{2i/d}}\right). \]

Each dim is a sinusoid of geometrically increasing wavelength. Allows linear maps to represent shifts: \(\text{PE}_{\text{pos}+k}\) is a linear function of \(\text{PE}_{\text{pos}}\).

9.4.2 Learned Absolute (BERT, GPT-1/2)

Learn a \(\text{max\_len} \times d\) embedding matrix. Simple but cannot extrapolate past max_len.

9.4.3 Relative Position (Shaw 2018, T5)

Bias attention scores by a learned function of \((i - j)\). T5 uses a small bucketed table shared across layers.

9.4.4 RoPE — Rotary Position Embedding (Su et al. 2021)

Apply a position-dependent rotation in \(\mathbb{R}^2\) pairs of dimensions to \(Q\) and \(K\) before the dot product. The dot product then depends only on \(i - j\):

\[ (R_\theta^i q)^\top (R_\theta^j k) = q^\top R_\theta^{j - i} k. \]

Used by LLaMA, Mistral, Falcon, Qwen, GPT-NeoX. Extrapolates somewhat with tricks like NTK-aware scaling and YaRN.

9.4.5 ALiBi (Press et al. 2021)

Add a head-specific linear penalty to the attention score: \(\text{score}_{ij} \mathrel{+}= -m \cdot |i - j|\). No positional embedding at all. Extrapolates well past training length.

9.5.1 Cross-Attention

In the decoder, an extra MHA sublayer where \(Q\) comes from the decoder and \(K, V\) come from the encoder output.

9.5.2 Complexity

Self-attention is \(O(n^2 d)\) in time and memory in the sequence length \(n\). This is the bottleneck for long contexts; remedies in §9.7.

MODULE 9.6: A Worked Forward Pass

Consider a 1-layer, 2-head, \(d=4\) transformer encoder block, single input token sequence of length 3 with embeddings

\[ X = \begin{bmatrix} 1 & 0 & 1 & 0 \\ 0 & 2 & 0 & 2 \\ 1 & 1 & 1 & 1 \end{bmatrix}. \]

Heads have \(d_h = 2\). Initialize \(W^Q, W^K, W^V \in \mathbb{R}^{4 \times 2}\) per head (in practice as a single \(4 \times 4\) projection then reshape). Compute \(Q = X W^Q\), etc. Scores \(S = QK^\top / \sqrt{2}\), \(A = \text{softmax}(S)\), \(H = A V\). Concat the two heads, project by \(W^O\), add residual and LayerNorm, FFN, add residual and LayerNorm. The same routine repeats \(N\) times.

MODULE 9.7: Efficient Attention Variants

For long contexts (8k–1M+), \(O(n^2)\) is fatal. Approaches:

  • Sparse / windowed: Longformer (sliding window + global tokens), BigBird (random + window + global).
  • Linear attention: Performer (FAVOR+ random features), Linformer (low-rank \(K, V\) projection), Linear Transformers (kernel form).
  • Recurrent / chunked: Transformer-XL (segment-level recurrence + relative pos), Compressive Transformer.
  • State-space models: S4, Mamba (selective SSM), Mamba-2 — linear in \(n\), strong on long-context, no attention.
  • FlashAttention (Dao et al. 2022, 2023): exact attention but IO-aware — tiles the computation to fit in SRAM, avoids materializing \(n^2\) matrices in HBM. 2–4× faster, much less memory. Now the default in modern LLM kernels.
  • Multi-Query Attention (MQA) / Grouped-Query Attention (GQA): share K, V across multiple heads. Reduces KV-cache memory at inference (huge for long sequences). LLaMA-2 70B uses GQA.
  • Sliding-window attention (Mistral): each token attends to the previous \(W\) tokens; receptive field grows with depth.

PART 10: FINE-TUNING, PEFT, INSTRUCTION TUNING, RLHF, DPO

MODULE 10.1: Full Fine-Tuning

Update all parameters of a pretrained model on a downstream task. Best quality, but:

  • Memory ∝ params × (params for grads + optimizer state). AdamW needs ~12 bytes/param for fp32 (params + grad + m + v); ~6 bytes/param in mixed precision.
  • A 7B model in full fine-tuning ≈ 80–100 GB GPU RAM.
  • Risk: catastrophic forgetting of pretraining knowledge.

Common recipes:

  • Lower LR than pretraining (e.g., \(2 \times 10^{-5}\) to \(5 \times 10^{-5}\) for BERT).
  • Linear warmup + decay schedule.
  • Layer-wise learning rate decay (ULMFiT/ELECTRA): higher LR for top layers.
  • Layer freezing: freeze the lower N layers entirely; train only the top.

MODULE 10.2: Parameter-Efficient Fine-Tuning (PEFT)

Update << 1% of parameters; achieve near-full-FT quality.

10.2.1 Adapters (Houlsby et al. 2019)

Insert a small bottleneck module after each sublayer:

\[ \text{Adapter}(h) = h + W_{\text{up}} \cdot \sigma(W_{\text{down}} h). \]

with \(W_{\text{down}} \in \mathbb{R}^{d \times r}\), \(W_{\text{up}} \in \mathbb{R}^{r \times d}\), \(r \ll d\).

10.2.2 LoRA — Hu et al. 2021

Freeze \(W\); learn a low-rank update \(\Delta W = B A\) with \(B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times k}\), then

\[ W' = W + \frac{\alpha}{r} B A. \]

Training optimizes only \(A, B\) — typically \(r \in \{4, 8, 16, 64\}\), \(\alpha \in \{8, 16, 32\}\). At inference, fold \(\Delta W\) into \(W\) — zero latency overhead. Applied most often to \(W^Q, W^K, W^V, W^O\) and/or FFN matrices.

from peft import LoraConfig, get_peft_model
cfg = LoraConfig(r=16, lora_alpha=32, target_modules=["q_proj","v_proj"],
                 lora_dropout=0.05, bias="none", task_type="CAUSAL_LM")
model = get_peft_model(base_model, cfg)
model.print_trainable_parameters()

10.2.3 QLoRA — Dettmers et al. 2023

Quantize the base model to 4-bit (NF4 — NormalFloat4, optimal for normally-distributed weights) and train LoRA adapters in fp16/bf16 on top. Adds:

  • Double quantization of the quantization constants themselves.
  • Paged optimizers to spill optimizer state to CPU on memory spikes.

Result: fine-tune 65B on a single 48 GB GPU.

10.2.4 Other PEFT Methods

  • Prefix Tuning (Li & Liang 2021) — prepend trainable “virtual tokens” to keys/values at every layer.
  • Prompt Tuning (Lester et al. 2021) — prepend trainable virtual tokens only at the input embedding; works best for large models.
  • P-Tuning v2 (Liu et al. 2021) — prefix tuning across all layers, with re-parameterization.
  • IA³ (Liu et al. 2022) — multiplicatively rescale activations with learned vectors. Very few params.
  • BitFit (Ben-Zaken et al. 2021) — only train biases. Strong baseline.

MODULE 10.3: Instruction Tuning & SFT

Supervised Fine-Tuning (SFT) on (instruction, response) pairs teaches a base LM to follow instructions.

Public datasets:

  • FLAN / FLAN-T5 / FLAN-Collection — instruction templates over hundreds of NLP tasks.
  • Self-Instruct (Wang et al. 2022) — bootstrap instructions from a teacher LLM.
  • Alpaca (52k examples generated by text-davinci-003).
  • Dolly, OpenAssistant (OASST), UltraChat, WizardLM, Orca.

Templates: ChatML (<|im_start|>system ... <|im_end|>), LLaMA-2 chat ([INST] ... [/INST]), Alpaca (### Instruction: ... ### Response:).

MODULE 10.4: RLHF — Ouyang et al. (InstructGPT) 2022

Three stages:

  1. SFT — fine-tune base on human demonstrations.
  2. Reward model — train \(r_\phi(x, y)\) on pairwise human preferences with the Bradley-Terry loss:

\[ \mathcal{L}_{\text{RM}} = -\mathbb{E}_{(x, y_w, y_l) \sim D}\!\left[ \log \sigma\big( r_\phi(x, y_w) - r_\phi(x, y_l) \big) \right]. \]

  1. PPO (Schulman et al. 2017) optimizes the policy \(\pi_\theta\) against the RM, with a per-token KL penalty from the SFT reference \(\pi_\text{ref}\) to prevent reward hacking:

\[ \max_\theta \mathbb{E}_{x \sim D, y \sim \pi_\theta}\!\left[ r_\phi(x, y) - \beta \, \mathrm{KL}\!\big[\pi_\theta(\cdot|x) \,\|\, \pi_\text{ref}(\cdot|x)\big] \right]. \]

PPO uses the clipped surrogate objective

\[ \mathcal{L}^{\text{CLIP}} = \mathbb{E}\!\left[ \min\!\big(r_t(\theta) \hat{A}_t,\; \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_t \big) \right], \]

with \(r_t(\theta) = \pi_\theta(a_t|s_t) / \pi_{\theta_\text{old}}(a_t|s_t)\) and advantage \(\hat{A}_t\).

RLAIF (Bai et al. 2022, “Constitutional AI”) replaces human preferences with AI-generated ones.

MODULE 10.5: DPO — Rafailov et al. 2023

Direct Preference Optimization avoids the RM and PPO entirely. Algebraic trick: the closed-form optimum of the RLHF objective is

\[ \pi^*(y|x) \propto \pi_\text{ref}(y|x) \exp\!\left( \frac{1}{\beta} r(x, y) \right), \]

so \(r(x, y) = \beta \log \frac{\pi^*(y|x)}{\pi_\text{ref}(y|x)} + \text{const}(x)\). Plug into Bradley-Terry:

\[ \mathcal{L}_\text{DPO} = -\mathbb{E}_{(x, y_w, y_l)}\!\left[ \log \sigma\!\left( \beta \log\frac{\pi_\theta(y_w|x)}{\pi_\text{ref}(y_w|x)} - \beta \log\frac{\pi_\theta(y_l|x)}{\pi_\text{ref}(y_l|x)} \right) \right]. \]

A simple cross-entropy on log-prob differences; no rollouts. Variants:

  • IPO (Azar et al.) — fixes DPO’s tendency to overfit pure preferences.
  • KTO (Ethayarajh) — uses Kahneman-Tversky utility; needs only binary good/bad signal.
  • ORPO (Hong et al.) — combines SFT and odds-ratio preference in one stage; no reference model.
  • SimPO — reference-free, length-normalized.
  • GRPO (DeepSeekMath / R1) — group-relative policy optimization for math/code RL.

PART 11: GENERATION STRATEGIES

Companion: Jay Alammar — “How GPT3 Works — Visualizations and Animations” and Lilian Weng — “Controllable Neural Text Generation”.

MODULE 11.1: Decoding Algorithms

11.1.1 Greedy

\(y_t = \arg\max_w p(w | y_{<t}, x)\). Cheap, deterministic, locally optimal. Tends to be repetitive and bland.

11.1.2 Beam Search

Maintain top-\(k\) partial hypotheses by cumulative log-prob. Standard for translation and summarization. Pitfalls: bland, repetitive (“the the the”), and biased toward shorter sequences.

  • Length penalty (Wu et al. 2016): divide by \(((5 + |y|)/6)^\alpha\) with \(\alpha \in [0.6, 1.0]\).
  • Coverage penalty to discourage repeating same source positions in MT.
  • No-repeat-n-gram blocking: forbid generating any n-gram already produced.
  • Diverse beam search (Vijayakumar et al.): split beam into groups with a diversity penalty.

11.1.3 Sampling

  • Pure sampling: \(w \sim p(\cdot | y_{<t})\). High diversity, low coherence.
  • Temperature: \(p_T(w) \propto p(w)^{1/T}\). \(T < 1\) sharpens (more greedy-like), \(T > 1\) flattens, \(T = 0\) is greedy.
  • Top-k (Fan et al. 2018): renormalize over the \(k\) most probable tokens.
  • Top-p / nucleus (Holtzman et al. 2019): smallest set whose probabilities sum to \(\ge p\); renormalize. Adapts the set size to entropy of the distribution.
  • Typical sampling (Meister et al.): sample tokens with probability close to expected information content.
  • Min-p sampling: only tokens with probability above \(p \cdot \max p\) are kept.
  • Repetition penalty (Keskar et al. 2019 — CTRL): divide the logit of recently seen tokens by \(\rho > 1\).

11.1.4 Constrained Decoding

  • Beam search with constraints (forbidden / required tokens).
  • Grammar-constrained decoding (e.g., llama.cpp JSON grammar, outlines).
  • Logit biasing (OpenAI API logit_bias).
  • Guided decoding with regex / DFA.

11.1.5 Speculative Decoding

Run a small draft model to propose \(k\) tokens; the target model verifies them in a single forward pass and accepts a prefix. 2–3× speedup with no quality loss (Leviathan et al. 2023; Chen et al. 2023). Variants: Medusa, EAGLE, lookahead decoding.

PART 12: EVALUATION

MODULE 12.1: Classification & Tagging Metrics

Already covered: accuracy, precision, recall, F1, macro/micro/weighted, MCC, ROC-AUC, PR-AUC. For sequence tagging, evaluate at the span/entity level with seqeval.

MODULE 12.2: Machine Translation — BLEU

BLEU (Papineni et al. 2002) — modified n-gram precision with brevity penalty.

  1. Modified n-gram precision: count clipped matches (each reference n-gram contributes at most its reference count).

\[ p_n = \frac{\sum_{C} \sum_{\text{n-gram} \in C} \text{Count}_\text{clip}(\text{n-gram})}{\sum_{C} \sum_{\text{n-gram} \in C} \text{Count}(\text{n-gram})}. \]

  1. Brevity Penalty to discourage too-short outputs:

\[ \text{BP} = \begin{cases} 1 & c > r \\ \exp(1 - r/c) & c \le r \end{cases} \]

with \(c\) = candidate length, \(r\) = closest-reference length.

  1. BLEU-N (usually \(N=4\)):

\[ \text{BLEU} = \text{BP} \cdot \exp\!\left( \sum_{n=1}^{N} w_n \log p_n \right), \quad w_n = 1/N. \]

Important caveats:

  • BLEU is computed on a corpus, not single sentences (sentence-BLEU is noisy).
  • BLEU between systems is comparable only with the same tokenizer; sacreBLEU standardizes this.
  • BLEU correlates weakly with human judgment for high-quality MT; modern metrics (COMET, chrF) are preferred.

MODULE 12.3: Summarization — ROUGE (Lin 2004)

  • ROUGE-N: n-gram recall of reference n-grams found in the candidate.
  • ROUGE-L: longest common subsequence (LCS) based F-measure.
  • ROUGE-Lsum: applies LCS per sentence, sums and normalizes.
  • ROUGE-W: weighted LCS, favors consecutive matches.

\[ \text{ROUGE-N}_\text{recall} = \frac{\sum_{S \in R} \sum_{g_n \in S} \text{Count}_\text{match}(g_n)}{\sum_{S \in R} \sum_{g_n \in S} \text{Count}(g_n)}. \]

By default ROUGE is recall-oriented; F-measure variants combine recall and precision.

from datasets import load_metric
rouge = load_metric("rouge")
rouge.compute(predictions=["the cat sat"], references=["a cat sat on the mat"])

MODULE 12.4: Other Generation Metrics

  • METEOR (Banerjee & Lavie) — unigram F-mean with stemming, synonymy (WordNet), word order penalty. Better human correlation than BLEU.
  • chrF / chrF++ (Popović) — character n-gram F-score; robust across languages, no tokenizer dependency.
  • TER — Translation Edit Rate (insertions + deletions + substitutions + shifts).
  • BERTScore (Zhang et al. 2020) — token-level cosine similarity between candidate and reference BERT embeddings; precision/recall/F1 over best matches.
  • MoverScore — earth-mover’s distance over BERT embeddings.
  • BLEURT (Sellam et al.) — learned BLEU; BERT fine-tuned on synthetic data + human ratings.
  • COMET (Rei et al. 2020) — cross-lingual reference-based; trained on Direct Assessment.
  • GEMBA / G-Eval / LLM-as-a-judge — use GPT-4 etc. to rate outputs (faster than humans, must be calibrated against humans).

MODULE 12.5: Language Modeling — Perplexity

\[ \text{PPL}(W) = \exp\!\left( -\frac{1}{N} \sum_{i=1}^{N} \log p(w_i | w_{<i}) \right) = \exp(\text{cross-entropy}). \]

Lower = better. Caveats:

  • Comparable only when tokenization is the same.
  • Drops automatically when vocabulary shrinks (use bits-per-byte instead for cross-tokenizer comparison).
  • Token-level PPL does not predict downstream task performance perfectly.

MODULE 12.6: Retrieval & Ranking

  • MRR — Mean Reciprocal Rank: \(\frac{1}{Q} \sum_i \frac{1}{\text{rank}_i}\).
  • Recall@k — fraction of queries whose relevant doc is in top-k.
  • Precision@k, MAP@k — averaged precision.
  • NDCG@k — Normalized Discounted Cumulative Gain:

\[ \text{DCG@k} = \sum_{i=1}^{k} \frac{2^{\text{rel}_i} - 1}{\log_2(i + 1)},\quad \text{NDCG@k} = \text{DCG@k} / \text{IDCG@k}. \]

MODULE 12.7: QA, Reasoning, Code

  • Exact Match (EM) and token-level F1 (SQuAD).
  • HumanEval / MBPP: pass@k = expected probability that at least one of \(k\) samples passes all unit tests. Unbiased estimator: \(\text{pass@k} = 1 - \binom{n - c}{k} / \binom{n}{k}\) with \(n\) samples, \(c\) correct.
  • MMLU / BBH / ARC / GSM8K / MATH — accuracy on multi-choice or numeric answers.

MODULE 12.8: Error Analysis & Behavioral Testing

  • Confusion matrix dive — which class pairs collapse?
  • Slice analysis — break F1 by entity length, domain, capitalization, sequence length.
  • SliceLine / DOMINO — automated slice discovery.
  • CheckList (Ribeiro et al. 2020) — behavioral tests:
    • MFT (Minimum Functionality Test) — simple sanity (negation flips sentiment).
    • INV (Invariance) — paraphrase shouldn’t change label.
    • DIR (Directional Expectation) — adding “not” should flip sentiment.
  • Contrast sets (Gardner et al. 2020) — manually perturbed test examples.
  • Adversarial sets — ANLI, AdvGLUE, HANS — diagnostic for shortcut learning.

PART 13: INTERPRETABILITY FOR NLP

Companion: Lilian Weng — “Attention? Attention!”, Jay Alammar — “The Illustrated Transformer” for visual intuition, and the Anthropic Transformer Circuits Thread for mechanistic interpretability.

MODULE 13.1: Attention Visualization

Attention weights \(\alpha_{ij}\) are often displayed as heatmaps or token-to-token arcs (BertViz). They show what the model attends to.

Important debate:

  • “Attention is not Explanation” (Jain & Wallace 2019) — attention weights are not unique; alternate attentions can produce the same prediction. Therefore attention is not a faithful explanation.
  • “Attention is not not Explanation” (Wiegreffe & Pinter 2019) — counter-argument: alternate attentions are hard to find without breaking the model, so the original attention does explain the model as trained.

Practical advice: treat attention as a hypothesis-generating view, not a faithful explanation. Validate with perturbation, ablation, or attribution.

MODULE 13.2: Gradient-Based Attribution

  • Vanilla gradient \(\partial f(x) / \partial x_i\). Noisy.
  • Input × Gradient \(x_i \cdot \partial f / \partial x_i\). A first-order Taylor expansion at zero.
  • SmoothGrad — average gradients over Gaussian-perturbed inputs.
  • Integrated Gradients (Sundararajan et al. 2017):

\[ \text{IG}_i(x) = (x_i - x'_i) \int_0^1 \frac{\partial f(x' + \alpha (x - x'))}{\partial x_i} d\alpha, \]

approximated by Riemann sum over \(\sim 50\) steps. Axioms: completeness (\(\sum_i \text{IG}_i = f(x) - f(x')\)), sensitivity, implementation invariance. Choosing a baseline \(x'\) (e.g., all-zero embeddings or a [PAD] sequence) matters.

  • Layer-wise Relevance Propagation (LRP) — propagate relevance backward through the network using conservation rules.
from captum.attr import IntegratedGradients
ig = IntegratedGradients(model)
attributions, delta = ig.attribute(inputs=input_ids, target=label,
                                   baselines=baseline_ids, return_convergence_delta=True)

MODULE 13.3: Perturbation Methods

  • LIME (Ribeiro et al. 2016) — locally fit a sparse linear model on perturbed inputs (words turned on/off). Easy to use, sometimes unstable.
  • SHAP for text (KernelSHAP, PartitionSHAP) — Shapley values over tokens or token groups.
  • Occlusion — drop or mask each token, measure change in output.

MODULE 13.4: Probing

Train a small classifier (a probe) on frozen representations to test what they encode (POS, dependency, NER, syntax tree depth, factual knowledge). Hewitt & Manning’s structural probe recovers dependency trees from BERT hidden states. Control tasks (Hewitt & Liang 2019) check that the probe isn’t learning the task itself.

MODULE 13.5: Mechanistic Interpretability

  • Circuit analysis (Anthropic Transformer Circuits) — find subnetworks responsible for specific behaviors (induction heads, indirect object identification).
  • Activation patching / causal tracing (Meng et al. ROME) — patch activations from a “clean” run into a “corrupt” run to localize where information lives.
  • Logit lens (nostalgebraist) — apply the unembedding directly to intermediate hidden states; visualize what the model “would say” at each layer.
  • Sparse autoencoders (Anthropic, OpenAI) — decompose activations into mono-semantic features.

PART 14: PRODUCTION — DRIFT, MONITORING, PRIVACY

MODULE 14.1: Monitoring NLP Systems in Production

Four kinds of drift to track:

  1. Data / input drift — the distribution of inputs changes. Detect with:
    • Population Stability Index (PSI) on summary stats (length, vocab coverage, language ID).
    • Kolmogorov–Smirnov / Anderson–Darling on continuous features.
    • Chi-square on categorical features.
    • Embedding drift — mean and covariance of input embeddings; MMD (Maximum Mean Discrepancy) on samples.
  2. Concept drift — the relationship \(P(y | x)\) changes (e.g., slang shifts, new product names, new entities). Detect via performance monitoring on labeled samples or proxy metrics (e.g., user thumbs-down rates).
  3. Output / prediction drift — distribution of predictions changes. Detect with histograms, KL divergence between current and reference output distributions.
  4. Latency / cost drift — token-per-second, p50/p95/p99, GPU utilization.

PSI:

\[ \text{PSI} = \sum_i (p_i - q_i) \ln(p_i / q_i), \]

over \(i\) bins of feature distribution. Rule of thumb: \(<0.1\) stable, \(0.1\)–\(0.25\) slight shift, \(>0.25\) major shift.

14.1.1 Detection Toolkits

  • Evidently AI, Arize, WhyLabs, Fiddler, NannyML — purpose-built model monitoring.
  • Alibi-Detect — drift, outlier, adversarial detection.
  • Prometheus + Grafana for metrics; OpenTelemetry for traces.

14.1.2 Retraining Strategies

  • Periodic retraining (weekly / monthly) on rolling windows.
  • Triggered retraining when drift exceeds a threshold or performance drops.
  • Online / incremental — partial fits on streaming data (river, vw).
  • Continual / lifelong learning — beware catastrophic forgetting; use rehearsal, EWC, LwF, or replay buffers.

14.1.3 Deployment Patterns

  • Shadow deployment — new model runs alongside old; outputs logged but not served.
  • Canary release — small % of traffic routed to new model; ramp up if healthy.
  • A/B testing — split traffic, measure metrics, statistical significance.
  • Blue-green — two identical envs, swap pointer.
  • Multi-armed bandits — adaptively route to best-performing model.
  • Feature flags for instant rollback.

14.2.2 Differential Privacy

A randomized algorithm \(\mathcal{A}\) is \((\varepsilon, \delta)\)-DP if for any neighboring datasets \(D, D'\) differing in one record, and any output set \(S\):

\[ P[\mathcal{A}(D) \in S] \le e^\varepsilon \cdot P[\mathcal{A}(D') \in S] + \delta. \]

In ML, DP-SGD (Abadi et al. 2016) clips per-example gradients to \(C\) and adds Gaussian noise \(\mathcal{N}(0, \sigma^2 C^2 I)\). Smaller \(\varepsilon\) = more privacy = more noise = worse utility. Track privacy budget with Rényi DP / moments accountant (Opacus, TensorFlow Privacy).

14.2.3 Federated Learning

Clients train locally on private data; only model updates are sent to a server. FedAvg (McMahan et al. 2017) averages updates. Combine with secure aggregation and DP for stronger guarantees.

14.2.4 Model & Data Security

  • Encryption — TLS in transit; KMS-managed keys at rest.
  • Access control — IAM, least privilege, audit logs.
  • Secret scanning — trufflehog, gitleaks in CI.
  • Prompt-injection defenses — input filtering, output filtering, defensive prompts, allowlists, tool-use sandboxing.
  • Jailbreak detection — LlamaGuard, Prompt Guard, output classifiers.
  • Model extraction defenses — rate limiting, output watermarking, query-cost-aware throttling.
  • Membership inference — DP and regularization reduce leakage.

MODULE 15.1: Where Bias Enters

Stage Mechanism
Data collection Over-/under-representation of demographics
Annotation Annotator demographics, prescriptive vs descriptive labels
Preprocessing Stripping dialectal features, name normalization
Modeling Capacity differences across groups, shortcut learning
Decoding Mode collapse to majority
Evaluation Benchmarks underrepresenting minorities
Deployment Feedback loops

MODULE 15.2: Diagnosing Bias in Embeddings & Representations

  • Analogy probing (Bolukbasi 2016): “man : programmer :: woman : ?” — uncovered “homemaker” in word2vec.
  • WEAT (Word Embedding Association Test) (Caliskan et al. 2017) — IAT analog: compute mean cosine of target words (“programmer”, “engineer”) with attribute words (“man”, “he”) vs (“woman”, “she”). Permutation test for significance.
  • SEAT (May et al. 2019) — sentence-level WEAT.
  • StereoSet (Nadeem et al.) and CrowS-Pairs (Nangia et al.) — pairs designed to reveal stereotype preferences.
  • WinoBias / WinoGender (Zhao et al. 2018) — coreference resolution test for gender bias around occupations.
  • BBQ (Parrish et al.) — bias benchmark for QA.
  • Distributional fairness — measure performance gaps across demographic slices (gender, race, dialect, country).

MODULE 15.3: Bias Mitigation

15.3.1 Pre-processing

  • Balanced sampling / reweighing to equalize representation.
  • Counterfactual Data Augmentation (CDA) — swap gendered words (“he/she”, “him/her”, “doctor/nurse”) and re-train.
  • Data cleaning — remove or rewrite biased sources; document-level filtering.
  • Synthetic data — generate examples to fill demographic gaps.

15.3.2 In-processing

  • Hard debiasing (Bolukbasi 2016) — identify a “gender direction” via PCA on gendered pairs, project gender-neutral words orthogonal to it. Caveat: Gonen & Goldberg (2019) show bias persists in clustering.
  • Soft debiasing — minimize gender direction component while preserving similarity.
  • Iterative Null-space Projection (INLP) (Ravfogel et al. 2020) — train a probe to predict the protected attribute; project representations onto its null-space; repeat until probes fail.
  • Adversarial debiasing (Zhang et al. 2018) — predictor + adversary trying to recover protected attribute; encoder trained to fool adversary.
  • Fair regularization — add penalty for performance gap across groups.

15.3.3 Post-processing

  • Calibrated equalized odds — adjust thresholds per group.
  • Reject option classification — defer to human when near decision boundary for protected group.
  • Output rewriting — paraphraser to neutralize biased generations.

15.3.4 Fairness Metrics

  • Demographic parity (statistical parity): \(P(\hat{Y} = 1 | A = 0) = P(\hat{Y} = 1 | A = 1)\).
  • Equalized odds: \(P(\hat{Y} = 1 | Y = y, A = 0) = P(\hat{Y} = 1 | Y = y, A = 1)\) for \(y \in \{0, 1\}\).
  • Equal opportunity: equalized odds restricted to \(y = 1\).
  • Predictive parity / calibration: \(P(Y = 1 | \hat{Y} = 1, A = a)\) equal across groups.
  • Counterfactual fairness (Kusner et al.) — predictions same under counterfactual change of protected attribute.
  • Impossibility theorem (Chouldechova 2017; Kleinberg et al. 2016): except in degenerate cases, calibration, equal FPR, and equal FNR cannot all hold simultaneously across groups.

MODULE 15.4: LLM-Specific Harms & Mitigations

  • Toxicity / hate — RealToxicityPrompts, Perspective API, ToxiGen. Mitigations: data filtering, RLHF, post-hoc classifier, system prompts.
  • Hallucination — retrieval-augmented generation (RAG), uncertainty calibration, “I don’t know” training, citations, self-consistency.
  • Misinformation — RAG, fact-checking, watermarking, content provenance (C2PA).
  • Privacy leakage — Carlini et al. show LMs memorize training data; mitigate with deduplication, DP-SGD, filtering, output classifier.
  • Adversarial attacks — see Lilian Weng’s “Adversarial Attacks on LLMs”. Categories:
    • Jailbreaks — Do-Anything-Now, persona injection, role-play.
    • Prompt injection — direct (user input) and indirect (poisoned tool output / RAG corpus).
    • Token smuggling / encoding attacks — base64, leet-speak, low-resource languages.
    • Multi-turn escalation.
    • Defenses — output safety classifier, dual-LLM (privileged + unprivileged), structured outputs, tool sandboxing, “spotlighting” (mark untrusted content), allowlists, content filters.
  • Provenance & accountability — model cards (Mitchell et al. 2019), system cards, third-party audits, red-teaming.

APPENDIX B: Cheat-Sheet — Equations You Should Have in Muscle Memory

  • TF-IDF: \(\text{tf-idf}(t, d) = \text{tf}(t, d) \cdot \log\frac{N}{\text{df}(t)}\).
  • BM25: \(\sum_t \text{idf}(t) \cdot \frac{\text{tf}(t,d)(k_1 + 1)}{\text{tf}(t,d) + k_1 (1 - b + b |d|/\text{avgdl})}\).
  • PMI: \(\log P(x, y) / (P(x) P(y))\).
  • Skip-Gram NS: \(\log \sigma(v'_c \cdot v_w) + \sum_{k} \mathbb{E}_{w_k \sim P_n}[\log \sigma(-v'_{w_k} \cdot v_w)]\).
  • GloVe: \(J = \sum f(X_{ij})(w_i^\top \tilde w_j + b_i + \tilde b_j - \log X_{ij})^2\).
  • Scaled Dot-Product Attention: \(\text{softmax}(QK^\top / \sqrt{d_k}) V\).
  • LoRA: \(W' = W + \frac{\alpha}{r} BA\).
  • Cross-Entropy: \(-\sum_i y_i \log \hat y_i\).
  • Perplexity: \(\exp(\text{cross-entropy})\).
  • BLEU: \(\text{BP} \cdot \exp(\sum_n w_n \log p_n)\).
  • Viterbi: \(v_t(j) = \max_i v_{t-1}(i) a_{ij} b_j(o_t)\).
  • DPO: \(-\log \sigma(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_\text{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_\text{ref}(y_l|x)})\).
  • PSI: \(\sum_i (p_i - q_i) \ln(p_i / q_i)\).
  • WEAT effect size: \(\frac{\text{mean}_{x \in X} s(x, A, B) - \text{mean}_{y \in Y} s(y, A, B)}{\text{std-dev}_{w \in X \cup Y} s(w, A, B)}\).

from datasets import load_dataset
from transformers import (AutoTokenizer, AutoModelForTokenClassification,
                          TrainingArguments, Trainer, DataCollatorForTokenClassification)
from seqeval.metrics import f1_score, classification_report
import numpy as np

ds = load_dataset("conll2003")
labels = ds["train"].features["ner_tags"].feature.names
id2label = dict(enumerate(labels)); label2id = {l:i for i,l in id2label.items()}

tok = AutoTokenizer.from_pretrained("bert-base-cased")
def align(batch):
    enc = tok(batch["tokens"], truncation=True, is_split_into_words=True)
    new_labels = []
    for i, lab in enumerate(batch["ner_tags"]):
        word_ids = enc.word_ids(batch_index=i)
        ids = []
        prev = None
        for wid in word_ids:
            if wid is None: ids.append(-100)
            elif wid != prev: ids.append(lab[wid])
            else: ids.append(lab[wid] if id2label[lab[wid]].startswith("I-") else -100)
            prev = wid
        new_labels.append(ids)
    enc["labels"] = new_labels
    return enc

ds = ds.map(align, batched=True, remove_columns=ds["train"].column_names)

model = AutoModelForTokenClassification.from_pretrained(
    "bert-base-cased", num_labels=len(labels), id2label=id2label, label2id=label2id)

def metrics(p):
    preds = np.argmax(p.predictions, axis=2)
    true_labels = [[id2label[l] for l in lab if l != -100] for lab in p.label_ids]
    true_preds = [[id2label[pi] for pi, li in zip(pl, ll) if li != -100]
                  for pl, ll in zip(preds, p.label_ids)]
    return {"f1": f1_score(true_labels, true_preds)}

args = TrainingArguments(output_dir="ner-bert", learning_rate=3e-5, num_train_epochs=3,
                         per_device_train_batch_size=16, eval_strategy="epoch")
trainer = Trainer(model=model, args=args, train_dataset=ds["train"],
                  eval_dataset=ds["validation"], tokenizer=tok,
                  data_collator=DataCollatorForTokenClassification(tok),
                  compute_metrics=metrics)
trainer.train()

D.6 Drift Monitoring Skeleton

import numpy as np
from scipy.stats import ks_2samp

def psi(reference, current, bins=10):
    breakpoints = np.linspace(0, 1, bins + 1)
    breakpoints[0], breakpoints[-1] = -np.inf, np.inf
    ref_hist, _ = np.histogram(reference, breakpoints)
    cur_hist, _ = np.histogram(current, breakpoints)
    ref_pct = ref_hist / max(ref_hist.sum(), 1)
    cur_pct = cur_hist / max(cur_hist.sum(), 1)
    eps = 1e-6
    return np.sum((cur_pct - ref_pct) * np.log((cur_pct + eps) / (ref_pct + eps)))

ref_lengths = np.random.normal(120, 30, 5000)        # training input lengths
prod_lengths = np.random.normal(140, 40, 1000)       # production input lengths
print("PSI:", psi(ref_lengths / 300, prod_lengths / 300))
print("KS p-value:", ks_2samp(ref_lengths, prod_lengths).pvalue)

APPENDIX G: Closing Notes

NLP is the broadest applied ML field — it spans linguistics, information retrieval, statistics, deep learning, systems, ethics, and law. The fastest way to operate at expert level is to think in pipelines: data sourcing → preprocessing → representation → modeling → decoding → evaluation → deployment → monitoring → mitigation. Each stage is a separate research literature, and each is testable.

When in doubt during the assessment:

  • Pick the most modern, widely deployed option unless the prompt restricts you (e.g., “without neural nets”).
  • For tagging / classification — encoder fine-tuning.
  • For generation — decoder LLM with appropriate decoding strategy.
  • For evaluation — pick the metric calibrated to the task (F1 for tagging, BLEU/ROUGE/BERTScore for generation, exact-match for QA, MRR/NDCG for retrieval, perplexity for LM).
  • For fairness — name the metric and the mitigation stage (pre / in / post-processing).
  • For production — name the drift type and a detector.

Good luck.
