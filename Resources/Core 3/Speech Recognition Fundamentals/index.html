Automatic Speech Recognition — From Waveforms to Words

A self-contained, end-to-end study companion for the Workera ELO / SAGE assessment in Speech Recognition Fundamentals. The goal is to move you from “I have heard of MFCCs” to “I can confidently reason about every block of a modern streaming Conformer-Transducer”. References used liberally throughout: Jurafsky & Martin SLP3 (ch. 16 ASR), Lilian Weng’s Speech Recognition notes, Mohri’s WFST tutorial, Povey et al. Kaldi, Graves CTC, Bahdanau Attention, Chan LAS, Gulati Conformer, Baevski wav2vec 2.0, Hsu HuBERT, Radford Whisper, Zhang USM, Chiu SOTA seq2seq ASR, and Jay Alammar’s Illustrated Transformer.

PART 1: AUDIO FUNDAMENTALS & SIGNAL PROCESSING

MODULE 1: FROM SOUND PRESSURE TO DIGITAL SAMPLES

1.1 What “speech” actually is

A speech signal is a continuous-time pressure variation \(p(t)\) that the vocal apparatus (lungs → vocal folds → vocal tract → lips/nostrils) imprints on the surrounding air. Two complementary views matter for ASR:

  1. Source–filter view (Fant 1960): an excitation source (periodic glottal pulses for voiced sounds, turbulent noise for unvoiced) is shaped by a linear time-varying filter (vocal tract resonances → formants).
  2. Phonetic view: the signal is a sequence of phonemes, syllables, words. ASR must invert the production process to recover symbolic units from the acoustic signal.

Speech is non-stationary globally (the vocal tract reconfigures every ~30–50 ms) but approximately stationary on short windows (~20–30 ms). This single observation justifies almost the entire short-time analysis pipeline used in ASR.

1.2 Sampling: from analog to digital

Recording a microphone signal yields a digital sequence \(x[n] = p(nT_s)\) obtained by sampling the analog waveform at rate \(f_s = 1/T_s\).

Nyquist–Shannon theorem. A bandlimited signal with maximum frequency \(f_{\max}\) can be perfectly reconstructed if \(f_s > 2 f_{\max}\). Sampling below this rate produces aliasing — energy above \(f_s/2\) folds back into the audible band, creating phantom tones.

Practical sample rates for speech:

Sample rate Nyquist Typical use Notes
8 kHz 4 kHz Narrowband telephony (G.711, GSM) Loses fricative energy above 4 kHz, hurts /s/ vs /f/ discrimination
16 kHz 8 kHz Standard ASR sample rate Captures essentially all phonetic content; default for Kaldi, Whisper
22.05 kHz 11.025 kHz Half CD quality, some TTS
44.1 kHz / 48 kHz 22.05 / 24 kHz CD / professional audio Downsampled before ASR; wastes compute otherwise

For ASR you should think in 16 kHz / 16-bit PCM mono by default. Re-sampling to 16 kHz with a proper anti-aliasing low-pass filter is one of the first preprocessing steps.

Bit depth and quantization. Each sample is quantized to \(B\) bits, giving \(2^B\) levels. The signal-to-quantization-noise ratio (SQNR) for a sinusoid filling the full scale is approximately

\[ \text{SQNR}_{\mathrm{dB}} \approx 6.02 B + 1.76. \]

So 16-bit PCM gives ~96 dB SQNR — plenty for speech, where the dynamic range of interest is ~60 dB. 8-bit PCM gives only ~50 dB, audibly degrading consonants. Floating-point WAV (32-bit float) is common in research code; production usually stores int16.

Dynamic range and clipping. A clipped sample is irrecoverable: the model sees a flat plateau at \(\pm 1\). Always check max(abs(x)) and normalize to a safe headroom (e.g. −3 dBFS) before feature extraction. Conversely, recordings that occupy only a small fraction of the range are dominated by quantization noise.

Dithering. Adding a small triangular noise (TPDF dither, ±1 LSB) before quantization decorrelates the quantization error from the signal. For 16-bit speech this is rarely necessary but matters when re-quantizing intermediate buffers to 8 bits.

PCM, ADPCM, μ-law / A-law.

  • PCM (Pulse-Code Modulation): linear quantization of the waveform.
  • μ-law / A-law (G.711): logarithmic 8-bit companding used in telephony — finer resolution near zero, coarse at extremes. Models 8-bit codec data better than naive linear 8-bit.
  • ADPCM: adaptive differential PCM (G.726). Encodes prediction residuals; 4 bits/sample at 32 kbps.
  • Lossy codecs (MP3, AAC, Opus): introduce time-frequency artefacts; a model trained on clean PCM degrades on lossy data unless trained with codec augmentation.

1.3 DC offset, pre-emphasis, dithering

DC removal. A constant offset \(\mu\) in the waveform shifts the spectrum at DC but contains no useful information. Subtract the mean over the utterance (or apply a high-pass filter at ~30 Hz). DC components inflate the energy feature and confuse downstream models.

Pre-emphasis. Speech spectra fall off by roughly \(-6\) dB/octave above ~500 Hz (radiation from the mouth and glottal source roll-off). To make high frequencies competitive with low frequencies for fixed-precision arithmetic, classical ASR applies a one-pole high-pass filter:

\[ y[n] = x[n] - \alpha x[n-1], \qquad H(z) = 1 - \alpha z^{-1}, \quad \alpha \approx 0.97. \]

The frequency response is \(|H(e^{j\omega})| = \sqrt{1 + \alpha^2 - 2\alpha\cos\omega}\), an approximately first-order high-pass. Modern neural front-ends usually drop pre-emphasis (the network learns equivalent weighting in early layers), but Kaldi MFCC pipelines still apply it.

import numpy as np
def preemphasis(x, alpha=0.97):
    return np.append(x[0], x[1:] - alpha * x[:-1])

MODULE 2: FILTERS AND THE FREQUENCY DOMAIN

2.1 Filter taxonomy

Filter type Passes Common ASR role
Low-pass \(f < f_c\) Anti-aliasing before downsampling
High-pass \(f > f_c\) DC removal, pre-emphasis, rumble rejection
Band-pass \(f_1 < f < f_2\) Telephony band-limiting, formant tracking
Band-stop / notch reject narrow band 50/60 Hz mains hum, fixed interferers
All-pass all magnitudes preserved Phase shaping, dereverberation post-processing
Mel filterbank overlapping triangular band-pass Feature extraction (Module 5)
Gammatone filterbank cochlear-shaped Auditory-front-end (CARFAC, PNCC)

A digital filter \(H(z) = \frac{\sum_k b_k z^{-k}}{1 + \sum_k a_k z^{-k}}\) is FIR if all \(a_k = 0\) (only feed-forward taps) and IIR otherwise. FIRs are always stable and have linear phase if symmetric; IIRs are more efficient for the same magnitude response but can be unstable and have non-linear phase.

2.2 Anti-aliasing filter

When downsampling from \(f_s\) to \(f_s'\) (e.g. 48 kHz → 16 kHz) you must apply a low-pass at \(f_s'/2\) first; otherwise aliases fold into the new band. Tools like librosa.resample, torchaudio.functional.resample, sox, and ffmpeg do this internally with a windowed-sinc kernel.

2.3 The Fourier viewpoint

Speech analysis lives in the frequency domain because the auditory system itself performs a (warped) Fourier-like decomposition along the basilar membrane.

  • Continuous Fourier transform: \(X(f) = \int_{-\infty}^{\infty} x(t) e^{-j 2\pi f t}\, dt\).
  • Discrete-Time Fourier Transform (DTFT): \(X(e^{j\omega}) = \sum_{n=-\infty}^{\infty} x[n] e^{-j\omega n}\).
  • Discrete Fourier Transform (DFT): \(X[k] = \sum_{n=0}^{N-1} x[n]\, e^{-j 2\pi k n / N}\), \(k = 0,\dots,N-1\).
  • Fast Fourier Transform (FFT): an \(O(N \log N)\) algorithm for the DFT. For \(N = 2^p\), Cooley–Tukey gives the textbook radix-2 form.

The DFT bin \(k\) corresponds to physical frequency \(f_k = k\, f_s / N\). With \(N = 512\) and \(f_s = 16\) kHz, bin spacing is \(\approx 31.25\) Hz.

Magnitude vs phase. \(X[k] = |X[k]| e^{j\angle X[k]}\). Almost all classical ASR features discard the phase — only \(|X[k]|\) enters the mel filterbank. Phase carries fine-grained temporal structure used by source-separation and some end-to-end models (SincNet, raw-waveform networks).

2.4 Short-Time Fourier Transform (STFT)

Speech is non-stationary, so we cut it into short windows and Fourier-transform each:

\[ X[m, k] = \sum_{n=0}^{N-1} w[n]\, x[n + m H]\, e^{-j 2\pi k n / N}, \]

where \(w[n]\) is the analysis window, \(N\) the window length, and \(H\) the hop. The STFT magnitude \(|X[m,k]|\) visualised as a heatmap is the familiar spectrogram. Squaring gives the power spectrogram. Taking the log gives the log-spectrogram, the typical scale used for human inspection.

2.5 Aliasing diagnostic

A common ASR bug: a 16 kHz model fed 8 kHz audio that was naively upsampled by zero-stuffing develops a brick-wall above 4 kHz. Plot the log-spectrogram — a flat black band above \(f_s/2\) of the original recording is the smoking gun. Always log the original sample rate alongside features.

PART 2: FEATURE EXTRACTION

MODULE 3: FRAMING, WINDOWING, AND THE OVERLAP-ADD PIPELINE

3.1 Why frame the signal?

Speech is quasi-stationary on ~20–30 ms windows. Framing means slicing the waveform into overlapping segments of length \(N\) samples with hop \(H\) samples:

  • \(N = 25\) ms (e.g. 400 samples at 16 kHz)
  • \(H = 10\) ms (e.g. 160 samples at 16 kHz)
  • frame rate \(= 1/H = 100\) frames/sec

The hop sets the temporal resolution of downstream features (one feature vector per 10 ms is the de facto ASR standard). The window length sets the spectral resolution: larger \(N\) → finer frequency bins, but blurred time localisation. This is the Heisenberg-style time-frequency tradeoff.

3.2 Window functions

Applying a finite window to a signal multiplies its spectrum by the window’s Fourier transform, leaking energy from a tone into neighbouring bins. A good window has a narrow main lobe and low side lobes.

Rectangular window \(w[n] = 1\) — narrowest main lobe, awful side lobes (−13 dB). Almost never used for ASR.

Hamming window (the ASR default):

\[ w[n] = 0.54 - 0.46 \cos\!\left( \frac{2\pi n}{N-1} \right), \quad 0 \le n \le N-1. \]

Side lobes around −43 dB.

Hann (Hanning) window:

\[ w[n] = 0.5 \left( 1 - \cos\!\left( \frac{2\pi n}{N-1} \right) \right). \]

Side lobes ~−31 dB but rolls off faster; standard in many neural pipelines.

Other windows: Blackman (better sidelobe attenuation), Kaiser (tunable \(\beta\) trades main-lobe width vs side lobes), flat-top (for amplitude calibration, rare in ASR).

3.3 Zero-padding and FFT size

You choose an FFT size \(N_{\text{fft}} \geq N\), padding zeros if needed (and rounding up to a power of two for FFT speed). Zero-padding does not add frequency information — it only interpolates the same DTFT more finely. For ASR, \(N_{\text{fft}}\) is usually 512 for a 400-sample frame at 16 kHz.

3.4 Overlap-add (OLA) and COLA

For analysis-only pipelines (features → recognizer) you never reconstruct the signal, so overlap-add is irrelevant. For synthesis (TTS vocoders, source separation, dereverberation) you need the Constant Overlap-Add property of the window: \(\sum_m w[n - mH] = C\) (constant). Hann with 50% overlap satisfies COLA; Hamming does not perfectly. Stick to Hann if you ever need to invert the STFT.

3.5 Code: framing pipeline

import numpy as np
import librosa

x, sr = librosa.load("utt.wav", sr=16000, mono=True)
x = librosa.effects.preemphasis(x, coef=0.97)

n_fft   = 512
win_len = int(0.025 * sr)  # 400
hop_len = int(0.010 * sr)  # 160
window  = np.hamming(win_len)

S = librosa.stft(x, n_fft=n_fft, hop_length=hop_len, win_length=win_len,
                 window="hamming", center=True)
magnitude = np.abs(S)        # (n_fft/2 + 1, T)
power     = magnitude ** 2

MODULE 4: THE MEL FILTERBANK AND PERCEPTUAL SCALES

4.1 Why perceptual scales?

The human cochlea has a non-linear frequency resolution: roughly logarithmic above ~1 kHz, linear below. Two tones an octave apart at 200 Hz sound much further apart than two tones an octave apart at 8 kHz. Mel (from “melody”) is one of several scales that re-map the physical frequency axis to a perceptual one.

The Stevens, Volkmann, Newman (1937) approximation used by HTK/Kaldi:

\[ m = 2595 \log_{10}\!\left( 1 + \frac{f}{700} \right), \qquad f = 700\,(10^{m/2595} - 1). \]

So 1000 Hz ≈ 1000 mel, 4000 Hz ≈ 2146 mel, 8000 Hz ≈ 2840 mel.

Other auditory scales:

  • Bark scale (Zwicker, 1961): 24 critical bands; \(z = 13 \arctan(0.00076 f) + 3.5 \arctan((f/7500)^2)\).
  • ERB (Equivalent Rectangular Bandwidth, Glasberg & Moore): \(\text{ERB}(f) = 24.7\,(0.00437 f + 1)\).
  • Gammatone filters: model cochlear filter shapes; impulse response \(g(t) = t^{n-1} e^{-2\pi b t} \cos(2\pi f_c t)\).

4.2 Constructing the mel filterbank

  1. Pick number of filters \(M\) (typically 23, 40, 80, 128).
  2. Choose lower and upper frequencies \(f_{\text{lo}}\), \(f_{\text{hi}}\) (e.g. 0 Hz and \(f_s/2\) for Whisper; 20 Hz–7600 Hz for Kaldi narrowband).
  3. Convert to mel: \(m_{\text{lo}}, m_{\text{hi}}\). Compute \(M+2\) equally spaced points on the mel axis. Convert back to Hz, then to FFT bin indices.
  4. For each filter \(m \in \{1, \dots, M\}\) build a triangle that rises from bin \(f(m-1)\) to \(f(m)\) and falls to \(f(m+1)\):

\[ H_m[k] = \begin{cases} 0 & k < f(m-1) \\ \frac{k - f(m-1)}{f(m) - f(m-1)} & f(m-1) \le k \le f(m) \\ \frac{f(m+1) - k}{f(m+1) - f(m)} & f(m) \le k \le f(m+1) \\ 0 & k > f(m+1) \end{cases} \]

  1. Mel-filterbank energies:

\[ E[m] = \sum_{k=0}^{N_{\text{fft}}/2} |X[k]|^2 \, H_m[k]. \]

  1. Log-mel features: \(\text{LM}[m] = \log(E[m] + \epsilon)\).
import librosa, numpy as np
mel_fb = librosa.filters.mel(sr=16000, n_fft=512, n_mels=80, fmin=0, fmax=8000)
power_spec = np.abs(librosa.stft(x, n_fft=512, hop_length=160, win_length=400))**2
mel_spec   = mel_fb @ power_spec       # (80, T)
log_mel    = np.log(mel_spec + 1e-6)

4.3 fbank vs MFCC: which to choose?

Log-mel filterbank (fbank) MFCC
Dim typically 40–128 typically 13 (+Δ+Δ²)
Correlation bins are correlated DCT decorrelates → diagonal covariance OK
Best for DNN/CNN/Transformer models (they re-learn decorrelation) GMM-HMM (needs diagonal covariance)
Phase / fine structure preserves more spectral envelope detail discards higher cepstral coefficients

Rule of thumb: GMM-HMM → MFCC, DNN-HMM hybrid and end-to-end → log-mel fbank.

MODULE 5: MFCC IN FULL DETAIL

The classical pipeline used everywhere from Sphinx to Kaldi:

\[ \text{waveform} \xrightarrow{\text{pre-emph}} \xrightarrow{\text{frame+window}} \xrightarrow{|\text{DFT}|^2} \xrightarrow{\text{mel filterbank}} \xrightarrow{\log} \xrightarrow{\text{DCT}} \text{MFCC}. \]

5.1 Step-by-step

  1. Pre-emphasis. \(y[n] = x[n] - 0.97 x[n-1]\).
  2. Frame+window. 25 ms Hamming windows, 10 ms hop.
  3. Power spectrum. \(P[k] = \frac{1}{N} |X[k]|^2\).
  4. Mel filterbank energies. \(E[m] = \sum_k P[k] H_m[k]\).
  5. Log. \(L[m] = \log E[m]\).
  6. DCT-II. Decorrelates the log-energies:

\[ c_n = \sum_{m=1}^{M} L[m] \cos\!\left( \frac{\pi n (m - 0.5)}{M} \right), \quad n = 0, 1, \dots, M - 1. \]

  1. Keep first \(D\) coefficients (\(D = 13\) classically). Drop \(c_0\) (overall energy) or replace it with the log-frame-energy.

5.2 Why the DCT?

The DCT is the asymptotic Karhunen–Loève transform for log-spectra of speech — it concentrates almost all variance into the first ~12 coefficients. In a GMM-HMM, modelling 40 correlated log-mel features requires full covariance matrices (expensive, data-hungry). Modelling 13 decorrelated MFCCs with diagonal covariance works much better.

5.3 Liftering (cepstral lifter)

To rebalance the dynamic range of cepstral coefficients:

\[ \tilde{c}_n = c_n \left( 1 + \frac{L}{2} \sin\frac{\pi n}{L} \right), \quad L = 22. \]

Not deeply principled but mild improvement in HMM-GMM. Irrelevant for neural models.

5.4 Delta and delta-delta features

To inject temporal context, append first and second time derivatives:

\[ \Delta c_t = \frac{\sum_{n=1}^{\Theta} n (c_{t+n} - c_{t-n})}{2 \sum_{n=1}^{\Theta} n^2}, \quad \Theta = 2. \]

So a classical MFCC vector is 39-dim = 13 static + 13 Δ + 13 ΔΔ.

5.5 CMVN — Cepstral Mean (and Variance) Normalisation

Speaker and channel effects show up as a roughly constant offset in the cepstral domain (convolution in time domain ↔︎ addition in log-spectrum ↔︎ same offset in cepstrum). Removing the per-utterance (or per-speaker) mean approximates channel compensation:

\[ \hat{c}_t = \frac{c_t - \mu_c}{\sigma_c}. \]

CMVN can be applied offline (over the whole utterance) or online (running mean with exponential decay, important for streaming).

5.6 PLP and RASTA-PLP

PLP (Perceptual Linear Prediction; Hermansky 1990). Uses Bark filterbank + equal-loudness pre-emphasis + intensity-loudness power law + LP autoregression on the auditory spectrum. Slightly more robust than MFCC for some noisy conditions; identical pipeline shape.

RASTA-PLP. Adds a bandpass filter in the modulation-frequency domain (1–10 Hz) to suppress slowly-varying channel artefacts and fast spurious peaks. Especially helpful for telephone speech.

5.7 Pitch (F0), formants, voicing, energy, ZCR

Even modern systems sometimes append:

  • F0 (fundamental frequency): estimated by autocorrelation, YIN, CREPE. Crucial for tonal languages (Mandarin) and for speaker/emotion side-channels.
  • Formants (\(F_1, F_2, F_3\)): vocal-tract resonances. Often extracted via LPC root analysis.
  • Voicing indicator: binary or probabilistic flag.
  • Frame energy / RMS: \(E_t = \log \sum_n x[t,n]^2\).
  • Zero-Crossing Rate (ZCR): \(\text{ZCR}_t = \frac{1}{N-1}\sum_n \mathbb{1}[ \text{sign}(x[n]) \neq \text{sign}(x[n-1]) ]\). High ZCR → unvoiced/fricative; low ZCR → voiced.

5.8 Dynamic range, quantization, and feature extraction

Mistakes that quietly destroy features:

  • Clipped input → flattened spectrum → no high-frequency cues.
  • Re-quantizing log-mel to int8 for storage → coarse log-magnitude steps → confuses softer phones.
  • Saving features in dB without sane floor (max(log(E), -100)) → -inf from zero-energy bins → NaN gradients.
  • Forgetting CMVN with a model that was trained with CMVN → ~5× WER overnight.

PART 3: HMM/GMM CLASSICAL ASR

MODULE 6: HIDDEN MARKOV MODELS

6.1 What an HMM is

A Hidden Markov Model is a doubly-stochastic process: a finite-state Markov chain over latent states \(\{q_t\}\) emits observations \(\{\mathbf{o}_t\}\) at each step. The parameters are:

  • Initial state distribution \(\pi_i = P(q_1 = i)\).
  • Transition probabilities \(a_{ij} = P(q_{t+1} = j \mid q_t = i)\).
  • Emission distributions \(b_j(\mathbf{o}) = P(\mathbf{o} \mid q_t = j)\).

In ASR each phoneme (or sub-phone) is modelled by a small left-to-right HMM with typically 3 emitting states (begin, middle, end of phone). The transition matrix is highly structured: \(a_{ij} = 0\) for \(j < i\) and \(j > i+1\).

6.2 The three canonical problems (Rabiner 1989)

  1. Evaluation — given \(\lambda = (\pi, A, B)\) and an observation sequence \(\mathbf{O} = (\mathbf{o}_1, \dots, \mathbf{o}_T)\), compute \(P(\mathbf{O} \mid \lambda)\). Solved by the forward algorithm.
  2. Decoding — find the most likely hidden state sequence \(\arg\max_{\mathbf{Q}} P(\mathbf{Q} \mid \mathbf{O}, \lambda)\). Solved by Viterbi.
  3. Learning — estimate \(\lambda\) from data. Solved by Baum–Welch (an instance of EM).

6.3 Forward and backward variables

Forward variable:

\[ \alpha_t(i) = P(\mathbf{o}_1, \dots, \mathbf{o}_t, q_t = i \mid \lambda). \]

Recursion:

\[ \alpha_1(i) = \pi_i b_i(\mathbf{o}_1), \qquad \alpha_{t+1}(j) = \left[ \sum_i \alpha_t(i) a_{ij} \right] b_j(\mathbf{o}_{t+1}). \]

Termination: \(P(\mathbf{O} \mid \lambda) = \sum_i \alpha_T(i)\).

Backward variable:

\[ \beta_t(i) = P(\mathbf{o}_{t+1}, \dots, \mathbf{o}_T \mid q_t = i, \lambda), \]

with \(\beta_T(i) = 1\) and \(\beta_t(i) = \sum_j a_{ij} b_j(\mathbf{o}_{t+1}) \beta_{t+1}(j)\).

State posterior: \(\gamma_t(i) = \frac{\alpha_t(i)\beta_t(i)}{P(\mathbf{O}|\lambda)}\).

Transition posterior: \(\xi_t(i,j) = \frac{\alpha_t(i) a_{ij} b_j(\mathbf{o}_{t+1}) \beta_{t+1}(j)}{P(\mathbf{O}|\lambda)}\).

6.4 Baum–Welch (EM for HMMs)

Iterate:

  1. E-step. Compute \(\alpha, \beta, \gamma, \xi\) with current \(\lambda\).
  2. M-step. Re-estimate:

\[ \hat{\pi}_i = \gamma_1(i), \quad \hat{a}_{ij} = \frac{\sum_t \xi_t(i,j)}{\sum_t \gamma_t(i)}, \]

and for GMM emissions update mixture weights, means, covariances using the GMM E-step weighted by \(\gamma_t(i)\).

EM monotonically increases \(P(\mathbf{O} \mid \lambda)\) but converges to a local optimum. ASR practice: initialise with a flat-start (uniform alignment) or with an existing model, run several iterations.

6.5 Viterbi decoding

Find \(\arg\max_{\mathbf{Q}} P(\mathbf{O}, \mathbf{Q} \mid \lambda)\) in \(O(T \cdot |S|^2)\):

Initialise:  δ_1(i) = π_i b_i(o_1);   ψ_1(i) = 0
Recurse t=2..T:
    for each j:
        δ_t(j) = max_i [ δ_{t-1}(i) · a_{ij} ] · b_j(o_t)
        ψ_t(j) = argmax_i [ δ_{t-1}(i) · a_{ij} ]
Terminate:   P* = max_i δ_T(i);  q*_T = argmax_i δ_T(i)
Backtrack:   q*_t = ψ_{t+1}(q*_{t+1})

In ASR we work in log-domain to avoid underflow: replace multiplications with additions, \(\max\) stays \(\max\).

MODULE 7: GAUSSIAN MIXTURE MODELS AS EMISSIONS

7.1 Why GMMs?

Each HMM state needs a probability distribution over the 39-dim MFCC vectors that it could emit. A single Gaussian is too restrictive; a mixture of Gaussians is universal in the limit of many components and analytically convenient.

\[ b_j(\mathbf{o}) = \sum_{k=1}^{K} w_{jk}\, \mathcal{N}\!\left(\mathbf{o};\, \boldsymbol{\mu}_{jk},\, \boldsymbol{\Sigma}_{jk}\right), \qquad \sum_k w_{jk} = 1. \]

Typical sizes: \(K = 8\) to \(32\) Gaussians per state, diagonal covariance (cheap and well-conditioned given MFCC decorrelation).

7.2 Tri-phone state tying via decision trees

A naive system with \(\sim 40\) phones and 3 states each has 120 HMM states — too coarse. To capture coarticulation, ASR uses tri-phones: each phone in the context of its left and right neighbours, e.g. b-eh+t (the /eh/ between /b/ and /t/). 40³ = 64 000 tri-phones, many unseen.

Solution: phonetic decision tree clustering. For each centre phone + state position, train a binary decision tree that splits tri-phones by phonetic questions (“is left context a nasal?”, “is right context a front vowel?”). Each leaf of the tree shares one GMM. Result: ~5000 tied states (“senones”) that cover all tri-phones, including unseen ones via the tree.

7.3 Speaker adaptation: MLLR, MAP, fMLLR

  • MAP (Maximum A Posteriori) adaptation. Reestimate GMM means using speaker-specific data with a prior centred on the speaker-independent mean.
  • MLLR (Maximum Likelihood Linear Regression). Learn an affine transform \(\hat{\boldsymbol{\mu}} = A \boldsymbol{\mu} + \mathbf{b}\) applied to all means (or to features). Generalises across unseen states.
  • fMLLR / CMLLR (feature-space MLLR). Apply the affine transform to the features themselves, \(\hat{\mathbf{o}} = A \mathbf{o} + \mathbf{b}\), so any acoustic model can consume them. Standard speaker adaptation step in Kaldi recipes.
  • VTLN (Vocal Tract Length Normalisation). Warps the frequency axis per speaker to compensate for vocal tract length variation.

7.4 The Kaldi HMM-GMM ladder

A canonical Kaldi recipe (s5/run.sh) trains successively richer models, each used to bootstrap the next:

  1. mono — monophone GMM-HMM with flat-start alignment.
  2. tri1 — tri-phone GMM-HMM with delta features, decision-tree state tying.
  3. tri2b — tri-phone with LDA+MLLT (transform applied to spliced features).
  4. tri3b — adds SAT (Speaker Adaptive Training) with fMLLR.
  5. tri4b — bigger model, more Gaussians.
  6. nnet — DNN-HMM hybrid on top of tri3b alignments and features.

This ladder is still the reference for low-resource languages where end-to-end systems struggle.

MODULE 8: NOISE ROBUSTNESS FOR CLASSICAL ASR

8.1 Spectral subtraction (Boll, 1979)

Assume \(|Y[k]|^2 = |S[k]|^2 + |N[k]|^2\) (uncorrelated noise). Estimate noise spectrum \(\hat{N}[k]\) from non-speech segments (via VAD), then:

\[ |\hat{S}[k]|^2 = \max\!\left( |Y[k]|^2 - \alpha\, |\hat{N}[k]|^2,\, \beta\, |Y[k]|^2 \right). \]

The floor \(\beta\) (~0.01) prevents musical noise — isolated time-frequency tiles with spurious gains.

8.2 Wiener filter

Optimal linear MMSE estimator in the frequency domain:

\[ \hat{S}[k] = \frac{\mathrm{SNR}[k]}{1 + \mathrm{SNR}[k]} Y[k]. \]

Requires an a priori SNR estimate — usually via decision-directed (Ephraim–Malah) approach.

8.3 MMSE-STSA (Ephraim & Malah, 1984)

Estimate the short-time spectral amplitude rather than the complex spectrum, assuming Gaussian speech and noise. Gives the celebrated minimum-mean-square-error log-spectral amplitude estimator — perceptually cleaner than spectral subtraction.

8.4 Microphone array beamforming

With \(M\) microphones you can spatially focus on the target direction.

  • Delay-and-Sum. Align microphone signals by their geometric delays and sum.
  • MVDR (Minimum Variance Distortionless Response). Minimise output power subject to a fixed unit response in the look direction. Closed form involving the noise covariance matrix.
  • GEV (Generalised Eigenvalue) beamformer. Maximise output SNR; principal eigenvector of \(\Phi_S \Phi_N^{-1}\) (cross-power-spectral-density matrices of speech and noise).

8.5 Dereverberation

  • WPE (Weighted Prediction Error; Nakatani et al.). Linear prediction in STFT domain to cancel late reverberation.
  • Neural dereverb. SE U-Nets, MetricGAN, etc.

8.6 Multi-condition training and augmentation

Modern noise robustness is rarely a separate front-end; instead training data is augmented:

  • Add noise from MUSAN (Music, Speech, Noise) at random SNRs.
  • Convolve clean speech with simulated or measured Room Impulse Responses (RIRs, e.g. RIRS_NOISES).
  • Apply SpecAugment time/frequency masking on features (Module 25).
  • Speed perturbation (\(\pm 10\%\)) for further variability.

PART 4: LANGUAGE MODELING FOR ASR

MODULE 9: STATISTICAL LANGUAGE MODELS

9.1 The decomposition

ASR solves

\[ \hat{W} = \arg\max_W P(W \mid \mathbf{O}) = \arg\max_W \underbrace{P(\mathbf{O} \mid W)}_{\text{acoustic model}} \underbrace{P(W)}_{\text{language model}}. \]

The language model (LM) scores plausibility of word sequences and is what distinguishes “wreck a nice beach” from “recognise speech”.

9.2 n-gram models

\(P(W) = \prod_i P(w_i \mid w_{i-n+1}, \dots, w_{i-1})\) with Markov assumption of order \(n\). MLE: \(P(w_i \mid \cdot) = \frac{c(w_{i-n+1}^i)}{c(w_{i-n+1}^{i-1})}\).

9.3 Smoothing

Raw counts assign zero probability to unseen \(n\)-grams — fatal at test time. Smoothing redistributes probability mass.

  • Laplace (add-one): add 1 to all counts. Too aggressive in practice.
  • Witten–Bell: \(P(w_i \mid h) = \frac{c(h, w_i)}{c(h) + N_{1+}(h)}\), with \(N_{1+}(h)\) the number of distinct continuations.
  • Kneser–Ney (modified Kneser–Ney is the SOTA n-gram smoothing). Lower-order distributions are continuation probabilities \(P_{\text{KN}}(w) \propto |\{ v : c(v,w) > 0 \}|\). Discount higher-order counts by absolute discount \(D\) and back-off appropriately.

Standard tool: KenLM (Heafield) — fast modified Kneser–Ney implementation.

lmplz -o 4 < corpus.txt > corpus.arpa     # 4-gram MKN
build_binary corpus.arpa corpus.bin

9.4 Perplexity

\[ \mathrm{PPL}(W) = P(W)^{-1/N} = \exp\!\left( -\frac{1}{N} \sum_i \log P(w_i \mid h_i) \right). \]

Lower PPL ⇒ better LM. Always compute PPL on held-out data; PPL on training data is misleading. PPL correlates with WER only loosely — a 10% PPL drop might be 1–3% WER drop, depending on AM strength.

MODULE 10: NEURAL LANGUAGE MODELS

10.1 Why neural LMs?

  • Continuous representation of words via embeddings → smooth generalisation across similar contexts (“a cat sat on the mat” helps with “a dog sat on the mat”).
  • Long context via RNN/Transformer rather than fixed \(n\).
  • Subword units (BPE, WordPiece, SentencePiece) reduce OOV problem.

10.2 Architectures

  • Feed-forward NLM (Bengio et al. 2003). Fixed window of past tokens → embedding → MLP → softmax over vocab.
  • RNN-LM (Mikolov 2010). \(h_t = \tanh(W_h h_{t-1} + W_x x_t)\); \(P(w_t) = \mathrm{softmax}(V h_{t-1})\).
  • LSTM-LM. Mitigates vanishing gradients; standard ~2015–2020.
  • Transformer-LM (Vaswani 2017; GPT-style). Self-attention over all past tokens, causal mask. Current SOTA for both LM and end-to-end ASR rescoring.

10.3 Neural vs statistical LM

n-gram Neural
Context fixed, short long, learned
Generalisation none across surface forms via embeddings
OOV smoothing/back-off subwords + softmax
Speed very fast lookup slower; needs GPU or quantisation
Memory gigabytes of ARPA / KenLM hundreds of MB to GB of weights
Streaming integration trivial needs careful caching

In production, hybrid systems often first-pass with a 3–4-gram for speed and rescore n-best/lattices with an LSTM- or Transformer-LM.

10.4 Why context matters

Local acoustic ambiguity is resolved by long-range semantics. “I need to ___ the parade” — acoustically /siː/ could be see or sea; LM context “I need to” makes see far more likely. End-to-end models internalise some of this (the decoder is itself a conditional LM), but external LMs still help — especially with domain mismatch.

MODULE 11: INTEGRATING THE LM INTO ASR

11.1 Shallow fusion

At decode time combine log-probabilities:

\[ \log P(W \mid \mathbf{O}) \propto \log P_{\text{AM}}(\mathbf{O} \mid W) + \lambda \log P_{\text{LM}}(W) + \gamma\, |W|. \]

  • \(\lambda\) is the LM weight (typical 0.3–0.6).
  • \(\gamma\) is an insertion penalty that compensates for the AM bias toward shorter hypotheses.
  • Tuned on dev set, not training set.

11.2 Cold fusion

Train the seq2seq decoder jointly with a pre-trained, frozen LM in the loop, so the AM learns to complement the LM (not duplicate it).

11.3 Deep fusion

Similar but the LM hidden state is concatenated to the decoder hidden state; only a small projection is trained.

11.4 Rescoring strategies

  • N-best rescoring. First-pass decoder emits the top \(N\) hypotheses; a stronger LM (or larger neural model) re-scores them. Trivial to implement; throughput friendly.
  • Lattice rescoring. First-pass produces a word lattice; LM is composed with the lattice (in WFST land) and shortest path re-extracted. Captures vastly more hypotheses than top-N.
  • Second-pass attention rescoring (used in modern RNN-T / Conformer-RNN-T systems): the streaming first-pass emits hypotheses; a non-streaming attention decoder produces full-context rescores.

PART 5: DECODING AND WFSTs

MODULE 12: BEAM SEARCH AND VITERBI IN ASR

12.1 The decoding problem

The decoder must explore an enormous space: at each frame, every HMM state in every word in the lexicon is a candidate. Exact search is intractable, so we use beam search — keep only the top-\(B\) partial hypotheses at each time step.

12.2 Time-synchronous Viterbi beam search (HMM-GMM)

For each frame t:
    For each active state s in beam:
        For each successor s':
            score' = score(s) + log a_{s,s'} + log b_{s'}(o_t)
            update best(s', t) if score' is higher
    Prune: keep only states within beam_width of best score

Beam pruning parameters in Kaldi: --beam (state beam) and --lattice-beam (lattice generation beam). Wider beam → better accuracy at higher latency.

12.3 Token passing

Conceptually, each surviving hypothesis is a token that carries its score and word-history pointer through the network. A back-pointer table reconstructs the best path at end of utterance.

12.4 Lattices

For research and rescoring, the decoder emits not just a single hypothesis but a lattice — a weighted DAG of all hypotheses surviving the beam. Lattices admit efficient n-best extraction, LM rescoring, and confidence estimation via posterior decoding.

12.5 CTC prefix beam search

For end-to-end CTC models the decoding handles the blank symbol and label collapsing:

Initialise beam = {("", 1.0)}
For each frame t:
    new_beam = {}
    For each prefix p in beam:
        For each label c (including blank):
            if c == blank:
                add (p, P(p) * P(blank|t)) to new_beam
            elif c == last_char(p):
                # extending repeat or appending same char
                handle blank-vs-non-blank end of prefix accordingly
            else:
                add (p+c, P(p) * P(c|t)) to new_beam
    Merge entries with same collapsed text; keep top-B

A clean reference: Hannun’s Sequence Modeling with CTC (Distill, 2017) gives the full pseudocode.

MODULE 13: WEIGHTED FINITE-STATE TRANSDUCERS

13.1 What is a WFST?

A WFST \(T = (\Sigma, \Delta, Q, q_0, F, E, \lambda, \rho)\) has input alphabet \(\Sigma\), output alphabet \(\Delta\), states \(Q\), transitions \(E\) each labelled with input symbol, output symbol, and a weight in a semiring \((\mathbb{K}, \oplus, \otimes, \bar{0}, \bar{1})\). The weight of a path is the \(\otimes\)-product of its arc weights; the weight of an input-output pair is the \(\oplus\)-sum over all matching paths.

Common semirings:

Semiring \(\oplus\) \(\otimes\) \(\bar{0}\) \(\bar{1}\)
Tropical (Viterbi best path) \(\min\) \(+\) \(\infty\) \(0\)
Log (sum over paths in log domain) \(\oplus_{\log}\) \(+\) \(\infty\) \(0\)
Probability \(+\) \(\times\) \(0\) \(1\)

Tropical is used at decode time (best path); log is used during training (forward score).

13.2 The HCLG composition

Modern WFST ASR decoders represent the entire decoding graph as a single composed transducer:

\[ HCLG = H \circ C \circ L \circ G \]

where

  • \(H\): HMM transducer — maps HMM state sequences to context-dependent phones.
  • \(C\): Context-dependency transducer — maps tri-phone identity to monophone.
  • \(L\): Lexicon transducer — maps phoneme sequences to words.
  • \(G\): Grammar / LM transducer — assigns LM weights to word sequences.

The composition is then determinized (every input label has a unique outgoing arc per state), minimized (collapse equivalent states), and weight-pushed (move weights toward the start for earlier pruning). The result: a single graph the decoder traverses with one synchronous beam search.

References: Mohri, Pereira, Riley, Speech Recognition with Weighted Finite-State Transducers (2002); Povey et al., The Kaldi Speech Recognition Toolkit (ASRU 2011).

13.3 Why WFSTs are so useful

  • Modularity. Swap \(G\) for a different LM without retraining the AM.
  • Optimisation. Determinisation+minimisation compresses the graph enormously.
  • Unified search. A single Viterbi over the composed graph handles AM scores, lexicon, and LM jointly.
  • Lattice generation. Naturally falls out as a sub-graph after decoding.

13.4 End-to-end and WFSTs

Even end-to-end CTC systems benefit from WFST decoding via tools like Kaldi’s latgen-faster-mapped or k2/icefall’s k2.intersect_dense. CTC’s output graph (allowing blanks and repeats) is itself a WFST.

PART 6: DEEP LEARNING ACOUSTIC MODELS (HYBRID)

MODULE 14: DNN-HMM HYBRID

14.1 Architecture

Replace the GMM emission \(b_j(\mathbf{o})\) in HMM-GMM with a neural network trained to output per-frame senone posteriors \(P(s \mid \mathbf{o}_t)\), then convert to scaled likelihoods using Bayes’ rule:

\[ \log P(\mathbf{o}_t \mid s) = \log P(s \mid \mathbf{o}_t) - \log P(s) + \text{const}. \]

Subtracting the prior \(\log P(s)\) (estimated from frame counts) is essential — without it, frequent silence states dominate.

14.2 Training

  • Get frame-level senone alignments from a trained GMM-HMM (e.g. Kaldi tri3b).
  • Treat each frame as an independent classification example.
  • Cross-entropy training, optionally followed by sequence-discriminative training (MMI / sMBR / bMMI) for direct WER reduction.

14.3 Architectures that dominated 2012–2017

  • DNN (Hinton et al. 2012). 6–7 fully-connected layers with sigmoid/ReLU. Splice ±5 frames as input.
  • TDNN (Peddinti et al. 2015). Time-Delay Neural Network — subsampled 1D convolutions over time, much wider effective context with fewer params.
  • TDNN-F. Factorised TDNN with a low-rank SVD-style decomposition; standard Kaldi recipe.
  • LSTM / BLSTM acoustic models. Capture longer temporal dependencies; BLSTM is non-streaming.
  • CNN-LSTM hybrids. CNN front-end captures local time-frequency, LSTM captures temporal.

14.4 LF-MMI

Lattice-Free Maximum Mutual Information (Povey et al. 2016) trains the AM directly with a sequence objective on a denominator graph (a 4-gram phone LM over tri-phones), without per-frame cross-entropy pre-training. Lowers WER and accelerates training; the default in modern Kaldi recipes.

PART 7: END-TO-END ASR

MODULE 15: CONNECTIONIST TEMPORAL CLASSIFICATION (CTC)

15.1 The setup

Inputs \(\mathbf{X} = (\mathbf{x}_1, \dots, \mathbf{x}_T)\), outputs \(\mathbf{y} = (y_1, \dots, y_U)\), \(U \leq T\). The model emits per-frame distributions \(P(\pi_t \mid \mathbf{X})\) over an extended vocabulary \(\Sigma' = \Sigma \cup \{\varnothing\}\) (blank). A path \(\pi \in \Sigma'^T\) collapses to a label sequence via the function \(\mathcal{B}\):

  1. Merge consecutive identical labels (a a b b → a b).
  2. Remove blanks (a ∅ a → a a; note distinct from above).

The probability of a target \(\mathbf{y}\) is the marginal over all paths that collapse to it:

\[ P(\mathbf{y} \mid \mathbf{X}) = \sum_{\pi \in \mathcal{B}^{-1}(\mathbf{y})} \prod_{t=1}^{T} P(\pi_t \mid \mathbf{X}). \]

15.2 Forward–backward over the CTC graph

Build the CTC trellis as \(\mathbf{y}\) with blanks inserted between and around every label: \(\mathbf{y}' = (\varnothing, y_1, \varnothing, y_2, \varnothing, \dots, y_U, \varnothing)\) of length \(L' = 2U + 1\).

Forward variable: \(\alpha_t(s) = \sum_{\pi : \mathcal{B}(\pi_{1:t}) = \mathbf{y}'_{1:s}} \prod_{t'} P(\pi_{t'} \mid \mathbf{X})\).

Allowed transitions from \((t, s)\) to \((t+1, s')\) with \(s' \in \{s, s+1, s+2\}\) — but \(s' = s+2\) only if \(\mathbf{y}'_{s} = \varnothing\) or \(\mathbf{y}'_{s+2} \neq \mathbf{y}'_s\) (cannot skip a non-blank repeated label).

Loss: \(\mathcal{L}_{\text{CTC}} = -\log P(\mathbf{y} \mid \mathbf{X}) = -\log(\alpha_T(L') + \alpha_T(L' - 1))\).

Gradients flow into the softmax via the same forward–backward variables.

15.3 Conditional independence assumption

CTC assumes per-frame outputs are conditionally independent given \(\mathbf{X}\). This is its biggest theoretical weakness — outputs cannot directly model \(P(y_u \mid y_{u-1}, \dots)\). In practice, encoders capture some of this context, but an external LM is necessary for high accuracy.

15.4 CTC decoding

  • Greedy / best-path. \(\arg\max_\pi P(\pi \mid \mathbf{X})\) then collapse. Cheap but suboptimal because of marginalisation.
  • Prefix beam search. Marginalise over paths reaching each prefix. Mandatory for high-accuracy CTC.
  • CTC + WFST. Compose a CTC topology with the lexicon and LM.

15.5 Pioneering paper

Graves, Fernández, Gomez, Schmidhuber (2006). “Connectionist Temporal Classification: Labelling Unsegmented Sequence Data with Recurrent Neural Networks.” Originally with BLSTM acoustic models; later with everything.

MODULE 16: ATTENTION-BASED SEQ2SEQ (LISTEN, ATTEND, AND SPELL)

16.1 Architecture

Chan, Jaitly, Le, Vinyals (2015). “Listen, Attend and Spell.”

  • Listener: pyramidal BLSTM encoder that consumes log-mel features and downsamples in time (typically 8×) producing high-level acoustic embeddings \(\mathbf{h}_1, \dots, \mathbf{h}_{T'}\).
  • AttendAndSpell: an LSTM decoder over output characters, attending to encoder outputs at each step.
  • Attention (Bahdanau-style):

\[ e_{u,t} = \mathbf{v}^\top \tanh(W_s s_u + W_h h_t),\quad \alpha_{u,t} = \mathrm{softmax}_t(e_{u,t}),\quad c_u = \sum_t \alpha_{u,t} h_t. \]

  • Output: \(P(y_u \mid y_{<u}, \mathbf{X}) = \mathrm{softmax}(W_y[s_u; c_u])\).

Training: cross-entropy on character (or BPE/wordpiece) tokens; teacher forcing.

16.2 Strengths and weaknesses

Strength Weakness
Models full output dependencies \(P(y_u \mid y_{<u})\) Non-streaming by default — attention is global
Conceptually simple Bad behaviour on long utterances (“attention runaway”)
Strong with limited LM Hard alignment failures: deletions/repetitions
Joint AM+LM in one model Sensitive to noise and OOV without LM fusion

16.3 Joint CTC-Attention (Watanabe et al. / ESPnet)

Multi-task loss

\[ \mathcal{L} = \lambda\, \mathcal{L}_{\text{CTC}} + (1 - \lambda)\, \mathcal{L}_{\text{att}}, \quad \lambda \approx 0.3. \]

CTC’s monotonic constraint regularises the attention so it doesn’t wander. At decode, both branches score hypotheses — best of both worlds.

MODULE 17: RNN-TRANSDUCER (RNN-T)

17.1 Motivation

CTC is monotonic and streamable but assumes output independence. Attention models dependencies but isn’t streamable. RNN-T (Graves, 2012) has both: monotonic alignment and output dependencies.

17.2 Architecture

  • Encoder (a.k.a. transcription network) \(f^{\text{enc}}(\mathbf{X})_t = \mathbf{h}_t^{\text{enc}}\).
  • Prediction network (label LM-like) \(f^{\text{pred}}(y_{<u}) = \mathbf{h}_u^{\text{pred}}\).
  • Joint network combines both:

\[ z_{t,u} = W_o \tanh(W_h h_t^{\text{enc}} + W_p h_u^{\text{pred}} + \mathbf{b}). \]

\(P(k \mid t, u) = \mathrm{softmax}(z_{t,u})\) over \(\Sigma \cup \{\varnothing\}\).

A path is a sequence of label/blank emissions on the \((t, u)\) lattice that ends at \((T, U)\). Forward–backward over this 2D lattice marginalises over alignments.

17.3 Streaming-friendly

The encoder can be causal (e.g. left-context-only LSTM, or chunked Conformer); the prediction network is causal by construction. At inference, you emit tokens whenever the joint says non-blank, moving \(u\) forward, and advance \(t\) on blank.

17.4 The current production workhorse

Google’s “Streaming end-to-end ASR for mobile devices” (He et al. 2019) is an RNN-T. Apple/Amazon/Microsoft all run RNN-T variants on-device. Conformer-Transducer (Conformer encoder + Transducer loss) dominates 2021-onward leaderboards (LibriSpeech, GigaSpeech, etc.).

MODULE 18: TRANSFORMERS AND CONFORMER

18.1 Self-attention recap (cf. Jay Alammar’s Illustrated Transformer)

For queries \(Q\), keys \(K\), values \(V\):

\[ \mathrm{Attention}(Q, K, V) = \mathrm{softmax}\!\left( \frac{Q K^\top}{\sqrt{d_k}} \right) V. \]

Multi-head: project to \(H\) subspaces, do attention in each, concatenate. Positional information is provided by sinusoidal, learned, or relative positional encodings (Shaw et al.; Transformer-XL).

18.2 Transformer for ASR

Replacing RNN encoders with Transformer encoders gives strong results — but raw Transformers struggle to model local acoustic patterns efficiently. Speech is more locally structured than text.

18.3 Conformer (Gulati et al. 2020)

A Conformer block = “Macaron” half-step FFN + multi-head self-attention + convolution module + Macaron half-step FFN, with residuals and LayerNorm everywhere.

Per-block:

\[ \tilde{x} = x + \tfrac{1}{2}\mathrm{FFN}(x) \] \[ y = \tilde{x} + \mathrm{MHSA}(\tilde{x}) \] \[ z = y + \mathrm{Conv}(y) \] \[ \mathrm{out} = \mathrm{LN}\!\left( z + \tfrac{1}{2}\mathrm{FFN}(z) \right) \]

The Conv module uses pointwise convs and a 1D depthwise conv with kernel 31, GLU activation, and BatchNorm. It captures local time-frequency patterns that pure attention misses, while attention captures long-range dependencies.

Conformer-L on LibriSpeech test-clean: ~1.9% WER without external LM, ~1.5% with. The Conformer encoder is now the default for both CTC and RNN-T ASR.

18.4 Transformer-Transducer

Same RNN-T loss but with a Transformer/Conformer encoder and (often) Transformer prediction network. Streaming via chunked attention + look-ahead context.

MODULE 19: SELF-SUPERVISED PRETRAINING

19.1 wav2vec 2.0 (Baevski et al. 2020)

Pretrain on raw audio without transcripts:

  1. CNN feature encoder \(f(\mathbf{x}) \in \mathbb{R}^{T \times d}\).
  2. Mask a random subset of time steps in \(f(\mathbf{x})\).
  3. Transformer context network \(g\) produces context vectors \(\mathbf{c}_t = g(\tilde{f}(\mathbf{x}))_t\).
  4. Quantization module discretises the unmasked features into \(K\) codewords via Gumbel-Softmax over product codebooks → targets \(\mathbf{q}_t\).
  5. Contrastive loss at masked positions: \(\mathbf{c}_t\) must identify the true \(\mathbf{q}_t\) among \(K\) distractors:

\[ \mathcal{L}_m = -\log \frac{\exp(\mathrm{sim}(c_t, q_t)/\kappa)}{\sum_{\tilde{q} \in Q_t} \exp(\mathrm{sim}(c_t, \tilde{q})/\kappa)}. \]

  1. Diversity loss encourages even codebook usage.

After pretraining, fine-tune with CTC on labelled data — even 10 minutes of labelled speech can yield usable WER. State-of-the-art with 100 hours of labels.

19.2 HuBERT (Hsu et al. 2021)

Sidesteps wav2vec’s contrastive instability with a masked prediction objective:

  1. Run k-means on MFCCs to create initial pseudo-labels per frame.
  2. Mask frames, predict pseudo-labels with cross-entropy.
  3. Re-cluster on intermediate transformer layer features; iterate.

Often outperforms wav2vec 2.0 and is more stable to train.

19.3 WavLM, data2vec, BEST-RQ

  • WavLM (Chen et al. 2022) — adds denoising (noise/overlap simulation) during pretraining; strong for speaker tasks.
  • data2vec (Baevski et al. 2022) — predict latent representations of unmasked teacher, unified across modalities.
  • BEST-RQ (Chiu et al. 2022) — random-projection quantizer; simple, scalable, strong.

19.4 Whisper (Radford et al. 2022)

Different paradigm: weakly-supervised pretraining on 680 000 hours of multilingual web audio with paired transcripts. Encoder-decoder Transformer. Single model handles ASR, translation, language ID, voice activity detection. Robust to noise, accents, and codecs due to scale and diversity.

Architecture: log-mel input (80 bins, 25 ms window, 10 ms hop) → 2 conv layers (stride 1 and 2) → Transformer encoder → Transformer decoder with cross-attention. Tokens include special tokens for language, task, and timestamps.

import whisper
model = whisper.load_model("medium")
result = model.transcribe("utt.wav", language="en")
print(result["text"])

19.5 USM (Universal Speech Model; Zhang et al. 2023)

Google’s 2-billion parameter model trained on 12 million hours of unlabelled audio across 300+ languages. BEST-RQ pretraining + multilingual supervised fine-tune + noisy student self-training. The model behind YouTube captions.

19.6 SpeechT5 and AudioLM

  • SpeechT5 (Ao et al. 2022). Unified encoder-decoder for speech/text I/O — ASR, TTS, speech translation, voice conversion in one model.
  • AudioLM (Borsos et al. 2022). Language-model-style modelling of audio tokens (SoundStream + w2v-BERT semantic tokens). Strong continuation/generation but used as a backbone for ASR-like tasks (Voice-style transfer).

PART 8: MULTIMODAL ASR

MODULE 20: AUDIO-VISUAL AND BEYOND

20.1 Why multimodal?

In noisy environments lip motion contains crucial complementary information (the McGurk effect). Multimodal models also help with overlapping speakers, far-field conditions, and disambiguation of homophones via on-screen text.

20.2 AV-ASR architectures

  • Early fusion: concatenate audio + visual features per frame; feed to joint encoder.
  • Late fusion: independent audio and video encoders; combine logits or hidden states near the output.
  • Cross-modal attention: audio queries attend to video keys/values and vice versa.

20.3 AV-HuBERT (Shi et al. 2022)

Extends HuBERT to audio + lip-video. Masked-prediction over joint audio-visual representations. Sets SOTA on LRS3 lipreading and audio-visual ASR with strong robustness to acoustic noise.

20.4 Video-grounded ASR

For scenarios like meetings or product reviews, the model can attend to slides/text overlays (OCR features) to anchor terminology. Also see speaker-attributed ASR (“who said what”) using face tracks plus speech embeddings (X-vector / ECAPA-TDNN).

20.5 Text-conditioned ASR

Provide a domain bias text (“biasing list” of contact names, product names, medical terms) to the decoder, attended to during decoding. Common in keyword-biased streaming ASR (e.g. Google Assistant contact names).

  • Output: timestamps of speech regions; saves compute by skipping silence and limits latency by avoiding waiting forever.

21.3 Endpointer

Decides “the user is done speaking.” Looks at decoder state — when blanks dominate and silence is detected for ~200–500 ms, finalise. Tradeoff: too eager cuts users off, too lazy adds latency.

21.4 ITN and punctuation

  • ITN (Inverse Text Normalisation): converts verbalised forms to written (“nineteen ninety five” → “1995”). Done by FST rules (Sparrowhawk / Nemo TN) or seq2seq neural models.
  • Punctuation & casing. Often a BERT-style token-classification model run on the raw decoder output.

21.5 Streaming vs non-streaming

Streaming Non-streaming

Decoder RNN-T greedy, CTC prefix Attention with full encoder access

Common pattern: streaming first pass + non-streaming second-pass rescoring (“two-pass ASR”, Sainath et al. 2019).

21.6 Chunked attention and look-ahead

  • Chunked attention. Process the input in fixed-size chunks (e.g. 320 ms); attention within and to previous chunks.
  • Look-ahead context. Allow each chunk to peek into the next \(L\) frames (e.g. 80 ms) for accuracy at the cost of latency.

21.7 Latency factors checklist

  • Algorithmic delay = look-ahead frames × hop.
  • Decoder beam width. Wider = more compute per frame.
  • LM rescoring. Adds end-of-utterance latency.
  • Model size & quantisation. FP16/INT8 inference, kernel fusion, ONNXRuntime, TensorRT, CoreML.
  • Batching. Server-side batching adds latency for fairness; on-device batch = 1.
  • Hardware. Mobile NPU (Apple Neural Engine, Google Tensor) vs CPU vs server GPU.
  • KV cache. For Transformer decoders, reuse keys/values across steps.
  • Memory bandwidth. Often the binding constraint on edge devices.

PART 10: EVALUATION, OPTIMISATION, AND EXPLAINABILITY

MODULE 22: WORD ERROR RATE AND FRIENDS

22.1 The metric

\[ \mathrm{WER} = \frac{S + I + D}{N}, \]

where \(S\), \(I\), \(D\) are substitutions, insertions, deletions in the minimum-edit-distance alignment between hypothesis and reference, and \(N\) is the number of words in the reference. WER can exceed 1.0 (e.g. many insertions on short references).

Levenshtein DP (the basis of WER):

def wer(ref, hyp):
    R, H = ref.split(), hyp.split()
    n, m = len(R), len(H)
    d = [[0]*(m+1) for _ in range(n+1)]
    for i in range(n+1): d[i][0] = i
    for j in range(m+1): d[0][j] = j
    for i in range(1, n+1):
        for j in range(1, m+1):
            cost = 0 if R[i-1] == H[j-1] else 1
            d[i][j] = min(d[i-1][j] + 1,        # deletion
                          d[i][j-1] + 1,        # insertion
                          d[i-1][j-1] + cost)   # sub / match
    return d[n][m] / max(1, n)

Standard tools: sclite (NIST), jiwer (Python), whisper.normalizers.

22.2 Errors decomposed

A useful diagnostic breakdown of WER into S / I / D rates. Common patterns:

  • High insertions: decoder too eager; LM weight too low; missing endpointing.
  • High deletions: truncated audio, overly aggressive VAD, missing context.
  • High substitutions: acoustic confusions, OOV, codeswitched names.

22.3 Character Error Rate (CER)

Same DP at the character level. Useful for languages without clear word boundaries (Mandarin, Japanese, Thai) and for evaluating subword/byte-level models.

22.4 Sentence Error Rate (SER)

Fraction of utterances with any error. A binary, much looser metric. Often quoted in command-and-control ASR.

22.5 BLEU and translation

For Speech Translation (e.g. Whisper translating non-English to English), BLEU is the standard metric — n-gram precision plus brevity penalty. WER does not apply because translations are not aligned.

22.6 Intent / Slot accuracy

For end-to-end Spoken Language Understanding (SLU): exact-match intent accuracy and slot F1. These end-task metrics often correlate poorly with WER — a wrong word that doesn’t affect intent doesn’t hurt downstream.

22.7 Normalisation matters

Before computing WER, both ref and hyp must be normalised the same way:

  • Lowercased.
  • Punctuation stripped.
  • Numbers verbalised consistently (or both pre-ITN).
  • Contractions handled (“can’t” vs “can not”).

Whisper publishes its EnglishTextNormalizer and BasicTextNormalizer exactly for fair WER comparison.

22.8 Confidence intervals

Always report bootstrap CIs on WER (resample utterances 1000×). Two systems with WER 5.3 vs 5.4 on the same eval set are statistically indistinguishable; report it.

MODULE 23: REAL-TIME FACTOR (RTF)

23.1 Definition

\[ \mathrm{RTF} = \frac{\text{wall-clock decoding time}}{\text{audio duration}}. \]

  • RTF < 1 → faster than real time.
  • RTF = 1 → exactly real time.
  • RTF > 1 → slower than real time (unacceptable for streaming).

23.2 What dominates RTF

  • Encoder forward pass (largest in modern models).
  • Decoder steps (especially attention with KV cache).
  • LM rescoring (often a separate stage, can be batched).
  • Audio I/O and resampling (small but non-zero).

23.3 Achieving low RTF

  • Model distillation / pruning. Distill a Conformer-L into a Conformer-S; near-equal WER.
  • Quantisation. Static INT8 on encoder; minor accuracy hit.
  • Streaming structure. Chunk-causal Conformer with limited look-ahead.
  • Beam pruning. Tune beam width; report RTF and WER together.
  • Hardware acceleration. ONNXRuntime CUDA EP, TensorRT, OpenVINO, CoreML, Apple Neural Engine.
  • Operator fusion. Fused MHSA + LayerNorm + bias; flash-attention.

23.4 Latency vs RTF

Not the same. A model can have RTF = 0.1 (very fast) but 800 ms algorithmic latency because it waits for 800 ms of look-ahead. Always report both, plus first-token latency (time from speech start to first hypothesis word) for streaming ASR.

MODULE 24: HYPERPARAMETER TUNING

24.1 Training hyperparameters

  • Learning rate & schedule. Warmup-decay (Noam schedule, \(\mathrm{lr} \propto d_{\text{model}}^{-0.5} \min(\mathrm{step}^{-0.5}, \mathrm{step} \cdot \mathrm{warmup}^{-1.5})\)) for Transformer/Conformer; cosine schedule with warmup is the modern default.
  • Optimiser. AdamW with \(\beta_1 = 0.9, \beta_2 = 0.98, \epsilon = 10^{-9}\) for Conformer.
  • Batch size. Sequence-bucketed batches (group by length) to maximise GPU utilisation.
  • Gradient clipping. Clip global norm to 5.0; essential for RNN-T stability.
  • Dropout. 0.1 on attention, FFN, conv; sometimes higher (0.3) on small datasets.
  • Label smoothing. \(\epsilon = 0.1\) for attention decoders; not used with CTC/RNN-T.
  • Mixed precision (FP16/BF16). Standard. Watch for NaN in attention logits — clamp or use BF16.

24.2 Decoding hyperparameters

  • Beam width. Typical 4–16 (RNN-T) up to 32–64 (CTC + LM).
  • LM weight \(\lambda\). Grid-search 0.1–0.8 on dev.
  • Insertion penalty \(\gamma\). Compensates for length bias.
  • Coverage penalty / EOS bias (attention models).
  • Endpointer thresholds. Silence duration, blank probability.

24.3 Practical workflow

  1. Train baseline; record WER, RTF, latency on dev.
  2. Run SpecAugment / speed perturbation ablations.
  3. Tune LM weight and beam on dev (cheap; just decoding).
  4. Investigate top error categories with substitution matrix; consider biasing list or vocabulary expansion.
  5. Quantise + benchmark on target hardware.

MODULE 25: DATA AUGMENTATION

25.1 SpecAugment (Park et al. 2019)

Apply directly to the log-mel spectrogram during training:

  1. Time warping (optional): warp the spectrogram in time by a random shift.
  2. Frequency masking. Choose \(f \sim U(0, F)\) and a starting bin \(f_0\); zero out \([f_0, f_0 + f)\). Repeat \(m_F\) times.
  3. Time masking. Choose \(t \sim U(0, T)\) and starting frame \(t_0\); zero out \([t_0, t_0 + t)\). Repeat \(m_T\) times, with \(t \leq p T\) for some upper bound \(p\).

Hyperparameters: \((F, T, m_F, m_T, p)\). Typical LibriSpeech-style “LD” policy: \(F = 27, T = 100, m_F = 1, m_T = 1, p = 1.0\); “LB” policy: \(F = 27, T = 100, m_F = 2, m_T = 2, p = 1.0\); “SM” policy reduces.

Massive WER gains essentially for free — SpecAugment is the single most cost-effective ASR augmentation.

25.2 Speed and tempo perturbation

Resample audio at \(0.9\times\), \(1.0\times\), \(1.1\times\) (triples the dataset) — changes both duration and pitch slightly. Cheap and effective for HMM-GMM and neural systems alike.

25.3 Volume / gain perturbation

Random per-utterance gain in \(\pm 10\) dB; or per-frame SpecAugment-style attenuation.

25.4 Noise and reverb injection

# Pseudo-code: add noise at random SNR
snr_db   = np.random.uniform(0, 20)
noise    = sample_from(MUSAN)
gain     = np.sqrt(signal_power / (noise_power * 10**(snr_db/10)))
noisy    = clean + gain * noise

# Reverb via convolution with sampled RIR
rir      = sample_from(RIRS_NOISES)
reverbed = scipy.signal.fftconvolve(clean, rir)[:len(clean)]

Standard corpora: MUSAN (Snyder et al.) and RIRS_NOISES (Ko et al.).

25.5 Mixup, SpecMix

Mix two utterances’ spectrograms with a Beta-sampled weight; mix labels accordingly. Helps regularisation, especially on small datasets.

25.6 Curriculum and length filtering

Drop utterances shorter than 1 s or longer than 30 s to stabilise training. Optionally, sort batches by length early and shuffle later (length curriculum).

MODULE 26: ERROR ANALYSIS AND EXPLAINABILITY

26.1 Why explainability for ASR?

ASR errors silently cascade into downstream NLU and decisions. Explainability tools answer:

  • Why did the model output “wreck” instead of “rec”?
  • Where in the audio did the substitution happen?
  • Which training subset is responsible?

26.2 Confusion / substitution matrix

Aggregate substitutions across the dev set. Top entries usually reveal:

  • Homophones the LM should have disambiguated.
  • Domain terms missing from the lexicon.
  • Acoustic confusables (/m/ vs /n/, /s/ vs /z/).
from collections import Counter
subs = Counter()
for ref, hyp in zip(refs, hyps):
    for r, h, op in align(ref, hyp):
        if op == "S": subs[(r, h)] += 1
print(subs.most_common(30))

26.3 Attention visualisation

For LAS / attention-based models, plot the attention matrix \(\alpha_{u,t}\). A clean diagonal (with monotonic increase) means healthy alignment; scattered or stuck attention means alignment failure.

For Whisper-style decoders, cross-attention layers ≈ word alignment — Whisper exposes word-level timestamps using exactly this signal.

26.4 Saliency on spectrograms

Compute \(\frac{\partial \log P(y \mid \mathbf{X})}{\partial \mathbf{X}}\) on the log-mel input. Heatmap the gradient magnitude over time-frequency tiles. Useful for adversarial robustness and for spotting reliance on artefacts (codec, room hum).

26.5 Integrated gradients

Average gradients along a path from a baseline (silence or noise) to the input. Sum to \(f(\mathbf{X}) - f(\mathbf{X}_{\text{baseline}})\), satisfying the completeness axiom; more robust than raw gradients.

26.6 Probing classifiers

Train a linear classifier on intermediate encoder representations to predict phones, speaker, gender, etc. Reveals what the encoder has implicitly learned at each depth.

26.7 Logging utterance-level diagnostics

Production ASR should log per-utterance:

  • WER and substitution list against any available ground truth.
  • Average frame-level blank probability (very high → encoder may be saturating).
  • Decoder beam diversity.
  • Endpointer trigger time vs true end.
  • Channel-level features (SNR estimate, reverberation time T60 estimate) for cohort analysis.

26.8 Error analysis tooling

  • Kaldi/ESPnet analyze_word_error.py — long-tail substitution & insertion lists.
  • Whisper word_timestamps=True — quick alignment for human inspection.
  • NeMo’s Speech Data Explorer — interactive browser for utterance-level errors.

MODULE 27: FORMULA SHEET

Pre-emphasis. \(y[n] = x[n] - \alpha x[n-1]\), \(\alpha \approx 0.97\).

STFT. \(X[m,k] = \sum_n w[n] x[n + mH] e^{-j2\pi k n / N}\).

Mel. \(m = 2595 \log_{10}(1 + f/700)\).

MFCC DCT. \(c_n = \sum_m L[m] \cos\!\left(\frac{\pi n (m - 0.5)}{M}\right)\).

Delta. \(\Delta c_t = \frac{\sum_{n=1}^{\Theta} n (c_{t+n} - c_{t-n})}{2 \sum n^2}\).

HMM forward. \(\alpha_{t+1}(j) = [\sum_i \alpha_t(i) a_{ij}] b_j(\mathbf{o}_{t+1})\).

Viterbi. \(\delta_t(j) = \max_i [\delta_{t-1}(i) a_{ij}] b_j(\mathbf{o}_t)\).

GMM emission. \(b_j(\mathbf{o}) = \sum_k w_{jk} \mathcal{N}(\mathbf{o}; \mu_{jk}, \Sigma_{jk})\).

Bayes rule for hybrid. \(\log P(\mathbf{o}_t \mid s) = \log P(s \mid \mathbf{o}_t) - \log P(s) + C\).

CTC marginalisation. \(P(\mathbf{y} \mid \mathbf{X}) = \sum_{\pi \in \mathcal{B}^{-1}(\mathbf{y})} \prod_t P(\pi_t \mid \mathbf{X})\).

Attention. \(\mathrm{Attn}(Q,K,V) = \mathrm{softmax}(QK^\top/\sqrt{d_k}) V\).

RNN-T joint. \(z_{t,u} = W_o \tanh(W_h h_t^{\text{enc}} + W_p h_u^{\text{pred}} + b)\).

Conformer block. \(\frac{1}{2}\mathrm{FFN} \rightarrow \mathrm{MHSA} \rightarrow \mathrm{Conv} \rightarrow \frac{1}{2}\mathrm{FFN} \rightarrow \mathrm{LN}\).

Shallow fusion. \(\arg\max_W [\log P_{\text{AM}}(\mathbf{O}|W) + \lambda \log P_{\text{LM}}(W) + \gamma |W|]\).

WER. \(\mathrm{WER} = (S + I + D)/N\).

RTF. \(\mathrm{RTF} = T_{\text{decode}} / T_{\text{audio}}\).

Perplexity. \(\mathrm{PPL} = \exp\!\left( -\frac{1}{N} \sum_i \log P(w_i \mid h_i) \right)\).

SQNR. \(\mathrm{SQNR}_{\mathrm{dB}} \approx 6.02 B + 1.76\).

MODULE 30: A “PIPELINE FROM SCRATCH” CODE WALKTHROUGH

A minimal, end-to-end illustrative pipeline that touches each block.

import numpy as np
import torch
import torch.nn as nn
import torchaudio
import torchaudio.transforms as T

# ---------- 1. Load and preprocess ----------
wav, sr = torchaudio.load("utt.wav")
if sr != 16000:
    wav = torchaudio.functional.resample(wav, sr, 16000); sr = 16000
wav = wav.mean(dim=0, keepdim=True)                  # mono
wav = wav - wav.mean()                                # DC remove
preemph = torchaudio.functional.preemphasis(wav, 0.97)

# ---------- 2. Feature extraction (log-mel) ----------
mel = T.MelSpectrogram(
    sample_rate=sr, n_fft=512, win_length=400, hop_length=160,
    n_mels=80, f_min=0, f_max=8000, power=2.0,
)(preemph)
log_mel = torch.log(mel + 1e-6)                       # (1, 80, T)

# ---------- 3. Per-utterance CMVN ----------
mean = log_mel.mean(dim=-1, keepdim=True)
std  = log_mel.std(dim=-1, keepdim=True) + 1e-5
log_mel = (log_mel - mean) / std

# ---------- 4. SpecAugment (training only) ----------
spec_aug = nn.Sequential(
    T.FrequencyMasking(freq_mask_param=27),
    T.TimeMasking(time_mask_param=100, p=1.0),
    T.TimeMasking(time_mask_param=100, p=1.0),
)
# log_mel_aug = spec_aug(log_mel)   # uncomment during training

# ---------- 5. Acoustic model (toy Conformer-CTC sketch) ----------
class ConvSubsample(nn.Module):
    def __init__(self, c_in, c_out):
        super().__init__()
        self.net = nn.Sequential(
            nn.Conv2d(1, c_out, 3, stride=2, padding=1), nn.ReLU(),
            nn.Conv2d(c_out, c_out, 3, stride=2, padding=1), nn.ReLU(),
        )
    def forward(self, x):                              # x: (B, 1, F, T)
        return self.net(x)

class ToyCTCModel(nn.Module):
    def __init__(self, n_mels=80, d=256, n_layers=4, vocab=29):
        super().__init__()
        self.sub  = ConvSubsample(1, d)
        self.proj = nn.Linear(d * (n_mels // 4), d)
        self.enc  = nn.TransformerEncoder(
            nn.TransformerEncoderLayer(d, nhead=4, dim_feedforward=4*d,
                                        batch_first=True, dropout=0.1),
            num_layers=n_layers,
        )
        self.head = nn.Linear(d, vocab)                # vocab includes <blank>
    def forward(self, log_mel):
        # log_mel: (B, n_mels, T)
        x = log_mel.unsqueeze(1)                       # (B,1,F,T)
        x = self.sub(x)                                # (B,d,F/4,T/4)
        B, d, F, Tp = x.shape
        x = x.permute(0, 3, 1, 2).reshape(B, Tp, d * F)
        x = self.proj(x)                               # (B, T/4, d)
        x = self.enc(x)
        return self.head(x)                            # (B, T/4, vocab)

# ---------- 6. CTC loss ----------
model = ToyCTCModel()
logits = model(log_mel)                                # (1, T', V)
log_probs = torch.log_softmax(logits, dim=-1).transpose(0, 1)  # (T', 1, V)
targets = torch.tensor([3, 8, 5, 12])                  # toy "HELL"
input_lengths  = torch.tensor([log_probs.size(0)])
target_lengths = torch.tensor([targets.size(0)])
loss = nn.functional.ctc_loss(log_probs, targets, input_lengths, target_lengths,
                              blank=0, zero_infinity=True)

# ---------- 7. Greedy decoding ----------
def ctc_greedy(log_probs, blank=0):
    # log_probs: (T, V)
    best = log_probs.argmax(dim=-1).tolist()
    out, prev = [], None
    for b in best:
        if b != prev and b != blank: out.append(b)
        prev = b
    return out

The principles in this toy model — CMVN, log-mel, conv subsampling, encoder, CTC loss, greedy/beam decode — generalise directly to production Conformer-CTC / Conformer-Transducer pipelines in torchaudio, NeMo, ESPnet, and Hugging Face Transformers.
