Preface — How to Use This Guide

This document is a single, self-contained reference on deep generative models, written as a book chapter that takes a reader from zero to expert. It is organized to mirror the assessment syllabus exactly so that every named skill has a dedicated section, but the content is presented as continuous study material — equations, derivations, code, pseudocode, comparisons, and worked numerical exercises sit side by side.

A few orientation notes before diving in:

  • Notation. \(x \in \mathbb{R}^d\) is the data point (image, audio, token sequence). \(z\) is a latent variable. \(\theta\) is a generic parameter vector for a neural network. \(p_{\text{data}}(x)\) is the true (unknown) data distribution; \(p_\theta(x)\) is the model’s distribution. \(q\) usually denotes a variational or forward-noise distribution. \(\mathbb{E}_{x\sim p}[f(x)]\) is an expectation. \(\mathrm{KL}(q\Vert p) = \mathbb{E}_q[\log q - \log p]\) is the Kullback–Leibler divergence. \(\mathcal{N}(\mu,\Sigma)\) is a Gaussian.
  • Math rendering. All math uses $...$ (inline) or $$...$$ (block) for Quarto/MathJax. Derivations are shown step by step, not just stated.
  • Code. PyTorch-style snippets are minimal but complete in the sense that the tensor shapes, loss expressions, and update rules can be read like a recipe. Tensor shapes are written as comments where they matter.
  • Pseudocode. Algorithm 1 / Algorithm 2 boxes follow the DDPM paper’s convention: a training loop and a sampling loop, side by side.
  • Comparison tables. Whenever multiple methods share a problem (e.g., DDPM vs DDIM, VAE vs GAN vs flow vs diffusion, FID vs IS vs Precision/Recall), a table makes the differences explicit.
  • “Assessment traps” sidebars highlight vocabulary distinctions that are easy to confuse: generative vs discriminative, ε-prediction vs v-prediction, classifier vs classifier-free guidance, mode collapse vs mode covering.
  • Worked exercises appear inline. They use small numbers (e.g., \(T=4\) timesteps, scalar latents) so you can carry the arithmetic in your head and verify each formula.

Read the parts in order on a first pass. On revisits, you can jump to a specific Module via the table of contents — every Module is self-contained enough that it stands alone after the foundational definitions in Part 1.

PART 1 — UNDERSTANDING GENERATIVE MODELING FOUNDATIONS

This part is the conceptual backbone. It establishes (i) what makes a model generative as opposed to discriminative; (ii) which downstream applications generative models actually solve; (iii) the unsupervised-learning principles (density estimation, latent variables, the manifold hypothesis, ELBO, contrastive learning, energy-based view) that underpin all of the architectures studied later; and (iv) the inventory of deep generative architectures — autoregressive, VAE, GAN, normalizing flow, diffusion / score-based, energy-based, consistency, flow matching / rectified flow — with their use cases and trade-offs.

MODULE 1: Comparing Generative and Discriminative Approaches

1.1 Distinguishing Generative from Discriminative Models in Data Contexts

The cleanest definition. Given data \((x, y)\) where \(x\) is an input (e.g., an image) and \(y\) is a label,

  • a discriminative model learns the conditional distribution \(p(y \mid x)\) directly. It answers “given this input, which class / value of \(y\) is likely?”.
  • a generative model learns the joint distribution \(p(x, y)\), or in the unsupervised case just \(p(x)\). It answers “how is the data itself distributed?” and can be sampled to produce new \(x\).

If you know the joint \(p(x,y)\), you can derive the discriminative answer via Bayes’ rule:

\[ p(y \mid x) \;=\; \frac{p(x \mid y)\, p(y)}{p(x)} \;=\; \frac{p(x, y)}{\sum_{y'} p(x, y')}. \]

So generative models contain discriminative ones as a special inference query, but the reverse is not true: a model trained only to output \(p(y \mid x)\) cannot synthesize new \(x\).

Generative vs discriminative — side by side.

Mental picture. A discriminative model draws a boundary through the input space. A generative model paints the density of the input space itself; the boundary is then a side-effect of comparing how much each class explains a given \(x\).

Bayes-rule sanity check. A Gaussian discriminant analysis classifier fits one Gaussian per class and combines them through Bayes’ rule:

\[ p(y=k \mid x) \;=\; \frac{\pi_k \, \mathcal{N}(x;\mu_k, \Sigma_k)}{\sum_{j} \pi_j \, \mathcal{N}(x;\mu_j, \Sigma_j)}. \]

It is generative because you can sample: pick a class \(k\) with probability \(\pi_k\), then draw \(x \sim \mathcal{N}(\mu_k,\Sigma_k)\). Logistic regression, by contrast, parametrizes \(p(y\mid x) = \sigma(w^\top x + b)\) directly without modeling \(p(x)\) — purely discriminative.

Common assessment trap — vocabulary.

  • Generative is about modeling the distribution; discriminative is about modeling the boundary.
  • “Generative AI” colloquially means “AI that makes new content”; technically, generative in the statistical sense is broader (anything that models \(p(x)\) or \(p(x,y)\)) but in this study guide the two senses coincide because we focus on models that do generate samples.
  • A GAN models \(p(x)\) but does not give you a tractable likelihood; it is still generative because you can sample from it.
  • A masked language model (BERT) is sometimes described as discriminative because its main downstream use is classification, but the pretext task itself is a (conditional) generative one.

1.2 Applications of Generative Models in Synthesis and Augmentation

Generative models are interesting not because they trace pretty contours of \(p(x)\) but because of the things you can do once you can sample from \(p(x)\) or \(p(x \mid c)\) for some condition \(c\). The applications below are the textbook list — each is worth knowing by name because the assessment loves “which of these is a generative-model application?” items.

Image synthesis. Unconditional: sample faces (StyleGAN), animals (BigGAN), arbitrary images. Conditional: text-to-image (Stable Diffusion, Imagen, DALL·E, Midjourney), class-conditional ImageNet (BigGAN, ADM, EDM), sketch-to-image, segmentation-to-image (Pix2Pix), pose-to-image (ControlNet).

Audio synthesis. Text-to-speech (Tacotron + WaveNet/HiFi-GAN vocoders, VALL-E, Bark), music generation (MusicLM, Jukebox, MusicGen, AudioLDM), sound effects, neural codecs (SoundStream, Encodec) that combine a generative decoder with a learned discrete representation.

Text generation. Autoregressive language models (GPT family, Llama, Claude, Gemini) are generative models of token sequences. Open-ended dialogue, summarization, translation, code generation are all sampling problems from \(p(x_{1:T} \mid \text{prompt})\).

Video generation and editing. Sora, Veo, Runway Gen-3, Stable Video Diffusion. Same diffusion or transformer toolkit, but on space-time tokens / latents.

Molecule and protein generation. Drug discovery (generate novel molecules with desired properties, e.g., MolGAN, EDM-for-molecules, RFdiffusion for protein backbones); de novo protein design (RFdiffusion, Chroma, ProteinMPNN). The data distribution lives on graphs or 3-D point clouds with equivariance constraints, but the underlying technology is still diffusion / flow / autoregressive.

Data augmentation. Synthesize extra training samples to expand or rebalance a dataset — especially powerful for rare classes (medical lesions, fraud examples), or for unsupervised representation learning where the augmentations themselves are the supervision signal (SimCLR, MoCo treat augmentations as positive pairs).

Simulation. World models for reinforcement learning (Dreamer, DayDreamer, GameNGen), simulators for self-driving (driving scenarios), physics surrogates that replace expensive PDE solvers with a learned generative model conditioned on parameters.

Anomaly detection. A generative model trained on “normal” data assigns low likelihood (or high reconstruction error) to anomalies. VAEs, flows, and diffusion models can all be used this way. The key signal is some scalar that estimates how far \(x\) is from the data manifold — exact likelihood, ELBO, score norm, or reconstruction loss.

Super-resolution. Upscale a low-resolution image to a high-resolution one. ESRGAN (GAN-based), SR3 (diffusion-based), Real-ESRGAN. The model learns \(p(x_{\text{HR}} \mid x_{\text{LR}})\).

Inpainting and editing. Fill in masked regions of an image: Pix2Pix (paired translation), LaMa, Stable Diffusion inpainting, RePaint (DDPM inpainting via masked diffusion). Same idea for audio (gap filling) and video (object removal).

Translation / domain transfer. CycleGAN (horses ↔︎ zebras, summer ↔︎ winter), Pix2Pix (paired sketch ↔︎ photo), neural machine translation, code translation. Conditional generative models with a strong cycle-consistency or paired-supervision signal.

Compression. Neural compression (Ballé et al., HiFiC) uses a learned generative decoder so that a small bitstream + decoder generates a perceptually faithful reconstruction.

Likelihood-based scientific modeling. Cosmology (generative models of galaxy images), particle physics (fast detector simulators replacing GEANT4), neuroscience (generative models of neural activity).

Why “data augmentation” is on the list. For supervised learning with very few labels per class, training a class-conditional generative model on the unlabeled pool and then sampling per class is a way to manufacture new training data. The catch: a generative model that has already learned the dataset cannot, in principle, add genuinely new information; but as a regularizer it often helps. The most successful modern variant of this idea is augmentation by generation (e.g., synthetic medical images by class-conditional diffusion).

1.3 Unsupervised Learning Principles for Generative Modeling

Generative modeling is the canonical unsupervised-learning problem. The principles below recur in every architecture later in the guide.

Principle 1 — Density estimation. The fundamental task is to estimate \(p_{\text{data}}(x)\). Given i.i.d. samples \(x^{(1)}, \dots, x^{(N)} \sim p_{\text{data}}\), learn \(p_\theta\) that is close to \(p_{\text{data}}\). “Close” is usually measured by KL divergence:

\[ \mathrm{KL}\!\left(p_{\text{data}} \,\Vert\, p_\theta\right) \;=\; \mathbb{E}_{x \sim p_{\text{data}}}\!\left[\log p_{\text{data}}(x) - \log p_\theta(x)\right]. \]

Since \(\log p_{\text{data}}(x)\) doesn’t depend on \(\theta\), minimizing this KL is equivalent to maximizing \(\mathbb{E}_{x\sim p_{\text{data}}}[\log p_\theta(x)]\), i.e., maximum likelihood. With \(N\) samples, this is the average log-likelihood

\[ \mathcal{L}_{\text{MLE}}(\theta) \;=\; \frac{1}{N} \sum_{i=1}^{N} \log p_\theta(x^{(i)}). \]

Three deep generative families optimize this (or a bound on it) directly: autoregressive models, normalizing flows, and VAEs (via ELBO). GANs optimize a different objective (Jensen–Shannon-like adversarial). Diffusion models optimize an ELBO that, in practice, is reduced to a denoising regression loss equivalent to score matching.

Principle 2 — Latent variable models. A latent-variable model writes the data distribution as a marginal:

\[ p_\theta(x) \;=\; \int p_\theta(x \mid z)\, p(z)\, dz, \]

where \(z\) is an unobserved “explanation” and \(p(z)\) is a prior (often \(\mathcal{N}(0, I)\)). This decomposition is conceptually powerful for three reasons:

  1. Sampling is easy. Draw \(z \sim p(z)\), then \(x \sim p_\theta(x\mid z)\).
  2. A simple prior + a complex decoder can model a very complex data distribution, because \(z\) encodes the slow modes of variation (pose, identity, content) and the decoder paints in details.
  3. Inference \(p_\theta(z \mid x)\) gives a useful representation that can be used for downstream tasks (clustering, retrieval, interpolation).

Examples: VAE (continuous \(z\), parametric decoder, learned approximate posterior); GAN (implicit decoder, no explicit likelihood); normalizing flow (decoder is invertible, so we can compute \(z = f^{-1}(x)\) exactly).

Principle 3 — The manifold hypothesis. Natural data — images, audio, language — does not fill \(\mathbb{R}^d\); it concentrates on a much lower-dimensional manifold \(\mathcal{M} \subset \mathbb{R}^d\). Two consequences:

  • Most random vectors in pixel space are noise; only a thin set are valid images. The job of a generative model is to learn that set.
  • A useful latent space \(z \in \mathbb{R}^{d'}\) with \(d' \ll d\) should give a chart of that manifold; “moving” in \(z\) should correspond to traversing the manifold smoothly (smile \(\to\) frown, day \(\to\) night).

Principle 4 — The ELBO (Evidence Lower Bound). For latent-variable models, the marginal likelihood \(p_\theta(x) = \int p_\theta(x,z)\,dz\) is intractable. The variational trick introduces a tractable approximate posterior \(q_\phi(z\mid x)\) and writes a lower bound on \(\log p_\theta(x)\).

Derivation. For any distribution \(q_\phi(z\mid x)\),

\[ \log p_\theta(x) = \log \int p_\theta(x, z)\, dz = \log \int q_\phi(z\mid x) \frac{p_\theta(x,z)}{q_\phi(z\mid x)}\, dz. \]

Applying Jensen’s inequality (because \(\log\) is concave):

\[ \log p_\theta(x) \;\ge\; \int q_\phi(z\mid x)\, \log \frac{p_\theta(x,z)}{q_\phi(z\mid x)}\, dz \;\equiv\; \mathrm{ELBO}(x;\theta,\phi). \]

Rearranging,

\[ \mathrm{ELBO}(x;\theta,\phi) = \underbrace{\mathbb{E}_{q_\phi(z\mid x)}\!\left[\log p_\theta(x\mid z)\right]}_{\text{reconstruction}} \;-\; \underbrace{\mathrm{KL}\!\left(q_\phi(z\mid x) \,\Vert\, p(z)\right)}_{\text{regularization}}. \]

The gap between \(\log p_\theta(x)\) and the ELBO is exactly \(\mathrm{KL}(q_\phi(z\mid x)\Vert p_\theta(z\mid x))\) — when the approximate posterior matches the true posterior, the bound is tight. The VAE optimizes the ELBO. Diffusion models also use an ELBO whose KL terms reduce to denoising losses (Module 3).

Principle 5 — Contrastive learning (an alternative unsupervised signal). Contrastive methods learn a representation by pulling together views of the same data and pushing apart views of different data. The InfoNCE objective is

\[ \mathcal{L}_{\text{InfoNCE}} \;=\; -\,\mathbb{E}\!\left[\log \frac{\exp(\mathrm{sim}(f(x), f(x^+))/\tau)}{\sum_{x^-} \exp(\mathrm{sim}(f(x), f(x^-))/\tau)}\right]. \]

Contrastive learning is not, strictly speaking, generative — it does not give you \(p(x)\) — but it is closely related: it is mutual-information maximization between views, and it produces representations that often work as drop-in features for downstream tasks. CLIP combines contrastive learning over (image, text) pairs to produce embeddings used in diffusion-model conditioning. The connection to generative modeling is the energy-based view below.

Principle 6 — Energy-based view. An energy-based model (EBM) writes the density as a normalized exponential:

\[ p_\theta(x) \;=\; \frac{\exp(-E_\theta(x))}{Z(\theta)}, \qquad Z(\theta) = \int \exp(-E_\theta(x'))\, dx'. \]

\(E_\theta : \mathbb{R}^d \to \mathbb{R}\) is a freely parametrized neural net (no normalization constraint). The headache is that \(Z(\theta)\) is intractable. Training is done with surrogates: contrastive divergence, score matching (avoid \(Z\) entirely by working with \(\nabla_x \log p_\theta\)), noise contrastive estimation, or by sampling via Langevin dynamics. Score-based diffusion models are formally a particular kind of energy-based model whose score is parametrized directly — that is why “denoising score matching” and “diffusion models” turn out to be the same thing.

Worked toy — MLE on a 1-D Gaussian. Suppose \(p_{\text{data}} = \mathcal{N}(\mu_*, 1)\), and the model is \(p_\theta = \mathcal{N}(\theta, 1)\). With \(N\) samples \(x^{(i)}\),

\[ \log p_\theta(x^{(i)}) = -\tfrac{1}{2}(x^{(i)} - \theta)^2 - \tfrac{1}{2}\log(2\pi). \]

Setting the derivative of the sum to zero,

\[ \sum_i (x^{(i)} - \theta) = 0 \;\;\Rightarrow\;\; \hat\theta_{\text{MLE}} = \frac{1}{N}\sum_i x^{(i)}. \]

This sample mean is the canonical MLE result. The same logic, with vastly more parameters, drives every likelihood-based deep generative model.

Common assessment trap — vocabulary.

  • Density estimation and generative modeling are nearly synonyms in this guide; “estimation” emphasizes the statistical view, “modeling” the engineering view.
  • Likelihood and evidence both mean \(p_\theta(x)\); the ELBO is a lower bound on \(\log p_\theta(x)\).
  • Latent variable \(z\) is unobserved by definition; a feature or embedding is similar in spirit but not necessarily probabilistic.
  • Unsupervised learning has no labels; self-supervised learning constructs pseudo-labels from the data itself (e.g., next-token prediction). Generative pretraining (GPT) is unsupervised in the classical sense and self-supervised in the modern sense.

MODULE 2: Introducing Main Generative Architectures

2.1 Core Types of Deep Generative Models

This is the menagerie. Each family has a defining trick — a way of writing \(p(x)\) or sampling from it that side-steps the intractable normalization constant. We’ll see each in much more depth later; here is the inventory, with the one-line idea you must keep straight on the assessment.

Autoregressive models (AR). Factorize \(p(x) = \prod_t p(x_t \mid x_{<t})\) and learn each conditional with a neural net. Tractable likelihood, slow sequential sampling. Examples: PixelRNN/PixelCNN (images, pixel-by-pixel), WaveNet (audio, sample-by-sample), GPT-style decoders (text, token-by-token), ImageGPT, Parti, Muse, MaskGIT (image tokens), MusicGen, VALL-E (audio codec tokens).

Variational Autoencoders (VAE). Latent-variable model with \(z\sim \mathcal{N}(0,I)\), decoder \(p_\theta(x\mid z)\), encoder \(q_\phi(z\mid x)\). Optimize the ELBO. Reparameterization trick makes gradients flow through stochastic \(z\). Closed-form Gaussian-Gaussian KL. Variants: β-VAE (disentanglement), VQ-VAE / VQ-VAE-2 (discrete latents, the workhorse for image tokenizers feeding latent diffusion / autoregressive transformers), NVAE, VDVAE. Strengths: fast sampling, interpretable latents, decent likelihoods, stable training. Weaknesses: samples tend to be blurry because the Gaussian decoder averages plausible reconstructions.

Generative Adversarial Networks (GAN). Train a generator \(G\) that maps noise \(z\) to \(G(z)\) and a discriminator \(D\) that distinguishes real from generated. Minimax game:

\[ \min_G \max_D \; \mathbb{E}_{x\sim p_{\text{data}}}\!\left[\log D(x)\right] + \mathbb{E}_{z\sim p(z)}\!\left[\log(1 - D(G(z)))\right]. \]

No explicit likelihood, but sharp samples. Variants: DCGAN (deep conv), WGAN/WGAN-GP (Wasserstein, gradient penalty), Progressive GAN, StyleGAN1/2/3 (state-of-the-art faces), BigGAN (class-conditional ImageNet), CycleGAN/Pix2Pix (image translation), ESRGAN (super-resolution), HiFi-GAN (audio), VQ-GAN (used as a tokenizer for autoregressive image models).

Normalizing flows. Define an invertible neural net \(f_\theta : z \to x\) with \(z\sim p(z)\). By the change-of-variables formula,

\[ \log p_\theta(x) \;=\; \log p\!\left(f_\theta^{-1}(x)\right) + \log\left|\det \frac{\partial f_\theta^{-1}}{\partial x}\right|. \]

Exact likelihood, exact inversion, free sampling. The art is in designing layers that are invertible and have tractable Jacobian determinants. Examples: NICE (additive coupling), RealNVP (affine coupling), Glow (1×1 invertible conv + affine coupling), MAF/IAF (autoregressive flows), FFJORD / Neural ODE (continuous-time flow), CTM/CNF (continuous normalizing flow).

Diffusion / score-based models. Define a forward process that gradually adds Gaussian noise to a data point over \(T\) steps, and learn the reverse process that denoises. The objective reduces to predicting the noise (or the data, or the velocity) from a noisy sample at a random step. The key papers fall into two equivalent formulations:

  • DDPM (Ho et al., 2020) — discrete-time, ELBO derivation, \(\epsilon\)-prediction.
  • NCSN (Song & Ermon, 2019) and score-based SDE (Song et al., 2021) — continuous-time stochastic differential equation; train a score network \(s_\theta(x,t) \approx \nabla_x \log p_t(x)\).

Variants: Improved DDPM (cosine schedule, learned variance, importance-weighted loss), ADM / Guided Diffusion (UNet improvements, classifier guidance), Latent Diffusion / Stable Diffusion (run diffusion in a VAE’s latent space), DDIM, DPM-Solver, EDM (fast and deterministic samplers), Imagen (text-to-image with T5), DiT (diffusion transformer), Cascaded Diffusion (low-res → super-res chain).

Energy-based models (EBM). \(p_\theta(x) = \exp(-E_\theta(x))/Z\) with intractable \(Z\). Trained by contrastive divergence (MCMC + sample comparison), score matching, or noise-contrastive estimation. Examples: classical Restricted Boltzmann Machines, modern deep EBMs (Du & Mordatch, “Implicit Generation and Modeling with Energy-Based Models”). EBMs are conceptually general but harder to train at scale than diffusion; nevertheless the score-based formulation has rehabilitated them — a diffusion model is, mathematically, a denoising score-matching EBM.

Consistency models. Introduced by Song et al., 2023 (“Consistency Models”). Learn a function \(f_\theta(x_t, t)\) that maps any noisy point on a probability-flow ODE trajectory back to the same starting point. The result is a model that can generate samples in one step (or a small number of steps) while remaining trainable from scratch or distilled from a diffusion teacher. Variants: Latent Consistency Models (LCM), CM++, Stable Diffusion XL Turbo (distilled consistency-like models for one-step text-to-image).

Flow matching / rectified flow. Learn a velocity field \(v_\theta(x_t, t)\) that transports samples from a simple prior \(p_0\) (Gaussian) to the data distribution \(p_1 = p_{\text{data}}\) along an ODE \[ \frac{dx}{dt} = v_\theta(x, t). \] Flow Matching (Lipman et al., 2022) trains \(v_\theta\) by regressing it on a closed-form conditional velocity along straight-line paths between samples and Gaussians. Rectified Flow (Liu et al., 2022) does the same with straighter trajectories that allow few-step sampling. Stable Diffusion 3 and many recent image models are rectified-flow models, not “diffusion” in the original DDPM sense — but the practical machinery (UNet/DiT, samplers, CFG) is essentially the same.

2.2 Key Use Cases for Each Generative Model Type

This subsection answers the recurring assessment question “which model family fits this use case?” It restates the families from 2.1 in terms of where each one is the standard choice today.

Autoregressive models — when to reach for them.

  • Discrete tokens (text, code, image tokens, audio codec tokens). The factorization is natural; no need for a continuous Gaussian likelihood.
  • Tasks where you want exact likelihood and likelihood-based decisions (compression, anomaly detection on sequences).
  • Long context, sequential streaming generation where you can pay the per-step cost (LLM chat).
  • Image and audio at low resolution / token-level (PixelCNN, WaveNet) when latency budgets are generous.
  • Not the first choice for high-resolution images and video at modern quality — diffusion / latent diffusion / flow matching dominate there because their parallel score predictions are much faster than per-pixel autoregression.

VAEs — when to reach for them.

  • Representation learning with a smooth latent space (interpolation, attribute manipulation, β-VAE for disentanglement).
  • Fast, stable, latent-variable models for moderate-fidelity generation (faces at low resolution, tabular synthesis, anomaly detection with a likelihood proxy).
  • As a tokenizer / latent codec for other generative models (VQ-VAE → autoregressive tokens for DALL·E, Parti; KL-VAE → latent diffusion for Stable Diffusion). This is, in 2026 practice, by far the biggest “use case” for VAE-style autoencoders: they compress images into a smaller latent space where the real generative model (diffusion, autoregressive transformer) operates.
  • Conditional VAEs (CVAE) for one-to-many prediction tasks (motion forecasting, missing-data imputation).

GANs — when to reach for them.

  • High-resolution face and portrait synthesis (StyleGAN family is still extremely strong here at modest compute).
  • Domain translation with cycle consistency (CycleGAN: horse↔︎zebra; UNIT/MUNIT).
  • Paired image-to-image translation (Pix2Pix: edges→shoes, segmentation→photo).
  • Super-resolution with sharp outputs (ESRGAN, Real-ESRGAN).
  • Audio vocoders (HiFi-GAN, MelGAN) where a few-millisecond inference and sharp waveforms matter.
  • As a tokenizer (VQ-GAN) feeding autoregressive transformers (Taming Transformers / Parti / MaskGIT pipeline).
  • Anywhere you need a 1-step generator with sharp output, you don’t need likelihood, and you can engineer around training instability.
  • Less competitive in 2026 as a general text-to-image model — diffusion / flow matching are state-of-the-art there.

Normalizing flows — when to reach for them.

  • Exact likelihood requirements (scientific applications, physics simulation surrogates, density estimation with calibrated uncertainty).
  • Invertible representations for normalizing-flow posteriors in Bayesian inference (variational inference with flows).
  • Lossless compression (bits-back coding with flow likelihoods).
  • Tabular density estimation, particle physics, cosmology — anywhere \(p(x)\) itself is the deliverable.
  • Continuous-time flows / Neural ODEs for time-series generative modeling.
  • Less competitive for natural images at high resolution — flow architectures struggle to match diffusion’s quality at the same compute, although flow matching (which is not exactly the same as a classical normalizing flow but shares the change-of-variables flavor) has now closed much of the gap.

Diffusion and score-based models — when to reach for them.

  • Text-to-image, text-to-video, image-to-image at modern quality (Stable Diffusion, Imagen, DALL·E 3, Sora).
  • Conditional image generation with rich control (ControlNet, T2I-Adapter, IP-Adapter, inpainting, SDEdit).
  • Audio diffusion for TTS, music, sound effects (AudioLDM, NaturalSpeech 2/3, Diff-TTS).
  • 3-D and protein structure generation (RFdiffusion, Chroma, EDM-for-molecules).
  • Solving inverse problems (super-resolution, inpainting, MRI reconstruction) via conditional samplers.
  • Anywhere mode coverage matters and you can pay 20–50 denoising steps, or you can distill to consistency / few-step models for latency.

Energy-based models — when to reach for them.

  • Composable conditioning (combine multiple energy terms additively to enforce constraints).
  • Theoretical / principled modeling when you want \(\nabla_x \log p(x)\) for sampling (Langevin) or downstream optimization (test-time guidance).
  • As the conceptual foundation for score-based diffusion: every diffusion model is “an energy-based model parametrized by its score”.
  • Less common as a standalone deployed system in 2026 because diffusion + consistency + flow matching cover the same theoretical ground with better optimization properties.

Consistency models — when to reach for them.

  • One-step or few-step image generation with quality close to a 20–50-step diffusion teacher (LCM, SDXL Turbo).
  • Real-time interactive generation (drawing apps, live preview).
  • Latency-critical text-to-image in mobile and edge deployments.

Flow matching / rectified flow — when to reach for them.

  • Modern image / video generative models (Stable Diffusion 3, Movie Gen, parts of Sora’s family).
  • Few-step sampling with straight-line probability paths.
  • Whenever you would have reached for diffusion but want simpler training (no noise schedule tuning) and faster inference.

Common assessment trap — vocabulary.

  • Likelihood vs ELBO: ELBO is a lower bound on log-likelihood. A model can have a high ELBO and still have a higher true likelihood.
  • Mode collapse (GAN failure) vs mode covering (likelihood-based behavior): mode collapse means the generator concentrates on a few modes and misses the rest; mode covering means the model spreads probability across all modes and may oversmooth them.
  • Sample quality vs likelihood: not the same thing. Flow models can have higher likelihood than GANs while producing visually worse samples; GANs can produce sharper images while having no defined likelihood at all. Diffusion models tend to do well on both, but FID and NLL are not perfectly correlated.

MODULE 3: Designing Diffusion Model Processes

3.1 The Forward Diffusion Process and Noise Addition

Definition. The forward diffusion (also called the noising or destruction) process takes a clean data point \(x_0 \sim p_{\text{data}}\) and gradually corrupts it over \(T\) steps into Gaussian noise. It is a fixed (not learned) Markov chain

\[ q(x_t \mid x_{t-1}) \;=\; \mathcal{N}\!\left(x_t;\; \sqrt{1-\beta_t}\, x_{t-1},\; \beta_t\, I\right), \qquad t = 1, \dots, T, \]

with a variance schedule \(0 < \beta_1 < \beta_2 < \dots < \beta_T < 1\). The schedule is what we’ll discuss in 3.3.

Why this particular Gaussian transition? Two reasons. First, Gaussians compose: a Gaussian step followed by another Gaussian step is still Gaussian, so \(q(x_t \mid x_0)\) has a closed form. Second, the scaling factor \(\sqrt{1-\beta_t}\) on \(x_{t-1}\) keeps the marginal variance bounded (so the process doesn’t blow up).

The closed form \(q(x_t \mid x_0)\). Define \(\alpha_t = 1 - \beta_t\) and the cumulative product \(\bar\alpha_t = \prod_{s=1}^t \alpha_s\). By induction,

\[ q(x_t \mid x_0) \;=\; \mathcal{N}\!\left(x_t;\; \sqrt{\bar\alpha_t}\, x_0,\; (1-\bar\alpha_t)\, I\right). \]

The “reparameterized” form is the one to memorize:

\[ x_t \;=\; \sqrt{\bar\alpha_t}\, x_0 + \sqrt{1-\bar\alpha_t}\, \epsilon, \qquad \epsilon \sim \mathcal{N}(0, I). \]

This single equation tells you how to construct a training example: pick a clean image, pick a random timestep \(t\), pick a random \(\epsilon\), and produce \(x_t\) as the noisy version. No simulation of the full Markov chain is required — you can jump straight to step \(t\) in one shot. This is the most important equation in diffusion modeling.

Derivation by induction. Assume \(q(x_{t-1}\mid x_0) = \mathcal{N}(\sqrt{\bar\alpha_{t-1}}\, x_0, (1-\bar\alpha_{t-1}) I)\). Then \(x_{t-1} = \sqrt{\bar\alpha_{t-1}}\, x_0 + \sqrt{1-\bar\alpha_{t-1}}\, \eta_{t-1}\) with \(\eta_{t-1}\sim\mathcal{N}(0,I)\). By the forward step,

\[ x_t = \sqrt{\alpha_t}\, x_{t-1} + \sqrt{\beta_t}\, \eta_t, \qquad \eta_t\sim\mathcal{N}(0,I). \]

Substituting and using \(\alpha_t \cdot (1-\bar\alpha_{t-1}) + \beta_t = 1 - \alpha_t \bar\alpha_{t-1} = 1 - \bar\alpha_t\):

\[ x_t = \sqrt{\alpha_t \bar\alpha_{t-1}}\, x_0 + \sqrt{1 - \alpha_t\bar\alpha_{t-1}}\,\tilde\eta = \sqrt{\bar\alpha_t}\, x_0 + \sqrt{1-\bar\alpha_t}\,\tilde\eta. \]

The two Gaussians combine by sum of independent variances, completing the induction.

Limit behaviour. As \(T \to \infty\) (or large enough), \(\bar\alpha_T \to 0\) and \(q(x_T\mid x_0) \to \mathcal{N}(0, I)\). The data has been forgotten; only Gaussian noise remains. This is the “prior” \(p(x_T)\) that the reverse process will start from at sampling time.

Properties of the forward process.

  • It is fixed, not learned. No trainable parameters in \(q\).
  • It is Markov: \(x_t\) depends only on \(x_{t-1}\).
  • It is Gaussian at every step, with linear mean and isotropic covariance.
  • Conditioned on \(x_0\), every \(x_t\) is jointly Gaussian, so any conditional like \(q(x_{t-1}\mid x_t, x_0)\) is also Gaussian (we’ll use this in 3.2 to write the posterior).

Worked example. Use a linear schedule with \(\beta_t\) varying from \(10^{-4}\) to \(0.02\) over \(T = 4\) steps:

\(t\) \(\beta_t\) \(\alpha_t = 1-\beta_t\) \(\bar\alpha_t\)
1 0.0001 0.9999 0.9999
2 0.0067 0.9933 0.99323
3 0.01335 0.98665 0.98000
4 0.02 0.98 0.96040

(Exact arithmetic: \(\bar\alpha_4 = 0.9999 \times 0.9933 \times 0.98665 \times 0.98\).) If \(x_0 = 1.0\) and we pick \(\epsilon = 0.3\),

\[ x_4 = \sqrt{0.96040}\cdot 1.0 + \sqrt{1 - 0.96040}\cdot 0.3 \approx 0.98 + 0.199 \cdot 0.3 \approx 1.04. \]

For a real-world \(T=1000\) schedule, \(\bar\alpha_T\) is much closer to \(0\) and \(x_T\) is essentially \(\mathcal{N}(0, I)\).

Minimal PyTorch — adding noise.

import torch

def add_noise(x0: torch.Tensor, t: torch.Tensor,
              sqrt_alpha_bar: torch.Tensor, sqrt_one_minus_alpha_bar: torch.Tensor):
    # x0:                       (B, C, H, W)  clean images, in [-1, 1]
    # t:                        (B,)          int64 timesteps in [0, T-1]
    # sqrt_alpha_bar:           (T,)          precomputed sqrt(alpha_bar_t)
    # sqrt_one_minus_alpha_bar: (T,)          precomputed sqrt(1 - alpha_bar_t)
    eps = torch.randn_like(x0)                # (B, C, H, W)
    a = sqrt_alpha_bar[t].view(-1, 1, 1, 1)   # (B, 1, 1, 1)
    b = sqrt_one_minus_alpha_bar[t].view(-1, 1, 1, 1)
    xt = a * x0 + b * eps                     # (B, C, H, W)
    return xt, eps

The two outputs xt and eps are exactly what the training loop needs: the noisy input and the ground-truth noise to predict.

3.2 The Reverse Denoising Process

Definition. The reverse process inverts the forward chain: starting from \(x_T \sim \mathcal{N}(0, I)\), take \(T\) Gaussian denoising steps to recover something distributed like \(x_0\). We parametrize it as

\[ p_\theta(x_{t-1} \mid x_t) \;=\; \mathcal{N}\!\left(x_{t-1};\; \mu_\theta(x_t, t),\; \Sigma_\theta(x_t, t)\right). \]

In DDPM (Ho et al., 2020), the covariance \(\Sigma_\theta\) is fixed (often \(\sigma_t^2 I\) with \(\sigma_t^2 = \beta_t\) or the optimal \(\tilde\beta_t\) below). Only the mean \(\mu_\theta(x_t, t)\) is learned. Improved DDPM and ADM let \(\Sigma_\theta\) be learned as an interpolation between two bounds.

Why Gaussian? When \(\beta_t\) is small, the true reverse \(q(x_{t-1}\mid x_t)\) is approximately Gaussian (a tangent-space approximation). With \(T\) large enough that each step is tiny, the Gaussian parametrization is provably very close to optimal.

The forward posterior \(q(x_{t-1}\mid x_t, x_0)\). This is where the algebra pays off. Conditioning on \(x_0\) makes the reverse step exactly Gaussian:

\[ q(x_{t-1}\mid x_t, x_0) \;=\; \mathcal{N}\!\left(x_{t-1};\; \tilde\mu_t(x_t, x_0),\; \tilde\beta_t\, I\right), \]

where

\[ \tilde\mu_t(x_t, x_0) \;=\; \frac{\sqrt{\bar\alpha_{t-1}}\,\beta_t}{1-\bar\alpha_t}\, x_0 + \frac{\sqrt{\alpha_t}\,(1-\bar\alpha_{t-1})}{1-\bar\alpha_t}\, x_t, \]

\[ \tilde\beta_t \;=\; \frac{1-\bar\alpha_{t-1}}{1-\bar\alpha_t}\,\beta_t. \]

These are the closed-form Gaussian conjugate-posterior formulas. Derivation: write the joint \(q(x_{t-1}, x_t\mid x_0)\) as a product of Gaussians, complete the square in \(x_{t-1}\).

ε-parametrization of \(\mu_\theta\). Plug \(x_0 = (x_t - \sqrt{1-\bar\alpha_t}\,\epsilon)/\sqrt{\bar\alpha_t}\) into the formula for \(\tilde\mu_t\). After algebra (use \(\beta_t = 1-\alpha_t\) and \(\bar\alpha_t = \alpha_t\bar\alpha_{t-1}\)):

\[ \tilde\mu_t(x_t, x_0) \;=\; \frac{1}{\sqrt{\alpha_t}}\!\left(x_t - \frac{\beta_t}{\sqrt{1-\bar\alpha_t}}\,\epsilon\right). \]

If we let the network predict \(\epsilon\) (call it \(\epsilon_\theta(x_t, t)\)), we get a clean expression for \(\mu_\theta\):

\[ \mu_\theta(x_t, t) \;=\; \frac{1}{\sqrt{\alpha_t}}\!\left(x_t - \frac{\beta_t}{\sqrt{1-\bar\alpha_t}}\,\epsilon_\theta(x_t, t)\right). \]

This is the heart of DDPM. Predict the noise, plug it into the closed-form posterior mean, sample a small Gaussian noise on top. That’s the entire reverse step.

Variational lower bound for the reverse process. The full ELBO for diffusion is

\[ \mathcal{L}_{\text{vlb}} = \mathbb{E}_q\!\left[ \underbrace{\mathrm{KL}(q(x_T\mid x_0)\Vert p(x_T))}_{L_T \approx \text{const}} + \sum_{t>1} \underbrace{\mathrm{KL}(q(x_{t-1}\mid x_t,x_0) \Vert p_\theta(x_{t-1}\mid x_t))}_{L_{t-1}} - \underbrace{\log p_\theta(x_0\mid x_1)}_{L_0}\right]. \]

The dominant terms are the \(L_{t-1}\) KL divergences between two Gaussians with matched variance, which (for fixed \(\Sigma_\theta\)) reduce to a weighted MSE between \(\tilde\mu_t\) and \(\mu_\theta\). After substituting the ε-parametrization above, the weighted MSE between \(\tilde\mu_t\) and \(\mu_\theta\) becomes a weighted MSE between true noise \(\epsilon\) and predicted noise \(\epsilon_\theta\).

The simplified loss \(L_{\text{simple}}\) (Ho et al., 2020). Ho et al. observed that dropping the time-dependent weight and using a plain MSE on noise gives better samples in practice:

\[ \boxed{\;\mathcal{L}_{\text{simple}}(\theta) \;=\; \mathbb{E}_{t,\,x_0,\,\epsilon}\!\left[\,\|\,\epsilon - \epsilon_\theta(x_t, t)\,\|^2\,\right],\;} \]

where \(t\sim \mathrm{Uniform}\{1,\dots,T\}\), \(x_0 \sim p_{\text{data}}\), \(\epsilon \sim \mathcal{N}(0, I)\), and \(x_t = \sqrt{\bar\alpha_t}\,x_0 + \sqrt{1-\bar\alpha_t}\,\epsilon\).

That is the only loss most practitioners actually use. It is equivalent (up to weighting) to denoising score matching (Song & Ermon, 2019; Vincent, 2011):

\[ \mathbb{E}_{t,x_0,\epsilon}\!\left[\lambda(t)\,\|\,s_\theta(x_t, t) - \nabla_{x_t} \log q(x_t\mid x_0)\,\|^2\right], \]

with the identity \(\nabla_{x_t} \log q(x_t\mid x_0) = -\epsilon/\sqrt{1-\bar\alpha_t}\) tying \(\epsilon\)-prediction and score prediction together: \(s_\theta(x_t,t) = -\epsilon_\theta(x_t, t)/\sqrt{1-\bar\alpha_t}\).

One reverse step in pseudocode.

Algorithm: DDPM reverse step  (Ho et al., 2020)
Input: x_t, t, network eps_theta, schedule { alpha_t, alpha_bar_t, beta_t, sigma_t }
1. Compute predicted noise: eps_hat = eps_theta(x_t, t)
2. Compute mean:
       mu = (1 / sqrt(alpha_t)) * (x_t - (beta_t / sqrt(1 - alpha_bar_t)) * eps_hat)
3. If t > 1: z ~ N(0, I)  else  z = 0
4. x_{t-1} = mu + sigma_t * z
Return x_{t-1}

Sampling loop pseudocode.

Algorithm 2: DDPM sampling
Input: trained eps_theta, schedule, T
1. x_T ~ N(0, I)
2. for t = T, T-1, ..., 1:
     x_{t-1} = DDPM_reverse_step(x_t, t, eps_theta, schedule)
3. return x_0

Worked numerical example — one reverse step. Take \(t=4\) from the schedule in 3.1 (\(\beta_4 = 0.02\), \(\alpha_4 = 0.98\), \(\bar\alpha_4 = 0.96040\)), and suppose \(x_4 = 0.50\), \(\epsilon_\theta(x_4, 4) = 0.10\).

  • Coefficient on \(\epsilon\): \(\beta_4 / \sqrt{1-\bar\alpha_4} = 0.02/\sqrt{0.0396} = 0.02/0.199 \approx 0.1005\).
  • \(\mu = (1/\sqrt{0.98})(0.50 - 0.1005 \cdot 0.10) = (1.0102)(0.50 - 0.01005) = 1.0102 \cdot 0.48995 \approx 0.4949\).
  • \(\sigma_4 = \sqrt{\beta_4} = \sqrt{0.02} \approx 0.1414\) (DDPM uses \(\sigma_t^2 = \beta_t\), though \(\tilde\beta_t\) is the other principled choice).
  • Sample \(z \sim \mathcal{N}(0,1)\), say \(z = -0.2\): \(x_3 = 0.4949 + 0.1414\cdot(-0.2) \approx 0.4666\).

That single number \(0.4666\) is one denoising step. In a real image model the same operation runs in parallel across every pixel and channel.

Common assessment trap — vocabulary.

  • ε-prediction (network outputs noise): the most common choice in DDPM.
  • \(x_0\)-prediction (network outputs the clean image directly): mathematically equivalent up to an algebraic re-arrangement, sometimes used for short-horizon training and conditioning.
  • \(v\)-prediction (Salimans & Ho, 2022): \(v = \sqrt{\bar\alpha_t}\,\epsilon - \sqrt{1-\bar\alpha_t}\, x_0\). Better-behaved at small SNR (very noisy steps) and at the boundary \(t \to T\); used in Imagen, Stable Diffusion 2.x, and most modern image models.
  • Forward process (noising): fixed, no parameters.
  • Reverse process (denoising): learned, parametrized by the network.

3.3 Noise Scheduling Strategies and Their Impact

The noise schedule \(\{\beta_t\}_{t=1}^T\) (equivalently \(\{\bar\alpha_t\}\) or the signal-to-noise ratio \(\mathrm{SNR}(t) = \bar\alpha_t/(1-\bar\alpha_t)\)) is one of the three knobs that change a diffusion model the most (the other two being architecture and parameterization). The schedule controls where the model spends its capacity along the noising trajectory.

Why the schedule matters.

  • Each \(t\) in \(\{1,\dots,T\}\) defines a sub-task: “given noise level \(\mathrm{SNR}(t)\), predict the noise”. Some sub-tasks are easy (very low or very high noise) and some are hard (intermediate noise). The schedule decides how often the model trains on each sub-task and how aggressive each step is at inference time.
  • A poorly chosen schedule wastes capacity at \(t\) where there is nothing useful to learn (e.g., \(t \approx 0\) where the network just has to copy the input, or \(t \approx T\) where the input is already nearly pure noise).
  • The schedule also determines whether \(x_T\) is truly Gaussian. If \(\bar\alpha_T\) is not small enough, \(x_T\) retains a faint trace of \(x_0\), and starting sampling from pure \(\mathcal{N}(0,I)\) creates a train/test mismatch.

Linear schedule (DDPM, 2020). \(\beta_t\) is a linear interpolation between \(\beta_{\min}\) and \(\beta_{\max}\), typically \(\beta_{\min}=10^{-4}\), \(\beta_{\max}=0.02\), \(T=1000\):

\[ \beta_t = \beta_{\min} + \frac{t-1}{T-1}(\beta_{\max} - \beta_{\min}). \]

Pros: simple, well-tested, gives \(\bar\alpha_T \approx 4\times 10^{-5}\) for \(T=1000\) which is close enough to \(0\). Cons: a lot of timesteps near \(t=T\) have almost no signal to extract; the model wastes capacity there. Especially noticeable at higher resolutions (more pixels destroyed faster).

Cosine schedule (Nichol & Dhariwal, “Improved Denoising Diffusion Probabilistic Models”, 2021). Define \(\bar\alpha_t\) directly via a cosine curve:

\[ \bar\alpha_t \;=\; \frac{f(t)}{f(0)}, \qquad f(t) \;=\; \cos^2\!\left(\frac{t/T + s}{1 + s}\cdot \frac{\pi}{2}\right), \]

with a small offset \(s = 0.008\) to avoid singularities. Then \(\beta_t = 1 - \bar\alpha_t/\bar\alpha_{t-1}\) (clipped to a maximum like \(0.999\)).

Effects:

  • \(\bar\alpha_t\) drops more slowly at small \(t\) and faster only near \(t=T\) — i.e., the easy late steps are skipped more aggressively.
  • The intermediate \(\mathrm{SNR}\) range is sampled more densely → the model learns the hard, useful denoising regime better.
  • Empirically improves both NLL and sample quality at the same compute. Especially important at \(T < 1000\), where the linear schedule is too aggressive at early steps.

Sigmoid schedule. Define \(\beta_t\) along a sigmoid in \(t/T\) between \(\beta_{\min}\) and \(\beta_{\max}\). Smoothly transitions, in between linear and cosine. Useful when you want a more S-shaped progression for \(\bar\alpha_t\).

Karras EDM schedule (Karras et al., “Elucidating the Design Space of Diffusion-Based Generative Models”, 2022). EDM reparametrizes everything in terms of the noise standard deviation \(\sigma\) rather than discrete \(t\). The noise levels at sampling time are

\[ \sigma_i \;=\; \left(\sigma_{\max}^{1/\rho} + \frac{i}{N-1}(\sigma_{\min}^{1/\rho} - \sigma_{\max}^{1/\rho})\right)^\rho, \qquad i = 0, \dots, N-1, \]

with \(\sigma_{\min}\approx 0.002\), \(\sigma_{\max}\approx 80\), \(\rho \approx 7\). The exponent \(\rho\) concentrates the steps at small noise (where the image is almost clean but the most perceptually important details are being painted in). Combined with EDM’s preconditioning of the network and second-order Heun sampler, this schedule yields state-of-the-art ImageNet FID at very low step counts.

Effect on SNR — the right way to read a schedule.

Define the log-signal-to-noise ratio

\[ \lambda(t) \;=\; \log \mathrm{SNR}(t) \;=\; \log \frac{\bar\alpha_t}{1-\bar\alpha_t}. \]

The training objective \(\mathcal{L}_{\text{simple}}\) is implicitly uniform in \(t\), but uniform in \(t\) is not uniform in \(\lambda\). A “balanced” schedule is one where the distribution of \(\lambda\) is approximately uniform over the meaningful range — this is what the cosine and EDM schedules give you. The Karras “EDM” framework also reweights the loss by a \(\sigma\)-dependent function (min-SNR weighting; see 5.2) to further balance contributions across noise levels.

Schedule comparison table.

Schedule Definition Pros Cons Typical use
Linear \(\beta_t\) linear in \(t\) Simple, robust Wasted capacity at high \(t\) Original DDPM (Ho et al., 2020)
Cosine \(\bar\alpha_t = \cos^2(\cdot)\) Better SNR coverage Slightly more parameters to set (\(s\)) Improved DDPM, ADM, Stable Diffusion
Sigmoid \(\beta_t = \mathrm{sigmoid}(\cdot)\) Smooth, S-shape Less explored Some research models
Karras / EDM \(\sigma\)-based, \(\rho\)-power SOTA FID at low steps Reparametrization, separate sampler EDM, modern image models

Choosing between linear and cosine in practice.

if T >= 1000 and resolution is small (32x32, 64x64):
    a linear schedule works fine
elif T <= 500 or resolution is large (256+):
    use a cosine (or Karras) schedule
if you want SOTA on ImageNet / generic images:
    use the Karras (EDM) schedule + EDM preconditioning + Heun sampler

The shorthand: linear is the easy default; cosine is the strict upgrade; Karras/EDM is the modern best practice.

3.4 Implementing Linear and Cosine Schedules for Tasks

This subsection turns 3.3 into actual code, because the assessment may ask you to identify or implement a schedule snippet.

Linear schedule — implementation.

import torch

def make_linear_schedule(T: int, beta_start: float = 1e-4, beta_end: float = 2e-2):
    """
    Returns dict with tensors of shape (T,):
      beta, alpha, alpha_bar, sqrt_alpha_bar, sqrt_one_minus_alpha_bar
    """
    beta = torch.linspace(beta_start, beta_end, T)        # (T,)
    alpha = 1.0 - beta                                    # (T,)
    alpha_bar = torch.cumprod(alpha, dim=0)               # (T,)
    return {
        "beta": beta,
        "alpha": alpha,
        "alpha_bar": alpha_bar,
        "sqrt_alpha_bar": torch.sqrt(alpha_bar),
        "sqrt_one_minus_alpha_bar": torch.sqrt(1.0 - alpha_bar),
    }

Cosine schedule — implementation.

import math
import torch

def make_cosine_schedule(T: int, s: float = 0.008):
    """
    Improved DDPM (Nichol & Dhariwal, 2021) cosine schedule.
    Returns the same dict shape as make_linear_schedule.
    """
    steps = torch.arange(T + 1, dtype=torch.float64) / T          # (T+1,)
    f = torch.cos((steps + s) / (1 + s) * math.pi / 2) ** 2       # (T+1,)
    alpha_bar = (f / f[0]).clamp(min=1e-12)                       # (T+1,)
    # beta_t = 1 - alpha_bar_t / alpha_bar_{t-1}, clipped for stability
    beta = 1.0 - alpha_bar[1:] / alpha_bar[:-1]                   # (T,)
    beta = beta.clamp(0.0, 0.999).float()
    alpha = 1.0 - beta
    alpha_bar = torch.cumprod(alpha, dim=0)
    return {
        "beta": beta,
        "alpha": alpha,
        "alpha_bar": alpha_bar,
        "sqrt_alpha_bar": torch.sqrt(alpha_bar),
        "sqrt_one_minus_alpha_bar": torch.sqrt(1.0 - alpha_bar),
    }

Task-driven selection.

  • Low-resolution image dataset, T=1000, prototype quickly → linear.
  • High-resolution image dataset, expecting cosine/EDM-style modern training → cosine.
  • Few-step image generator, EDM-style or distillation target → Karras \(\sigma\)-schedule and \(v\)-prediction.
  • Audio / waveform diffusion at high resolution → cosine or a tailored schedule (e.g., variance-preserving cosine; DiffWave originally used linear).

Diagnostics — how to tell if your schedule is “wrong”.

  • Plot \(\log \mathrm{SNR}(t)\) vs \(t\). Healthy curves cover a wide range of \(\lambda \in [-10, +10]\) (or similar) with most mass around \(\lambda \approx 0\), where the denoising task is hardest and most informative.
  • If the model’s per-\(t\) training loss is very low for the highest \(t\), it’s not learning anything from those timesteps — the schedule is wasting them.
  • If samples have a characteristic “noisy at the seams between regions” look, the schedule is dropping signal too late and the late steps are not contributing detail.

MODULE 4: Constructing Diffusion Model Architectures

4.1 Designing U-Net-Based Architectures

Why U-Net? The reverse process \(\mu_\theta(x_t, t)\) takes a noisy image and outputs another image of the same shape. The task is image-to-image, with strong locality (denoising is mostly a local operation, modulated by global context like content and lighting). The U-Net (Ronneberger et al., 2015, originally for biomedical segmentation) is the canonical architecture for this:

  • Symmetric encoder–decoder with progressively halved spatial resolution and doubled channels going down, then mirrored going up.
  • Skip connections between matching encoder/decoder levels preserve high-frequency detail that would otherwise be destroyed by downsampling.
  • Easy to inject time and conditioning at every level via normalization or addition.

Block diagram (text form).

input image (B, C, H, W)
   │
   ├── stem conv -> (B, C0, H, W)
   │
   ▼
[ResBlock + (optional Attention)] x N0    -- level 0
   │ downsample
   ▼
[ResBlock + (optional Attention)] x N1    -- level 1, (B, C1, H/2, W/2)
   │ downsample
   ▼
[ResBlock + Attention] x N2               -- level 2, (B, C2, H/4, W/4)
   │ downsample
   ▼
[ResBlock + Attention] x N3               -- bottleneck (B, C3, H/8, W/8)
   │
   │ upsample ──── skip connection from level 2 ─┐
   ▼                                              │
[ResBlock + Attention] x N2                       │
   │                                              │
   │ upsample ──── skip from level 1 ─────────────┐
   ▼                                              │
[ResBlock + Attention] x N1                       │
   │                                              │
   │ upsample ──── skip from level 0 ─────────────┐
   ▼                                              │
[ResBlock + (optional Attention)] x N0
   │
   ▼
GroupNorm + SiLU + Conv -> (B, C_out, H, W)

Channel and depth choices.

  • \(C_0 \in \{64, 128, 192, 256\}\) typical; doubling per downsampling stage (or a multiplier list like [1,2,2,4,4]).
  • Number of ResBlocks per stage: 2–4. Heavier networks (Imagen, ADM, Stable Diffusion) use 2–3.
  • Attention at low resolutions (e.g., 16×16, 8×8) only — full self-attention on a 64×64 feature map is \(\mathcal{O}(64^4) = \mathcal{O}(\text{16M})\) pairs per head, expensive but doable; on 256×256 it’s prohibitive without windowing or linear attention.

Conditioning injection.

  • Timestep \(t\): encode with sinusoidal embedding \(\to\) small MLP \(\to\) a per-block bias/scale (AdaGN or AdaLN). Added to every ResBlock.
  • Class label / text embedding \(c\): same idea — small MLP applied to the conditioning vector and added to every ResBlock’s AdaGN. For text, the standard pattern is cross-attention: at certain U-Net resolutions, replace self-attention by attention where queries come from image features and keys/values from a text-encoder (e.g., CLIP, T5) output.

Modern variants.

  • ADM (Dhariwal & Nichol) — wider U-Net, AdaGN, multi-head attention, learned variance head.
  • Stable Diffusion U-Net — same skeleton, with cross-attention at multiple resolutions for text conditioning, and the whole network operating in a 4-channel VAE latent space (8× downsampling).
  • DiT (Diffusion Transformer, Peebles & Xie, 2022) — replace the U-Net with a transformer over flattened patches. No skip connections in the U-Net sense; uses AdaLN-Zero modulation. Scales like a transformer; SOTA on class-conditional ImageNet and the basis for Sora, SD3.
  • U-ViT — a U-Net-shaped vision transformer.
  • MM-DiT — multi-modal DiT used in Stable Diffusion 3, fusing image and text tokens in the same transformer.

Common assessment trap — U-Net vs DiT vs ResNet. The U-Net is the architecture for the reverse network, not for the diffusion process itself. The diffusion process is fully specified by the schedule. Swapping U-Net for DiT does not change DDPM into something else — it changes the implementation, not the math.

4.2 Sinusoidal Timestep Encodings

Definition. Sinusoidal positional encodings borrowed from transformers (Vaswani et al., 2017). For a timestep \(t\) and embedding dimension \(d\) (even):

\[ \mathrm{PE}(t)_{2i} \;=\; \sin\!\left(\frac{t}{10000^{2i/d}}\right), \qquad \mathrm{PE}(t)_{2i+1} \;=\; \cos\!\left(\frac{t}{10000^{2i/d}}\right), \]

for \(i = 0, \dots, d/2 - 1\).

Why this form? The sinusoidal embedding has two useful properties:

  1. Smoothness across \(t\). Embeddings of nearby timesteps are similar; far-away timesteps are uncorrelated. The network can easily interpolate.
  2. Unique encoding. Every \(t\) has a distinct vector. Together with a learned linear projection, the network can extract any monotonic feature of \(t\) it needs (e.g., SNR, \(\beta_t\)).
  3. Frequency coverage. Different dimensions oscillate at different rates, so the embedding contains both fine-grained and coarse-grained information about \(t\).

Standard recipe. Concatenate or use the sinusoidal embedding as input to a small two-layer MLP (commonly with SiLU/GELU activations) that produces a higher-dimensional time embedding \(\tau \in \mathbb{R}^{D}\). This \(\tau\) is injected into each ResBlock (see 4.3).

PyTorch implementation.

import math, torch
import torch.nn as nn

def sinusoidal_time_embedding(t: torch.Tensor, dim: int) -> torch.Tensor:
    # t:   (B,)         int or float timesteps
    # out: (B, dim)
    device = t.device
    half = dim // 2
    freqs = torch.exp(
        -math.log(10000) * torch.arange(0, half, device=device).float() / half
    )                                                # (half,)
    args = t.float()[:, None] * freqs[None, :]       # (B, half)
    emb  = torch.cat([torch.sin(args), torch.cos(args)], dim=-1)  # (B, dim)
    if dim % 2 == 1:
        emb = torch.nn.functional.pad(emb, (0, 1))
    return emb

class TimeEmbeddingMLP(nn.Module):
    def __init__(self, dim_in: int, dim_out: int):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(dim_in, dim_out),
            nn.SiLU(),
            nn.Linear(dim_out, dim_out),
        )
    def forward(self, t_emb: torch.Tensor) -> torch.Tensor:
        # t_emb: (B, dim_in)  →  (B, dim_out)
        return self.net(t_emb)

Continuous \(t\) vs discrete \(t\). DDPM with \(T=1000\) uses integer timesteps \(t \in \{0, \dots, 999\}\) and the formula above plugs them in directly. EDM-style models use continuous \(\sigma\) and feed \(\log \sigma\) through the same sinusoidal-then-MLP path. Either way, the conditioning vector \(\tau\) is a smooth function of the noise level.

Why not learn a separate embedding per integer \(t\)? You can — and some early DDPM implementations did — but a learned embedding table doesn’t generalize to subsampled timesteps at inference time (e.g., when using DDIM with \(N < T\)). Sinusoidal encodings interpolate gracefully.

4.3 ResNet and Attention Blocks for Diffusion

ResBlock with time-embedding injection. Each ResBlock takes a feature map and the time embedding \(\tau\), applies a normalization-conv stack with a time-conditioned shift, and adds a residual connection.

A widely-used pattern (used in ADM, Stable Diffusion, EDM):

input h (B, Cin, H, W),  tau (B, Dt)
   │
GroupNorm
   │
SiLU
   │
Conv 3x3 -> (B, Cmid, H, W)
   │
add (Linear(SiLU(tau)) reshaped to (B, Cmid, 1, 1)) as bias       ← time injection
   │
GroupNorm
   │
SiLU
   │
Dropout
   │
Conv 3x3 -> (B, Cout, H, W)
   │
add skip = Conv1x1(input) if Cin != Cout else input
   │
output (B, Cout, H, W)

PyTorch implementation.

import torch.nn as nn
import torch.nn.functional as F

class ResBlock(nn.Module):
    def __init__(self, in_ch, out_ch, time_dim, groups=32, dropout=0.0):
        super().__init__()
        self.norm1 = nn.GroupNorm(groups, in_ch)
        self.conv1 = nn.Conv2d(in_ch, out_ch, 3, padding=1)
        self.time_proj = nn.Linear(time_dim, out_ch)
        self.norm2 = nn.GroupNorm(groups, out_ch)
        self.conv2 = nn.Conv2d(out_ch, out_ch, 3, padding=1)
        self.drop  = nn.Dropout(dropout)
        self.skip  = nn.Identity() if in_ch == out_ch else nn.Conv2d(in_ch, out_ch, 1)

    def forward(self, h, tau):
        # h:   (B, in_ch, H, W)
        # tau: (B, time_dim)
        x = self.conv1(F.silu(self.norm1(h)))            # (B, out_ch, H, W)
        x = x + self.time_proj(F.silu(tau))[:, :, None, None]
        x = self.conv2(self.drop(F.silu(self.norm2(x)))) # (B, out_ch, H, W)
        return x + self.skip(h)

Self-attention block. At low resolutions (e.g., 32×32 or smaller in feature space), insert a self-attention block that mixes information across spatial positions.

class SelfAttention2D(nn.Module):
    def __init__(self, channels, heads=4):
        super().__init__()
        assert channels % heads == 0
        self.heads = heads
        self.norm = nn.GroupNorm(32, channels)
        self.qkv  = nn.Conv2d(channels, channels * 3, 1)
        self.proj = nn.Conv2d(channels, channels, 1)

    def forward(self, x):
        # x: (B, C, H, W)
        B, C, H, W = x.shape
        h = self.norm(x)
        qkv = self.qkv(h).reshape(B, 3, self.heads, C // self.heads, H * W)
        q, k, v = qkv[:,0], qkv[:,1], qkv[:,2]            # each (B, heads, C/heads, HW)
        attn = torch.einsum("bhdn,bhdm->bhnm", q, k) * (C // self.heads) ** -0.5
        attn = attn.softmax(dim=-1)
        out  = torch.einsum("bhnm,bhdm->bhdn", attn, v)   # (B, heads, C/heads, HW)
        out  = out.reshape(B, C, H, W)
        return x + self.proj(out)

Cross-attention for conditioning (used in Stable Diffusion U-Net). Queries are image features, keys/values are text-encoder outputs:

class CrossAttention2D(nn.Module):
    def __init__(self, channels, text_dim, heads=4):
        super().__init__()
        self.heads = heads
        self.norm  = nn.GroupNorm(32, channels)
        self.to_q  = nn.Conv2d(channels, channels, 1)
        self.to_k  = nn.Linear(text_dim, channels)
        self.to_v  = nn.Linear(text_dim, channels)
        self.proj  = nn.Conv2d(channels, channels, 1)

    def forward(self, x, text):
        # x:    (B, C, H, W)
        # text: (B, L, text_dim)
        B, C, H, W = x.shape
        h = self.norm(x)
        q = self.to_q(h).reshape(B, self.heads, C // self.heads, H * W)
        k = self.to_k(text).reshape(B, -1, self.heads, C // self.heads).permute(0,2,3,1)
        v = self.to_v(text).reshape(B, -1, self.heads, C // self.heads).permute(0,2,3,1)
        attn = torch.einsum("bhdn,bhdl->bhnl", q, k) * (C // self.heads) ** -0.5
        attn = attn.softmax(dim=-1)
        out  = torch.einsum("bhnl,bhdl->bhdn", attn, v).reshape(B, C, H, W)
        return x + self.proj(out)

AdaGN / AdaLN-Zero modulation (modern modulation). Instead of (or in addition to) adding the time embedding as a bias, modern diffusion models use adaptive group norm: the time embedding produces per-channel scale \(\gamma(\tau)\) and shift \(\beta(\tau)\) that modulate the normalized features:

\[ \mathrm{AdaGN}(h, \tau) \;=\; \gamma(\tau)\odot \mathrm{GroupNorm}(h) + \beta(\tau). \]

DiT uses AdaLN-Zero: the same idea with layer norm and the residual scale initialized to zero so each block starts as an identity.

Putting it together — block stack at one U-Net level.

input feature h, time embedding tau, (optional) text embedding c

ResBlock(h, tau) -> h'
SelfAttention(h') -> h''         (only at low resolutions)
CrossAttention(h'', c) -> h'''   (only if conditioning on text)

A diffusion U-Net is essentially this block stacked at several resolutions, with downsampling/upsampling and skip connections joining mirrored levels.

MODULE 5: Training and Optimizing Diffusion Models

5.1 Preparing Datasets and Noise Samples

Data normalization. Diffusion models are trained to denoise from a Gaussian prior \(\mathcal{N}(0, I)\). Therefore the data should also have approximately zero mean and unit variance. Standard practice for images:

  1. Load image with pixel values in \(\{0, 1, \dots, 255\}\) → divide by \(255\) → in \([0,1]\).
  2. Scale to \([-1, 1]\): \(x \leftarrow 2x - 1\).
  3. Optionally normalize per-channel mean/std if the dataset is far from symmetric.

For audio / waveforms: scale to \([-1, 1]\) or normalize per clip.

For text/tokens (which is rare; diffusion-LMs exist but are less common than autoregressive LMs): treat the token embeddings or one-hot vectors as continuous and add noise; or use categorical diffusion (D3PM, masked diffusion, structured discrete diffusion).

Batch sampling.

For each minibatch:
  1. Sample N clean data points x0  ~ p_data
  2. Sample N timesteps t ~ Uniform{1, ..., T}      (or a non-uniform sampler; see below)
  3. Sample N noise vectors eps ~ N(0, I)           with same shape as x0
  4. Form x_t = sqrt(alpha_bar_t) x0 + sqrt(1 - alpha_bar_t) eps
  5. Forward pass:  eps_hat = eps_theta(x_t, t, [conditioning])
  6. Loss:          L = || eps - eps_hat ||^2       (or v-loss / x0-loss)

Non-uniform \(t\) sampling. Improved DDPM proposes importance sampling over \(t\), biasing draws toward \(t\) with high per-step loss. In practice, uniform \(t\) + min-SNR-\(\gamma\) loss reweighting (Hang et al., 2023) achieves a similar effect with simpler code. (We will write down min-SNR explicitly in 5.2.)

EMA of weights. Track an exponential moving average of the model parameters:

\[ \theta_{\text{EMA}} \leftarrow \rho\, \theta_{\text{EMA}} + (1 - \rho)\, \theta, \qquad \rho \in [0.999, 0.9999]. \]

At sampling time, use \(\theta_{\text{EMA}}\), not the raw training weights. EMA is the single most-impactful trick for diffusion stability and FID; without it, samples are visibly noisier. Standard \(\rho\) values: \(0.9999\) (very smooth, slow to converge) to \(0.999\) (faster, slightly noisier). EDM uses a more elaborate “power EMA” with multiple time constants.

Data augmentations. Diffusion models tend to benefit from light augmentation only — they already see many noise levels of every image, so heavy augmentation can distort the learned distribution.

  • Standard for images: horizontal flips (when class-symmetric), random crops (when training on patches), color jitter (sparingly).
  • “Augmentation conditioning” (EDM): condition the network on the augmentation parameters so that augmentation does not pollute the learned \(p(x)\). At sampling time, set the conditioning to “no augmentation”.

Conditioning dropout for classifier-free guidance. For conditional models (text-to-image, class-conditional), randomly drop the conditioning during training with probability \(p_{\text{drop}} \in [0.05, 0.2]\):

with probability p_drop:
    c = null_embedding   # an explicit "no condition" embedding

This trains the same network to be both conditional and unconditional, which is what classifier-free guidance (CFG) requires at sampling time (7.3).

Mixed precision and gradient accumulation. Image diffusion training is GPU-bound (memory and FLOPs). Standard tricks:

  • bf16 / fp16 mixed precision with automatic gradient scaling.
  • Gradient accumulation to fake large batch sizes when the batch doesn’t fit in memory.
  • Gradient checkpointing across U-Net blocks at the cost of recomputation.
  • ZeRO / FSDP for very large models.

5.2 Formulating and Minimizing Diffusion Objectives

This subsection collects every loss you’ll encounter, in one place.

Full ELBO (variational lower bound).

Starting from \(\log p_\theta(x_0) \ge \mathrm{ELBO}\) and expanding,

\[ \mathcal{L}_{\text{vlb}}(\theta) \;=\; \mathbb{E}_q\!\Big[ \underbrace{\mathrm{KL}\!\left(q(x_T\mid x_0)\Vert p(x_T)\right)}_{L_T} + \sum_{t=2}^{T}\underbrace{\mathrm{KL}\!\left(q(x_{t-1}\mid x_t, x_0) \Vert p_\theta(x_{t-1}\mid x_t)\right)}_{L_{t-1}} - \underbrace{\log p_\theta(x_0\mid x_1)}_{L_0} \Big]. \]

  • \(L_T\) is essentially constant w.r.t. \(\theta\) (it measures how close \(q(x_T\mid x_0)\) is to the prior \(\mathcal{N}(0,I)\), which depends only on the schedule).
  • Each \(L_{t-1}\) is the KL between two Gaussians. With fixed \(\Sigma_\theta = \sigma_t^2 I\), this KL is proportional to \(\|\tilde\mu_t - \mu_\theta\|^2 / (2\sigma_t^2)\).

Simplified loss \(L_{\text{simple}}\).

After substituting \(\mu_\theta\) in \(\epsilon\)-form (3.2), each \(L_{t-1}\) becomes

\[ L_{t-1} \;\propto\; \frac{\beta_t^2}{2\sigma_t^2 \alpha_t (1-\bar\alpha_t)}\, \|\epsilon - \epsilon_\theta(x_t, t)\|^2. \]

Dropping the \(t\)-dependent coefficient and averaging uniformly over \(t\) gives

\[ \mathcal{L}_{\text{simple}}(\theta) \;=\; \mathbb{E}_{t,x_0,\epsilon}\!\left[\|\epsilon - \epsilon_\theta(x_t, t)\|^2\right]. \]

This is what most DDPMs actually minimize. Empirically it generates better-looking samples than the full ELBO, although it does not maximize true log-likelihood exactly.

\(v\)-prediction. Define \(v = \sqrt{\bar\alpha_t}\,\epsilon - \sqrt{1-\bar\alpha_t}\,x_0\). Then training on

\[ \mathcal{L}_{v}(\theta) \;=\; \mathbb{E}_{t,x_0,\epsilon}\!\left[\|v - v_\theta(x_t, t)\|^2\right] \]

is equivalent (up to a \(t\)-dependent constant) to \(L_{\text{simple}}\) but is better-conditioned at the boundary \(t\to T\) (where \(\epsilon\)-prediction becomes the trivial identity \(\epsilon_\theta(x_t,t)\approx x_t\) and the network’s effective capacity is wasted on this trivial task). Stable Diffusion 2.0+, Imagen, EDM-related code, and most modern image diffusion models use \(v\)-prediction.

You can convert between predictions:

  • \(\epsilon = \sqrt{\bar\alpha_t}\,v + \sqrt{1-\bar\alpha_t}\,x_t\),
  • \(x_0 = \sqrt{\bar\alpha_t}\,x_t - \sqrt{1-\bar\alpha_t}\,v\).

\(x_0\)-prediction. The network outputs \(\hat x_0(x_t, t)\) directly, and the loss is \(\|x_0 - \hat x_0(x_t, t)\|^2\). This is mathematically equivalent to \(\epsilon\)-prediction but the loss surface is differently weighted across \(t\), which can matter for short schedules or distillation.

Min-SNR weighting. The simplified loss treats every \(t\) uniformly, but the “useful signal” of each \(t\) depends on its SNR. Hang et al. (2023, “Efficient Diffusion Training via Min-SNR Weighting”) propose weighting each step by

\[ w(t) \;=\; \min\!\left(\mathrm{SNR}(t),\;\gamma\right), \]

with \(\gamma \in [1,5]\). This caps the loss contribution at low noise (high SNR) where the gradient is dominated by easy steps, freeing capacity for the genuinely hard middle steps. It is the simplest, sharpest improvement on \(L_{\text{simple}}\) at unchanged code complexity.

EDM preconditioning (Karras et al., 2022).

EDM rewrites the network as

\[ D_\theta(x, \sigma) \;=\; c_{\text{skip}}(\sigma)\, x + c_{\text{out}}(\sigma)\, F_\theta\!\left(c_{\text{in}}(\sigma)\, x, c_{\text{noise}}(\sigma)\right), \]

with \(\sigma\)-dependent coefficients chosen so that the internal network always sees inputs and targets with unit variance. Combined with \(\sigma\)-uniform-in-\(\log\sigma\) training and a Heun second-order sampler, this gives the best image-FID-per-FLOP available.

Score matching connection. With \(s_\theta(x_t, t) = -\epsilon_\theta(x_t, t)/\sqrt{1-\bar\alpha_t}\), the simplified loss is denoising score matching:

\[ \mathcal{L}_{\text{DSM}}(\theta) \;=\; \mathbb{E}_{t,x_0,\epsilon}\!\left[ \lambda(t)\, \|s_\theta(x_t, t) - \nabla_{x_t} \log q(x_t\mid x_0)\|^2 \right]. \]

The two views — DDPM as variational denoising, and NCSN/score-SDE as score matching — are mathematically identical up to reparameterization (Song et al., “Score-Based Generative Modeling Through Stochastic Differential Equations”, 2021). The continuous-time SDE view writes the forward process as

\[ dx \;=\; f(x,t)\, dt + g(t)\, dw, \]

with a Variance-Preserving (VP) or Variance-Exploding (VE) choice of drift/diffusion. The reverse is a corresponding reverse-time SDE

\[ dx \;=\; \left[f(x,t) - g(t)^2\, \nabla_x \log p_t(x)\right] dt + g(t)\, d\bar w, \]

and the probability-flow ODE counterpart removes the stochastic term to give a deterministic trajectory whose marginals match. Plugging \(s_\theta\) in for \(\nabla_x \log p_t\) recovers the DDPM/DDIM sampler.

End-to-end DDPM training pseudocode.

Algorithm 1: DDPM training
Inputs: dataset D, network eps_theta, schedule { sqrt(alpha_bar_t), sqrt(1 - alpha_bar_t) }, T
Repeat:
    1. Sample x_0 ~ D
    2. Sample t ~ Uniform{1, ..., T}
    3. Sample eps ~ N(0, I)
    4. x_t = sqrt(alpha_bar_t) * x_0 + sqrt(1 - alpha_bar_t) * eps
    5. Compute loss L = || eps - eps_theta(x_t, t) ||^2          (or v / x0 / min-SNR variants)
    6. Take a gradient step on L w.r.t. theta
    7. Update theta_EMA <- rho * theta_EMA + (1 - rho) * theta

Minimal PyTorch — DDPM training step.

def ddpm_train_step(model, x0, schedule, opt, ema_model, ema_rho=0.9999):
    """
    model:    eps_theta(x_t, t) -> predicted noise
    x0:       (B, C, H, W), normalized to [-1, 1]
    schedule: dict from make_linear_schedule or make_cosine_schedule
    """
    B = x0.size(0)
    T = schedule["beta"].shape[0]
    t = torch.randint(0, T, (B,), device=x0.device)

    sqrt_ab    = schedule["sqrt_alpha_bar"][t].view(B, 1, 1, 1)
    sqrt_1mab  = schedule["sqrt_one_minus_alpha_bar"][t].view(B, 1, 1, 1)

    eps = torch.randn_like(x0)
    xt  = sqrt_ab * x0 + sqrt_1mab * eps                    # (B, C, H, W)
    eps_hat = model(xt, t)                                  # (B, C, H, W)
    loss = ((eps - eps_hat) ** 2).mean()

    opt.zero_grad()
    loss.backward()
    torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
    opt.step()

    with torch.no_grad():
        for p_ema, p in zip(ema_model.parameters(), model.parameters()):
            p_ema.data.mul_(ema_rho).add_(p.data, alpha=1 - ema_rho)

    return loss.item()

5.3 Applying Gradient Descent to Optimize Models

Optimizer choice. The standard for diffusion is AdamW with \(\beta_1 = 0.9\), \(\beta_2 = 0.999\), \(\epsilon = 10^{-8}\), weight decay \(\sim 10^{-2}\) on the network parameters (and zero on biases / norm parameters). Learning rates are conservative — \(10^{-4}\) is the usual starting point, falling to \(10^{-5}\) for very large networks.

Why AdamW and not SGD? The loss surface of diffusion is full of large, infrequent gradients (e.g., when very high or very low noise levels lead to nearly-zero or nearly-saturated gradients). Adam’s per-parameter adaptive scaling absorbs that variance.

Learning rate schedule.

  • Warmup: start with LR \(0\), linearly ramp up to the target over the first ~10k steps. Diffusion U-Nets are sensitive to early instability.
  • Constant LR after warmup is common for U-Net diffusion (no decay needed; the noise variance in the loss already acts as regularization).
  • Cosine decay to zero is common for transformer-based diffusion (DiT, Stable Diffusion).

Gradient clipping. Clip the global gradient norm at \(1.0\) (or sometimes \(5.0\)). Without clipping, occasional large gradients can spike weights and destroy training; with clipping, the model is much more forgiving of schedule and learning-rate choices.

Batch sizes.

  • Small datasets (CIFAR-10): batch 128–512, single GPU.
  • Medium datasets (FFHQ 256×256): batch 128–256, multi-GPU.
  • Large text-to-image (Stable Diffusion-class): effective batch 2k–8k via gradient accumulation across many GPUs.

Iteration counts.

  • CIFAR-10 DDPM: ~500k–1M steps.
  • ImageNet 64×64 ADM: ~2M steps.
  • Stable Diffusion v1.x: ~500k–1M optimizer steps at very large effective batch.

Loss curves to expect.

  • \(L_{\text{simple}}\) drops quickly in the first ~10k steps to a moderate plateau and then improves slowly. Don’t be alarmed by the slow tail — sample quality keeps improving long after the loss looks “flat” because the EMA-smoothed network keeps refining.
  • Per-\(t\) loss is highly informative: plot it during training. A bowl shape (high at intermediate \(t\), low at extreme \(t\)) is normal.

Failure modes and debugging.

  • Diverging loss / NaNs. Check: is the schedule reversed? Are alpha_bar tensors on the right device? Is the network outputting Inf/NaN? Lower LR or add gradient clipping.
  • Samples look right but FID is bad. Likely EMA is not applied at sampling time. Always sample from the EMA weights.
  • Samples are noisy at high resolution. Schedule has too few steps for the resolution; try \(T=1000\) with a cosine schedule, or move to latent diffusion.
  • Samples have repeating patterns. Often a too-low-resolution attention layer (the model can’t see globally), or under-trained network.
  • CFG samples become saturated / cartoonish at very high guidance scale (8+). Expected behaviour; reduce CFG or apply CFG rescaling (e.g., divide guided prediction by its standard deviation ratio to preserve overall scale).

5.4 Accelerating Image Generation by Reducing Inference Timesteps

The naïve DDPM sampler runs \(T = 1000\) denoise steps to produce one image. Modern systems run as few as 1 step with comparable quality. This section catalogs the techniques.

1. DDIM — deterministic, non-Markovian sampler (Song, Meng, Ermon, 2021).

DDIM observes that the training objective only constrains \(\epsilon_\theta(x_t, t)\) to denoise \(x_t\), not the specific Markov reverse chain \(p_\theta(x_{t-1}\mid x_t)\). A whole family of non-Markovian reverse processes share the same training objective. The DDIM update is

\[ x_{t-1} \;=\; \sqrt{\bar\alpha_{t-1}}\!\left(\frac{x_t - \sqrt{1-\bar\alpha_t}\,\epsilon_\theta(x_t, t)}{\sqrt{\bar\alpha_t}}\right) + \sqrt{1-\bar\alpha_{t-1} - \sigma_t^2}\,\epsilon_\theta(x_t, t) + \sigma_t z, \]

where \(z\sim \mathcal{N}(0,I)\) and \(\sigma_t \in [0, \sqrt{(1-\bar\alpha_{t-1})/(1-\bar\alpha_t)}\cdot\sqrt{1-\bar\alpha_t/\bar\alpha_{t-1}}]\).

Setting \(\sigma_t = 0\) gives the deterministic DDIM:

\[ \boxed{\;x_{t-1} \;=\; \sqrt{\bar\alpha_{t-1}}\,\hat x_0(x_t, t) + \sqrt{1-\bar\alpha_{t-1}}\,\epsilon_\theta(x_t, t),\;} \]

where \(\hat x_0(x_t, t) = (x_t - \sqrt{1-\bar\alpha_t}\,\epsilon_\theta)/\sqrt{\bar\alpha_t}\) is the predicted clean image. Because the chain is deterministic (given \(x_T\)), you can run it on a subsequence of timesteps \(\{t_0, t_1, \dots, t_N\}\) with \(t_0 = T\) and \(t_N = 0\) — typically \(N \in \{10, 20, 50\}\) — and produce a sample in \(N\) network evaluations instead of \(T\).

DDIM also gives an encoder: starting from \(x_0\), running the deterministic chain forward yields a \(x_T\) that, fed back into the sampler, returns \(x_0\) exactly. This is useful for editing and interpolation.

Worked one-step DDIM. Using the schedule from 3.1, jump from \(t = 4\) to \(t = 0\) in one step. Suppose \(x_4 = 0.50\), \(\epsilon_\theta = 0.10\). Then \(\hat x_0 = (0.50 - 0.199\cdot 0.10)/\sqrt{0.96040} = 0.48005/0.98 \approx 0.4898\). With \(t_{\text{prev}} = 0\) and \(\bar\alpha_0 = 1\): \(x_0 = 1\cdot 0.4898 + 0\cdot 0.10 = 0.4898\). (For non-zero \(t_{\text{prev}}\) you’d interpolate properly.) The point is: one network call, one step.

2. DPM-Solver and DPM-Solver++ (Lu et al., 2022). Treat the probability-flow ODE as the object to integrate and use a higher-order solver (exponential integrator). DPM-Solver achieves DDIM-like quality at 5–10 steps; DPM-Solver++ (a multi-step variant) is even faster and is the default in most production diffusion deployments (Stable Diffusion, Comfy, Diffusers).

3. Euler, Heun, LMS, UniPC. A family of ODE/SDE solvers from EDM and follow-ups. Heun (2nd order) is the EDM default; UniPC (Unified Predictor-Corrector, Zhao et al., 2023) is a competitive multi-step solver that often beats DPM-Solver++ at very low step counts.

4. Distillation methods. Use a trained “teacher” diffusion model to train a “student” that produces the same result in fewer steps.

  • Progressive distillation (Salimans & Ho, 2022). Train a student that takes 2 student-steps per 1 teacher-step. Repeat to halve the step count each round. Goes from 1024 → 1 step over ~10 rounds.
  • Consistency distillation (Song et al., 2023, “Consistency Models”). Train \(f_\theta\) to satisfy self-consistency: \(f_\theta(x_t, t) = f_\theta(x_{t'}, t')\) for any two points on the same ODE trajectory. Result: one-step sampling.
  • Latent Consistency Models (LCM, Luo et al., 2023). Apply consistency distillation in the Stable Diffusion latent space; gives 1–4-step text-to-image.
  • SDXL Turbo / Adversarial Diffusion Distillation (ADD, Sauer et al., 2023). Combine distillation with an adversarial loss to bake very-low-step generation into a single network; near-real-time text-to-image.

Run diffusion in the latent space of a pretrained VAE, not in pixel space.

encode:        x (B, 3, 512, 512)  --VAE-->  z (B, 4, 64, 64)
diffuse on z:  DDPM/DDIM/DPM-Solver in z-space
decode:        z'  --VAE-->  x'

The latent space is \(8\times\) smaller per spatial dimension, giving a \(64\times\) reduction in FLOPs. Combined with cross-attention text conditioning, this is the architecture of Stable Diffusion 1.x / 2.x / SDXL.

6. Cascaded diffusion. Generate a low-resolution image first, then super-resolve in several stages, each itself a diffusion model. Imagen uses 64×64 → 256×256 → 1024×1024 cascade. Lighter per-stage networks, but more total network evaluations.

MODULE 6: Working with GANs and Alternative Models

6.1 The Adversarial Game and Core GAN Architecture

Definition. A Generative Adversarial Network (GAN; Goodfellow et al., 2014) is a pair of neural networks trained against each other:

  • a generator \(G_\theta: \mathbb{R}^{d_z} \to \mathbb{R}^{d_x}\) that maps a random latent \(z\sim p(z)\) to a synthesized data point \(G_\theta(z)\);
  • a discriminator \(D_\phi: \mathbb{R}^{d_x} \to [0,1]\) that estimates the probability that an input is real (i.e., from \(p_{\text{data}}\)) rather than fake.

The two networks play a zero-sum game:

\[ \min_\theta \max_\phi \;\; V(D_\phi, G_\theta) \;=\; \mathbb{E}_{x\sim p_{\text{data}}}\!\left[\log D_\phi(x)\right] + \mathbb{E}_{z\sim p(z)}\!\left[\log(1 - D_\phi(G_\theta(z)))\right]. \]

The discriminator maximizes; the generator minimizes.

Optimal discriminator. For any fixed \(G\), the discriminator that maximizes \(V\) is

\[ D^*(x) \;=\; \frac{p_{\text{data}}(x)}{p_{\text{data}}(x) + p_g(x)}, \]

where \(p_g(x)\) is the distribution induced by \(G(z)\). Substituting this into \(V\) gives

\[ V(D^*, G) \;=\; 2\,\mathrm{JSD}(p_{\text{data}} \,\Vert\, p_g) - 2\log 2, \]

where \(\mathrm{JSD}\) is the Jensen–Shannon divergence:

\[ \mathrm{JSD}(p \,\Vert\, q) \;=\; \tfrac{1}{2}\mathrm{KL}(p\Vert m) + \tfrac{1}{2}\mathrm{KL}(q\Vert m), \qquad m = (p+q)/2. \]

So under perfect alternation, the GAN is implicitly minimizing the JS divergence between the model distribution \(p_g\) and the data distribution \(p_{\text{data}}\). The global optimum is \(p_g = p_{\text{data}}\), at which \(D^* = 1/2\) everywhere.

The original (“saturating”) generator loss. By symmetry, the generator should minimize the discriminator’s objective:

\[ \mathcal{L}_G^{\text{(saturating)}} = \mathbb{E}_{z}\!\left[\log(1 - D_\phi(G_\theta(z)))\right]. \]

Problem: early in training, \(D_\phi\) assigns very low probability to \(G(z)\), so \(1 - D \approx 1\) and \(\log(1-D) \approx 0\) everywhere — the gradient w.r.t. \(\theta\) vanishes.

The non-saturating (“heuristic”) generator loss. Switch to maximizing \(\log D(G(z))\) instead of minimizing \(\log(1-D(G(z)))\):

\[ \mathcal{L}_G^{\text{(non-saturating)}} = -\,\mathbb{E}_{z}\!\left[\log D_\phi(G_\theta(z))\right]. \]

Same fixed-point dynamics (still \(D^* = 1/2\) at the optimum), but the gradient w.r.t. \(\theta\) is large precisely when \(D\) is confident that \(G(z)\) is fake — the regime that needs the most training signal. This is the version used in practice.

Training algorithm (Goodfellow et al., 2014).

Algorithm: GAN training
Inputs: data D, generator G_theta, discriminator D_phi, noise prior p(z)
Repeat:
    -- D step: k inner iterations (usually k=1)
    1. Sample minibatch of real x ~ D, fake z ~ p(z)
    2. Update phi to maximize: log D_phi(x) + log(1 - D_phi(G_theta(z)))
    -- G step

Architecture conventions.

  • DCGAN (Radford et al., 2016): strided conv for downsampling, transposed conv for upsampling, BatchNorm, LeakyReLU, no fully-connected layers except at the entry/exit.
  • Progressive GAN (Karras et al., 2018): start at low resolution (4×4), gradually fade in additional layers and resolution.
  • StyleGAN1/2/3 (Karras et al., 2019/2020/2021): map noise \(z\) through a mapping network to a “style” code \(w\); inject \(w\) at every layer via AdaIN; introduce a learned constant input.
  • BigGAN (Brock et al., 2018): large class-conditional GAN with self-attention, spectral norm, hierarchical noise injection, truncation trick at inference.
  • Conditional GAN (cGAN, Mirza & Osindero, 2014): both \(G\) and \(D\) take an extra conditioning input \(c\) (label, text, image).

Common assessment trap — vocabulary.

  • Minimax and non-saturating losses converge to the same fixed point but have different optimization dynamics; non-saturating is what you use.
  • JS divergence interpretation only holds under the optimal discriminator; in practice \(D\) is far from optimal during training, which is why other losses (WGAN, hinge) often work better empirically.
  • GAN models \(p(x)\) implicitly (you can sample but not evaluate density). This is a defining property that often appears in comparison questions.

6.2 GAN Loss Functions and Training Workflows

GAN training is a notoriously delicate balancing act. The history of GANs after 2014 is largely a history of loss functions designed to stabilize that balance. Know each by name.

Vanilla GAN (Goodfellow 2014). Already covered above. Suffers from vanishing generator gradients and mode collapse.

Non-saturating loss. Standard fix to the vanishing-gradient problem; this is the practical baseline.

Least-Squares GAN (LSGAN, Mao et al., 2017). Replace the binary-cross-entropy on the discriminator with a least-squares loss:

\[ \mathcal{L}_D = \tfrac{1}{2}\mathbb{E}_{x\sim p_{\text{data}}}[(D(x) - 1)^2] + \tfrac{1}{2}\mathbb{E}_{z}[(D(G(z)))^2], \]

\[ \mathcal{L}_G = \tfrac{1}{2}\mathbb{E}_{z}[(D(G(z)) - 1)^2]. \]

Equivalent to minimizing a Pearson \(\chi^2\) divergence; gives non-vanishing gradients far from the decision boundary and often makes training more stable.

Wasserstein GAN (WGAN, Arjovsky et al., 2017). Replace JS divergence with the Earth Mover’s Distance / Wasserstein-1 distance. By the Kantorovich-Rubinstein duality,

\[ W_1(p_{\text{data}}, p_g) \;=\; \sup_{\|f\|_L \le 1} \;\mathbb{E}_{x\sim p_{\text{data}}}[f(x)] - \mathbb{E}_{x\sim p_g}[f(x)]. \]

Use a critic \(f_\phi\) (not a discriminator — it’s not bounded to \([0,1]\)) constrained to be 1-Lipschitz. The loss is

\[ \mathcal{L}_D = -\mathbb{E}_{x\sim p_{\text{data}}}[f_\phi(x)] + \mathbb{E}_{z}[f_\phi(G_\theta(z))], \qquad \mathcal{L}_G = -\mathbb{E}_{z}[f_\phi(G_\theta(z))]. \]

In WGAN, the Lipschitz constraint is enforced by weight clipping (clamp \(\phi\) to \([-c, c]\)), which is crude.

WGAN with Gradient Penalty (WGAN-GP, Gulrajani et al., 2017). Enforce the Lipschitz constraint by a soft penalty instead of weight clipping. Sample interpolations \(\hat x = \alpha x + (1-\alpha) G(z)\) with \(\alpha \sim U[0,1]\) and add

\[ \mathcal{L}_{\text{GP}} = \lambda\, \mathbb{E}_{\hat x}\!\left[(\|\nabla_{\hat x} f_\phi(\hat x)\|_2 - 1)^2\right] \]

with \(\lambda = 10\). Vastly more stable than WGAN. Standard for serious GAN training pre-2020.

Hinge loss (BigGAN, StyleGAN2). A hinge-style critic loss:

\[ \mathcal{L}_D = \mathbb{E}_{x\sim p_{\text{data}}}[\max(0, 1 - D(x))] + \mathbb{E}_{z}[\max(0, 1 + D(G(z)))], \]

\[ \mathcal{L}_G = -\mathbb{E}_{z}[D(G(z))]. \]

Empirically very strong, especially with spectral normalization (Miyato et al., 2018), which normalizes each weight matrix’s spectral norm (top singular value) to enforce a soft Lipschitz constraint.

Two-time-scale update rule (TTUR, Heusel et al., 2017). Use different learning rates for \(G\) and \(D\) (commonly \(D\) faster, e.g., \(\eta_D = 4\eta_G\)). Provably converges to a Nash equilibrium under mild assumptions and is the default in most modern GAN training.

Other tricks.

  • Spectral normalization (mentioned above): impose Lipschitz constraint by dividing each layer’s weights by its spectral norm. Stabilizes training without weight clipping or explicit penalties.
  • R1 / R2 gradient penalty. Penalize the discriminator’s gradient norm only on real (R1) or only on fake (R2) samples; cheaper than two-sided WGAN-GP and the default in StyleGAN2.
  • Path length regularization (StyleGAN2): regularize the Jacobian of \(G\) at random points in latent space toward a constant; gives smoother latent traversals.
  • Equalized learning rate. Multiply the weights of a layer at runtime by \(\sqrt{2/\text{fan\_in}}\) so the Adam-style adaptive LR is balanced across layers.
  • Truncation trick (BigGAN). At inference, truncate \(z\) to within a smaller ball; sacrifices diversity for sample quality.

Failure modes.

  • Mode collapse. \(G\) produces a small set of outputs that fool \(D\) rather than the full data distribution. Symptoms: \(G\) generates the same image regardless of \(z\); FID is high; diversity is low; inception score is low.
  • Vanishing discriminator gradients. \(D\) becomes too strong and assigns near-zero probability to all fakes; gradient to \(G\) flat-lines.
  • Cycling. \(G\) and \(D\) oscillate without converging to equilibrium.
  • Overtraining \(D\). A perfect \(D\) has no useful gradient signal for \(G\). Use \(k=1\) inner \(D\) steps per \(G\) step (modern default).

Diagnosing mode collapse.

  • Track the generator output diversity (e.g., minibatch standard deviation of features) over time.
  • Use Minibatch standard deviation (Karras et al., 2018) as a discriminator feature: \(D\) sees the standard deviation of each feature across the minibatch, so a generator that always outputs the same thing is easily detected.
  • Use PacGAN (Lin et al., 2018) — feed \(D\) tuples of samples; penalizes lack of diversity.

Choosing a loss in 2026.

StyleGAN family:               non-saturating + R1 + path-length + EMA on G.
Older / academic baseline:     WGAN-GP.
Audio vocoders (HiFi-GAN):     hinge + multi-resolution feature matching.

Minimal PyTorch — GAN training step (non-saturating + spectral norm).

import torch, torch.nn as nn

def gan_train_step(G, D, opt_G, opt_D, x_real, z_dim):
    """
    G: noise (B, z_dim) -> image
    D: image -> logit  (use spectral_norm on every Conv2d/Linear)
    """
    B = x_real.size(0)
    device = x_real.device

    # ---- D step ----
    z = torch.randn(B, z_dim, device=device)
    x_fake = G(z).detach()
    d_real = D(x_real)
    d_fake = D(x_fake)
    loss_D = (
        nn.functional.binary_cross_entropy_with_logits(d_real, torch.ones_like(d_real))
      + nn.functional.binary_cross_entropy_with_logits(d_fake, torch.zeros_like(d_fake))
    )
    opt_D.zero_grad()
    loss_D.backward()
    opt_D.step()

    # ---- G step (non-saturating) ----
    z = torch.randn(B, z_dim, device=device)
    x_fake = G(z)
    d_fake = D(x_fake)
    loss_G = nn.functional.binary_cross_entropy_with_logits(d_fake, torch.ones_like(d_fake))
    opt_G.zero_grad()
    loss_G.backward()
    opt_G.step()

    return loss_D.item(), loss_G.item()

6.3 Applications Where GANs Are Appropriate

GANs are not the modern default for generic text-to-image, but they remain best-in-class for several specific tasks, and you should recognize each by name.

Faces and portraits — StyleGAN family (1/2/3).

  • StyleGAN1 (2019): mapping network + style injection via AdaIN.
  • StyleGAN2 (2020): removed AdaIN artifacts via weight demodulation; added path-length regularization.
  • StyleGAN3 (2021): removed aliasing for translation/rotation-equivariance, smoother latent interpolations.

State-of-the-art for high-fidelity portrait synthesis at modest compute; widely used in research and for content generation. The StyleGAN family’s latent space (“\(\mathcal{W}\) space”) is extremely well-behaved for editing: directions in \(\mathcal{W}\) correspond to interpretable attributes (smile, age, glasses) and many editing techniques (GANSpace, StyleCLIP, e4e) operate by linear arithmetic in \(\mathcal{W}\).

Class-conditional ImageNet — BigGAN.

Large, class-conditional GAN with self-attention and spectral norm. Once held the FID record on ImageNet, surpassed by ADM (diffusion) in 2021. Truncation trick controls the sample-quality-vs-diversity trade-off at inference time.

Paired image-to-image translation — Pix2Pix (Isola et al., 2017).

Conditional GAN where the generator takes an input image (sketch, segmentation map, edge map) and outputs a paired target image (photo). Trained on aligned (input, output) pairs with an L1 + adversarial loss.

Unpaired image-to-image translation — CycleGAN (Zhu et al., 2017).

Two generators \(G : A\to B\) and \(F : B\to A\), two discriminators \(D_A, D_B\), and a cycle consistency loss \(\|F(G(x_A)) - x_A\|_1 + \|G(F(x_B)) - x_B\|_1\). Allows translation without paired data (horses↔︎zebras, summer↔︎winter, monet↔︎photo). The cycle loss is what makes the unsupervised setting tractable.

Super-resolution — ESRGAN, Real-ESRGAN (Wang et al., 2018/2021).

Generator upsamples a low-resolution image; discriminator distinguishes from high-resolution real images. Combined with perceptual loss (features from a frozen VGG) and adversarial loss. Real-ESRGAN extends to real-world degradations.

Neural vocoders — HiFi-GAN (Kong et al., 2020), MelGAN, BigVGAN.

Generator maps mel spectrogram to waveform. Discriminator is multi-resolution — several discriminators operate at different resolutions of the waveform to catch artifacts at different time scales. Sub-real-time, very high-quality TTS voices.

Deepfakes.

Faceswap, FSGAN, SimSwap — typically built on a face-encoder/decoder backbone with adversarial fine-tuning. Strong realism with tight identity preservation. (Ethical concerns are a real-world topic; technically these are still GANs at heart.)

As tokenizers — VQ-GAN (Esser et al., 2021).

A VQ-VAE-style autoencoder with an additional discriminator + perceptual loss. The quantized codes are the tokens fed to an autoregressive transformer (Taming Transformers, Parti, MaskGIT) for image generation. So even in the “diffusion era”, GAN-style training is the workhorse for high-quality image tokenizers.

Other applications worth knowing by name.

  • Music timbre transfer, voice conversion (CycleGAN-VC).
  • Inpainting (DeepFill, CoModGAN) — predates and is now mostly superseded by diffusion inpainting.
  • Video generation (TGAN, MoCoGAN; mostly superseded by video diffusion).
  • 3D shape generation (3D-GAN).
  • GAN inversion — given a real image \(x\), find \(z\) such that \(G(z) \approx x\); used for editing.

When to pick a GAN today.

  • You need a single forward pass at inference (no diffusion steps).
  • You’re working with faces or another well-bounded domain where StyleGAN-class models are SOTA.
  • You need sharp outputs (super-resolution, audio vocoder).
  • You need a discrete tokenizer with sharp reconstructions (VQ-GAN).
  • You have a small amount of paired translation data (Pix2Pix).
  • You have unpaired translation data with a clear cycle (CycleGAN).

When not to pick a GAN.

  • You need likelihood (use a flow or autoregressive model).
  • You’re doing generic text-to-image / video — diffusion / flow matching are now dominant.
  • You want stable, easy training without architectural tricks — use diffusion or a VAE.

Do I need exact likelihood?
  yes  -> Normalizing flow (or autoregressive)
  no   -> continue

Do I have abundant paired data with a clear input-output map AND need very fast inference?
  yes  -> conditional GAN (Pix2Pix), or feed-forward regression
  no   -> continue

Do I need SOTA image / audio / video quality?
  yes  -> Diffusion / latent diffusion / flow matching
  no   -> continue

Do I need a smooth, low-dimensional latent for downstream manipulation?
  yes  -> VAE (β-VAE or VQ-VAE)
  no   -> any of the above; choose by stability / compute budget

One-line cheat sheet to memorize for the assessment.

  • VAE — ELBO, fast, blurry, latent codec.
  • GAN — sharp, unstable, no likelihood, sharp niches (faces, vocoders).
  • Flow — exact likelihood, invertible, modest quality.
  • Diffusion — top quality, stable, slow without fast samplers.

MODULE 7: Evaluating and Improving Model Performance

7.1 Assessing Generative Model Outputs for Quality

Every generative-modeling paper compares on a small set of canonical metrics. Knowing each by name, what it measures, and its weaknesses is essential.

FID — Fréchet Inception Distance (Heusel et al., 2017). - Pass real and generated samples through a frozen Inception-v3, take the 2048-dim pool layer features, fit a multivariate Gaussian to each set, and compute the Fréchet (2-Wasserstein-like) distance:

\[ \mathrm{FID} \;=\; \|\mu_r - \mu_g\|^2 + \mathrm{Tr}\!\left(\Sigma_r + \Sigma_g - 2(\Sigma_r \Sigma_g)^{1/2}\right). \]

  • Lower is better. Sensitive to both quality (different mean) and diversity (different covariance).
  • Requires ~10k–50k samples for a stable estimate; FID at 1k samples is too noisy.
  • Inception-v3 was trained on ImageNet, so FID is partially ImageNet-biased; not perfect on faces, line art, or out-of-distribution domains.
  • The single most-quoted metric for image generation.

KID — Kernel Inception Distance (Bińkowski et al., 2018). - Same features as FID, but computes a kernel-based MMD instead of a Gaussian Fréchet distance. - Unbiased estimator; works better at small sample counts than FID. - Lower is better.

Inception Score (IS, Salimans et al., 2016). - \(\mathrm{IS} = \exp\!\left(\mathbb{E}_x\!\left[\mathrm{KL}(p(y\mid x) \,\Vert\, p(y))\right]\right)\), where \(p(y\mid x)\) is the Inception-v3 classifier output on a generated sample. - High IS = each sample looks like one class (sharp) AND the distribution over classes is uniform (diverse). - Higher is better. - Major weakness: blind to memorization (IS rewards a generator that memorizes one image per class). - Largely replaced by FID in modern papers, but still cited.

sFID — spatial FID. - FID computed on lower-level (spatial) Inception features (e.g., the early mixed_7c-equivalent block) instead of the deep pool features. Detects spatial artifacts (textures, local patches) that pool-FID misses.

Precision and Recall for Generative Models (Kynkäänniemi et al., 2019). - Construct nearest-neighbor manifolds for both real and fake feature sets in Inception space. - Precision = fraction of generated samples that lie inside the real-data manifold (i.e., look like real samples). - Recall = fraction of real samples that lie inside the generated manifold (i.e., diversity). - Decomposes FID into a quality / diversity 2-D space; very useful for diagnosing mode collapse (high precision, low recall) vs oversmoothing (low precision, high recall).

CLIP-Score / CLIP-FID. - For text-conditional models: cosine similarity between CLIP image embedding of the generated image and CLIP text embedding of the prompt, averaged over a benchmark prompt set. - A higher CLIP-Score means the model follows the prompt better; orthogonal to FID, which measures realism.

Perplexity (text). - Standard for language models. \(\mathrm{PPL}(x) = \exp\!\left(-\frac{1}{N}\sum_i \log p_\theta(x_i)\right)\). Lower is better. - Used for autoregressive language models; doesn’t apply to image diffusion.

NLL / BPD — Negative Log-Likelihood / Bits-Per-Dimension. - \(\mathrm{BPD}(x) = -\log_2 p_\theta(x)\,/\,d\), where \(d\) is the data dimensionality. Lower is better. - Comparable across likelihood-based models (autoregressive, flow, VAE, diffusion). - Not available for GANs.

Human evaluation. - Side-by-side preference tests; ELO ratings; specific attribute judgments (“does this image follow the prompt?”, “is this image free of artifacts?”). - Gold standard for text-to-image, but expensive and noisy.

MOS — Mean Opinion Score (audio). - Scale of 1 (bad) to 5 (excellent). Humans rate generated audio for naturalness. - The de facto standard for TTS and music generation.

Other metrics worth knowing by name.

  • PSNR, SSIM, LPIPS for reconstruction quality (super-resolution, inpainting). LPIPS uses deep features for perceptual similarity; PSNR/SSIM are pixel-level.
  • FVD (Fréchet Video Distance) — FID’s analog for video, using a video classifier (I3D).
  • FAD (Fréchet Audio Distance) — FID’s analog for audio.
  • Diversity score: pairwise distance among generated samples.
  • Coverage: variant of recall, the fraction of real samples whose nearest generated neighbor is within a small radius.

Common assessment trap — vocabulary.

  • Precision and recall in generative-model literature are not the same as in retrieval; they refer to manifold-based estimates of quality and diversity.
  • FID is a population-level metric; you cannot compute “the FID of a single image”.
  • Perplexity and NLL / BPD are equivalent up to a base change; perplexity is more common in text, BPD in image likelihood papers.

7.2 Adapting Inference Schedules for Model Performance

This subsection is about the inference-time knobs that change the output of a fixed trained model.

Number of sampling steps. Fewer steps = faster but lower quality (until you go below ~4 steps for non-distilled diffusion, where artifacts dominate). More steps = slower but diminishing returns past ~50 steps for most modern models. Always tune to a quality / latency Pareto front.

Sampler choice. See Module 5.4 for the full menu. The rough hierarchy in 2026:

Best quality at high steps:      DDPM (1000) or EDM Heun (35)
Best quality at moderate steps:  DPM-Solver++ (15-20) or UniPC (10-15)
Best quality at low steps:       Consistency model / LCM / SDXL Turbo (1-4 steps)
Editing / inversion:             DDIM (deterministic)

Classifier-free guidance (CFG) scale — \(w\) or \(s\). For text- or class-conditional models trained with conditioning dropout (5.1), the guided prediction at inference is

\[ \hat\epsilon_{\text{cfg}}(x_t, t, c) \;=\; (1+w)\,\epsilon_\theta(x_t, t, c) \,-\, w\,\epsilon_\theta(x_t, t, \varnothing), \]

where \(\varnothing\) is the null conditioning (“empty prompt”) and \(w \ge 0\) is the guidance scale (\(s = 1+w\) in some papers). \(w=0\) means unconditional (\(\epsilon_\theta(x_t,t,\varnothing)\)); \(w=1\) means exactly the conditional model; \(w > 1\) “exaggerates” the conditional direction.

  • Low CFG (\(w \le 1\)): faithful to the prompt but soft / averaged.
  • Mid CFG (\(w \in [2, 6]\)): the sweet spot for most prompts.
  • High CFG (\(w \ge 8\)): over-saturated, posterized, sometimes degraded.

CFG rescaling (Lin et al., “Common Diffusion Noise Schedules and Sample Steps are Flawed”, 2023) corrects the variance shift introduced by high \(w\) by rescaling the guided noise prediction so that its \(\sigma\) matches that of the conditional one. Stable Diffusion XL and most modern models use this.

Churn parameter (EDM stochasticity). EDM allows you to inject a small amount of noise at each step of the deterministic sampler — controlled by parameters \(S_{\text{churn}}\), \(S_{\text{tmin}}\), \(S_{\text{tmax}}\), \(S_{\text{noise}}\). Adding stochasticity can sometimes improve FID at the cost of reproducibility.

Ancestral vs deterministic samplers. Ancestral samplers add a small noise \(\sigma_t z\) at each step (DDPM, ancestral DDIM with \(\sigma>0\)). Deterministic samplers do not (DDIM with \(\sigma=0\), DPM-Solver, Heun without churn). Deterministic samplers are reproducible and allow inversion (DDIM inversion for editing). Ancestral can give slightly higher diversity.

Prompt weighting. In text-to-image, syntactically promote (or demote) parts of a prompt at inference. Stable Diffusion supports “()” / “[]” syntax, AUTOMATIC1111 supports weighted multi-prompt, and ComfyUI exposes per-token weight vectors. Mechanism: scale the cross-attention contribution from a subset of text tokens.

7.3 Integrating Text or Conditional Inputs for Flexible Generation

The defining advance from “generative model” to “controllable generative model” is conditioning. This subsection collects every major conditioning mechanism.

Classifier guidance (Dhariwal & Nichol, 2021).

Given a separately trained classifier \(p_\phi(c\mid x_t)\) (trained on noisy images to predict class), the guided score is

\[ \nabla_{x_t}\log p_t(x_t \mid c) \;=\; \nabla_{x_t}\log p_t(x_t) + s\,\nabla_{x_t} \log p_\phi(c\mid x_t), \]

where \(s\) is the guidance scale. The second term pulls samples toward high-classifier-probability regions.

Pros: works with an unconditional diffusion model + a small classifier. Cons: needs a classifier robust to noise across all \(t\); class-only signal is brittle for free-form text.

Classifier-free guidance (CFG; Ho & Salimans, 2022).

Train a single conditional network with random conditioning dropout (5.1) so that at inference

\[ \hat\epsilon_{\text{cfg}} = \epsilon_\theta(x_t, t, c) + w\,\big(\epsilon_\theta(x_t, t, c) - \epsilon_\theta(x_t, t, \varnothing)\big). \]

Equivalent to classifier guidance with \(\log p_\phi(c\mid x_t) \approx \log p_\theta(c\mid x_t)\) implied by the model itself. No separate classifier. Used by every modern text-to-image model.

Cross-attention conditioning.

Inside the U-Net, replace some self-attention layers with cross-attention where the queries are image features and the keys/values come from a text encoder (e.g., CLIP, T5, OpenCLIP). The text encoder outputs a sequence of token embeddings, and the U-Net attends to that sequence at multiple resolutions. This is the conditioning mechanism in Stable Diffusion 1.x / 2.x / SDXL / SD3.

ControlNet (Zhang et al., 2023).

Adds a parallel branch to a pretrained U-Net that takes a structured condition (Canny edges, depth map, pose skeleton, segmentation mask) and injects feature additions at every U-Net stage. The parallel branch is initialized from the U-Net’s encoder and its outputs are added with zero-initialized convolutions, so the model starts as identity and gradually learns to use the new condition.

  • Train the ControlNet on (image, condition) pairs while freezing the base U-Net.
  • Multiple ControlNets can be combined at inference (e.g., depth + pose).
  • The conditioning is spatially aligned, unlike text which is global.

T2I-Adapter (Mou et al., 2023).

Lighter alternative to ControlNet — a small adapter network that produces feature additions at each U-Net resolution. Similar effect, fewer parameters, slightly less expressive.

IP-Adapter (Image Prompt Adapter, Ye et al., 2023).

Conditioning on a reference image (rather than text). A separate cross-attention path attends to features from a CLIP image encoder of the reference. Lets you say “produce something in the style of this image” or “this person’s face in this scene”.

Textual Inversion (Gal et al., 2022).

Learn a new word in the text encoder’s embedding space that represents a concept from a small set of images. Add a pseudo-token <sks> to the vocabulary; freeze the diffusion model and the text encoder; optimize only the embedding of <sks> to reconstruct the input images. Then use <sks> in prompts. Tiny (~kilobytes) per-concept files.

DreamBooth (Ruiz et al., 2022).

Personalize a diffusion model to a specific subject by fine-tuning the full diffusion model (or, in practice, a LoRA on top of it) on a few images of that subject, paired with a unique token. Includes a prior-preservation loss to prevent catastrophic forgetting on the general class.

LoRA on diffusion (Hu et al., 2021, applied to diffusion in Dreambooth-LoRA).

Apply low-rank adapters \(\Delta W = AB^\top\) with \(A \in \mathbb{R}^{d\times r}\), \(B \in \mathbb{R}^{d \times r}\), \(r \ll d\), to the cross-attention layers (and optionally other layers) of the U-Net. Train only these adapters. ~1–10 MB per “style” / “character” LoRA. The default mechanism for community fine-tunes of Stable Diffusion / SDXL.

Image-to-image / SDEdit (Meng et al., 2022).

Given an input image \(x_0\), add a controlled amount of noise to it: \(x_t = \sqrt{\bar\alpha_t} x_0 + \sqrt{1-\bar\alpha_t}\epsilon\) for some intermediate \(t\), then run the reverse process from there. The result is a sample that preserves the coarse structure of \(x_0\) but takes on details from the conditioning prompt. Parameter: “strength” / “noising level” \(\in [0,1]\) — higher means more deviation from the input.

Inpainting.

Mask a region of the input; everywhere outside the mask, replace the diffusion intermediate with the (noised) original at each step. The model fills the masked region while respecting the unmasked context. Variants:

  • RePaint (Lugmayr et al., 2022): re-condition on the unmasked context by resampling.
  • Stable Diffusion Inpainting (a fine-tuned U-Net with explicit mask channels).
  • GLIDE / DALL·E 2 inpainting (built-in).

Pix2Pix / Conditional U-Net.

Train a paired image-to-image diffusion model: condition on the input image by concatenating it to \(x_t\) in the U-Net’s input channels (no special architecture needed). Used in cascaded diffusion super-resolution and many translation tasks.

7.4 Comparing Diffusion Models with GANs for Tractability and Application Fit

Putting the GAN material (Module 6) and diffusion material (Modules 3–5) side by side answers the assessment’s diffusion-vs-GAN comparison questions.

Tractability of likelihood. - Diffusion: tractable variational lower bound; can compute exact NLL via the probability-flow ODE (Song et al., 2021). - GAN: no likelihood at all (implicit model).

Sample quality (FID). - Diffusion: state-of-the-art on ImageNet, COCO text-to-image, faces (with right tuning), audio, video. - GAN: still SOTA for faces (StyleGAN3); strong for super-resolution and vocoders; surpassed elsewhere.

Mode coverage. - Diffusion: strong (training objective is mode-covering by construction). - GAN: weak; mode collapse is the canonical failure mode.

Training stability. - Diffusion: stable, scales smoothly with model size and data. - GAN: notoriously unstable; needs spectral norm, R1 penalty, TTUR, careful tuning.

Sampling speed. - Diffusion (vanilla): \(T = 50\text{–}1000\) network calls. - Diffusion (with DPM-Solver++ / UniPC / consistency distillation): 1–20 calls. - GAN: 1 call. Still faster than distilled diffusion in raw network forward passes per sample, although distilled diffusion is now competitive in wall-clock for many tasks.

Architecture and training complexity. - Diffusion: one network (epsilon predictor), one loss (MSE). Stable, simple. - GAN: two networks (G, D), one min-max game, several regularizers. Brittle.

Conditioning. - Diffusion: classifier guidance, CFG, ControlNet, IP-Adapter, LoRA, inpainting, SDEdit, image-to-image — a rich ecosystem. - GAN: cGAN, AC-GAN, projection discriminator; style-based latent edits (StyleGAN). Less plug-and-play.

Inversion (mapping a real image back to latent). - Diffusion: DDIM inversion (deterministic, lossy in practice). Editing via prompt-to-prompt, null-text inversion, SDEdit. - GAN: optimization-based or encoder-based inversion (e4e, pSp); StyleGAN \(\mathcal{W}/\mathcal{W}^+\) space.

Compute and memory. - Diffusion at high resolution requires latent diffusion to be tractable (Stable Diffusion encodes 512×512 images to 64×64 latents). - GAN at high resolution is feasible without a latent compression step (StyleGAN3 trains directly in pixel space at 1024×1024).

When to pick which (cheat sheet).

Pick a diffusion model when:
  - You want a single, stable training recipe.
  - You want strong text conditioning out of the box.
  - You can afford 10-50 network calls per sample, or you can distill.
  - You need mode coverage / diversity.

Pick a GAN when:
  - You need 1-call inference at maximum quality (especially faces).
  - You need a discrete tokenizer (VQ-GAN).
  - You need a super-resolution / vocoder backbone.
  - You have paired image-to-image data with strict 1-to-1 mapping.

The one-sentence summary. Diffusion is the default; GANs survive where you specifically need 1-pass inference, sharp textures, or a discrete tokenizer.

8.1 Variational Autoencoders in Depth

Definition. A VAE is a latent-variable generative model with:

  • Prior \(p(z) = \mathcal{N}(0, I)\), \(z \in \mathbb{R}^{d_z}\).
  • Decoder (generative network) \(p_\theta(x \mid z)\), often \(\mathcal{N}(x;\,\mu_\theta(z), \sigma_x^2 I)\) for continuous data, or a Bernoulli / Categorical for binary / discrete data.
  • Encoder (recognition / inference network) \(q_\phi(z \mid x) = \mathcal{N}(z;\,\mu_\phi(x), \mathrm{diag}(\sigma_\phi^2(x)))\).

Both encoder and decoder are neural networks. Training maximizes the ELBO jointly over \(\theta\) (decoder) and \(\phi\) (encoder).

Full ELBO derivation, with the reparameterization step worked in.

Start from the marginal log-likelihood,

\[ \log p_\theta(x) = \log \int p_\theta(x, z)\,dz = \log \mathbb{E}_{p(z)}[p_\theta(x\mid z)]. \]

Introduce \(q_\phi(z\mid x)\) and multiply/divide:

\[ \log p_\theta(x) \;=\; \log \mathbb{E}_{q_\phi(z\mid x)}\!\left[\frac{p_\theta(x\mid z)\,p(z)}{q_\phi(z\mid x)}\right] \;\ge\; \mathbb{E}_{q_\phi(z\mid x)}\!\left[\log \frac{p_\theta(x\mid z)\,p(z)}{q_\phi(z\mid x)}\right] \]

by Jensen’s inequality (because \(\log\) is concave). The right-hand side is the ELBO, which splits cleanly:

\[ \mathrm{ELBO}(x;\theta,\phi) \;=\; \underbrace{\mathbb{E}_{q_\phi(z\mid x)}[\log p_\theta(x\mid z)]}_{\text{reconstruction}} \;-\; \underbrace{\mathrm{KL}(q_\phi(z\mid x) \,\Vert\, p(z))}_{\text{regularizer}}. \]

The VAE loss is \(-\mathrm{ELBO}\). The two terms have intuitive meanings:

  • Reconstruction: how well the decoder reconstructs \(x\) from \(z\sim q_\phi(z\mid x)\).
  • KL regularizer: how close the approximate posterior is to the prior. Keeps the latent space from collapsing into isolated point masses, and ensures that sampling from \(p(z)\) at generation time yields plausible \(x\).

Closed-form Gaussian–Gaussian KL. For \(q_\phi(z\mid x) = \mathcal{N}(\mu, \mathrm{diag}(\sigma^2))\) and \(p(z) = \mathcal{N}(0, I)\) in \(d_z\) dimensions,

\[ \mathrm{KL}(q_\phi \Vert p) \;=\; \tfrac{1}{2}\sum_{j=1}^{d_z}\!\left(\mu_j^2 + \sigma_j^2 - 1 - \log \sigma_j^2\right). \]

This is computed exactly in code — no Monte Carlo needed. Most VAE codebases parametrize \(\log\sigma^2\) directly (so \(\sigma^2 = e^{\log\sigma^2}\)) for numerical stability.

The reparameterization trick. We need gradients of the reconstruction term w.r.t. \(\phi\), which sits inside the sampling of \(z\). The naïve score-function estimator (REINFORCE) has very high variance. The reparameterization trick rewrites \(z\sim \mathcal{N}(\mu_\phi(x), \mathrm{diag}(\sigma_\phi^2(x)))\) as a deterministic function of a parameter-free noise source:

\[ z \;=\; \mu_\phi(x) \,+\, \sigma_\phi(x) \odot \epsilon, \qquad \epsilon \sim \mathcal{N}(0, I). \]

Now \(z\) is a differentiable function of \((\phi, x, \epsilon)\), and \(\nabla_\phi \mathbb{E}_{\epsilon}[f(z)] = \mathbb{E}_\epsilon[\nabla_\phi f(z)]\). The gradient flows from the reconstruction term through the decoder, through \(z\), into the encoder’s parameters.

For more general posteriors, variants of the trick exist: Gumbel–Softmax for categorical \(z\); IAF (inverse autoregressive flow) for autoregressive posteriors; planar / radial flows for more expressive posteriors.

Worked example — VAE forward pass. Take a tiny 2-D VAE: \(d_x = d_z = 2\). Suppose for one input \(x\) the encoder outputs \(\mu_\phi(x) = (0.1, -0.2)\) and \(\log \sigma_\phi^2(x) = (-2, -1)\), so \(\sigma_\phi = (e^{-1}, e^{-0.5}) \approx (0.368, 0.607)\). Sample \(\epsilon = (0.5, -0.3)\). Then

\[ z = \big(0.1 + 0.368\cdot 0.5,\; -0.2 + 0.607\cdot(-0.3)\big) \;=\; (0.284,\; -0.382). \]

If the decoder is the identity for illustration, \(\hat x = z\). The Gaussian (with \(\sigma_x = 1\)) reconstruction NLL is \(\tfrac{1}{2}\|x - \hat x\|^2 + \text{const}\).

The KL term is, in nats,

\[ \mathrm{KL} \;=\; \tfrac{1}{2}\big(0.1^2 + 0.368^2 - 1 - (-2) + (-0.2)^2 + 0.607^2 - 1 - (-1)\big) \;\approx\; \tfrac{1}{2}\big(0.01 + 0.135 + 1 + 0.04 + 0.368 + 0\big) \;\approx\; 0.78. \]

The total loss is reconstruction + 0.78 per example.

Minimal PyTorch — VAE encoder, decoder, and training step.

import torch
import torch.nn as nn
import torch.nn.functional as F

class Encoder(nn.Module):
    def __init__(self, d_in: int, d_h: int, d_z: int):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(d_in, d_h), nn.ReLU(),
            nn.Linear(d_h, d_h), nn.ReLU(),
        )
        self.mu        = nn.Linear(d_h, d_z)
        self.log_sigma = nn.Linear(d_h, d_z)              # log of standard deviation

    def forward(self, x):
        # x: (B, d_in)
        h = self.net(x)                                    # (B, d_h)
        return self.mu(h), self.log_sigma(h)               # each (B, d_z)

class Decoder(nn.Module):
    def __init__(self, d_z: int, d_h: int, d_out: int):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(d_z, d_h), nn.ReLU(),
            nn.Linear(d_h, d_h), nn.ReLU(),
            nn.Linear(d_h, d_out),
        )

    def forward(self, z):
        # z: (B, d_z)  ->  x_hat: (B, d_out)
        return self.net(z)

def vae_elbo_loss(x, x_hat, mu, log_sigma, beta: float = 1.0):
    """
    Returns the (negative) ELBO averaged over the batch.
      recon: per-example sum-of-squares (Gaussian decoder with sigma_x = 1)
      kl:    closed-form KL between N(mu, sigma^2) and N(0, 1) per dim
    """
    B = x.size(0)
    recon = F.mse_loss(x_hat, x, reduction="sum") / B
    kl = 0.5 * (mu.pow(2) + (2 * log_sigma).exp() - 1 - 2 * log_sigma).sum() / B
    return recon + beta * kl, recon.item(), kl.item()

def vae_train_step(enc, dec, opt, x, beta: float = 1.0):
    mu, log_sigma = enc(x)
    eps = torch.randn_like(mu)                              # (B, d_z)
    z = mu + log_sigma.exp() * eps                          # reparameterization
    x_hat = dec(z)
    loss, r, k = vae_elbo_loss(x, x_hat, mu, log_sigma, beta=beta)
    opt.zero_grad(); loss.backward(); opt.step()
    return loss.item(), r, k

Failure modes.

  • Posterior collapse. \(q_\phi(z\mid x) \approx p(z)\) for all \(x\): the encoder ignores \(x\), the decoder ignores \(z\), and the model degenerates to a non-conditional decoder. Symptom: KL term drops to nearly zero and reconstruction barely improves over a marginal autoregressive baseline. Mitigations: KL annealing (start with \(\beta \approx 0\), ramp up); free bits (don’t penalize KL below a small threshold per dim); a less-powerful decoder; use a more expressive prior.
  • Blurry samples. A Gaussian decoder’s MAP output is the mean over plausible reconstructions, which averages across modes and blurs details. Mitigations: discrete-latent VAE (VQ-VAE) + autoregressive prior on top; pair the VAE with a sharp downstream generator; or just use the VAE as a latent codec and not as the final generator.
  • Imbalanced ELBO. When \(d_x \gg d_z\), the reconstruction term dwarfs the KL term, and the KL has negligible regularization effect; or vice versa. β-VAE with \(\beta < 1\) (more reconstruction) or \(\beta > 1\) (more KL / disentanglement) trades these off.
  • Mismatched decoder noise. Using a Gaussian decoder with \(\sigma_x = 1\) when the data is binary, or vice versa, gives a badly-calibrated likelihood. Match the likelihood to the data type: Bernoulli for \(\{0,1\}\), categorical for tokens, Gaussian/Logistic for continuous in \(\mathbb{R}^d\).

Variants worth knowing by name.

  • β-VAE (Higgins et al., 2017). ELBO with KL weighted by \(\beta\). Larger \(\beta\) encourages disentanglement of factors of variation; smaller \(\beta\) recovers more reconstruction fidelity.
  • VQ-VAE (van den Oord et al., 2017). Replace continuous Gaussian latents with discrete codebook entries; train with a straight-through estimator and a commitment loss. The latents become categorical tokens that downstream autoregressive transformers or masked-token models can predict (DALL·E v1, Parti, MaskGIT, MusicGen).
  • NVAE / VDVAE (Vahdat & Kautz, 2020; Child, 2020). Hierarchical VAEs with very deep latent hierarchies; competitive sample quality.
  • VAE-GAN (Larsen et al., 2016). Add a discriminator on \(\hat x\); trains a sharper decoder.
  • Conditional VAE (CVAE). Condition encoder and decoder on a label \(y\): \(q_\phi(z\mid x, y)\), \(p_\theta(x\mid z, y)\). Used for one-to-many prediction (motion forecasting, imputation).

Modern role: latent codec for diffusion and transformers. The dominant 2024–2026 use of VAEs in image generation is not as the end-to-end generator. Instead, train a small VAE (KL autoencoder) or VQ-VAE to compress images to a low-resolution latent (e.g., 512×512 RGB → 64×64×4 for Stable Diffusion’s latent space). Then train the real generative model — a latent diffusion U-Net (Stable Diffusion), a DiT (DiT, SD3), or an autoregressive transformer over discrete tokens (Parti, MaskGIT, Muse) — in that latent space.

8.2 Normalizing Flows in Depth

Definition. A normalizing flow is a parametrized invertible function \(f_\theta: \mathbb{R}^d \to \mathbb{R}^d\) that pushes a simple base density \(p_z(z)\) (typically \(\mathcal{N}(0, I)\)) onto the data density:

\[ x = f_\theta(z), \qquad z = f_\theta^{-1}(x). \]

Change of variables. For an invertible differentiable \(f\), the densities are related by

\[ p_\theta(x) \;=\; p_z(f_\theta^{-1}(x))\,\left|\det \frac{\partial f_\theta^{-1}(x)}{\partial x}\right|, \]

equivalently (taking \(\log\) and using \(\det J_{f^{-1}} = 1/\det J_f\)),

\[ \log p_\theta(x) \;=\; \log p_z(f_\theta^{-1}(x)) \;-\; \log \left|\det \frac{\partial f_\theta(z)}{\partial z}\right|_{z = f_\theta^{-1}(x)}. \]

This is exact — no bound, no Monte Carlo. The likelihood is fully tractable provided we can (i) invert \(f\) and (ii) compute \(\log|\det J_f|\) cheaply.

The architectural trick: special invertible layers. A general neural net is not invertible and has an intractable Jacobian. Flows compose specially designed layers, each of which is invertible and has a tractable Jacobian determinant. The most common building block is the affine coupling layer (RealNVP, Dinh et al., 2016):

  1. Split the input along a feature axis into two halves: \(z = (z_a, z_b)\).
  2. Compute scale \(s\) and shift \(t\) from \(z_a\) via a (non-invertible) neural net: \((s, t) = \mathrm{NN}(z_a)\).
  3. Transform only \(z_b\):

\[ z_a' = z_a, \qquad z_b' = z_b \odot \exp\!\big(s(z_a)\big) + t(z_a). \]

Why this is invertible: \(z_b = (z_b' - t(z_a'))\odot \exp(-s(z_a'))\), with \(z_a' = z_a\) unchanged. The neural net is not inverted; the coupling structure makes inversion trivial.

Why the Jacobian is tractable: \(\partial z'/\partial z\) is triangular in the partition, so

\[ \det \frac{\partial z'}{\partial z} \;=\; \prod_i \exp\!\big(s_i(z_a)\big) \;=\; \exp\!\Big(\sum_i s_i(z_a)\Big), \]

and the log-det is simply \(\sum_i s_i(z_a)\) — one sum, no matrix factorization.

A flow then stacks several coupling layers, alternating which half is transformed (or applying a permutation / learned invertible 1×1 convolution between layers to mix dimensions).

Worked toy — Jacobian of a one-layer coupling. Take \(d = 2\), \(z = (z_1, z_2)\), with the coupling layer transforming \(z_2\) given \(z_1\). Suppose for the chosen \(z_1\) the net returns \(s(z_1) = 0.3\), \(t(z_1) = 0.5\). Then

\[ J_f = \begin{pmatrix} 1 & 0 \\ \partial t/\partial z_1 \cdot \text{etc.} & e^{0.3} \end{pmatrix}, \]

a lower-triangular matrix, so \(\det J_f = 1 \cdot e^{0.3} = e^{0.3} \approx 1.350\). The contribution to \(\log p_\theta(x)\) from this layer is \(-0.3\).

Minimal PyTorch — affine coupling layer.

import torch
import torch.nn as nn

class AffineCoupling(nn.Module):
    def __init__(self, d: int, d_hidden: int = 128):
        super().__init__()
        assert d % 2 == 0
        self.d = d
        self.net = nn.Sequential(
            nn.Linear(d // 2, d_hidden), nn.ReLU(),
            nn.Linear(d_hidden, d_hidden), nn.ReLU(),
            nn.Linear(d_hidden, d),                       # outputs (s, t) concatenated
        )

    def forward(self, z):
        # z: (B, d)
        za, zb = z.chunk(2, dim=1)                        # each (B, d/2)
        s, t = self.net(za).chunk(2, dim=1)               # each (B, d/2)
        s = torch.tanh(s)                                 # tanh for numerical stability
        zb_out = zb * torch.exp(s) + t                    # (B, d/2)
        x = torch.cat([za, zb_out], dim=1)                # (B, d)
        log_det = s.sum(dim=1)                            # (B,)
        return x, log_det

    def inverse(self, x):
        xa, xb = x.chunk(2, dim=1)
        s, t = self.net(xa).chunk(2, dim=1)
        s = torch.tanh(s)
        zb = (xb - t) * torch.exp(-s)
        return torch.cat([xa, zb], dim=1), -s.sum(dim=1)

A complete flow stacks several AffineCouplings with alternating partitions (achieved by reversing or shuffling the dimensions between layers, or a learned 1×1 convolution in the Glow style).

Training objective. Direct maximum likelihood:

\[ \mathcal{L}(\theta) \;=\; -\,\frac{1}{N}\sum_{i=1}^{N}\Bigg[\log p_z(f_\theta^{-1}(x^{(i)})) + \sum_{\ell} \log |\det J_{f_\ell}^{-1}|\Bigg]. \]

In practice you accumulate the per-layer log-determinants as the data is pushed backwards through the flow (from \(x\) to \(z\)), or in the forward direction if you parametrize the inverse map.

Sampling. One forward pass: draw \(z \sim \mathcal{N}(0, I)\) and compute \(x = f_\theta(z)\).

Important named flows.

  • NICE (Dinh et al., 2015). Additive coupling only (\(s = 0\)). Jacobian determinant is exactly 1; volume-preserving.
  • RealNVP (Dinh et al., 2016). Affine coupling with both \(s\) and \(t\); tractable, non-volume-preserving.
  • Glow (Kingma & Dhariwal, 2018). RealNVP + invertible 1×1 convolutions to mix channels at each level. SOTA flow for images at the time.
  • MAF / IAF (Papamakarios et al., 2017; Kingma et al., 2016). Masked autoregressive flows. MAF is fast at density evaluation, slow at sampling; IAF is the reverse — a useful tool for variational posteriors.
  • FFJORD (Grathwohl et al., 2018). Continuous normalizing flow parametrized by an ODE; the Jacobian trace is computed via Hutchinson’s trace estimator.
  • Neural Spline Flow (Durkan et al., 2019). Expressive monotonic-spline transformations of each coordinate, conditioned on the others.

Where flows are used in 2026.

  • Scientific density estimation (cosmology, particle physics surrogates, neuroscience): when you want a calibrated \(p(x)\) with uncertainty.
  • Bayesian inference with flow-based variational posteriors (IAF for VAE posteriors).
  • Lossless compression (bits-back coding with a flow likelihood).
  • Flow matching / rectified flow — not a classical flow but uses the same change-of-variables idea applied to an ODE. The most successful 2024–2026 application of flow-style modeling lives here (Stable Diffusion 3, Movie Gen).

Trade-offs.

Flow strength Cost
Exact log-likelihood Architectural constraint (invertibility + tractable Jacobian)
Free inversion / encoding (\(z = f^{-1}(x)\)) Modest sample quality vs diffusion / GAN at the same compute
Sampling in 1 forward pass High parameter count for expressive flows
Composability (stack flows like layers) Numerical stability depends on log-det not exploding

8.3 Autoregressive Models in Depth

Definition. An autoregressive model factorizes the joint as a product of conditionals over some ordering of the dimensions:

\[ p_\theta(x) \;=\; \prod_{t=1}^{N} p_\theta(x_t \mid x_{<t}). \]

Each conditional is parametrized by a neural network. The ordering can be left-to-right (text), raster scan (images), or any fixed permutation; the model is causal — at position \(t\) it only looks at earlier positions \(x_{<t}\).

Why this works.

  • The factorization is exact (chain rule of probability).
  • Each conditional \(p_\theta(x_t\mid x_{<t})\) is a low-dimensional distribution (categorical for tokens, Gaussian mixture for continuous), tractable in closed form.
  • Training reduces to next-token classification (or regression); cross-entropy loss; very stable.

Training objective. Maximum likelihood, per token:

\[ \mathcal{L}(\theta) \;=\; -\,\frac{1}{N}\sum_{t=1}^{N}\log p_\theta(x_t \mid x_{<t}). \]

For categorical tokens this is the familiar cross-entropy loss. The whole-sequence likelihood is the product of per-step probabilities, which is also how perplexity is defined.

Architectures.

  • PixelRNN / PixelCNN (van den Oord et al., 2016): RNN / CNN over image pixels in raster scan order. Use masked convolutions to ensure causality.
  • WaveNet (van den Oord et al., 2016): dilated causal convolutions over raw audio samples (16 kHz). One sample at a time.
  • GPT / Llama / Mistral / Qwen / Claude / Gemini (2018–2026): decoder-only transformers over discrete tokens (BPE / SentencePiece / tiktoken). The defining architecture of modern language modeling.
  • ImageGPT (Chen et al., 2020): GPT-style transformer over flattened low-resolution pixel sequences.
  • Parti (Yu et al., 2022): autoregressive transformer over VQ-GAN image tokens, conditioned on T5 text encoder output.
  • MaskGIT / Muse (Chang et al., 2022, 2023): parallel masked-token decoding — autoregressive in spirit (a sequence of unmasking steps) but updates many positions per step at low quality cost.
  • VALL-E (Wang et al., 2023): autoregressive transformer over Encodec audio tokens for TTS.
  • MusicGen / AudioGen (Copet et al., 2023): autoregressive transformer over audio codec tokens.

Sampling. Sequential at the token level: predict \(x_1\), then \(x_2 \mid x_1\), and so on. Standard sampling tricks apply: temperature, top-\(k\), top-\(p\) / nucleus, beam search, speculative decoding, contrastive search. Parallel sampling (MaskGIT) updates many positions per step at a small quality cost.

Connection to diffusion.

  • AR models are fully observed at every step (no latent), while diffusion has \(T\) latent intermediate states.
  • AR likelihood is exact; diffusion’s is a lower bound.
  • AR sampling cost scales with the number of tokens; diffusion sampling cost scales with the number of denoising steps (which can be far smaller than the number of pixels because all positions are predicted in parallel per step).

Where AR dominates.

  • Text generation (LLMs).
  • Code generation.
  • Tokenized audio (codec-token LMs are competitive with audio diffusion).
  • Discrete-token image generation (Parti, MaskGIT; competitive with diffusion at scale and far faster to sample with parallel decoding).

Where AR is not the modern default.

  • High-resolution continuous-pixel image / video / 3-D generation, because the per-pixel sequential cost is prohibitive. Diffusion / flow matching parallelize across all positions per step.

8.4 Latent Diffusion — Encoder + Diffusion + CFG, Together

The Stable Diffusion / latent-diffusion architecture combines a VAE encoder (a continuous-latent variant of VQ-VAE — sometimes called a KL autoencoder), a diffusion U-Net or DiT operating in the latent space, and a text encoder (CLIP / OpenCLIP / T5) feeding the U-Net via cross-attention. The end-to-end pipeline:

text prompt c
   │
   ├──► text encoder (CLIP / T5) ──► text embeddings (B, L, D_text)
   │
input image x (B, 3, H, W) in [-1, 1]
   │
VAE encoder E ──► latent z = E(x)             # e.g., (B, 4, H/8, W/8)
   │
[ training: add noise to z, train U-Net to denoise, cross-attending to text ]
   │
sampling: start z_T ~ N(0, I) in latent space, run N denoising steps with CFG
   │
final latent z_0
   │
VAE decoder D ──► reconstructed image x' = D(z_0)       # (B, 3, H, W)

Minimal PyTorch — Latent Diffusion training step.

def latent_diffusion_train_step(
    vae,          # frozen autoencoder; .encode -> latent, .decode -> image
    unet,         # eps_theta(z_t, t, text_emb) -> predicted noise
    text_encoder, # tokenized prompts -> text embeddings (B, L, D)
    schedule,     # dict from make_*_schedule
    opt,
    x_image,      # (B, 3, H, W), normalized to [-1, 1]
    prompt_tokens,
    p_drop=0.1,   # conditioning-dropout probability for CFG
):
    B = x_image.size(0)

    # 1. Encode image to latent (no gradient through VAE)
    with torch.no_grad():
        z0 = vae.encode(x_image).sample()                  # (B, 4, H/8, W/8)
        z0 = z0 * 0.18215                                  # SD scaling factor

    # 2. Text conditioning with random CFG dropout
    text_emb = text_encoder(prompt_tokens)                 # (B, L, D_text)
    drop = (torch.rand(B, device=z0.device) < p_drop)
    null_emb = text_encoder.null_embedding.expand_as(text_emb)
    text_emb = torch.where(drop[:, None, None], null_emb, text_emb)

    # 3. Sample t and noise in latent space
    T = schedule["beta"].shape[0]
    t = torch.randint(0, T, (B,), device=z0.device)
    sqrt_ab    = schedule["sqrt_alpha_bar"][t].view(B, 1, 1, 1)
    sqrt_1mab  = schedule["sqrt_one_minus_alpha_bar"][t].view(B, 1, 1, 1)
    eps = torch.randn_like(z0)
    z_t = sqrt_ab * z0 + sqrt_1mab * eps

    # 4. Predict noise and minimize MSE
    eps_hat = unet(z_t, t, text_emb)
    loss = ((eps - eps_hat) ** 2).mean()

    opt.zero_grad(); loss.backward()
    torch.nn.utils.clip_grad_norm_(unet.parameters(), 1.0)
    opt.step()
    return loss.item()

Minimal PyTorch — DDIM sampler with classifier-free guidance and VAE decode.

@torch.no_grad()
def ddim_sample_cfg(
    unet, text_encoder, vae, schedule,
    prompt_tokens, null_tokens,
    num_steps: int = 30,
    cfg_scale: float = 7.5,
    shape=(1, 4, 64, 64),
    device: str = "cuda",
):
    """
    Text-conditional DDIM sampler with classifier-free guidance.

    Returns a decoded image tensor of shape (B, 3, 8H, 8W).
    """
    text_emb = text_encoder(prompt_tokens)        # (B, L, D)
    null_emb = text_encoder(null_tokens)          # (B, L, D)

    T = schedule["beta"].shape[0]
    timesteps = torch.linspace(T - 1, 0, num_steps + 1, dtype=torch.long, device=device)

    z = torch.randn(shape, device=device)         # (B, 4, h, w)
    for i in range(num_steps):
        t_now = timesteps[i].expand(shape[0])
        t_prv = timesteps[i + 1].expand(shape[0])

        # CFG: run the U-Net twice and combine
        eps_cond   = unet(z, t_now, text_emb)
        eps_uncond = unet(z, t_now, null_emb)
        eps = eps_uncond + cfg_scale * (eps_cond - eps_uncond)

        ab_now = schedule["alpha_bar"][t_now].view(-1, 1, 1, 1)
        ab_prv = schedule["alpha_bar"][t_prv].view(-1, 1, 1, 1)

        # Predicted clean latent x0 from eps
        x0_pred = (z - torch.sqrt(1 - ab_now) * eps) / torch.sqrt(ab_now)

        # Deterministic DDIM step (sigma_t = 0)
        z = torch.sqrt(ab_prv) * x0_pred + torch.sqrt(1 - ab_prv) * eps

    img = vae.decode(z / 0.18215)
    return img.clamp(-1, 1)

This single function captures three of the most important concepts in modern diffusion deployment: DDIM as the sampler, CFG as the conditioning mechanism, and latent diffusion as the architecture (the diffusion loop operates in (B, 4, h, w) and only the final latent gets decoded back to pixel space).

Classifier-free guidance — standalone code snippet.

def classifier_free_guidance(unet, z_t, t, text_emb, null_emb, cfg_scale: float):
    """
    Returns the CFG-combined noise prediction.
      hat_eps_cfg = (1 + w) * eps(z, t, c) - w * eps(z, t, null)
    Using cfg_scale s = (1 + w), this becomes
      hat_eps_cfg = eps_uncond + s * (eps_cond - eps_uncond).
    """
    eps_cond   = unet(z_t, t, text_emb)
    eps_uncond = unet(z_t, t, null_emb)
    return eps_uncond + cfg_scale * (eps_cond - eps_uncond)

APPENDIX A — Glossary

A consolidated list of named concepts, in the order they tend to appear. Use this as a “did I learn this?” checklist.

  • \(p_{\text{data}}(x)\) — the true data distribution; unknown but sampled.
  • \(p_\theta(x)\) — the model distribution, parametrized by \(\theta\).
  • Generative model — models the data distribution (joint or marginal); can synthesize new samples.
  • Discriminative model — models the conditional \(p(y\mid x)\); cannot synthesize \(x\).
  • Bayes’ rule — \(p(y\mid x) = p(x\mid y)p(y)/p(x)\).
  • Latent variable — unobserved variable in a generative model (e.g., \(z\) in a VAE).
  • Prior \(p(z)\) — distribution over latent variable, often \(\mathcal{N}(0, I)\).
  • Posterior \(p_\theta(z\mid x)\) — distribution over latents given an observation.
  • Approximate posterior \(q_\phi(z\mid x)\) — variational distribution used to bound the marginal likelihood.
  • ELBO — Evidence Lower Bound, \(\mathrm{ELBO} = \mathbb{E}_{q}[\log p_\theta(x\mid z)] - \mathrm{KL}(q_\phi(z\mid x)\Vert p(z))\).
  • KL divergence — \(\mathrm{KL}(p\Vert q) = \mathbb{E}_p[\log p - \log q]\).
  • Jensen–Shannon divergence (JSD) — symmetric version of KL.
  • Wasserstein distance — Earth Mover’s distance; minimum work to morph one distribution into another.
  • Score \(\nabla_x \log p(x)\) — gradient of log-density; the quantity score-based models predict.
  • Score matching — fitting \(s_\theta(x)\) to \(\nabla_x \log p(x)\) without computing the normalizer.
  • Denoising score matching — score matching using noised inputs; equivalent to DDPM’s noise-prediction objective.
  • Energy-based model (EBM) — \(p_\theta(x) = e^{-E_\theta(x)}/Z\); unnormalized.
  • Manifold hypothesis — natural data lies on a low-dimensional manifold inside a high-dimensional ambient space.
  • Reparameterization trick — sample \(z = \mu + \sigma\cdot\epsilon\) with \(\epsilon \sim \mathcal{N}(0, I)\) to allow gradients through stochastic sampling.
  • Reconstruction term — \(\mathbb{E}_q[\log p_\theta(x\mid z)]\) in the ELBO.
  • Regularizer / KL term — \(\mathrm{KL}(q_\phi(z\mid x) \Vert p(z))\) in the ELBO.
  • Posterior collapse — VAE failure where \(q_\phi(z\mid x) \approx p(z)\) for all \(x\); the latents become uninformative.
  • β-VAE — VAE with weighted KL term (\(\beta \cdot \mathrm{KL}\)); higher \(\beta\) encourages disentanglement, often at the cost of reconstruction.
  • VQ-VAE — Vector-Quantized VAE; discrete latents from a learned codebook.
  • GAN — Generative Adversarial Network — generator + discriminator playing a minimax game.
  • Generator \(G\) — maps \(z\) to a synthetic sample.
  • Discriminator \(D\) — distinguishes real from fake.
  • Critic — discriminator in WGAN; outputs a real number rather than a probability.
  • Non-saturating loss — generator loss \(-\log D(G(z))\); avoids vanishing gradients.
  • Mode collapse — generator covers only some modes of the data distribution.
  • Mode covering — likelihood-based models spread mass over all modes (potentially oversmoothing).
  • WGAN / WGAN-GP — Wasserstein-distance-based GAN with weight clipping (WGAN) or gradient penalty (WGAN-GP).
  • Hinge loss (GAN) — margin-based critic loss.
  • Spectral normalization — divide each weight matrix by its top singular value; enforces a soft Lipschitz constraint.
  • R1 / R2 penalty — gradient penalty on the discriminator at real (R1) or fake (R2) samples.
  • TTUR (two-time-scale update rule) — different learning rates for \(G\) and \(D\).
  • EMA (exponential moving average) — slow average of model weights; used at sampling time for stability.
  • Normalizing flow — invertible neural net used as a generative model.
  • Change of variables — \(\log p_\theta(x) = \log p(z) + \log|\det(\partial f^{-1}/\partial x)|\).
  • Coupling layer — invertible building block (NICE additive, RealNVP affine).
  • Jacobian determinant — factor in the change-of-variables formula; designed to be tractable in flows.
  • Glow — RealNVP-style flow with invertible 1×1 convolutions.
  • MAF / IAF — autoregressive normalizing flows; MAF fast at density, IAF fast at sampling.
  • Neural ODE / FFJORD — continuous-time normalizing flow.
  • Diffusion model — model that learns to denoise samples noised by a fixed Gaussian Markov chain.
  • Forward process \(q(x_t\mid x_{t-1})\) — noising; fixed, no parameters.
  • Reverse process \(p_\theta(x_{t-1}\mid x_t)\) — denoising; learned.
  • \(\alpha_t, \beta_t, \bar\alpha_t\) — schedule quantities; \(\alpha_t = 1-\beta_t\), \(\bar\alpha_t = \prod_{s\le t}\alpha_s\).
  • SNR — signal-to-noise ratio \(\bar\alpha_t/(1-\bar\alpha_t)\).
  • Linear schedule — \(\beta_t\) linear in \(t\).
  • Cosine schedule — \(\bar\alpha_t\) follows a cosine curve.
  • Sigmoid schedule — \(\beta_t\) follows a sigmoid in \(t/T\).
  • Karras / EDM schedule — power-law \(\sigma\) spacing; SOTA at low step count.
  • ε-prediction — network predicts the noise added to \(x_0\).
  • \(x_0\)-prediction — network predicts the clean image directly.
  • \(v\)-prediction — network predicts \(v = \sqrt{\bar\alpha_t}\epsilon - \sqrt{1-\bar\alpha_t}x_0\).
  • \(L_{\text{simple}}\) — un-weighted MSE between predicted and true \(\epsilon\).
  • Min-SNR weighting — cap each timestep’s loss weight at \(\min(\mathrm{SNR}, \gamma)\).
  • EDM preconditioning — Karras et al. unit-variance reparameterization of the network input/output.
  • U-Net — symmetric encoder-decoder with skip connections; standard diffusion backbone.
  • AdaGN / AdaLN-Zero — adaptive normalization conditioning on time / class embedding.
  • DiT (Diffusion Transformer) — transformer-based diffusion network (no U-Net).
  • Latent diffusion — run diffusion in a VAE’s latent space (Stable Diffusion).
  • Cascaded diffusion — chain of diffusion models from low to high resolution.
  • DDPM sampler — ancestral, \(T\)-step.
  • DDIM sampler — deterministic, non-Markovian, \(N\)-step where \(N<T\).
  • DPM-Solver, DPM-Solver++ — high-order ODE samplers.
  • UniPC — unified predictor-corrector sampler.
  • Heun sampler — second-order ODE sampler; EDM default.
  • Progressive distillation — train a student to halve the step count of a teacher diffusion model.
  • Consistency model — one-step (or few-step) generative model trained from scratch or distilled from a diffusion teacher.
  • Latent Consistency Model (LCM) — consistency model in a latent space.
  • SDXL Turbo / Adversarial Diffusion Distillation (ADD) — distillation with adversarial loss for very-low-step text-to-image.
  • Classifier guidance — guide samples toward \(c\) using a trained classifier’s gradient.
  • Classifier-free guidance (CFG) — guide via the difference between conditional and unconditional predictions.
  • Cross-attention — attention from image queries to text keys/values; standard text conditioning in U-Net.
  • ControlNet — side branch injecting spatially-aligned conditioning into a pretrained U-Net.
  • T2I-Adapter — lighter version of ControlNet.
  • IP-Adapter — image-prompt conditioning via CLIP image embedding.
  • Textual Inversion — learn a new word embedding for a concept.
  • DreamBooth — fine-tune the diffusion model to a specific subject.
  • LoRA — low-rank adapters; the practical way to fine-tune large diffusion models.
  • SDEdit — partial noising of an input image + denoising for editing.
  • Inpainting — fill masked regions of an image.
  • Pix2Pix — paired image-to-image translation (conditional GAN; also a diffusion variant).
  • CycleGAN — unpaired image-to-image translation with cycle-consistency loss.

APPENDIX C — Self-Test Bank (50 Questions)

Use these to verify you can answer every named skill cold. Each question is followed by a short answer and, when useful, a pointer to the section that develops the topic in depth.

Section 1 — Foundations (1.1–1.3)

Q1. Match: \(p(x,y)\), \(p(y\mid x)\), \(p(x)\), \(p_\theta(z\mid x)\). A. Joint (generative), discriminative conditional, marginal (also generative target), approximate or true posterior over latents. (§1.1, §1.3.)

Q2. MCQ. Which of the following is a discriminative model? (a) Naive Bayes (b) GPT-style decoder (c) Logistic regression (d) Normalizing flow A. (c). All others model \(p(x)\) or \(p(x,y)\) in some form. (§1.1.)

Q3. Why can a generative model classify but a discriminative model cannot synthesize? A. Because \(p(y\mid x)\) can be derived from \(p(x,y)\) via Bayes’ rule, but the reverse derivation requires \(p(x\mid y)\) which is not modeled by a pure discriminator. (§1.1.)

Q4. Describe at least eight downstream applications of generative models. A. Image / audio / text / video synthesis, molecule and protein design, data augmentation, simulation, anomaly detection, super-resolution, inpainting, domain transfer, compression, scientific density modeling. (§1.2.)

Q5. State the ELBO and identify its two components. A. \(\log p_\theta(x) \ge \mathbb{E}_q[\log p_\theta(x\mid z)] - \mathrm{KL}(q_\phi(z\mid x)\Vert p(z))\). Reconstruction + KL regularization. (§1.3.)

Q6. What is the manifold hypothesis and why does it matter? A. Natural data concentrates on a low-dimensional manifold \(\mathcal{M}\subset\mathbb{R}^d\). It motivates latent-variable models, low-dimensional latent codes, and explains why arbitrary points in pixel space look like noise while data points have structure. (§1.3.)

Q7. Contrastive learning is unsupervised but is it “generative”? A. No. Contrastive methods learn a representation that aligns positive pairs and separates negatives, but they do not give you \(p(x)\). They are related to generative modeling via mutual-information bounds and the energy-based view. (§1.3.)

Q8. Define an energy-based model and explain why \(Z(\theta)\) is the central computational headache. A. \(p_\theta(x) = e^{-E_\theta(x)}/Z(\theta)\) with \(Z(\theta) = \int e^{-E_\theta(x')}dx'\). \(Z\) is a high-dimensional integral that is intractable in general; training and density evaluation must side-step it via score matching, contrastive divergence, or NCE. (§1.3.)

Section 2 — Architectures Inventory (2.1–2.3)

Q9. List the eight generative-model families covered in this guide. A. Autoregressive, VAE, GAN, normalizing flow, diffusion / score-based, EBM, consistency, flow matching / rectified flow. (§2.1.)

Q10. Match each model to its defining trick: PixelCNN, WaveNet, GPT-style decoder, RealNVP, Glow, DDPM, NCSN, DiT, Imagen, Stable Diffusion, StyleGAN, BigGAN, CycleGAN, ESRGAN, VQ-GAN, FFJORD, Flow Matching, Rectified Flow, Consistency Models. A. See the labeled family inventory in §2.1.

Q11. Which families give exact likelihood? Which give a lower bound only? Which give none? A. Exact: autoregressive, normalizing flow. Lower bound: VAE, diffusion. None (or unnormalized): GAN, EBM, consistency. (§2.3.)

Q12. Why are GANs prone to mode collapse but diffusion models prone to mode covering? A. GANs are trained adversarially; the generator can collapse onto a few modes that fool the discriminator (reverse-KL-like behaviour). Diffusion / likelihood models optimize a forward-KL-like objective which spreads probability mass — mode covering — at the risk of oversmoothing. (§2.3.)

Q13. List the typical sampling speed of each family from fastest to slowest baseline. A. GAN / VAE / flow / consistency (1 step) → flow matching (few steps) → autoregressive (token / pixel sequential) → diffusion (many steps). (§2.3.)

Q14. Two short paragraphs: name three use cases where you would not pick a GAN today. A. Generic text-to-image (use diffusion / flow matching); exact density estimation (use a flow); training stability under tight budgets (use a VAE or diffusion). (§2.2, §6.3.)

Section 3 — Diffusion Process (3.1–3.4)

Q15. Write down the closed-form \(q(x_t\mid x_0)\). A. \(q(x_t\mid x_0) = \mathcal{N}(\sqrt{\bar\alpha_t}\,x_0, (1-\bar\alpha_t)I)\), i.e., \(x_t = \sqrt{\bar\alpha_t}\,x_0 + \sqrt{1-\bar\alpha_t}\,\epsilon\). (§3.1.)

Q16. Compute \(\bar\alpha_t\) for a linear schedule with \(T=4\), \(\beta_{\min}=10^{-4}\), \(\beta_{\max}=0.02\). A. See the worked table in §3.1. The product \(0.9999 \times 0.9933 \times 0.98665 \times 0.98 \approx 0.96040\). (§3.1.)

Q17. State the formula for \(\mu_\theta(x_t, t)\) when the network predicts \(\epsilon\). A. \(\mu_\theta(x_t, t) = (1/\sqrt{\alpha_t})\big(x_t - (\beta_t/\sqrt{1-\bar\alpha_t})\,\epsilon_\theta(x_t,t)\big)\). (§3.2.)

Q18. What does \(\tilde\beta_t\) equal? Why is it a reasonable choice for \(\Sigma_\theta\)? A. \(\tilde\beta_t = (1-\bar\alpha_{t-1})/(1-\bar\alpha_t)\cdot \beta_t\). It is the exact variance of \(q(x_{t-1}\mid x_t, x_0)\), the true forward posterior; using it makes the reverse step’s variance match the posterior’s variance. (§3.2.)

Q19. Why is \(L_{\text{simple}}\) preferred over the full ELBO in practice? A. Dropping the \(t\)-dependent weight effectively up-weights difficult intermediate timesteps and empirically yields better samples. The \(L_{\text{simple}}\) loss is equivalent to denoising score matching up to a per-\(t\) weight. (§3.2, §5.2.)

Q20. Define the linear and cosine schedules and explain when each is preferable. A. Linear: \(\beta_t\) linear from \(\beta_{\min}\) to \(\beta_{\max}\). Cosine: \(\bar\alpha_t = \cos^2\)-curve. Cosine is preferable at higher resolutions or shorter \(T\) because it samples the intermediate-SNR range more densely. (§3.3.)

Q21. Why does EDM use \(\sigma\) instead of \(t\) as the noise parameter? A. Because \(\sigma\) is the physically meaningful quantity (noise standard deviation); the network’s preconditioning, the sampler, and the loss weighting can all be expressed cleanly in terms of \(\sigma\). This decouples the schedule from the discretization. (§3.3.)

Q22. Implement the cosine schedule in pseudocode. A. See make_cosine_schedule in §3.4.

Section 4 — Diffusion Architectures (4.1–4.3)

Q23. Why is a U-Net the natural backbone for diffusion? A. The task is image-to-image (predict noise of the same shape as input); the U-Net’s encoder-decoder + skip connections capture both local and global structure efficiently. (§4.1.)

Q24. Where in a U-Net is self-attention typically inserted? A. At low-resolution levels (e.g., 16×16 or 8×8 feature maps), to add global context without paying quadratic attention cost at high resolutions. (§4.1, §4.3.)

Q25. Write down the sinusoidal timestep encoding formula. A. \(\mathrm{PE}(t)_{2i} = \sin(t/10000^{2i/d})\), \(\mathrm{PE}(t)_{2i+1} = \cos(t/10000^{2i/d})\). Typically followed by a 2-layer MLP. (§4.2.)

Q26. Why not learn a separate embedding per integer \(t\)? A. Discrete embeddings don’t generalize to subsampled or continuous timesteps used by DDIM / EDM samplers. Sinusoidal encodings interpolate smoothly. (§4.2.)

Q27. Describe an AdaGN-style ResBlock with time injection. A. A residual block with GroupNorm + SiLU + Conv, where the time embedding’s MLP projection is added as a per-channel bias (or, in AdaGN, as scale-and-shift modulation of the normalized features). (§4.3.)

Q28. What is the difference between DiT and a U-Net diffusion network? A. DiT replaces the U-Net with a transformer over flattened image patches, using AdaLN-Zero modulation for time/class conditioning. No skip connections in the U-Net sense. Scales like a transformer. (§4.1, §4.3.)

Section 5 — Diffusion Training (5.1–5.4)

Q29. Why normalize images to \([-1,1]\) for diffusion training? A. So that the data distribution has zero mean and unit-ish variance, matching the Gaussian prior \(\mathcal{N}(0, I)\) at \(x_T\) and making the noise/data relationship symmetric. (§5.1.)

Q30. What is EMA and why is it nearly mandatory? A. Exponential moving average of model weights, \(\theta_{\text{EMA}} \leftarrow \rho \theta_{\text{EMA}} + (1-\rho)\theta\) with \(\rho \in [0.999, 0.9999]\). Sample from \(\theta_{\text{EMA}}\), not raw \(\theta\). Reduces sample noise and improves FID dramatically. (§5.1.)

Q31. What is conditioning dropout and why is it needed? A. During training, with probability \(p_{\text{drop}}\) replace the conditioning \(c\) with a learned null embedding \(\varnothing\). This trains the same network to be conditional and unconditional, enabling classifier-free guidance at inference. (§5.1, §7.3.)

Q32. Derive \(L_{\text{simple}}\) from the ELBO in one paragraph. A. Each \(L_{t-1}\) KL between two Gaussians with matched variance reduces to a weighted MSE on the means. Substituting the ε-parametrization of \(\mu_\theta\) turns the MSE on means into a \(t\)-weighted MSE on noise. Dropping the \(t\)-weight gives \(L_{\text{simple}}\). (§5.2.)

Q33. State min-SNR weighting. A. Weight each timestep’s loss by \(w(t) = \min(\mathrm{SNR}(t), \gamma)\) with \(\gamma \in [1,5]\). Caps easy steps’ contribution and frees capacity for hard intermediate steps. (§5.2.)

Q34. What is \(v\)-prediction? Why prefer it? A. \(v = \sqrt{\bar\alpha_t}\epsilon - \sqrt{1-\bar\alpha_t}x_0\). Equivalent to ε-prediction up to a \(t\)-dependent reweighting but better-behaved at the high-\(t\) boundary; standard in Stable Diffusion 2.x+, Imagen, EDM. (§5.2.)

Q35. Pseudocode for one DDPM training step. A. Sample \(x_0\), \(t\), \(\epsilon\); form \(x_t\); compute \(\hat\epsilon = \epsilon_\theta(x_t, t)\); backprop \(\|\epsilon - \hat\epsilon\|^2\). (§5.2.)

Q36. Why does gradient clipping matter in diffusion training? A. The per-step loss has high variance over \(t\); occasional huge gradients can destabilize training. Clipping at norm 1.0 (or similar) caps the influence of these outliers. (§5.3.)

Q37. List five techniques to accelerate diffusion sampling. A. DDIM (training-free), DPM-Solver / UniPC (high-order ODE solvers), latent diffusion (smaller spatial size), progressive distillation, consistency distillation, adversarial distillation (SDXL Turbo). (§5.4.)

Q38. Write the deterministic DDIM update. A. \(x_{t-1} = \sqrt{\bar\alpha_{t-1}}\hat x_0(x_t, t) + \sqrt{1-\bar\alpha_{t-1}}\epsilon_\theta(x_t, t)\), with \(\hat x_0 = (x_t - \sqrt{1-\bar\alpha_t}\epsilon_\theta)/\sqrt{\bar\alpha_t}\). (§5.4.)

Q39. How does latent diffusion reduce compute? A. Run diffusion in a VAE’s latent space (e.g., \(8\times\)-downsampled spatial dims), which cuts FLOPs by \(\sim 64\times\) for image generation. (§5.4.)

Section 6 — GANs (6.1–6.4)

Q40. Write the minimax GAN objective. A. \(\min_G \max_D \mathbb{E}_{x\sim p_{\text{data}}}[\log D(x)] + \mathbb{E}_{z}[\log(1 - D(G(z)))]\). (§6.1.)

Q41. Derive the optimal discriminator \(D^*\). A. For fixed \(G\), \(D^*(x) = p_{\text{data}}(x)/(p_{\text{data}}(x) + p_g(x))\), found by setting the functional derivative of \(V\) w.r.t. \(D\) to zero. (§6.1.)

Q42. What does the GAN objective reduce to at the optimal discriminator? A. \(2\mathrm{JSD}(p_{\text{data}}\Vert p_g) - 2\log 2\). (§6.1.)

Q43. Why use the non-saturating generator loss? A. The saturating loss \(\log(1-D(G(z)))\) gives near-zero gradients when \(D\) is confident (early in training). The non-saturating loss \(-\log D(G(z))\) gives strong gradients exactly in that regime. (§6.1.)

Q44. Compare WGAN and WGAN-GP. A. Both use the Wasserstein-1 distance with a Lipschitz critic. WGAN enforces the constraint by weight clipping; WGAN-GP by a one-sided gradient penalty on samples interpolated between real and fake. WGAN-GP is much more stable in practice. (§6.2.)

Q45. Define spectral normalization. A. Divide each weight matrix \(W\) by its top singular value \(\sigma_{\max}(W)\) (or an iteratively estimated upper bound), enforcing \(\|W\|_2 = 1\). Imposes a soft Lipschitz constraint on the network. (§6.2.)

Q46. List five named GAN architectures and what each is best for. A. StyleGAN1/2/3 (faces); BigGAN (class-conditional ImageNet); CycleGAN (unpaired image-to-image translation); Pix2Pix (paired translation); ESRGAN / Real-ESRGAN (super-resolution); HiFi-GAN (audio vocoders); VQ-GAN (image tokenizer). (§6.3.)

Q47. Side-by-side: GAN vs diffusion on (likelihood, sample quality, sampling speed, training stability, mode coverage). A. GAN: no likelihood, sharp, 1-step, unstable, mode-collapses. Diffusion: ELBO, top quality, slow but distillable, very stable, mode-covering. (§6.4.)

Section 7 — Evaluation and Conditioning (7.1–7.4)

Q48. Define FID, IS, and Precision/Recall for generative models. A. FID: Fréchet distance between Gaussian fits to Inception features of real and fake sets. IS: \(\exp(\mathbb{E}_x[\mathrm{KL}(p(y\mid x)\Vert p(y))])\). Precision/Recall: nearest-neighbor manifold-based estimates of quality and diversity, respectively. (§7.1.)

Q49. State the classifier-free guidance formula and explain \(w\). A. \(\hat\epsilon_{\text{cfg}} = (1+w)\epsilon_\theta(x_t,t,c) - w\,\epsilon_\theta(x_t,t,\varnothing)\). \(w=0\) is unconditional; \(w=1\) is exactly conditional; \(w>1\) “exaggerates” the conditional direction at the cost of saturation. (§7.3.)

Q50. Match each conditioning method to its mechanism: ControlNet, T2I-Adapter, IP-Adapter, Textual Inversion, DreamBooth, LoRA, SDEdit, Inpainting. A. ControlNet: side branch with zero-conv injection of spatially-aligned features. T2I-Adapter: lightweight adapter network with per-resolution feature additions. IP-Adapter: separate cross-attention path on a reference image’s CLIP embedding. Textual Inversion: learn a single new token embedding for a concept. DreamBooth: fine-tune the diffusion model on a few images of a subject with a unique token (often via LoRA). LoRA: low-rank adapters \(\Delta W = AB^\top\) on cross-attention. SDEdit: partial noising + denoising of an input image. Inpainting: mask-aware denoising that preserves the unmasked region. (§7.3.)
