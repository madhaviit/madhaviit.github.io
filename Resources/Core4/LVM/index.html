Preface — How to use this guide

This document is a self-contained, dense reference on the modern field of large vision models (LVMs): the architectures, training techniques, generative systems, and vision-language models that underpin contemporary computer vision. It is organized as four Parts that mirror the assessment syllabus, each broken into Modules and Sections that name and explain a specific skill. The goal is to take a reader from “I know basic CNNs” to “I can read a 2024 vision paper, reimplement the algorithm from scratch, choose between competing approaches, and debug training” — without leaving this file.

Conventions used throughout:

  • Mathematical notation uses Quarto’s KaTeX rendering. Inline math is written as $x$; block math as $$x$$.
  • Code blocks use fenced Python (python) or text (text) blocks. Tensor shapes appear in comments. None of the code is meant to be “production-ready”; it is meant to be small enough to read but complete enough to rewrite from.
  • Pseudocode is used for algorithm summaries when raw code would obscure the idea.
  • Comparison tables appear at the end of each module to crystallise the trade-offs.
  • Worked numerical examples appear in many sections. Do these by hand; the assessment likes “compute the number of patches…”, “compute attention complexity…” style items.
  • “Common assessment traps” callouts highlight the vocabulary distinctions that look identical but mean different things.
  • A glossary, a full reading list, and ~50 self-test questions sit at the end as Appendices.

Read top-to-bottom on a first pass to build the mental scaffolding. Use the table of contents as a lookup later. The skill names match the syllabus verbatim.

PART 1 — DESIGNING VISION MODEL ARCHITECTURES

The first Part is about what a modern vision model looks like inside. The transformer revolution that swept NLP in 2017 reached vision in 2020 with the Vision Transformer (ViT), and since then the dominant architectures fall into four families: pure transformers (ViT, Swin, DiNAT), multi-scale / mobile-friendly hybrids (MobileViT, CrossViT, capsules), CNN+Transformer hybrids (ConvNeXt, CoAtNet, MaxViT), and autoregressive image models (Gated PixelCNN, Image Transformer, Parti, MaskGIT). Part 1 covers all four.

MODULE 1 — Applying Vision Transformer Models

1.1 Implement Vision Transformers (ViTs) for image recognition

Definition and motivation. A Vision Transformer (ViT, Dosovitskiy et al., 2020, “An Image is Worth 16×16 Words”) applies a vanilla Transformer encoder — the same architecture that powers BERT and GPT — directly to images. The architecture replaces the convolutional inductive biases of CNNs (locality, translation equivariance, hierarchical receptive fields) with patch tokens and global self-attention. The motivation was simple: if scaling laws say that bigger transformers trained on more data dominate domain-specific architectures in NLP, do they also dominate in vision? The answer turned out to be yes, given enough data. ViT-Large pretrained on JFT-300M matched or beat the best CNNs on ImageNet-1k.

The full mechanism, step by step. ViT processes an image \(\mathbf{x}\in\mathbb{R}^{H\times W\times C}\) as follows:

  1. Patchification. Split the image into a grid of non-overlapping square patches of side \(P\). With \(H = W = 224\) and \(P = 16\), you get \(N = HW/P^2 = 196\) patches, each a tensor of shape \(P\times P\times C = 16\times 16\times 3 = 768\) values when flattened.
  2. Patch embedding. Flatten each patch to a vector \(\mathbf{x}_p^{(i)} \in\mathbb{R}^{P^2 C}\) and linearly project it to dimension \(D\) (the model’s embedding dimension): \[\mathbf{z}_0^{(i)} = \mathbf{E}\,\mathbf{x}_p^{(i)},\qquad \mathbf{E}\in\mathbb{R}^{D\times P^2 C}.\] In practice this is implemented with a single 2D convolution with kernel size \(P\) and stride \(P\), which is exactly equivalent.
  3. Class token. Prepend a learnable \([\text{CLS}]\) token \(\mathbf{z}_{\text{cls}}\in\mathbb{R}^D\) to the patch tokens. After all encoder layers, its final value will be the image-level representation used for classification. The sequence now has \(N+1\) tokens.
  4. Positional embedding. Add a learnable positional embedding \(\mathbf{E}_{\text{pos}}\in\mathbb{R}^{(N+1)\times D}\) to encode where each patch came from. Without it, the transformer would treat patches as a bag. \[\mathbf{z}_0 = [\mathbf{z}_{\text{cls}};\mathbf{E}\mathbf{x}_p^{(1)};\dots;\mathbf{E}\mathbf{x}_p^{(N)}] + \mathbf{E}_{\text{pos}}.\]
  5. Transformer encoder. Pass \(\mathbf{z}_0\) through \(L\) identical encoder blocks. Each block applies pre-norm multi-head self-attention (MSA), then pre-norm MLP, both with residual connections: \[\mathbf{z}'_\ell = \mathrm{MSA}(\mathrm{LN}(\mathbf{z}_{\ell-1})) + \mathbf{z}_{\ell-1},\] \[\mathbf{z}_\ell = \mathrm{MLP}(\mathrm{LN}(\mathbf{z}'_\ell)) + \mathbf{z}'_\ell.\]
  6. Classification head. Take the final value of the \([\text{CLS}]\) token, \(\mathbf{z}_L^{(0)}\), layer-normalize it, and feed it through a single linear layer to produce class logits.

Scaled dot-product attention. The core operation inside MSA. Given queries, keys, and values \(\mathbf{Q}, \mathbf{K}, \mathbf{V}\in\mathbb{R}^{n\times d_k}\):

\[\mathrm{Attn}(\mathbf{Q},\mathbf{K},\mathbf{V}) = \mathrm{softmax}\!\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right)\mathbf{V}.\]

Multi-head attention splits the channel dimension into \(h\) heads, computes attention independently per head, then concatenates and projects:

\[\mathrm{MSA}(\mathbf{z}) = \mathrm{Concat}(\mathrm{head}_1,\dots,\mathrm{head}_h)\mathbf{W}^O,\quad \mathrm{head}_i = \mathrm{Attn}(\mathbf{z}\mathbf{W}^Q_i,\mathbf{z}\mathbf{W}^K_i,\mathbf{z}\mathbf{W}^V_i).\]

ViT-Base configuration (the canonical baseline). \(L=12\) layers, \(D=768\) hidden, \(h=12\) heads, MLP hidden \(4D=3072\), patch \(P=16\), image \(224\times 224\), so 196 patch tokens + 1 cls = 197 tokens. Total parameters ≈ 86M.

Worked example: patch count. With \(H=W=224, P=16\), the patch grid is \(14\times 14\), so \(N = 196\) patches. Switching to \(P=14\) gives \(16\times 16 = 256\) patches; switching to a \(384\times 384\) image with \(P=16\) gives \(24\times 24 = 576\) patches.

Worked example: attention complexity. Self-attention is \(O(N^2 D)\). For ViT-B at \(224\times 224\): \(N=197\), \(D=768\), so attention is ~\(197^2 \cdot 768 \approx 30\text{M}\) multiplies per layer per head before the softmax-times-V step. Doubling resolution to \(448\times 448\) gives \(N=785\), so attention cost grows by \((785/197)^2 \approx 16\times\) — the quadratic blow-up that motivates window attention (Swin) and neighborhood attention (NAT/DiNAT).

Minimal PyTorch ViT implementation.

import torch, torch.nn as nn

class PatchEmbed(nn.Module):
    """Image -> patch tokens via a strided conv."""
    def __init__(self, img_size=224, patch=16, in_ch=3, dim=768):
        super().__init__()
        self.proj = nn.Conv2d(in_ch, dim, kernel_size=patch, stride=patch)
        self.n_patches = (img_size // patch) ** 2

    def forward(self, x):                # x: (B, 3, 224, 224)
        x = self.proj(x)                 # (B, dim, 14, 14)
        x = x.flatten(2).transpose(1, 2) # (B, 196, dim)
        return x

class MLP(nn.Module):
    def __init__(self, dim, hidden, drop=0.0):
        super().__init__()
        self.fc1 = nn.Linear(dim, hidden); self.fc2 = nn.Linear(hidden, dim)
        self.act = nn.GELU(); self.drop = nn.Dropout(drop)
    def forward(self, x):
        return self.drop(self.fc2(self.drop(self.act(self.fc1(x)))))

class Attention(nn.Module):
    def __init__(self, dim, heads=12, drop=0.0):
        super().__init__()
        self.h = heads; self.dk = dim // heads
        self.qkv = nn.Linear(dim, dim * 3)
        self.proj = nn.Linear(dim, dim); self.drop = nn.Dropout(drop)

    def forward(self, x):                # x: (B, N, D)
        B, N, D = x.shape
        qkv = self.qkv(x).reshape(B, N, 3, self.h, self.dk).permute(2, 0, 3, 1, 4)
        q, k, v = qkv[0], qkv[1], qkv[2]              # each (B, h, N, dk)
        attn = (q @ k.transpose(-2, -1)) / self.dk ** 0.5
        attn = attn.softmax(dim=-1)
        out  = (attn @ v).transpose(1, 2).reshape(B, N, D)
        return self.drop(self.proj(out))

class Block(nn.Module):
    def __init__(self, dim, heads, mlp_ratio=4.0, drop=0.0):
        super().__init__()
        self.ln1 = nn.LayerNorm(dim); self.attn = Attention(dim, heads, drop)
        self.ln2 = nn.LayerNorm(dim); self.mlp  = MLP(dim, int(dim*mlp_ratio), drop)
    def forward(self, x):
        x = x + self.attn(self.ln1(x))
        x = x + self.mlp(self.ln2(x))
        return x

class ViT(nn.Module):
    def __init__(self, img=224, patch=16, dim=768, depth=12, heads=12, n_classes=1000):
        super().__init__()
        self.patch = PatchEmbed(img, patch, 3, dim)
        N = self.patch.n_patches
        self.cls  = nn.Parameter(torch.zeros(1, 1, dim))
        self.pos  = nn.Parameter(torch.zeros(1, N + 1, dim))
        self.blocks = nn.ModuleList([Block(dim, heads) for _ in range(depth)])
        self.ln   = nn.LayerNorm(dim)
        self.head = nn.Linear(dim, n_classes)
        nn.init.trunc_normal_(self.pos, std=0.02)
        nn.init.trunc_normal_(self.cls, std=0.02)

    def forward(self, x):                         # (B, 3, 224, 224)
        x = self.patch(x)                         # (B, 196, dim)
        cls = self.cls.expand(x.size(0), -1, -1)  # (B, 1, dim)
        x = torch.cat([cls, x], dim=1) + self.pos # (B, 197, dim)
        for blk in self.blocks: x = blk(x)
        x = self.ln(x)
        return self.head(x[:, 0])                 # (B, n_classes)

Algorithm — ViT forward (pseudocode).

Input:  image x of shape (H, W, C); patch size P
        learnable matrices E, E_pos, W^{Q,K,V,O} per layer
Output: class logits

z0 = PatchEmbed(x)               # split, flatten, linear-project each patch
z0 = [CLS] :: z0                  # prepend learnable class token
z0 = z0 + E_pos                   # add position embeddings
for l = 1..L:
    z'_l = z_{l-1} + MSA(LN(z_{l-1}))
    z_l  = z'_l + MLP(LN(z'_l))
y = LN(z_L); logits = Head(y[CLS])

Engineering considerations.

  • Resolution vs compute. Doubling the image side quadruples the token count and quadruples again the attention cost (quadratic in \(N\)). ViTs at \(384\times 384\) or \(512\times 512\) require either window attention, gradient checkpointing, or token pruning.
  • Position embeddings. Original ViT used learnable absolute positions. Modern variants prefer 2D sin–cos positions or learnable 2D positions; some go further with relative position biases (added directly to attention logits). When fine-tuning at a higher resolution, you must interpolate the position embedding grid (bicubic) to the new patch grid.
  • CLS vs global-average-pool head. ViT uses a class token; DeiT (Touvron et al., 2021) showed average pooling of the patch tokens works equally well. Both appear in the literature; know they are interchangeable.
  • Warmup and LR schedule. ViTs are touchy. Use AdamW with \(\beta_1=0.9, \beta_2=0.999\), weight decay 0.05, cosine LR schedule, warmup of ~10% of training, and gradient clipping at 1.0.
  • Tooling. Ross Wightman’s timm library is the canonical reference implementation; the Hugging Face transformers ViTModel mirrors it for inference.

Common assessment trap. Patch token vs class token vs positional embedding. A patch token is the projection of one image patch. The class token is a learnable vector prepended to the sequence and trained as the global summary. The positional embedding is a vector added to every token to encode its grid position; it is not a token of its own.

1.2 Configure hierarchical vision processing using Swin Transformers

Definition and motivation. The Swin Transformer (Liu et al., 2021, “Swin Transformer: Hierarchical Vision Transformer using Shifted Windows”) addresses two ViT limitations: (a) attention complexity grows quadratically with token count, so high-resolution images and dense-prediction tasks (segmentation, detection) become infeasible; (b) ViT produces a flat token grid with no multi-scale feature pyramid that detection/segmentation heads expect. Swin restores hierarchy by computing self-attention inside non-overlapping local windows and progressively merging patches to build a feature pyramid, exactly like a CNN’s stages but using attention as the unit of computation.

The full mechanism.

  1. Patch partitioning. Like ViT, the image is split into patches (Swin uses \(4\times 4\) patches by default, then projects them to dimension \(C\)).
  2. Stages with patch merging. The network has four stages. Each stage applies several Swin blocks at one resolution, then patch merging halves the spatial resolution and doubles the channel count (à la CNN downsampling). After stage 1 the resolution is \(H/4 \times W/4\); after stage 4 it is \(H/32 \times W/32\).
  3. Window self-attention (W-MSA). Inside each Swin block, partition tokens into non-overlapping \(M\times M\) windows (usually \(M=7\)). Apply self-attention only within each window. Complexity per layer drops from \(O((HW)^2)\) to \(O(M^2 \cdot HW)\) — linear in image size.
  4. Shifted window self-attention (SW-MSA). Alternating blocks shift the window grid by \((\lfloor M/2 \rfloor, \lfloor M/2 \rfloor)\). This makes tokens at window edges in one layer interact with tokens that lay in a neighbouring window the previous layer, providing the cross-window connections that pure W-MSA lacks.
  5. Cyclic shift + masking trick. A naïve shift creates new windows on the edges that are smaller than \(M\times M\), which would waste compute. The implementation instead cyclically rolls the feature map and applies a precomputed attention mask that prevents tokens originally on opposite edges from attending to each other. The result is identical to “real” shifted-window attention but processes uniform \(M\times M\) windows.

Window-attention math. Let \(N\) be the number of tokens. Standard self-attention costs: \[\Omega(\mathrm{MSA}) = 4ND^2 + 2N^2 D.\] Window self-attention with window size \(M\): \[\Omega(\mathrm{W\text{-}MSA}) = 4ND^2 + 2M^2 N D.\] The quadratic-in-\(N\) term is replaced by a quadratic-in-\(M\) term, and \(M\) is a small constant (7).

Relative position bias. Swin replaces ViT’s absolute positions with a learnable bias \(\mathbf{B}\in\mathbb{R}^{M^2\times M^2}\) added to the attention logits: \[\mathrm{Attn} = \mathrm{softmax}(\mathbf{Q}\mathbf{K}^\top/\sqrt{d}+\mathbf{B})\mathbf{V}.\] The bias is parameterised by a smaller table indexed by relative coordinates within a window, so the same bias generalises across windows.

Patch merging. Take a \(2\times 2\) neighbourhood of tokens, concatenate along channel (yielding \(4C\) channels per merged token), and linearly project to \(2C\). Net effect: spatial \(\div 2\), channels \(\times 2\). This mirrors the strided-conv downsampling stage of a ResNet.

Trace one Swin shifted-window block (worked example). Take a \(14\times 14\) token grid with \(M=7\). The first block uses non-shifted \(7\times 7\) windows, producing 4 windows. The next block shifts by \((3,3)\). After the cyclic shift, the same 4-window partition is used, but with the attention mask the tokens that were “wrapped around” do not attend to each other. The set of attending pairs differs from the first block, so by alternating W-MSA and SW-MSA, the receptive field grows across blocks and information flows globally without ever computing a full \(196\times 196\) attention matrix.

Variants.

  • Swin v2 (Liu et al., 2022) introduces log-spaced continuous position bias (a small MLP from continuous relative coordinates to bias) and scaled cosine attention (\(\cos(\mathbf{q},\mathbf{k})/\tau\) instead of dot product), enabling stable training at very high resolutions and up to 3B parameters.
  • Swin-T/S/B/L correspond to roughly ResNet-50/101/152/larger-parameter budgets. Swin-T has channel widths \(C=96\) and depths \(\{2,2,6,2\}\).

PyTorch sketch of a Swin block.

def window_partition(x, M):              # x: (B, H, W, C) -> (num_windows*B, M, M, C)
    B, H, W, C = x.shape
    x = x.view(B, H // M, M, W // M, M, C)
    x = x.permute(0, 1, 3, 2, 4, 5).contiguous().view(-1, M, M, C)
    return x

def window_reverse(w, M, H, W):          # inverse of window_partition
    B = int(w.shape[0] / (H * W / M / M))
    x = w.view(B, H // M, W // M, M, M, -1)
    return x.permute(0, 1, 3, 2, 4, 5).contiguous().view(B, H, W, -1)

class SwinBlock(nn.Module):
    def __init__(self, dim, heads, M=7, shift=0, mlp_ratio=4.0):
        super().__init__()
        self.M, self.shift = M, shift
        self.ln1 = nn.LayerNorm(dim); self.attn = Attention(dim, heads)
        self.ln2 = nn.LayerNorm(dim); self.mlp  = MLP(dim, int(dim * mlp_ratio))

    def forward(self, x, H, W, mask=None):   # x: (B, H*W, C)
        B, _, C = x.shape
        x_in = x
        x = self.ln1(x).view(B, H, W, C)
        if self.shift:
            x = torch.roll(x, shifts=(-self.shift, -self.shift), dims=(1, 2))
        w = window_partition(x, self.M).view(-1, self.M * self.M, C)
        w = self.attn(w)                       # masked attention if mask is provided
        x = window_reverse(w.view(-1, self.M, self.M, C), self.M, H, W)
        if self.shift:
            x = torch.roll(x, shifts=(self.shift, self.shift), dims=(1, 2))
        x = x.view(B, H * W, C)
        x = x_in + x
        x = x + self.mlp(self.ln2(x))
        return x

Algorithm — Swin shifted-window forward.

Input: token map z of shape (B, H, W, C); window size M
Block 2k (W-MSA):
  windows  = partition(z, M)              # disjoint MxM windows
  z'       = WindowAttn(windows) + z
  z2       = MLP(LN(z')) + z'
Block 2k+1 (SW-MSA):
  z_shift  = roll(z2, (-M/2, -M/2))
  windows  = partition(z_shift, M)
  z'       = WindowAttn(windows, mask=cross_window_mask) + z_shift
  z3       = MLP(LN(z')) + z'
  z3       = roll(z3, (M/2, M/2))
Patch merging: concat 2x2 neighbours along C; Linear(4C, 2C)

Common assessment trap. Window vs shifted window. Window attention computes attention inside fixed \(M\times M\) windows — there is no cross-window communication. Shifted-window attention offsets the window grid by \(\lfloor M/2\rfloor\) so that tokens that were in separate windows in the previous block now share a window, giving information flow across windows without paying global-attention cost.

1.3 Apply DINAT (Dilated Neighborhood Attention Transformer) for efficient attention

Definition and motivation. Hassani et al. introduced Neighborhood Attention (NAT, 2022) and then Dilated Neighborhood Attention (DiNAT, 2023). The motivation is to combine the linear-in-image-size scaling of windowed attention with a more flexible receptive field. NAT replaces global self-attention with attention to a small neighborhood around each query (much like a depthwise convolution with self-attention semantics). DiNAT adds a dilation axis: alternating layers use dilation 1, 2, 4, etc., so that local layers capture nearby detail and dilated layers reach across longer distances — analogous to dilated convolutions in segmentation networks.

Neighborhood attention (NA). For each query token at position \((i,j)\), attend to a \(k\times k\) neighborhood centred on \((i,j)\) (padded at the boundary, no wrap-around). This is not a windowed partition; every token has its own \(k\times k\) context. Complexity is \(O(N k^2)\), still linear in token count.

NA differs from Swin’s W-MSA in three important ways: (1) windows are not disjoint — each token has its own window; (2) there is no need for shifting because adjacent tokens already share most of their neighborhood; (3) it generalises convolutions: setting \(k=1\) recovers a linear projection per token, while increasing \(k\) smoothly increases the receptive field.

Dilated NA. A NA layer with dilation \(r\) attends to the \(k\times k\) tokens around \((i,j)\) taken every \(r\) steps. So a \(7\times 7\) NA with dilation 2 covers a \(13\times 13\) region of the image while still computing only 49 attention weights per token. By interleaving layers with dilations \(\{1, 2, 4, 8\}\), DiNAT builds an exponentially growing receptive field with constant per-layer cost.

Algorithm — DiNAT block (pseudocode).

for each token (i, j):
    neighbors = { (i + r*di, j + r*dj) : di, dj in [-k/2, k/2] }
    q = W_Q x[i,j];  k_n = W_K x[neighbors];  v_n = W_V x[neighbors]
    attn = softmax(q . k_n / sqrt(d) + B_rel(di, dj))
    y[i,j] = attn . v_n

The block alternates dilation rates between layers; e.g., a 4-block stage uses dilations \((1, 2, 1, 4)\) or \((1, 2, 4, 8)\). Patch merging and overall hierarchy match Swin.

Why DiNAT works. It is equivariant under translation (because every position has the same kernel), preserves locality (capturing fine-grained detail like a CNN), and provides global reach through dilation (like atrous convolutions in DeepLab). On COCO detection / instance segmentation, DiNAT consistently outperforms Swin at comparable FLOPs, particularly on small objects.

Practical notes.

  • The reference implementation uses a custom CUDA kernel (natten) because naïve gather/scatter would be slow. As of 2023 the kernels are open-source and integrated with HuggingFace.
  • Choose \(k=7\) as a default; dilations follow the pattern \(\{1, 1, r, 1\}\) where \(r\) doubles with depth.
  • DiNAT-S, DiNAT-Base, DiNAT-Large correspond to comparable parameter budgets to Swin-S/B/L.

MODULE 2 — Building Multi-scale Vision Models

2.1 Implement MobileViT for mobile vision tasks

Definition and motivation. MobileViT (Mehta & Rastegari, 2021, “MobileViT: Light-weight, General-purpose, and Mobile-friendly Vision Transformer”) targets edge devices where pure transformers are too heavy and pure mobile CNNs (MobileNetV2/V3) lack global context. It interleaves cheap inverted-residual (MV2) blocks — the workhorse of MobileNetV2 — with a tiny transformer block that operates on unfolded spatial patches, so the model gets CNN-like locality + transformer-like global reasoning at competitive FLOPs and parameter counts.

MV2 block recap. MobileNetV2’s inverted residual: \(1\times 1\) pointwise expand \(\to\) \(3\times 3\) depthwise \(\to\) \(1\times 1\) pointwise project, with a skip connection when input/output shapes match. Expansion ratio \(t\) (commonly 4 or 6) controls compute.

MobileViT block (the key idea). Given a feature map \(\mathbf{X}\in\mathbb{R}^{H\times W\times C}\):

  1. Local processing. Apply a \(3\times 3\) conv \(\to\) a \(1\times 1\) conv to lift channels to \(d\).
  2. Unfold to patches. Reshape the \(H\times W\times d\) map into \(N\) non-overlapping \(h\times w\) patches; “unfold” so that tokens at the same position within different patches are aligned: shape \((P, N, d)\) where \(P=hw\) and \(N = HW/P\). The trick: each of the \(P\) “channels” of patch tokens is then transformed independently by a transformer over the \(N\) patches.
  3. Transformer over patches. Apply \(L\) transformer blocks across the \(N\) axis. This gives each within-patch position a global view across all patches.
  4. Fold back. Reshape back to \(H\times W\times d\).
  5. Local fusion. \(1\times 1\) conv to project back to \(C\), concat with the input \(\mathbf{X}\) along channels, then \(3\times 3\) conv to fuse.

The unfold/fold trick is what makes MobileViT cheap: instead of a full attention on \(HW\) tokens, attention runs on \(N = HW/P\) tokens per position-within-patch, and the within-patch positions are handled by the surrounding convs.

MobileViT-XXS/XS/S. Three sizes: 1.3M, 2.3M, 5.6M parameters. MobileViT-S at 224 reaches ~78.4% ImageNet top-1 — better than MobileNetV3-Large at similar parameter count.

Practical notes.

  • On-device latency is what matters more than FLOPs. The transformer block is the bottleneck on phones lacking matmul-optimised kernels.
  • MobileViT-v2 (2022) removes the fold/unfold via separable self-attention (linear-time attention with explicit decay).
  • For object detection, MobileViT as a SSD-lite backbone is competitive with EfficientNet at lower latency.

Algorithm — MobileViT block.

X = MV2(X)                              # local features
Y = Conv_3x3(X); Y = Conv_1x1_to_d(Y)
P = unfold(Y, patch=h, w=w)             # (P, N, d) with P=h*w, N=H*W/(h*w)
for i in range(L):
    P = TransformerBlock(P, axis=N)     # attend across patches
Y = fold(P, H, W)                       # (H, W, d)
Y = Conv_1x1_to_C(Y); Y = Concat([X, Y])
return Conv_3x3(Y)

2.2 Integrate CrossViT for multi-scale vision tasks

Definition and motivation. CrossViT (Chen et al., 2021) processes the image at two scales simultaneously using a dual-branch design. A small-patch branch (e.g., patch=12) sees fine details with many tokens, while a large-patch branch (e.g., patch=16) sees coarse structure with fewer tokens. The two branches exchange information through a cross-attention fusion module that uses each branch’s class token as the query into the other branch.

Architecture.

  • L-branch (large patch, deep transformer, fewer tokens): captures global semantics.
  • S-branch (small patch, shallow transformer, more tokens): captures local detail.
  • Each branch has its own class token. Periodically, the class tokens are swapped between branches and used as queries against the patch tokens of the other branch — this is the cross-attention step.

Cross-attention fusion. Project the L-branch’s class token into the S-branch’s dimension. Use it as the query, with the S-branch patch tokens as keys/values. Aggregate the result back and add to the original class token. Do the symmetric operation for the S-branch. Mathematically, with \(\mathbf{x}_{\text{cls}}^L\) from the L-branch and patches \(\mathbf{x}_{\text{patch}}^S\) from the S-branch:

\[\mathbf{y}_{\text{cls}}^L \gets \mathbf{x}_{\text{cls}}^L + \mathrm{Attn}(\mathbf{x}_{\text{cls}}^L \mathbf{W}^Q,\; \mathbf{x}_{\text{patch}}^S \mathbf{W}^K,\; \mathbf{x}_{\text{patch}}^S \mathbf{W}^V).\]

The cost is small because the query is a single token; cross-attention is \(O(N_S\cdot d)\), not \(O(N^2 d)\).

Result. CrossViT-15 beats DeiT-S on ImageNet at similar FLOPs (~81.5% top-1). The dual-scale design is also strong for detection/segmentation where multi-scale features matter.

2.3 Implement Capsule Networks for capturing spatial hierarchies

Definition and motivation. Capsule Networks (Sabour, Frosst, Hinton 2017, “Dynamic Routing Between Capsules”) were proposed as an alternative to CNNs that explicitly represents part–whole relationships and equivariance to viewpoint. Where a CNN feature map records “there is something at position \((i,j)\)”, a capsule layer records “there is an instance of entity \(e\) with pose vector \(\mathbf{u}\)”. Activation of a capsule is encoded by the length of its activity vector (between 0 and 1 after squashing), while the direction of the vector encodes pose (orientation, scale, deformation, lighting, etc.).

Capsule. A capsule is a group of neurons whose output is a vector \(\mathbf{v}\in\mathbb{R}^d\). Activity length = probability the entity exists; direction = instantiation parameters.

Squashing nonlinearity. Keeps the length in \([0,1)\): \[\mathbf{v} = \frac{\|\mathbf{s}\|^2}{1+\|\mathbf{s}\|^2}\,\frac{\mathbf{s}}{\|\mathbf{s}\|}.\]

Routing-by-agreement (Dynamic Routing). Lower-level capsule \(i\) predicts a “vote” for higher-level capsule \(j\): \[\hat{\mathbf{u}}_{j\mid i} = \mathbf{W}_{ij}\,\mathbf{u}_i.\] Output of capsule \(j\) is a weighted sum of all votes: \[\mathbf{s}_j = \sum_i c_{ij}\,\hat{\mathbf{u}}_{j\mid i},\qquad \mathbf{v}_j = \mathrm{squash}(\mathbf{s}_j).\] The coupling coefficients \(c_{ij}\) are normalised by softmax over \(j\): \[c_{ij} = \frac{\exp(b_{ij})}{\sum_k \exp(b_{ik})}.\] Logits \(b_{ij}\) are updated by the agreement between vote and output: \[b_{ij}\gets b_{ij} + \hat{\mathbf{u}}_{j\mid i}\cdot \mathbf{v}_j.\] Iterate (typically 3 rounds). Capsules whose vote aligns with the eventual output get larger coupling; capsules whose vote disagrees get smaller coupling. This is what makes routing “agreement-based” rather than “max-pool”.

Loss. Margin loss per class (existence) plus reconstruction loss (decoder from the active capsule pose vector). Margin loss for class \(k\): \[L_k = T_k\max(0, m^+ - \|\mathbf{v}_k\|)^2 + \lambda(1-T_k)\max(0, \|\mathbf{v}_k\| - m^-)^2,\] with \(m^+=0.9, m^-=0.1, \lambda=0.5\).

EM routing. Hinton/Sabour/Frosst 2018 follow-up replaces the iterative-dot-product update with Expectation-Maximisation over a mixture-of-Gaussians model where each higher-level capsule is a Gaussian over predicted poses. More principled, but slower and trickier to implement.

Equivariance. Because pose information lives in the direction of the vector, a small rotation of the input ideally rotates the capsule’s activity vector predictably — i.e., the representation is equivariant, not just invariant. This is the key conceptual departure from max-pooling, which discards pose.

Practical reality. Capsule networks did not scale to ImageNet-class problems. They handle small datasets (MNIST, smallNORB) elegantly but do not match modern CNNs/Transformers on large images. The ideas — routing, agreement, equivariance — nonetheless show up in subsequent work (transformers’ attention is a form of soft routing).

Pseudocode — Dynamic Routing.

Inputs: predictions hat_u[i,j] for all (i, j)  # primary -> digit capsule votes
b[i,j] = 0
for r in 1..3:
    c[i,:] = softmax(b[i,:])                   # over j (parents)
    s[j]   = sum_i c[i,j] * hat_u[i,j]
    v[j]   = squash(s[j])
    b[i,j] += hat_u[i,j] . v[j]
return v

MODULE 3 — Developing Hybrid Vision Architectures

3.1 Designing architectures that combine CNNs with Transformers

The hybrid family was the response to two observations: (1) ViT needs vast data to outperform CNNs because it lacks the convolutional inductive bias; (2) CNNs lack the global-attention capability that helps with long-range reasoning. Hybrids put both pieces together. The dominant designs:

ConvNeXt (Liu et al., 2022). “A ConvNet for the 2020s.” The team started from a vanilla ResNet-50, then incrementally adopted Transformer-era choices: \(7\times 7\) depthwise convolution (large kernel, like a window of attention), inverted bottleneck, fewer activations and norms, LayerNorm (not BatchNorm), GELU activations, separate downsampling layer between stages. The result is a pure CNN that matches Swin-T at the same FLOPs (82.1% vs 81.3% ImageNet top-1) and beats it on transfer tasks. ConvNeXt v2 (Woo et al., 2023) adds Global Response Normalization and MAE-style pretraining for further gains. Lesson: much of the ViT/Swin advantage came from training recipe + macro design, not from attention per se.

CoAtNet (Dai et al., 2021). “CoAtNet: Marrying Convolution and Attention for All Data Sizes.” Stages: conv-conv-attention-attention with depthwise convolutions augmented by relative attention biases. The first two stages use MBConv (mobile inverted bottleneck) for local features at high resolution; the last two stages use transformer blocks with attention now affordable on the downsampled feature maps. CoAtNet-7 reaches 90.88% ImageNet top-1 with JFT-3B pretraining — among the highest reported.

MaxViT (Tu et al., 2022). Multi-axis attention: each block computes block attention (within disjoint \(7\times 7\) windows, like Swin) followed by grid attention (sample every \(\sqrt{N}/M\) positions to form a sparse grid, attend within it). Block attention captures local context; grid attention captures global context. Combined with MBConv at each stage, MaxViT is a strong general-purpose backbone.

BoTNet (Srinivas et al., 2021). Replace the final \(3\times 3\) convolutions in a ResNet’s last stage with multi-head self-attention. Cheap surgery, meaningful gains on detection (the global attention helps with long-range object reasoning at the top of the pyramid where resolution is already low).

LeViT (Graham et al., 2021). Hybrid optimised for inference latency: convolutional stem, downsampling via strided attention, attention bias from a learned table (no positional embedding tokens). Beats EfficientNet on the latency-accuracy frontier at small sizes.

MobileViT — covered in 2.1; a CNN+Transformer hybrid by construction.

Design themes common to all hybrids.

  • Use convolutions at high resolution (cheap, locality-friendly, good with little data).
  • Use attention at low resolution (manageable cost, global context).
  • Replace BatchNorm with LayerNorm in the transformer-style parts.
  • Adopt large-kernel depthwise convs (\(7\times 7\) or larger) to mimic window attention’s receptive field without softmax.
  • Stagewise downsampling to build a feature pyramid for downstream dense-prediction heads.

3.2 Integrate attention mechanisms into convolutional architectures

Before ViT, the question was the reverse: how to inject attention into CNNs? Several mechanisms became canonical.

Squeeze-and-Excitation (SE) block (Hu et al., 2018). Per-channel reweighting based on global context. For a feature map \(\mathbf{X}\in\mathbb{R}^{C\times H\times W}\):

  1. Squeeze: global average pool over space, giving a \(C\)-dimensional vector \(\mathbf{z}\in\mathbb{R}^C\), \(z_c=\frac{1}{HW}\sum_{i,j}X_{c,i,j}\).
  2. Excite: two-layer MLP with a bottleneck ratio \(r\): \(\mathbf{s} = \sigma(\mathbf{W}_2\,\delta(\mathbf{W}_1\mathbf{z}))\), where \(\sigma\) is sigmoid, \(\delta\) is ReLU.
  3. Reweight: multiply each channel by its gate: \(\tilde{X}_{c,:,:} = s_c\,X_{c,:,:}\).

SE blocks cost very little but consistently improve ImageNet accuracy by 1–2%. SE-ResNet-50 outperforms ResNet-101 at lower FLOPs.

CBAM — Convolutional Block Attention Module (Woo et al., 2018). Sequential channel attention (SE-like) followed by spatial attention:

  • Spatial attention: compute channel-wise max-pool and average-pool to get two \(1\times H\times W\) maps, concatenate, apply a \(7\times 7\) conv, and sigmoid. Multiply with the input across all channels. This tells the network where to look in addition to which channels matter.

Non-local block (Wang et al., 2018). Self-attention applied to convolutional feature maps: \[y_i = \frac{1}{\mathcal{C}(\mathbf{x})}\sum_j f(x_i, x_j)\,g(x_j),\] with \(f\) commonly the embedded Gaussian (dot-product followed by softmax) — i.e., scaled dot-product attention over spatial positions. Captures long-range dependencies (e.g., for action recognition).

Axial attention (Wang et al., 2020; Ho et al., 2019). Decompose 2D attention into a sequence of attention along rows and along columns. Cost drops from \(O((HW)^2)\) to \(O(H\cdot W^2 + W\cdot H^2) = O(HW(H+W))\). Used in Axial-DeepLab.

Self-attention pooling / GLOM-style aggregators. Later work (e.g., GLOM, ViT’s CLS token) generalises pooling via attention readout.

Worked example: count parameters of an SE block. With \(C=256\), \(r=16\): hidden dim is 16. \(\mathbf{W}_1\) is \(C\times C/r = 256\times 16 = 4096\) params; \(\mathbf{W}_2\) is \(C/r\times C = 16\times 256 = 4096\) params. Total ≈ 8k params per SE block — negligible compared to a \(3\times 3\) conv with 256 in/out channels (~590k params).

Code — SE block.

class SEBlock(nn.Module):
    def __init__(self, C, r=16):
        super().__init__()
        self.avg = nn.AdaptiveAvgPool2d(1)
        self.fc  = nn.Sequential(
            nn.Linear(C, C // r), nn.ReLU(inplace=True),
            nn.Linear(C // r, C), nn.Sigmoid())
    def forward(self, x):                         # (B, C, H, W)
        b, c, _, _ = x.shape
        s = self.avg(x).view(b, c)                # (B, C)
        s = self.fc(s).view(b, c, 1, 1)           # (B, C, 1, 1)
        return x * s                              # broadcast multiply

Common assessment trap. Channel attention vs spatial attention vs self-attention. SE = channel attention. CBAM = channel + spatial. Non-local = full spatial self-attention. Don’t conflate them.

MODULE 4 — Developing Autoregressive Vision Models

4.1 Implement Gated PixelCNNs for autoregressive image modeling

Definition and motivation. Autoregressive image models factorise the image likelihood as a product of pixel-level conditionals in raster-scan order: \[p(\mathbf{x}) = \prod_{i=1}^{HW}p(x_i\mid x_1,\dots,x_{i-1}).\] PixelRNN (van den Oord et al., 2016) used RNNs to model this. PixelCNN replaced the RNN with a deep stack of masked convolutions for parallel training. Gated PixelCNN (van den Oord et al., 2016b) fixed the blind spot problem of vanilla PixelCNN and added gated activations, becoming the de-facto autoregressive baseline.

Masked convolutions. To ensure causality (each pixel can only depend on previously-seen pixels), the convolutional kernel is masked.

  • Mask type A (used on the first layer): zero out the centre pixel and every position after the centre in raster order. Used because the input at the centre is the very pixel we’re predicting and must not be seen.
  • Mask type B (subsequent layers): zero out positions strictly after the centre in raster order, but allow the centre. Used because by the time we are in layer 2+, the input at the centre is a feature derived from already-known pixels.

Blind spot. A naïve stack of mask-B convolutions leaves a triangular region of pixels that no kernel can ever see — the blind spot. The receptive field grows as a parallelogram instead of the desired causal pentagon.

Vertical/horizontal stack fix. Gated PixelCNN splits the convolution into two paths:

  1. Vertical stack uses \(n\times n\) kernels that look only at rows above the current pixel (so they cover the entire row above with no constraint on horizontal position). Output of the vertical stack is fed into the horizontal stack.
  2. Horizontal stack uses \(1\times n\) kernels masked to look only at pixels to the left of the current pixel (and optionally include the centre via mask B). It receives the vertical-stack feature as an additive input via a \(1\times 1\) conv.

This decomposition fixes the blind spot — every previously-seen pixel can reach the current one through one of the two stacks.

Gated activation. Instead of ReLU, use a multiplicative gate: \[\mathbf{y} = \tanh(\mathbf{W}_f \ast \mathbf{x}) \odot \sigma(\mathbf{W}_g \ast \mathbf{x}),\] where \(\ast\) is masked convolution. The gate \(\sigma\) controls how much of the \(\tanh\) feature flows through. This is the same gating idea as LSTM / GLU and consistently outperforms ReLU here.

Conditional variant. Class-conditional PixelCNN adds a class embedding \(\mathbf{h}\) to the gated activation: \[\mathbf{y} = \tanh(\mathbf{W}_f \ast \mathbf{x} + \mathbf{V}_f \mathbf{h}) \odot \sigma(\mathbf{W}_g \ast \mathbf{x} + \mathbf{V}_g \mathbf{h}).\]

Sampling. Truly sequential. For each pixel in raster order, run a full forward pass with the current (partial) image as input, take the softmax output, and sample the next pixel. Cost: \(O(HW)\) forward passes per image — slow.

Code — masked convolutional layers.

class MaskedConv2d(nn.Conv2d):
    """Convolution that respects raster-scan causality."""
    def __init__(self, mask_type, *args, **kw):
        super().__init__(*args, **kw)
        assert mask_type in ("A", "B")
        _, _, kH, kW = self.weight.shape
        mask = torch.ones(kH, kW)
        mask[kH // 2, kW // 2 + (mask_type == "B"):] = 0
        mask[kH // 2 + 1:, :] = 0
        self.register_buffer("mask", mask[None, None])

    def forward(self, x):
        self.weight.data *= self.mask
        return super().forward(x)

Algorithm — Gated PixelCNN sampling.

x = zeros(H, W, C)
for i = 0..H-1:
    for j = 0..W-1:
        for c = 0..C-1:
            logits = PixelCNN(x)
            x[i, j, c] = sample(softmax(logits[i, j, c]))
return x

4.2 Apply autoregressive techniques for image completion

Inpainting and super-resolution are natural fits for autoregressive models: condition on the known pixels and sample only the unknown ones. The same algorithm covers both.

PixelCNN++ (Salimans et al., 2017). Replaces the 256-way softmax over each pixel intensity with a mixture of logistics over a continuous output, downsamples internally, and uses skip connections. Sharper images, faster training.

Image Transformer (Parmar et al., 2018). Replaces masked convolutions with masked self-attention over pixel tokens. Restricted to local 1D or 2D windows to keep complexity tractable. Better quality at the cost of training-time compute.

Parti (Yu et al., 2022, Google). Treats text-to-image generation as autoregressive prediction of image tokens (from a VQ-GAN tokenizer) conditioned on a text prompt. Scales to 20B params. Quality competitive with diffusion at the time of release.

MaskGIT (Chang et al., 2022). Generates images by iteratively unmasking tokens in a bidirectional transformer rather than left-to-right. Each step predicts all masked tokens in parallel and keeps the most confident predictions, then re-masks the rest with lower probability. Sampling is much faster than raster-scan PixelCNN — comparable to diffusion samplers — while quality matches strong diffusion baselines.

Inpainting with autoregressive models. For pixels inside the mask, sample conditional on every other pixel using the trained model; outside the mask, leave pixels unchanged. For raster-order models this is exact only when the mask is causal (top-left); for arbitrary masks, MaskGIT or BERT-style bidirectional autoregressive models are more natural.

Super-resolution. Treat the high-resolution pixels as the variable to sample; condition on the low-resolution input via a learned encoder that injects features into the transformer.

PART 2 — DEVELOPING GENERATIVE VISION MODELS

Generative vision models fall into three major families: likelihood-based (VAEs, autoregressive, diffusion), adversarial (GANs), and score-based (closely related to diffusion). This Part focuses on diffusion and GANs — the two dominant paradigms in modern image generation — plus data augmentation and style transfer (which historically grew out of the same toolbox).

MODULE 5 — Training Diffusion Models

5.1 Train diffusion models for generative image tasks

Definition and motivation. Denoising Diffusion Probabilistic Models (DDPM, Ho et al., 2020) define a fixed forward process that gradually corrupts data with Gaussian noise, and learn a reverse process that recovers data from noise. They became state-of-the-art for image synthesis around 2021–2022, displacing GANs for tasks that demand sample diversity and stable training.

Forward process. Starting from data \(\mathbf{x}_0\sim q(\mathbf{x}_0)\), define a Markov chain over \(T\) steps: \[q(\mathbf{x}_t\mid \mathbf{x}_{t-1}) = \mathcal{N}(\mathbf{x}_t;\sqrt{1-\beta_t}\mathbf{x}_{t-1},\beta_t\mathbf{I}),\] with a fixed noise schedule \(\{\beta_t\}_{t=1}^{T}\) (e.g., linear \(\beta_1=10^{-4}\to\beta_T=0.02\), or cosine). Convenient closed form using \(\alpha_t = 1-\beta_t\) and \(\bar\alpha_t = \prod_{s\le t}\alpha_s\): \[q(\mathbf{x}_t\mid \mathbf{x}_0) = \mathcal{N}(\mathbf{x}_t;\sqrt{\bar\alpha_t}\mathbf{x}_0,(1-\bar\alpha_t)\mathbf{I}).\] So sampling \(\mathbf{x}_t\) given \(\mathbf{x}_0\) is one-shot: \[\mathbf{x}_t = \sqrt{\bar\alpha_t}\,\mathbf{x}_0 + \sqrt{1-\bar\alpha_t}\,\boldsymbol\epsilon,\quad \boldsymbol\epsilon\sim\mathcal{N}(0,\mathbf{I}).\]

Reverse process. A learned Gaussian: \[p_\theta(\mathbf{x}_{t-1}\mid\mathbf{x}_t) = \mathcal{N}(\mathbf{x}_{t-1};\boldsymbol\mu_\theta(\mathbf{x}_t,t),\boldsymbol\Sigma_\theta(\mathbf{x}_t,t)).\] DDPM reparametrises \(\boldsymbol\mu_\theta\) in terms of a noise predictor \(\boldsymbol\epsilon_\theta\): \[\boldsymbol\mu_\theta(\mathbf{x}_t,t) = \frac{1}{\sqrt{\alpha_t}}\left(\mathbf{x}_t - \frac{\beta_t}{\sqrt{1-\bar\alpha_t}}\boldsymbol\epsilon_\theta(\mathbf{x}_t,t)\right).\] The variance is usually fixed to \(\beta_t\) or \(\tilde\beta_t = \beta_t\,(1-\bar\alpha_{t-1})/(1-\bar\alpha_t)\), or learned (Nichol & Dhariwal, 2021).

Simplified training loss. Drop weights, train the noise predictor by MSE: \[\mathcal{L}_{\text{simple}}(\theta) = \mathbb{E}_{t,\mathbf{x}_0,\boldsymbol\epsilon}\left[\|\boldsymbol\epsilon - \boldsymbol\epsilon_\theta(\sqrt{\bar\alpha_t}\mathbf{x}_0 + \sqrt{1-\bar\alpha_t}\boldsymbol\epsilon,\,t)\|^2\right].\] Sample \(t\sim\mathrm{Uniform}\{1,\dots,T\}\), sample \(\mathbf{x}_0\) from the dataset, sample \(\boldsymbol\epsilon\), form \(\mathbf{x}_t\), run the network, compute MSE.

\(\epsilon\)-prediction vs \(v\)-prediction. Two equivalent parameterisations.

  • \(\epsilon\)-prediction: predict the noise (DDPM original).
  • \(v\)-prediction (Salimans & Ho, 2022): predict \(\mathbf{v} = \sqrt{\bar\alpha_t}\boldsymbol\epsilon - \sqrt{1-\bar\alpha_t}\mathbf{x}_0\). Better behaviour at high noise levels; standard in distilled samplers and v-parameterised Stable Diffusion.

Score-matching view. Define \(s_\theta(\mathbf{x}_t,t)\approx\nabla_{\mathbf{x}_t}\log p_t(\mathbf{x}_t)\). The noise predictor and the score are related by: \[s_\theta(\mathbf{x}_t,t) = -\frac{\boldsymbol\epsilon_\theta(\mathbf{x}_t,t)}{\sqrt{1-\bar\alpha_t}}.\] Score-matching (Song & Ermon 2019; Song et al. 2020) provides the continuous-time SDE formulation that DDIM, DPM-Solver, and EDM (Karras et al., 2022) build on.

Architecture. Almost universally a U-Net with attention blocks at low-resolution scales. The U-Net takes \(\mathbf{x}_t\) as input, conditions on the timestep \(t\) via a sinusoidal embedding \(\to\) MLP \(\to\) added to every residual block, and (for conditional models) on a class label or text embedding via cross-attention or AdaGN.

Latent diffusion (Rombach et al., 2022, “Stable Diffusion”). Train a VAE on images so it produces a compact latent (e.g., \(64\times 64\times 4\) for a \(512\times 512\) RGB image). Run the diffusion model in latent space instead of pixel space. Order-of-magnitude reduction in compute; minimal loss in quality if the autoencoder is good.

EMA. Always maintain an Exponential Moving Average of the model weights (decay 0.999 or 0.9999) and sample from the EMA copy. Without EMA, samples are noticeably worse.

Worked numerical example. With \(\bar\alpha_t = 0.5\), \(\mathbf{x}_0\) a particular image pixel of value 0.8, and \(\epsilon = -0.3\): \(x_t = \sqrt{0.5}\cdot 0.8 + \sqrt{0.5}\cdot(-0.3) = 0.707\cdot 0.5 = 0.354\). The network sees \(x_t = 0.354\) at time \(t\) and is trained to predict \(\epsilon = -0.3\).

Algorithm — DDPM training.

repeat:
    x0 ~ data;  t ~ Uniform{1..T};  eps ~ N(0, I)
    xt = sqrt(alphabar_t) * x0 + sqrt(1 - alphabar_t) * eps
    L  = || eps - eps_theta(xt, t) ||^2
    optimize L
    EMA_theta <- 0.9999 * EMA_theta + 0.0001 * theta

Algorithm — DDPM sampling (ancestral).

xT ~ N(0, I)
for t = T..1:
    eps_hat = eps_theta(xt, t)
    mu      = (xt - beta_t / sqrt(1 - alphabar_t) * eps_hat) / sqrt(alpha_t)
    if t > 1: xt-1 = mu + sqrt(beta_t) * z, z ~ N(0, I)
    else:      x0  = mu
return x0

import torch, torch.nn.functional as F

    """x: (B, 3, 512, 512) image; betas: (T,) noise schedule."""
    with torch.no_grad():
        z0 = vae.encode(x).sample()                # (B, 4, 64, 64) latent
    B, T = z0.size(0), betas.size(0)
    t = torch.randint(0, T, (B,), device=z0.device)
    alphabar = torch.cumprod(1 - betas, dim=0)[t]  # (B,)
    eps = torch.randn_like(z0)
    a = alphabar.view(-1, 1, 1, 1).sqrt()
    b = (1 - alphabar).view(-1, 1, 1, 1).sqrt()
    zt = a * z0 + b * eps
    eps_hat = model(zt, t)                          # U-Net predicts noise
    return F.mse_loss(eps_hat, eps)

Engineering considerations.

  • Noise schedule matters. Linear schedules over-noise small images by mid-training; cosine schedule (Nichol & Dhariwal) keeps signal-to-noise reasonable longer.
  • Training stability. Use \(\beta_2=0.999\) in Adam, weight decay 0, mixed-precision (bf16 or fp32 master weights), gradient clipping at 1.0.
  • Resolution. Train at fixed resolution (256 or 512) for thousands of GPU-hours; high-resolution variants (1024+) use cascade or latent diffusion to stay tractable.
  • Memory. U-Net activations dominate. Use gradient checkpointing in the attention blocks if working at \(\ge 512\).

5.2 Implement sampling strategies for diffusion model inference

The trained network defines a family of samplers. Choose one based on compute budget and quality target.

Ancestral (DDPM) sampling. As above. Stochastic; \(T\) steps (often 1000). High quality but slow.

DDIM (Song et al., 2021). Deterministic, non-Markovian. Skips steps using: \[\mathbf{x}_{t-1} = \sqrt{\bar\alpha_{t-1}}\,\hat{\mathbf{x}}_0 + \sqrt{1-\bar\alpha_{t-1}-\sigma_t^2}\,\boldsymbol\epsilon_\theta(\mathbf{x}_t,t) + \sigma_t \mathbf{z},\] where \(\hat{\mathbf{x}}_0 = (\mathbf{x}_t - \sqrt{1-\bar\alpha_t}\,\boldsymbol\epsilon_\theta)/\sqrt{\bar\alpha_t}\) is the network’s estimate of the clean image. Setting \(\sigma_t = 0\) gives deterministic sampling; identical-looking samples with the same noise seed but $$10–50 steps instead of 1000.

DPM-Solver (Lu et al., 2022). Treats the reverse process as an ODE and solves it with a high-order solver. Reaches DDPM-level quality in 10–25 steps. Variants: DPM-Solver++, multistep DPM-Solver-2M.

Euler / Euler-ancestral / Heun. Karras et al. (2022) recast diffusion as a probability-flow ODE in EDM form. Euler is first-order; Heun is a second-order corrector (predict-then-correct), more accurate per step. Euler-ancestral injects stochasticity at each step à la DDPM. These solvers are standard in Automatic1111 / k-diffusion.

Quality vs steps. Empirically:

  • 1–4 steps: good with consistency models or LCM-distilled checkpoints; otherwise unusable.
  • 10–25 steps: DPM-Solver++ or DDIM with a strong CFG can produce visibly high-quality samples.
  • 50 steps: near-asymptotic quality for most diffusion checkpoints.
  • 100+ steps: diminishing returns.

Algorithm — DDIM sampling.

choose decreasing schedule t_0 > t_1 > ... > t_S = 0 (subset of 1..T)
x = N(0, I)
for s = 0..S-1:
    eps = eps_theta(x, t_s)
    x0_hat = (x - sqrt(1 - alphabar(t_s)) * eps) / sqrt(alphabar(t_s))
    x = sqrt(alphabar(t_{s+1})) * x0_hat + sqrt(1 - alphabar(t_{s+1})) * eps
return x

5.3 Apply conditioning techniques in diffusion models

Conditioning adds an external signal — a class label, a text caption, a depth map, an identity image — to steer generation.

Classifier guidance (Dhariwal & Nichol, 2021). Train a noise-robust classifier \(p_\phi(y\mid\mathbf{x}_t)\) on noisy images. At inference, modify the score by the classifier gradient: \[\hat{\boldsymbol\epsilon} = \boldsymbol\epsilon_\theta(\mathbf{x}_t,t) - \sqrt{1-\bar\alpha_t}\,\lambda\,\nabla_{\mathbf{x}_t}\log p_\phi(y\mid\mathbf{x}_t).\] \(\lambda\) is the guidance scale. Pros: works with any pretrained unconditional model. Cons: requires a separate noisy-image classifier.

Classifier-free guidance (CFG, Ho & Salimans, 2022). Train the same noise predictor on both conditional and unconditional inputs by randomly dropping the condition (say 10% of the time). At inference: \[\hat{\boldsymbol\epsilon}(\mathbf{x}_t,t,c) = \boldsymbol\epsilon_\theta(\mathbf{x}_t,t,\varnothing) + w\,\bigl(\boldsymbol\epsilon_\theta(\mathbf{x}_t,t,c) - \boldsymbol\epsilon_\theta(\mathbf{x}_t,t,\varnothing)\bigr).\] With \(w=1\) we recover the conditional model. With \(w>1\) we extrapolate “more strongly along \(c\)”, at the cost of reduced diversity. Stable Diffusion typically uses \(w\in[5,12]\).

Cross-attention conditioning. Text embeddings (e.g., CLIP text encoder) are fed into the U-Net as keys/values; spatial tokens act as queries. This is how Stable Diffusion turns a prompt into the image: \[\mathrm{Attn}(\mathbf{Q}_{\text{image}},\mathbf{K}_{\text{text}},\mathbf{V}_{\text{text}}).\]

ControlNet (Zhang et al., 2023). Adds a parallel copy of the U-Net’s encoder, initialised from the base model, with zero-initialised projection layers to inject control signals (Canny edges, depth, pose, scribble, segmentation). The base U-Net is frozen; only the ControlNet branch is trained. Zero-init projections mean the network is identical to the base model at the start of training, so it cannot diverge.

T2I-Adapter (Mou et al., 2023). A lighter alternative to ControlNet: a small adapter network produces residual features that are added to the U-Net’s encoder at multiple resolutions. Cheaper and smaller than ControlNet, with comparable quality on many control tasks.

IP-Adapter (Ye et al., 2023). Conditions on an image (e.g., a reference image whose style or subject you want to mimic). Uses a separate image encoder (CLIP image) and a parallel cross-attention layer that pumps image features into the U-Net. Decouples text and image conditioning so prompts and reference images can be combined.

MODULE 6 — Implementing GAN Architectures

6.1 Implement generator and discriminator architectures

Definition. A Generative Adversarial Network (Goodfellow et al., 2014) consists of two networks playing a minimax game: the generator \(G(\mathbf{z})\) maps noise \(\mathbf{z}\sim p_z\) to images; the discriminator \(D(\mathbf{x})\) scores how real an image looks. They are trained jointly until the discriminator can no longer distinguish generated from real samples.

Vanilla GAN objective. \[\min_G\max_D \mathbb{E}_{\mathbf{x}\sim p_{\text{data}}}[\log D(\mathbf{x})] + \mathbb{E}_{\mathbf{z}\sim p_z}[\log(1-D(G(\mathbf{z})))].\]

DCGAN (Radford et al., 2015). First convolutional architecture for GANs that actually trained well. Recipe: strided convolutions instead of pooling; batch norm in both networks (except last G layer and first D layer); ReLU in G except final tanh; LeakyReLU(0.2) in D; Adam with \(\beta_1=0.5\).

StyleGAN (Karras et al., 2019/2020). Architectural innovation: replace the input-noise-then-convolve pattern with a mapping network that takes \(\mathbf{z}\) to a “style” \(\mathbf{w}\), followed by a synthesis network whose convolutional layers are modulated by \(\mathbf{w}\) via Adaptive Instance Normalization (StyleGAN1) or weight modulation (StyleGAN2). Adds per-layer noise inputs for stochastic detail. StyleGAN2 fixes characteristic “droplet” artefacts via weight demodulation. StyleGAN3 (2021) addresses aliasing by enforcing equivariance, removing the texture-sticking pathology when interpolating in \(\mathbf{w}\) space.

BigGAN (Brock et al., 2018). Scales class-conditional GANs to ImageNet at 512×512. Class conditioning via class-conditional BatchNorm and a projection discriminator. Truncation trick: at inference, sample \(\mathbf{z}\) from a truncated normal to trade diversity for quality.

ProgGAN (Karras et al., 2017). Progressive Growing: start training at \(4\times 4\), then \(8\times 8\), \(16\times 16\), …, \(1024\times 1024\), fading in new layers smoothly. Stabilises training of high-resolution GANs.

Discriminator design. Convolutional encoder mirroring G, often with spectral normalization on each weight matrix (see 6.3). Output is one scalar per image (or a patch-wise score map for “PatchGAN” in image-to-image tasks).

Generator design. Either: (a) MLP from \(\mathbf{z}\) to a small feature map, then transposed-conv / upsample-then-conv layers; or (b) StyleGAN-style learned constant input + style modulation.

6.2 Train GAN models using adversarial loss and monitor training dynamics

Adversarial loss variants.

  • Vanilla (Goodfellow 2014): as above. Generator minimises \(\log(1-D(G(\mathbf{z})))\) — saturates when D is too strong.
  • Non-saturating (Goodfellow 2014, “the trick”): generator maximises \(\log D(G(\mathbf{z}))\) instead. Identical fixed point, better gradients early in training.
  • WGAN (Arjovsky et al., 2017): uses Wasserstein-1 distance under a 1-Lipschitz discriminator. Loss: \(\mathbb{E}[D(\mathbf{x})] - \mathbb{E}[D(G(\mathbf{z}))]\). Originally enforced Lipschitz by weight clipping (crude).
  • WGAN-GP (Gulrajani et al., 2017): enforces Lipschitz by adding a gradient penalty to the discriminator loss: \[\mathcal{L}_{\text{GP}} = \lambda\,\mathbb{E}_{\hat{\mathbf{x}}\sim p_{\hat{\mathbf{x}}}}\Bigl[(\|\nabla_{\hat{\mathbf{x}}} D(\hat{\mathbf{x}})\|_2 - 1)^2\Bigr],\] where \(\hat{\mathbf{x}}\) is a random interpolation between real and fake samples. \(\lambda=10\) typically.
  • Hinge loss (Lim & Ye, 2017; standard in BigGAN/StyleGAN2): \[L_D = \mathbb{E}[\max(0, 1 - D(\mathbf{x}))] + \mathbb{E}[\max(0, 1 + D(G(\mathbf{z})))],\quad L_G = -\mathbb{E}[D(G(\mathbf{z}))].\]
  • R1 regularization (Mescheder et al., 2018): \[\mathcal{L}_{R1} = \tfrac{\gamma}{2}\mathbb{E}_{\mathbf{x}\sim p_{\text{data}}}[\|\nabla_{\mathbf{x}} D(\mathbf{x})\|^2].\] Standard in StyleGAN2/3.

Tracking metrics.

  • Inception Score (IS): \(\exp(\mathbb{E}_{\mathbf{x}}\,\mathrm{KL}(p(y\mid\mathbf{x})\,\|\,p(y)))\). Higher is better. Sensitive to class diversity but biased toward ImageNet categories.
  • Fréchet Inception Distance (FID): Fit Gaussians to Inception features of real and generated samples; compute Wasserstein-2 between them: \[\mathrm{FID} = \|\mu_r - \mu_g\|^2 + \mathrm{Tr}(\Sigma_r + \Sigma_g - 2(\Sigma_r\Sigma_g)^{1/2}).\] Lower is better. The de-facto standard.
  • Precision/Recall (Kynkäänniemi et al., 2019): disentangles sample quality (precision) from diversity (recall).

Training dynamics — failure modes to watch for.

  • Mode collapse: generator outputs few distinct samples. Diagnose by low intra-batch variance and FID plateau.
  • Vanishing gradients: discriminator wins too fast; generator gradient signal disappears.
  • Oscillation: losses bounce; samples flip between failure modes.
  • NaNs: usually from unbounded D output; spectral norm or hinge loss fixes most cases.

6.3 Adjust training strategies to stabilize GAN learning

Spectral normalization (Miyato et al., 2018). Constrain each weight matrix to have spectral norm 1 by dividing by the largest singular value (estimated with one power iteration per step). Lipschitz-bounds the discriminator without weight clipping. Cheap, default-on in most modern GANs.

Two-time-scale update rule (TTUR, Heusel et al., 2017). Use different learning rates for G and D — typically D’s larger than G’s (e.g., \(\eta_D = 4\times 10^{-4}, \eta_G = 1\times 10^{-4}\)). Helps the discriminator stay informative without overpowering.

Gradient penalty. As in WGAN-GP and R1 — penalises the norm of the discriminator’s input gradient at real (or interpolated) points to keep it smooth.

Exponential moving averages of G’s weights. Sample from EMA-of-G, not raw G. Reduces sample variance dramatically.

Progressive growing (ProgGAN). Train at low resolution, then progressively add layers and grow to higher resolutions.

Equalised learning rate (StyleGAN). Scale the weight initialisation by the He factor at runtime rather than at init time, so all weights have similar gradient scales.

Path length regularization (StyleGAN2). Encourages small steps in \(\mathbf{w}\) space to correspond to small image-space changes.

Code — WGAN-GP training step.

def wgan_gp_step(G, D, x_real, opt_G, opt_D, lam=10.0):
    z = torch.randn(x_real.size(0), Z_DIM, device=x_real.device)
    x_fake = G(z).detach()
    eps = torch.rand(x_real.size(0), 1, 1, 1, device=x_real.device)
    x_hat = eps * x_real + (1 - eps) * x_fake
    x_hat.requires_grad_(True)
    d_hat = D(x_hat)
    grads = torch.autograd.grad(d_hat.sum(), x_hat, create_graph=True)[0]
    gp = ((grads.flatten(1).norm(2, dim=1) - 1) ** 2).mean()
    loss_D = D(x_fake).mean() - D(x_real).mean() + lam * gp
    opt_D.zero_grad(); loss_D.backward(); opt_D.step()
    z = torch.randn_like(z); x_fake = G(z)
    loss_G = -D(x_fake).mean()
    opt_G.zero_grad(); loss_G.backward(); opt_G.step()

MODULE 7 — Applying Data Augmentation and Synthesis

7.1 Explore automated data augmentation techniques

Definition. Data augmentation creates new training examples by applying label-preserving transformations to existing ones. Modern computer vision uses a stack of automated augmentation policies that are themselves learned or randomly sampled.

AutoAugment (Cubuk et al., 2018). Search a policy of sub-policies (each sub-policy is two image operations with probabilities and magnitudes) using reinforcement learning on a small proxy task. Discovered policies (e.g., for ImageNet) transfer to other datasets and improve accuracy by 1–2%.

RandAugment (Cubuk et al., 2020). Two hyperparameters: \(N\) (number of operations to apply per image) and \(M\) (magnitude). Sample \(N\) operations uniformly from the AutoAugment op set; apply each at magnitude \(M\). No expensive search; matches AutoAugment.

TrivialAugment (Müller & Hutter, 2021). Apply one operation, sampled uniformly, with magnitude sampled uniformly. Even simpler than RandAugment; competitive on most benchmarks.

AugMix (Hendrycks et al., 2020). Generate \(k\) augmented copies via random chains of operations, then mix them linearly with Dirichlet-sampled weights. Improves robustness to distribution shift (ImageNet-C).

Mixup (Zhang et al., 2018). Train on convex combinations of pairs: \[\tilde{\mathbf{x}} = \lambda \mathbf{x}_i + (1-\lambda)\mathbf{x}_j,\quad \tilde{\mathbf{y}} = \lambda \mathbf{y}_i + (1-\lambda)\mathbf{y}_j,\quad \lambda\sim\mathrm{Beta}(\alpha,\alpha).\] With \(\alpha=0.2\) or \(0.4\) it consistently improves ImageNet accuracy and calibration.

CutMix (Yun et al., 2019). Cut a rectangle from one image and paste it onto another; mix labels by area: \[\tilde{\mathbf{x}} = \mathbf{M}\odot\mathbf{x}_i + (\mathbf{1}-\mathbf{M})\odot\mathbf{x}_j,\quad \tilde{\mathbf{y}} = \lambda \mathbf{y}_i + (1-\lambda)\mathbf{y}_j.\] \(\mathbf{M}\) is a random rectangular mask of area \(1-\lambda\).

Cutout / Random Erasing (DeVries & Taylor, 2017; Zhong et al., 2020). Set a random rectangle to a constant (Cutout) or to random pixels (Random Erasing). Regularises against feature reliance on small patches.

Practical recipe (DeiT-III, ConvNeXt, ViT). RandAugment(N=2, M=9) + Mixup(0.8) + CutMix(1.0) + Random Erasing(0.25) + Label smoothing(0.1). This recipe alone closes most of the gap between vanilla ViT and the strongest hybrids.

Worked example: Mixup label. Two classes (cat, dog), \(\lambda=0.7\), \(\mathbf{y}_i=\text{one-hot(cat)}\), \(\mathbf{y}_j=\text{one-hot(dog)}\). \(\tilde{\mathbf{y}} = (0.7, 0.3)\). The loss is cross-entropy with this soft label.

7.2 Apply style transfer for domain adaptation

Neural Style Transfer (Gatys et al., 2015). Given a content image \(\mathbf{C}\) and a style image \(\mathbf{S}\), find an image \(\mathbf{x}\) that matches \(\mathbf{C}\) in content features and \(\mathbf{S}\) in style features of a pretrained VGG network.

  • Content loss: MSE between VGG features (typically conv4_2) of \(\mathbf{x}\) and \(\mathbf{C}\).
  • Style loss: MSE between Gram matrices of \(\mathbf{x}\) and \(\mathbf{S}\) at multiple layers. The Gram matrix at layer \(\ell\) is \[G^\ell_{ij} = \sum_k F^\ell_{ik}F^\ell_{jk},\] where \(F^\ell\in\mathbb{R}^{C_\ell\times (H_\ell W_\ell)}\) is the flattened feature map. Gram captures channel-channel correlations — texture without content.

Total objective: \[\mathcal{L} = \alpha\,\mathcal{L}_{\text{content}} + \beta\,\mathcal{L}_{\text{style}}.\] Optimise pixel values of \(\mathbf{x}\) via L-BFGS or Adam. Iterations per image: ~500.

AdaIN (Huang & Belongie, 2017). Replace optimisation with a feed-forward network: align the per-channel mean and standard deviation of content features to those of style features: \[\mathrm{AdaIN}(\mathbf{f}_c,\mathbf{f}_s) = \sigma(\mathbf{f}_s)\left(\frac{\mathbf{f}_c-\mu(\mathbf{f}_c)}{\sigma(\mathbf{f}_c)}\right) + \mu(\mathbf{f}_s).\] Real-time style transfer with arbitrary styles.

CycleGAN (Zhu et al., 2017). Unpaired image-to-image translation between two domains \(X\) and \(Y\). Two generators \(G:X\to Y\), \(F:Y\to X\) and two discriminators \(D_X, D_Y\). The trick is cycle consistency: \[\mathcal{L}_{\text{cyc}} = \mathbb{E}_{x}[\|F(G(x)) - x\|_1] + \mathbb{E}_{y}[\|G(F(y)) - y\|_1].\] Combined with adversarial losses, this provides supervision without paired data. Used for photo→painting, horse→zebra, day→night.

MUNIT / DRIT++. Disentangle content (shared across domains) and style (domain-specific). Allows one-to-many translation by sampling a style code.

Diffusion-based style transfer. Use a pretrained text-to-image diffusion model with an image prompt (IP-Adapter) or with a control image (ControlNet); inject style by ip-adapter weight or by conditioning on a style image embedded via CLIP. SDXL + IP-Adapter is the modern workhorse.

Worked example — Gram matrix (tiny). Suppose a layer has \(C=2\) channels and \(HW=3\) positions, with feature matrix \[F = \begin{pmatrix}1 & 0 & 2\\0 & 1 & 1\end{pmatrix}.\] Then \(G = FF^\top = \begin{pmatrix}5 & 2\\ 2 & 2\end{pmatrix}\) — a \(2\times 2\) Gram matrix capturing channel-channel correlations.

MODULE 8 — Utilizing Self-Supervised Learning

8.1 Implement self-supervised pretraining techniques

Definition and motivation. Self-Supervised Learning (SSL) trains a model on unlabeled data using pretext tasks whose targets are constructed from the input itself. The resulting features transfer to downstream tasks with little or no fine-tuning. SSL is what makes “foundation models for vision” possible — you can pretrain on hundreds of millions of unlabeled images, then fine-tune on a tiny labeled dataset.

Pretext-task SSL (the first wave).

  • Rotation prediction (Gidaris et al., 2018): rotate the image by \(\{0°,90°,180°,270°\}\) and predict the rotation as a 4-way classification. Strong baseline.
  • Jigsaw puzzles (Noroozi & Favaro, 2016): split the image into 9 patches, shuffle them by one of a fixed set of permutations, and predict which permutation was used.
  • Colorization (Zhang et al., 2016): predict the ab channels of a Lab image from the L channel.
  • Context prediction (Doersch et al., 2015): predict the spatial relation between two patches.

These were soon eclipsed by contrastive and masked methods, but they shaped the vocabulary.

Joint-embedding SSL.

  • BYOL (Grill et al., 2020): Bootstrap Your Own Latent. Two networks: an online network and a target network (EMA of the online network). Online predicts the target’s projection. No negative samples required. Trained with MSE between online prediction and target projection (after stop-gradient on the target side).
  • SimSiam (Chen & He, 2021): Simplified BYOL — no EMA target, just stop-gradient. Surprisingly works. Confirmed that the stop-gradient is the key ingredient, not the negative samples.
  • DINO (Caron et al., 2021): Self-distillation with no labels. Student and teacher both ViTs; teacher is EMA of student. Loss is cross-entropy between teacher and student predictions over a learned cluster assignment (centered/sharpened temperature). DINO features are notably good for image segmentation without any pixel-level supervision (attention maps “find” objects).
  • DINOv2 (Oquab et al., 2023): scale up DINO to 142M-image curated dataset, use Sinkhorn-Knopp centering, stronger augmentation, larger ViTs. State-of-the-art frozen features for classification, segmentation, depth, and retrieval.
  • I-JEPA (Assran et al., 2023): Image Joint-Embedding Predictive Architecture. Predict feature representations of masked target blocks from a context block, in latent space (not pixel space). Avoids the pixel-reconstruction overhead of MAE while keeping the benefits.

Algorithm — DINO pseudocode.

student, teacher = ViT, EMA(student)
for image in data:
    x1, x2 = aug(image), aug(image)             # two global views
    crops  = [x1, x2] + small_crops(image, k)   # multi-crop
    z_s    = [softmax(student(c) / tau_s) for c in crops]
    z_t    = [softmax((teacher(g) - center) / tau_t) for g in (x1, x2)]
    loss   = sum( -z_t[i] . log(z_s[j])  for i in [0,1], j != i over all crops)
    optimize
    teacher.weights = m * teacher.weights + (1 - m) * student.weights
    center = c_m * center + (1 - c_m) * mean(teacher outputs)

8.2 Utilize contrastive learning for enhanced feature representation

Contrastive learning. Pull together representations of “positive” pairs and push apart representations of “negatives”. For images, a positive pair is two random augmentations of the same image; negatives are augmentations of other images.

InfoNCE / NT-Xent. Given a batch of \(N\) images, generate \(2N\) augmentations (two per image). For a positive pair \((i,j)\): \[\ell_{ij} = -\log\frac{\exp(\mathrm{sim}(\mathbf{z}_i,\mathbf{z}_j)/\tau)}{\sum_{k=1}^{2N}\mathbf{1}_{[k\ne i]}\exp(\mathrm{sim}(\mathbf{z}_i,\mathbf{z}_k)/\tau)}.\] \(\mathrm{sim}\) is cosine similarity; \(\tau\in[0.05, 0.2]\) is the temperature. Total loss: \(\mathcal{L} = \frac{1}{2N}\sum_{(i,j)\text{ positive}} (\ell_{ij}+\ell_{ji})\).

SimCLR (Chen et al., 2020). Two-augmentation, in-batch negatives, projection head (\(f \to g(f)\) with two MLP layers + ReLU), \(\tau=0.1\), very strong augmentations (color jitter + Gaussian blur). Requires huge batch sizes (4096+) to get enough negatives.

MoCo v1/v2/v3 (He et al., 2020 / Chen et al., 2020 / Chen et al., 2021). Maintain a queue of negatives (typically 65k) so the batch size can stay small. The encoder for the queue is a momentum-updated copy of the main encoder (EMA). MoCo v2 adopts SimCLR’s MLP head and augmentations. MoCo v3 extends to ViTs and fixes the instability that arises when scaling.

Worked numerical example — InfoNCE with 4 examples. Two augmentations of each of two images: \(\mathbf{z}_1, \mathbf{z}_2\) from image A; \(\mathbf{z}_3, \mathbf{z}_4\) from image B. Suppose cosine similarities (after dividing by \(\tau\)):

\(\mathrm{sim}(\mathbf{z}_1,\mathbf{z}_2)/\tau = 5\) (positive), \(\mathrm{sim}(\mathbf{z}_1,\mathbf{z}_3)/\tau = 1\), \(\mathrm{sim}(\mathbf{z}_1,\mathbf{z}_4)/\tau = 0\).

Then \(\ell_{12} = -\log\frac{e^5}{e^5 + e^1 + e^0} = -\log\frac{148.4}{148.4 + 2.72 + 1} = -\log\frac{148.4}{152.1} \approx -\log(0.976) \approx 0.025\) — small, because the positive dominates.

If instead similarities were \(1, 5, 4\) (positive not dominating), the loss would be \(-\log\frac{e^1}{e^1 + e^5 + e^4}\approx -\log\frac{2.72}{208.8}\approx 4.34\) — large, reflecting the wrong arrangement.

Code — SimCLR loss.

def nt_xent(z, tau=0.1):
    """z: (2N, d) L2-normalised embeddings; positives at (i, i+N)."""
    N = z.size(0) // 2
    sim = z @ z.T / tau                                    # (2N, 2N)
    mask = torch.eye(2 * N, device=z.device).bool()
    sim.masked_fill_(mask, -1e9)                            # exclude self
    targets = torch.arange(2 * N, device=z.device)
    targets = (targets + N) % (2 * N)                       # positive index
    return F.cross_entropy(sim, targets)

MODULE 9 — Applying Masked Image Modeling

9.1 Develop vision models using masked image modeling (MIM)

Definition and motivation. Masked Image Modeling (MIM) adapts BERT-style masked-language modeling to vision. Mask a fraction of image patches, ask the model to reconstruct (pixels) or predict (tokens / features) the hidden ones. Unlike contrastive SSL, MIM does not need carefully designed augmentations or a negative pool — the pretext task is intrinsic to a single image.

MAE — Masked Autoencoders (He et al., 2022).

  • Encoder is a ViT that processes only the visible patches (typically 25% of patches; 75% are masked).
  • Decoder is a small ViT (8 blocks, narrow) that takes the visible-patch encodings plus learnable mask tokens placed at masked positions, then reconstructs the masked patches in pixel space.
  • Asymmetric design: because the encoder skips 75% of tokens, training is ~3× cheaper than processing all tokens.
  • Loss: MSE on the normalized pixel values of masked patches only (subtract patch mean, divide by patch std before computing MSE). Computing MSE only on masked patches gives a clean signal; normalising makes the task harder and improves transfer.

\[\mathcal{L}_{\text{MAE}} = \frac{1}{|\mathcal{M}|}\sum_{i\in\mathcal{M}}\|\hat{\mathbf{p}}_i - \mathbf{p}_i\|_2^2,\] where \(\mathcal{M}\) is the set of masked patches and \(\mathbf{p}_i\) is normalised.

BEiT (Bao et al., 2021). Predict discrete visual tokens (from a pretrained DALL-E-style VQ-VAE tokenizer) at masked positions, with a softmax classifier — the most BERT-like formulation. Predates MAE but uses a heavier objective.

SimMIM (Xie et al., 2022). Closer to MAE but processes all tokens (masked + visible) in the encoder, with a very simple decoder (a single linear layer) projecting back to pixels. Showed that the decoder complexity matters less than the masking strategy.

MaskFeat (Wei et al., 2022). Predict HOG features at masked positions instead of pixels. HOG is a hand-crafted feature; turns out to be a very effective target for video pretraining.

iBOT (Zhou et al., 2021). Combines DINO-style self-distillation with masked image modeling — a teacher network provides the target tokens, a student predicts them at masked positions. Strong results across many downstream tasks.

Worked example — count masked vs visible patches. A ViT-B/16 on \(224\times 224\) has 196 patches. MAE masks 75% → 49 visible, 147 masked. Encoder runs on 49 tokens + 1 CLS = 50 tokens (a 4× smaller sequence). Decoder runs on all 196 tokens (49 visible features + 147 mask tokens + positional embeddings).

Code — MAE sketch.

class MAE(nn.Module):
    def __init__(self, encoder, decoder, mask_ratio=0.75, patch=16):
        super().__init__()
        self.enc, self.dec = encoder, decoder
        self.mask_ratio = mask_ratio
        self.patch = patch
        self.mask_tok = nn.Parameter(torch.zeros(1, 1, decoder.dim))

    def random_mask(self, x):                       # x: (B, N, D)
        B, N, D = x.shape
        keep = int(N * (1 - self.mask_ratio))
        noise = torch.rand(B, N, device=x.device)
        idx_shuffle = noise.argsort(dim=1)
        idx_keep = idx_shuffle[:, :keep]
        idx_restore = idx_shuffle.argsort(dim=1)
        x_visible = x.gather(1, idx_keep.unsqueeze(-1).expand(-1, -1, D))
        mask = torch.ones(B, N, device=x.device); mask[:, :keep] = 0
        mask = mask.gather(1, idx_restore)
        return x_visible, mask, idx_restore

    def forward(self, image):                       # (B, 3, H, W)
        x = self.enc.patch(image) + self.enc.pos[:, 1:]
        x_vis, mask, ids = self.random_mask(x)
        z = self.enc.encode_tokens(x_vis)            # (B, keep, D)
        mask_tokens = self.mask_tok.expand(z.size(0), x.size(1) - z.size(1), -1)
        z_all = torch.cat([z, mask_tokens], dim=1)
        z_all = z_all.gather(1, ids.unsqueeze(-1).expand(-1, -1, z.size(-1)))
        recon = self.dec(z_all)                      # (B, N, patch*patch*3)
        return recon, mask

9.2 Apply random masking strategies for effective pretraining

Random patch masking. Mask each patch independently with probability \(p\). Standard for MAE (p=0.75) and SimMIM.

Block masking (BEiT). Mask contiguous rectangles. Forces the model to reconstruct large, contiguous structure rather than relying on adjacent pixels.

Asymmetric encoder–decoder. The reason MAE can mask 75% of patches (vs BERT’s 15%) is that images have far more spatial redundancy than text; the model can still recover global structure from a quarter of the patches. The 75% setting also makes the encoder cheap.

Token vs patch granularity. BEiT operates on VQ tokens (predict token IDs); MAE operates on pixels (predict normalised values). VQ token targets are easier (a 8192-way classification with sharp categories) but require an external tokenizer; pixel targets are harder but require no extra training.

9.3 Implement reconstruction objectives for masked image regions

Pixel-space MSE (MAE, SimMIM). \(\mathcal{L} = \mathbb{E}\|\hat{\mathbf{p}} - \mathbf{p}\|^2\) on masked patches only.

Normalised pixel target (MAE). Subtract patch mean and divide by patch std before computing MSE. Improves linear-probe accuracy by 1–2%.

Discrete-token target (BEiT). Cross-entropy over the VQ codebook for each masked patch: \[\mathcal{L} = -\sum_{i\in\mathcal{M}}\log p_\theta(z_i\mid \mathbf{x}^{\text{vis}}).\]

Feature target (MaskFeat, iBOT, data2vec). Predict the feature vectors that an EMA teacher (or a hand-crafted descriptor like HOG) computes on the full image at the masked positions. Bridges MIM and self-distillation.

Loss weighting. A common trick is to scale the loss inversely with the variance of the target so high-frequency details (which have higher variance after normalisation) don’t dominate.

MODULE 10 — Implementing Curriculum Learning

10.1 Develop curriculum learning strategies

Definition. Curriculum learning (Bengio et al., 2009) trains a model on easy examples first and progressively introduces harder ones. The intuition mirrors human learning: start with the basics; build up.

In practice for vision.

  • Sample difficulty curricula. Rank training samples by a difficulty signal (loss under a pretrained model, classifier confidence, distillation gap) and present them in order. Often improves final accuracy by 0.5–1% and speeds convergence.
  • Resolution curricula (EfficientNetV2). Start training at low resolution and small augmentation magnitude; progressively increase both as training proceeds. Saves wall-clock time while reaching strong accuracy.
  • Domain curricula. When training a model on a mixture of datasets, start with the cleanest, smallest, most general dataset; introduce noisier or more domain-specific data later.

Curriculum for SSL pretraining. Recent results (e.g., RHO-Loss; Mindermann et al., 2022) suggest that selecting learnable, worth-learning, not-yet-learned examples accelerates pretraining. Implementations require an auxiliary holdout-loss model.

10.2 Implement progressive learning techniques

Progressive Growing GANs (Karras et al., 2017). Train at \(4\times 4\) for some iterations, then add a \(8\times 8\) block by fading it in (alpha-blending the new high-res output with an upsampled low-res output), continue training, repeat up to \(1024\times 1024\). This was the unlock that made high-resolution GANs trainable.

EfficientNetV2 (Tan & Le, 2021). Progressive learning: jointly increase image resolution and regularisation strength (RandAugment magnitude, dropout, Mixup) across training. Starts at 128 px with mild augmentation, finishes at 380 px with strong augmentation. Halves training time vs naïve fixed-size training to reach the same accuracy.

Progressive learning in ViT. Less standard; some works (e.g., Token Labelling, TokenLearner) gradually increase token count or patch granularity. Most ViT training pipelines use fixed resolution + cosine schedule.

Curriculum sampling in diffusion. Stable Diffusion training first focuses on smaller timesteps (less noise), then expands to the full range, which is a curriculum over noise levels.

MODULE 11 — Applying Unsupervised Learning for GANs

11.1 Implement adversarial training for unsupervised GAN models

The classical GAN setup (Module 6) is itself unsupervised — the discriminator only needs unlabeled real samples. Variants:

  • Conditional GAN (cGAN). Condition both G and D on labels \(y\); loss as before with \(D(\mathbf{x}, y)\) and \(G(\mathbf{z}, y)\).
  • Unsupervised GAN. Pure noise → image; no labels at all. Vanilla GAN, DCGAN, StyleGAN1 in unconditional mode.
  • Self-supervised GAN (Chen et al., 2019). Add a rotation-prediction loss to the discriminator as an auxiliary task; improves training stability and quality without using labels.

Why unsupervised GANs are tricky. Without labels, modes can collapse to common “easy” classes; long-tail content gets pruned. Mitigations include strong data augmentation in D (StyleGAN-ADA), spectral norm, R1 regularization, and TTUR.

11.2 Apply cycle consistency loss for unpaired image translation

CycleGAN (Zhu et al., 2017). Two domains \(X\) (e.g., horses) and \(Y\) (e.g., zebras), unpaired. Train:

  • \(G:X\to Y\) (horse → zebra)
  • \(F:Y\to X\) (zebra → horse)
  • \(D_X\) (real-vs-fake on \(X\)), \(D_Y\) (real-vs-fake on \(Y\))

Adversarial losses: \[\mathcal{L}_{\text{GAN}}(G, D_Y, X, Y) = \mathbb{E}_{y}[\log D_Y(y)] + \mathbb{E}_{x}[\log(1 - D_Y(G(x)))].\] And symmetrically for \(F, D_X\).

Cycle-consistency loss (the core idea): \[\mathcal{L}_{\text{cyc}}(G,F) = \mathbb{E}_x\|F(G(x)) - x\|_1 + \mathbb{E}_y\|G(F(y)) - y\|_1.\] Forces \(G\) and \(F\) to be approximate inverses, which provides supervision in the absence of paired data.

Identity loss (optional, helps colour preservation): \[\mathcal{L}_{\text{id}} = \mathbb{E}_y\|G(y) - y\|_1 + \mathbb{E}_x\|F(x) - x\|_1.\] Encourages the generators to act as identity when given inputs already in the target domain.

Total objective. \[\mathcal{L} = \mathcal{L}_{\text{GAN}}(G,D_Y) + \mathcal{L}_{\text{GAN}}(F,D_X) + \lambda_{\text{cyc}}\mathcal{L}_{\text{cyc}} + \lambda_{\text{id}}\mathcal{L}_{\text{id}}.\] Typical \(\lambda_{\text{cyc}}=10\), \(\lambda_{\text{id}}=0.5\lambda_{\text{cyc}}\).

Algorithm — CycleGAN step.

sample x ~ X;  sample y ~ Y
fake_y = G(x);  rec_x = F(fake_y)
fake_x = F(y);  rec_y = G(fake_x)

# Discriminator updates
L_DY = -log D_Y(y) - log(1 - D_Y(fake_y.detach()))
L_DX = -log D_X(x) - log(1 - D_X(fake_x.detach()))
opt_D.step()

# Generator updates
L_G  = -log D_Y(fake_y) - log D_X(fake_x)               # adversarial
L_cyc = ||rec_x - x||_1 + ||rec_y - y||_1
L_id  = ||G(y) - y||_1 + ||F(x) - x||_1
L_total = L_G + 10 * L_cyc + 5 * L_id
opt_G.step()

Failure modes. Hallucination of objects (a horse with no rider may be mapped to a zebra-with-rider if zebra training images often contain riders); checkerboard artefacts from transposed convolutions; mode collapse on rare domains.

11.3 Utilize feature matching techniques in unsupervised GAN training

Feature matching loss (Salimans et al., 2016). Replace the generator’s adversarial loss with the MSE between the discriminator’s intermediate feature activations on real and fake images: \[\mathcal{L}_{\text{FM}}(G) = \|\,\mathbb{E}_{\mathbf{x}\sim p_{\text{data}}}D_f(\mathbf{x}) - \mathbb{E}_{\mathbf{z}}D_f(G(\mathbf{z}))\|_2^2,\] where \(D_f\) is the activation at some layer of \(D\). Generator now matches statistics of real data in feature space rather than fooling the discriminator outright. Often stabilises training, especially when used alongside the standard adversarial loss.

Perceptual loss (Johnson et al., 2016). Same idea but uses a pretrained VGG network as the feature extractor. Used in pix2pixHD, super-resolution GANs, video frame synthesis. Image is “perceptually similar” if VGG features at multiple layers match.

LPIPS (Zhang et al., 2018). Learned Perceptual Image Patch Similarity. Trains a small head on top of VGG/SqueezeNet/AlexNet features to better predict human perceptual judgement. The current default for “do these two images look the same to a human?”.

MODULE 12 — Building Cross-Modal Representation

12.1 Implement vision-language models for complex multimodal tasks

Definition. A Vision-Language Model (VLM) is any model that learns a joint representation of images and text — typically by training on large image-caption corpora. The dominant paradigms:

  • Dual-encoder / contrastive (CLIP-style). Encode image and text into a shared embedding space; train with a symmetric contrastive loss so that paired image-caption embeddings have high cosine similarity and unpaired ones have low. Excellent at retrieval and zero-shot classification; cannot generate text.
  • Fusion / cross-attention (BLIP, ALBEF, Flamingo). Image features feed into a text decoder via cross-attention; the model can generate captions, answer questions, and reason. Heavier than dual-encoders.
  • Late-fusion LLM (LLaVA, MiniGPT-4, InstructBLIP). Take a pretrained LLM, freeze it, and learn a projection from a vision encoder (CLIP/SigLIP/EVA) to the LLM’s token embedding space. Use the LLM as the language backbone; finetune lightly on instruction data.

CLIP (Radford et al., 2021). Trained on ~400M image-text pairs scraped from the web. Two encoders: a ViT image encoder and a transformer text encoder. Loss (the foundational vision-language contrastive loss):

\[\mathcal{L} = \tfrac{1}{2}\bigl(\mathrm{CE}(\mathrm{logits}_{I\to T}, y) + \mathrm{CE}(\mathrm{logits}_{T\to I}, y)\bigr),\]

with logits = \(\frac{\mathbf{I}\mathbf{T}^\top}{\tau}\), \(y\) the diagonal index in a batch of \(N\) pairs, and \(\tau\) a learned temperature. Equivalent to InfoNCE applied symmetrically.

ALIGN (Jia et al., 2021). Same recipe as CLIP but at much larger scale (1.8B pairs) and noisier text. Confirms that scale + minimal cleaning beats careful curation at smaller scale.

SigLIP (Zhai et al., 2023). Replaces the softmax-based InfoNCE with a sigmoid loss applied independently to every pair. Removes the dependence on batch size for negatives, allows smaller batches with no degradation, and is numerically more stable.

BLIP (Li et al., 2022). Three losses: image-text contrastive (CLIP-style), image-text matching (binary classifier), image-grounded language modeling (caption generation). One model can do retrieval, matching, and captioning.

BLIP-2 (Li et al., 2023). Frozen image encoder (ViT-G/EVA) + frozen LLM (FlanT5, OPT, Vicuna), bridged by a lightweight Q-Former (a transformer that takes learnable query tokens and cross-attends to the vision features, producing a fixed-size sequence of “soft prompts” for the LLM). Hugely reduces training cost.

Flamingo (Alayrac et al., 2022). Interleaves a frozen vision encoder and a frozen LLM with Perceiver Resampler (compress variable-size image features to a fixed number of tokens) and gated cross-attention layers inserted into the LLM. Supports multi-image multi-turn dialogue; pioneered in-context few-shot for VLMs.

LLaVA / LLaVA-1.5 (Liu et al., 2023). Project CLIP-ViT-L features through a two-layer MLP into Vicuna/LLaMA token space; instruction-tune on synthetic GPT-4-generated visual conversations. Cheap, effective, and the foundation of much follow-up work.

InternVL, Qwen-VL, Gemini, GPT-4V/4o. Modern VLMs scale these recipes up: hundreds of billions of parameters, multi-modal pretraining from scratch, native interleaved data.

Algorithm — CLIP training step.

sample batch of N (image, text) pairs
I = ImageEncoder(images)          # (N, d)
T = TextEncoder(texts)            # (N, d)
I = I / ||I||;  T = T / ||T||      # L2 normalize
logits = I @ T.T * exp(tau)       # (N, N) similarity matrix
labels = arange(N)
loss = (CE(logits, labels) + CE(logits.T, labels)) / 2
optimize loss

Code — CLIP-style bi-encoder training step.

def clip_step(img_enc, txt_enc, images, texts, log_tau, opt):
    I = F.normalize(img_enc(images), dim=-1)      # (N, d)
    T = F.normalize(txt_enc(texts),  dim=-1)      # (N, d)
    logits = (I @ T.t()) * log_tau.exp()          # (N, N)
    targets = torch.arange(I.size(0), device=I.device)
    loss = 0.5 * (F.cross_entropy(logits, targets) +
                  F.cross_entropy(logits.t(), targets))
    opt.zero_grad(); loss.backward(); opt.step()
    log_tau.data.clamp_(max=math.log(100))         # keep temperature bounded
    return loss.item()

12.2 Implement few-shot and zero-shot learning in vision-language models

Zero-shot classification with CLIP. Build a class-name “prompt” for each candidate class — e.g., “a photo of a {class}.” — encode all prompts with the text encoder, and assign each image to the class whose text embedding has the highest cosine similarity with the image embedding. No fine-tuning; works for any class name that CLIP saw during pretraining.

def clip_zero_shot(image, classnames, image_encoder, text_encoder):
    prompts = [f"a photo of a {c}." for c in classnames]
    T = F.normalize(text_encoder(prompts), dim=-1)
    I = F.normalize(image_encoder(image.unsqueeze(0)), dim=-1)
    return classnames[(I @ T.t()).argmax(dim=-1).item()]

Prompt engineering for VLMs. Ensemble many prompt templates (“a photo of a {c}”, “an image of a {c}”, “a sketch of a {c}”, …) and average their embeddings; consistently improves zero-shot accuracy by 1–3%.

Few-shot in-context (Flamingo). Interleave \(k\) image-text demonstration pairs in the prompt before the new image; the model attends to them at inference time. Performance scales with shot count up to 32 shots on most VQA benchmarks.

Few-shot via linear probing. Train a single linear classifier on top of frozen image embeddings using \(k\) examples per class. Strong baseline; CLIP’s linear-probe accuracy on ImageNet at 16 examples per class is ~80%.

Open-vocabulary detection / segmentation. Replace fixed class heads with text embeddings produced by a CLIP-like text encoder, so the model can localise any class described in natural language. ViLD, OWL-ViT, OpenSeg, Grounding DINO, GLIP, OWLv2.

Common assessment trap. Zero-shot uses zero downstream labels (only the class names as text). Few-shot uses a small number of labelled examples (linear probe, in-context). Both rely on a strongly pretrained VLM.

MODULE 13 — Executing Multimodal Tasks

13.1 Perform image captioning with vision-language models

Task. Given an image, produce a natural-language description.

Encoder-decoder (Vinyals et al., 2015, “Show and Tell”). CNN encodes image to a vector \(\to\) LSTM decodes a caption word by word, conditioning each token on the image vector and previously generated tokens. Trained with cross-entropy; optional reinforcement learning with CIDEr reward.

Attention-based (Xu et al., 2015, “Show, Attend and Tell”). Decoder attends over CNN feature map positions at each timestep — predates Transformer attention but is the same idea applied to captioning.

ViT + GPT. Replace the CNN with a ViT, replace the LSTM with a Transformer decoder. The ViT outputs patch features; the decoder cross-attends to them. Used in CLIPCap and many modern captioning systems.

BLIP / BLIP-2. Trained for captioning as a built-in capability — feed the image, prompt with “a photo of”, and generate.

LLaVA-style. Use a powerful LLM as the decoder; the captioning ability emerges from general instruction-following.

Evaluation metrics. BLEU-4, METEOR, ROUGE-L, CIDEr, SPICE, CLIPScore (CLIP-based, label-free). CIDEr is the most commonly reported.

13.2 Implement text-to-image generation models

Stable Diffusion (Rombach et al., 2022). Latent diffusion conditioned on CLIP text embeddings via cross-attention. Trained on LAION-5B. SD 1.5 → SD 2 → SDXL → SD 3 (rectified flow + dual text encoders); the workhorse open-weight family for text-to-image.

DALL-E 2 (Ramesh et al., 2022). Two-stage: (a) prior mapping CLIP text embedding to CLIP image embedding; (b) decoder (a diffusion model) generating an image from the CLIP image embedding. The CLIP latent is a semantic-level bottleneck.

DALL-E 3 (OpenAI, 2023). Native captioner-improved training and a more powerful decoder with strong text rendering.

Imagen (Saharia et al., 2022). Cascade of diffusion models: text \(\to\) 64×64 base \(\to\) 64×64 → 256×256 super-res \(\to\) 256×256 → 1024×1024 super-res. Uses a frozen T5-XXL text encoder (insight: a strong text encoder dominates over a strong image decoder).

Parti (Yu et al., 2022). Autoregressive over VQ-GAN tokens (see 4.2). Two-stage architecture: VQ tokenizer + transformer decoder.

Muse (Chang et al., 2023). MaskGIT-style parallel decoding over VQ tokens conditioned on T5 text embeddings. An order of magnitude faster than diffusion at comparable quality.

Key conditioning mechanism (Stable Diffusion). A frozen CLIP-text encoder produces a sequence of token embeddings. Each cross-attention block in the U-Net uses these as keys/values and the spatial latent as queries. Classifier-free guidance is applied at inference (Section 5.3).

Inversion / editing. Methods like DreamBooth (fine-tune the diffusion model on a few images of a subject), Textual Inversion (learn a new token embedding only), Null-text Inversion, Prompt-to-Prompt, InstructPix2Pix, and SDS (used for text-to-3D) enable editing or personalisation.

13.3 Perform transformer-based image segmentation

DETR — Detection Transformer (Carion et al., 2020). First end-to-end object detector that bypasses NMS and anchor design. Architecture: CNN backbone \(\to\) Transformer encoder over flattened feature map \(\to\) Transformer decoder with \(N\) learnable object queries that cross-attend to the encoder output. Each query outputs (class, box) or “no object”. Loss: Hungarian matching between predictions and ground-truth, then box-regression + classification per matched pair.

Mask2Former (Cheng et al., 2022). Unified architecture for semantic, instance, and panoptic segmentation. Same DETR-style decoder, but with masked attention that restricts each query’s attention to its predicted mask, and per-query mask predictions through a small mask head. Outperforms specialised architectures on all three segmentation tasks.

SAM — Segment Anything (Kirillov et al., 2023). A promptable segmentation model.

  • Image encoder: ViT-H/14, runs once per image (heavy).
  • Prompt encoder: lightweight, encodes points, boxes, masks, or text into embeddings.
  • Mask decoder: two-layer transformer that, in tens of milliseconds, takes the image embedding + prompt embedding and outputs a binary mask (plus IoU prediction).

Trained on SA-1B: 11M images, 1.1B masks generated by an iterative human-in-the-loop pipeline. Generalises to almost any image; segments objects, parts, regions on demand.

SAM 2 (Ravi et al., 2024). Extends SAM to video and to interactive tracking. Memory bank stores past frames so the model maintains object identity through time.

Segmentation taxonomy (the assessment trap).

  • Semantic segmentation: every pixel gets a class label. Same class instances are not distinguished.
  • Instance segmentation: each object instance has its own mask. Class labels too.
  • Panoptic segmentation: combines both — “stuff” classes (sky, road) get semantic masks; “things” classes (people, cars) get instance masks.
  • Interactive segmentation: user clicks / boxes / scribbles guide the mask. SAM is the canonical model.

class SAMPromptEncoder(nn.Module):
    def __init__(self, dim=256):
        super().__init__()
        self.point_emb = nn.Embedding(2, dim)              # fg / bg
        self.box_emb   = nn.Embedding(2, dim)              # tl / br
        self.no_mask   = nn.Embedding(1, dim)
        self.mask_conv = nn.Sequential(
            nn.Conv2d(1, dim // 4, 2, 2), nn.GELU(),
            nn.Conv2d(dim // 4, dim, 2, 2))

    def forward(self, points=None, boxes=None, mask=None):
        emb = []
        if points is not None:
            emb.append(self.point_emb(points["labels"]) + sinusoidal_pe(points["coords"]))
        if boxes is not None:
            emb.append(self.box_emb.weight + sinusoidal_pe(boxes))
        sparse = torch.cat(emb, dim=1) if emb else None
        dense  = self.mask_conv(mask) if mask is not None else self.no_mask.weight.view(1, -1, 1, 1)
        return sparse, dense

class SAMMaskDecoder(nn.Module):
    """Two-block transformer decoder that maps image+prompt to a mask."""
    def __init__(self, dim=256):
        super().__init__()
        self.iou_token  = nn.Parameter(torch.zeros(1, 1, dim))
        self.mask_token = nn.Parameter(torch.zeros(1, 3, dim))     # multi-mask
        self.blocks = nn.ModuleList([CrossAttnBlock(dim) for _ in range(2)])
        self.upscale = nn.Sequential(
            nn.ConvTranspose2d(dim, dim // 4, 2, 2), nn.GELU(),
            nn.ConvTranspose2d(dim // 4, dim // 8, 2, 2), nn.GELU())
        self.mlp_iou   = nn.Linear(dim, 1)
        self.mlp_mask  = nn.Linear(dim, dim // 8)

    def forward(self, img_emb, sparse, dense):                    # img_emb: (B, dim, H/16, W/16)
        B, C, H, W = img_emb.shape
        img_emb = img_emb + dense
        tokens  = torch.cat([self.iou_token, self.mask_token], dim=1).expand(B, -1, -1)
        if sparse is not None:
            tokens = torch.cat([tokens, sparse], dim=1)
        x = img_emb.flatten(2).transpose(1, 2)                    # (B, H*W, C)
        for blk in self.blocks: tokens, x = blk(tokens, x)
        mask_feat = self.upscale(x.transpose(1, 2).view(B, C, H, W))
        masks = (self.mlp_mask(tokens[:, 1:4]) @ mask_feat.flatten(2)).view(B, 3, H * 4, W * 4)
        iou   = self.mlp_iou(tokens[:, 0])
        return masks, iou

13.4 Perform image-text retrieval using cross-attention mechanisms

Bi-encoder retrieval (CLIP). Pre-compute image embeddings, pre-compute text embeddings, then do nearest-neighbour search in the shared space. Lightning fast; mediocre precision because the encoders never see each other’s content.

Cross-encoder reranking (BLIP, ALBEF, X-VLM). A second-stage model takes (image, text) as joint input and computes a single matching score with full cross-attention. Reranks the bi-encoder’s top-N.

ALBEF (Li et al., 2021). Align Before Fuse. Stage 1: contrastive alignment of separate image/text encoders (CLIP-like). Stage 2: cross-attention fusion module + image-text matching head. Stage 3 (during training): momentum distillation for noisy web pairs.

BLIP image-text matching head sketch.

class ITMHead(nn.Module):
    def __init__(self, dim):
        super().__init__()
        self.cross = nn.MultiheadAttention(dim, num_heads=8, batch_first=True)
        self.fc = nn.Linear(dim, 2)                       # matched vs not
    def forward(self, text_emb, image_emb):
        # text_emb: (B, L_t, D); image_emb: (B, L_i, D)
        attn, _ = self.cross(text_emb, image_emb, image_emb)
        cls = attn[:, 0]                                   # use [CLS]
        return self.fc(cls)

Retrieval evaluation. R@1 / R@5 / R@10 on MS-COCO and Flickr30K for both directions (image→text and text→image).

MODULE 14 — Advancing Visual Reasoning

14.1 Implement models for Visual Question Answering (VQA)

Task. Given an image and a free-form question, produce an answer (open-ended text or a class index out of fixed candidates).

Pre-transformer baselines. CNN image encoder + LSTM question encoder + element-wise fusion or bilinear pooling + classifier (LXMERT-precursors, MCB).

Two-stream transformers.

  • LXMERT (Tan & Bansal, 2019). Separate language and vision transformer streams plus a cross-modal transformer; trained on multiple pretext tasks including masked language modeling, masked object prediction, and image-text matching.
  • ViLBERT (Lu et al., 2019). Similar two-stream architecture with co-attentional transformer layers.

Modern VLM-based VQA.

  • BLIP-2. Frozen image encoder + Q-Former + frozen LLM. Q-Former produces soft prompt tokens that condition the LLM; LLM generates the answer.
  • LLaVA / LLaVA-1.5. CLIP-ViT-L features projected into Vicuna’s embedding space; instruction-tuned on VQAv2, GQA, OK-VQA, ScienceQA. Strong zero-shot and instruction-following performance.
  • InstructBLIP (Dai et al., 2023). BLIP-2 architecture with an instruction-aware Q-Former (instructions also enter the Q-Former) and broad instruction-tuning across 26 datasets.

Evaluation datasets. VQAv2 (general), GQA (compositional), OK-VQA (external knowledge), TextVQA (reading text in images), MMMU (multimodal university exams), MathVista (visual math).

14.2 Apply visual grounding techniques

Visual grounding. Localise the region of an image that a natural-language expression refers to. Sub-tasks:

  • Referring Expression Comprehension (REC): input image + text; output a bounding box.
  • Referring Expression Segmentation (RES): output a mask instead of a box.
  • Phrase grounding: localise every noun phrase in a caption.

MDETR (Kamath et al., 2021). DETR with text queries. The decoder’s object queries are augmented with token embeddings from the text; outputs boxes aligned to the words/phrases.

GLIP (Li et al., 2022). Reformulates object detection as a grounding task: each candidate class name is treated as a phrase, and the detector predicts boxes for all phrases in a “prompt” listing classes. Trained on large image-text-grounding corpora; achieves open-vocabulary detection.

Grounding DINO (Liu et al., 2023). Combines a DETR-style detector with strong vision-language pretraining (BERT text encoder + Swin image encoder + cross-modal decoder). Achieves state-of-the-art open-vocabulary detection and integrates naturally with SAM (Grounded-SAM) for open-vocabulary segmentation.

Kosmos-2 (Peng et al., 2023). A general-purpose multimodal LLM that learns to output bounding-box tokens alongside text — grounding is just another token type. Trained on GRIT, a corpus of grounded image-text pairs.

Visual grounding vs VQA — the trap. VQA expects a textual answer to a question. Visual grounding expects a spatial answer (box or mask) to a description. A system can do one without the other; modern VLMs (Kosmos-2, Qwen-VL) do both.

14.3 Implement visual reasoning systems with compositional logic

Compositional visual reasoning. Many questions require multi-step reasoning over discrete sub-tasks: “How many red objects are to the left of the cube and the same size as the sphere?” Classic monolithic VQA models fail on this; the field developed modular and neuro-symbolic alternatives.

Neural Module Networks (Andreas et al., 2016). Parse the question into a layout of differentiable modules (Find, Filter, Compare, Count) and dynamically assemble a network for that question.

NS-VQA — Neuro-Symbolic VQA (Yi et al., 2018). A scene parser converts the image to a structured scene graph (objects, attributes, relations). A program generator parses the question into a symbolic program. An executor runs the program against the scene graph. State-of-the-art on the CLEVR benchmark.

MMReasoner / chain-of-thought for VLMs. Prompt or fine-tune VLMs to emit a step-by-step rationale before the final answer. Helps on compositional questions and reduces shortcut behaviour.

Set-of-Mark prompting (Yang et al., 2023). Pre-annotate the image with numbered marks (e.g., from SAM segments) and reference those marks in the prompt: “Object 3 is to the left of Object 5; is Object 3 red?” Lets VLMs reason about spatial relations more reliably.

Visual Programming / ViperGPT (Surís et al., 2023). Use a code-writing LLM to emit Python that calls vision tools (object detector, depth estimator, captioner) and composes their outputs. The LLM does the reasoning; the vision tools do the perception.

Common assessment trap. Neuro-symbolic approaches mix learned perception with explicit programs/graphs. Monolithic VQA pushes everything through a single network and relies on the network’s capacity (and CoT prompting) to handle compositionality.

APPENDIX C — Self-Test Bank (~50 Questions)

Use these to gauge readiness. Each question is followed by a short answer or a section pointer. Mix of MCQ, match-the-pairs, ordering, and descriptive prompts.

Architectures and ViT family.

  1. What three components define ViT input processing? — Patch embedding (Conv with stride=patch), prepended learnable [CLS] token, added positional embedding. (§1.1)

  2. Compute the number of patch tokens (excluding [CLS]) for a 384×384 image with patch size 16. — \(24\times 24 = 576\). (§1.1)

  3. Why does ViT use a [CLS] token instead of global average pooling? — Convention from BERT; DeiT showed average pooling works equally well. (§1.1)

  4. How does Swin reduce attention cost compared to ViT? — Computes attention inside disjoint \(M\times M\) windows; cost is \(O(M^2 \cdot N)\) rather than \(O(N^2)\). (§1.2)

  5. What problem do “shifted windows” solve, and how? — They re-introduce cross-window communication. Alternating blocks shift the window grid by \(\lfloor M/2\rfloor\) so tokens that were in different windows now share one. (§1.2)

  6. Match: patch merging vs patch embedding vs positional embedding. — Patch embedding: image→token. Patch merging: 2×2 tokens→1 token (Swin downsampling). Positional embedding: per-token spatial encoding. (§§1.1–1.2)

  7. Difference between NAT and DiNAT? — NAT: per-token sliding \(k\times k\) neighborhood attention. DiNAT: same but with dilation rates that vary by layer, expanding receptive field. (§1.3)

  8. Don’t confuse DINO with DiNAT — explain. — DINO is a self-supervised distillation method (Caron 2021); DiNAT is an architecture with dilated neighborhood attention (Hassani 2023). (§1.3, §8.1)

Multi-scale, hybrid, capsules.

  1. What is the role of the unfold/fold trick in MobileViT? — Reorders patches so the transformer attends across spatial positions while keeping the per-position channel compute small; enables global reasoning at mobile-FLOP budgets. (§2.1)

  2. Describe the dual-branch design of CrossViT. — Small-patch branch with many tokens (detail) + large-patch branch with few tokens (semantics). Class tokens are swapped between branches and used as queries against the other branch’s patches. (§2.2)

  3. Describe routing-by-agreement in capsules in 3 lines. — Lower capsules predict votes for higher capsules; coupling coefficients (softmax of accumulated logits) weight the votes; logits are updated by the dot product of vote and current parent output; iterate (typically 3 rounds). (§2.3)

  4. What modern CNN matches Swin-T at the same FLOPs? — ConvNeXt-T. (§3.1)

  5. Compare SE, CBAM, and non-local blocks. — SE: channel attention only. CBAM: channel + spatial attention. Non-local: full self-attention over spatial positions of a feature map. (§3.2)

Autoregressive image models.

  1. Explain the “blind spot” in PixelCNN and how Gated PixelCNN fixes it. — Stacking mask-B convolutions leaves a triangular set of pixels invisible. Gated PixelCNN uses parallel vertical and horizontal stacks that together cover every previously-seen pixel. (§4.1)

  2. Write the gated activation used in Gated PixelCNN. — \(\tanh(W_f \ast x) \odot \sigma(W_g \ast x)\). (§4.1)

  3. Order: PixelRNN, PixelCNN, Gated PixelCNN, PixelCNN++, Image Transformer. — Chronological as listed (2016 → 2017 → 2018). (§4.2)

  4. How does MaskGIT differ from autoregressive raster decoding? — It is bidirectional, predicts all masked tokens in parallel each step, and keeps the most confident predictions while re-masking the rest. Far fewer steps. (§4.2)

Diffusion.

  1. Write the closed-form forward sampling in DDPM. — \(x_t = \sqrt{\bar\alpha_t}x_0 + \sqrt{1-\bar\alpha_t}\,\epsilon\). (§5.1)

  2. State the simplified DDPM training loss. — \(\mathcal{L} = \mathbb{E}_{t,x_0,\epsilon}\|\epsilon - \epsilon_\theta(x_t,t)\|^2\). (§5.1)

  3. What does “\(v\)-prediction” mean? — Predict \(v=\sqrt{\bar\alpha}\epsilon - \sqrt{1-\bar\alpha}\,x_0\) instead of \(\epsilon\). Better numerics at high noise; standard in distillation. (§5.1)

  4. Why train in latent space (Stable Diffusion)? — Image compression by the VAE reduces resolution by ~8×, so diffusion runs on a much smaller tensor (e.g., \(64\times 64\) instead of \(512\times 512\)). Order-of-magnitude compute savings. (§5.1)

  5. Write classifier-free guidance. — \(\hat\epsilon = \epsilon_\theta(x_t,t,\varnothing) + w(\epsilon_\theta(x_t,t,c) - \epsilon_\theta(x_t,t,\varnothing))\). (§5.3)

  6. Difference between classifier guidance and classifier-free guidance. — Classifier guidance uses a separately trained noise-robust classifier and modifies the score with \(\nabla\log p(y\mid x_t)\). CFG instead trains one model both conditionally and unconditionally and extrapolates the conditional direction at inference; no extra classifier. (§5.3)

  7. Why does ControlNet zero-initialise its projections? — At initialisation, the ControlNet branch contributes zero, so the system is identical to the frozen base model — guarantees stable warm-start. (§5.3)

  8. Order: DDPM ancestral, DDIM, DPM-Solver, single-step (LCM). — Roughly decreasing number of steps required for comparable quality. (§5.2)

GANs.

  1. Write the WGAN-GP gradient penalty. — \(\lambda \mathbb{E}_{\hat x}[(\|\nabla_{\hat x} D(\hat x)\|_2 - 1)^2]\) with \(\hat x\) a random interpolation between real and fake; \(\lambda=10\). (§6.2)

  2. Match: spectral norm / R1 / TTUR / EMA. — Spectral norm: Lipschitz constraint on D. R1: gradient penalty at real samples. TTUR: different learning rates for G and D. EMA: average G’s weights for sampling. (§6.3)

  3. Define FID in one line. — Fréchet distance between the Gaussians fit to Inception features of real and generated images. Lower is better. (§6.2)

  4. Explain mode collapse and one mitigation. — Generator outputs only a few distinct samples; diagnose by low intra-batch diversity. Mitigate by spectral norm + non-saturating loss + augmentation in D (StyleGAN-ADA). (§6.2–6.3)

Augmentation & style transfer.

  1. Write the Mixup formula. — \(\tilde x = \lambda x_i + (1-\lambda)x_j\), \(\tilde y = \lambda y_i + (1-\lambda) y_j\), \(\lambda\sim\mathrm{Beta}(\alpha,\alpha)\). (§7.1)

  2. Difference between Mixup and CutMix? — Mixup blends pixel intensities across the whole image; CutMix pastes a random rectangle from one image onto another and mixes labels by area. (§7.1)

  3. Write the Gram matrix definition. — \(G_{ij}=\sum_k F_{ik}F_{jk}\). (§7.2)

  4. Why does CycleGAN need cycle-consistency loss? — Provides supervision in the absence of paired data; without it, the generators could map any image to any image in the target domain. (§11.2)

Self-supervised.

  1. Write the InfoNCE loss for one positive pair. — \(\ell = -\log\frac{\exp(\mathrm{sim}(z_i,z_j)/\tau)}{\sum_{k\ne i}\exp(\mathrm{sim}(z_i,z_k)/\tau)}\). (§8.2)

  2. Match: SimCLR / MoCo / BYOL / SimSiam. — SimCLR: in-batch negatives, big batches. MoCo: queue of negatives + momentum encoder. BYOL: no negatives; EMA target + stop-grad. SimSiam: no negatives; stop-grad only, no EMA. (§8.2)

  3. Why does MAE mask 75% of patches but BERT only ~15%? — Images are spatially redundant; a model can still reconstruct from 25% visible patches. Text tokens carry more independent information; high mask ratios hurt. (§9.1)

  4. State MAE’s loss target. — Normalised pixel values of masked patches only, MSE. (§9.1)

  5. Compare BEiT vs MAE vs SimMIM vs MaskFeat in one line each. — BEiT: predict VQ tokens. MAE: pixel target with asymmetric encoder-decoder. SimMIM: pixel target with linear decoder; encoder sees all tokens. MaskFeat: HOG-feature target. (§9.1–9.3)

  6. Contrastive vs masked SSL — name the key conceptual difference. — Contrastive pulls/pushes embeddings of augmentation pairs; masked predicts hidden parts of the input from visible parts. Different pretext tasks. (§§8.2, 9)

Vision-Language.

  1. Write the CLIP symmetric InfoNCE loss. — \(\mathcal{L} = \tfrac{1}{2}(\mathrm{CE}(IT^\top/\tau, y) + \mathrm{CE}(TI^\top/\tau, y))\) with \(y\) the diagonal index. (§12.1)

  2. What is SigLIP’s key change over CLIP? — Sigmoid (per-pair) loss instead of softmax-over-batch; removes batch-size sensitivity, more numerically stable. (§12.1)

  3. Explain Q-Former (BLIP-2). — A small transformer with learnable query tokens that cross-attends to the frozen image encoder’s features; its outputs are projected into the frozen LLM’s token embedding space as a fixed-size “soft prompt”. (§12.1)

  4. Zero-shot CLIP classification — how does it work? — Encode each class name as a prompt; encode the image; assign the class whose text embedding has highest cosine similarity to the image embedding. (§12.2)

  5. What is Flamingo’s main architectural innovation? — Perceiver Resampler to compress variable-size image features to a fixed token count, plus gated cross-attention layers inserted into a frozen LLM. Enables few-shot in-context multimodal prompting. (§12.1)

Segmentation & grounding.

  1. Define semantic vs instance vs panoptic segmentation. — Semantic: per-pixel class only. Instance: per-object mask + class. Panoptic: both, with stuff classes (semantic) and thing classes (instance). (§13.3)

  2. What makes SAM “promptable”? — Separate prompt encoder for points/boxes/masks/text + lightweight mask decoder. The heavy image encoder runs once; the decoder runs per prompt in tens of ms. (§13.3)

  3. Visual grounding vs VQA — what’s the output type? — Grounding outputs a spatial region (box or mask). VQA outputs text. (§14.2)

  4. Order the modern open-vocab detection stack: MDETR, GLIP, Grounding DINO. — Chronological (2021 → 2022 → 2023); progressively stronger language conditioning and larger pretraining. (§14.2)

Reasoning.

  1. What is the Set-of-Mark prompting trick? — Pre-segment the image (e.g., with SAM), overlay numbered marks, and refer to those numbers in prompts; helps VLMs reason about spatial layout. (§14.3)

  2. Neuro-symbolic VQA vs end-to-end VQA — describe the difference. — Neuro-symbolic systems parse the image to a structured scene graph and execute a symbolic program parsed from the question. End-to-end systems push image + question through a single network. (§14.3)
